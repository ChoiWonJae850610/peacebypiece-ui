# scripts 폴더 안내

- 기준 앱 버전: `0.23.72`
- 용도: 개발·시드·검증 보조 스크립트 보관
- 운영 런타임 화면 코드와 분리된 영역으로 본다.

## 현재 포함 파일

- DB/R2 테스트 환경을 구성하는 simulator 명령은 `tools/simulator/`로 이동했다.
- `scripts/`에는 감사·계약·테스트 실행 진입점만 둔다.

## 테스트 불가 기간 기준

사용자가 테스트 가능하다고 명시하기 전까지는 이 폴더의 실행 동작을 바꾸지 않는다. 이 기간에는 다음 작업만 우선한다.

- 실행 전제와 위험 범위 문서화
- secret/URL 노출 방지 기준 점검
- 생성물 위치와 `.gitignore` 기준 확인
- 실제 DB/API/R2 동작을 바꾸지 않는 주석·문서 보강

## 생성물 기준

스크립트가 만드는 로컬 생성물은 `.tmp/` 아래에 둔다. `.tmp/`는 Git 추적 대상이 아니다.


## DB/API smoke test

- `smoke-db-api.mjs`
  - 개발 또는 초기화 가능한 테스트 DB에서 DB/API 계약을 자동 점검한다.
  - 실행 명령: `npm run test:smoke:db-api`
  - 모든 쓰기는 트랜잭션 안에서 실행하고 마지막에 `ROLLBACK`한다.
  - 운영 DB에서 실행하지 않는다.

## 주의

이 폴더의 스크립트는 개발 보조용이다. 실제 고객사 데이터, 운영 DB, 운영 R2, 실제 secret으로 실행하지 않는다.
