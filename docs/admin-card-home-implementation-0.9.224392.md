# 0.9.224392 — 고객관리자 카드형 홈 1차 구현

## 목적

고객관리자 화면의 좌측 패널을 렌더링하지 않고, `/admin`을 카드형 홈 중심으로 전환한다.

이번 버전은 실제 권한 DB, 멤버 초대, 권한 관리 UI를 구현하지 않는다. 대신 카드 registry에 permission code를 포함해 이후 권한 기반 홈으로 확장할 수 있는 구조를 먼저 둔다.

## 반영 내용

### 1. 고객관리자 Shell 좌측 패널 제거

`AdminShell`에서 `AdminSidebar` 렌더링을 제거했다.

기존 `/admin/*` 페이지들이 넘기던 `navigationItems` prop은 호환을 위해 optional로 남겼다. 이 prop은 더 이상 고객관리자 Shell에서 표시하지 않는다.

### 2. 상단 이동 버튼 추가

좌측 패널을 제거했기 때문에 `AdminTopbar`에 최소 이동 버튼을 추가했다.

- 홈 → `/admin`
- 작업지시서 → `/worker`

이 버튼은 다음 단계에서 작업지시서 화면의 홈 버튼 설계와 연결될 수 있다.

### 3. 카드형 홈 registry 추가

`lib/admin/adminWorkspaceCards.ts`를 추가했다.

카드는 다음 필드를 가진다.

- `id`
- `label`
- `description`
- `permission`
- `href`
- `status`
- `statusLabel`

현재는 실제 권한 필터를 적용하지 않는다. 고객관리자 기준으로 필요한 카드를 표시한다.

### 4. 기본 관리 카드

고객관리자 기본 관리 카드는 다음으로 제한했다.

1. 협력업체관리
2. 저장소관리
3. 통계정보
4. 환경설정
5. 멤버관리 후보

작업지시서는 기본 관리 카드에서 제외하고, 별도 업무 진입 카드로 분리했다.

### 5. 권한 기반 확장 후보

아래 항목은 고객관리자 전용 메뉴로 고정하지 않고, 향후 멤버 권한에 따라 표시할 후보로 분리했다.

1. 단위표준
2. 외주공정
3. 생산품유형

## 미반영 항목

이번 버전에서는 아래 항목을 구현하지 않는다.

1. 권한 DB
2. 역할 DB
3. 멤버 초대/가입 흐름
4. 권한 관리 UI
5. `/workspace` 공통 홈
6. API 권한 검사
7. 작업지시서 화면 홈 버튼
