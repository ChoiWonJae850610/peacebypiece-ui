# 0.10.8 작업지시서 홈/워크스페이스 흐름 보정

## 목표

작업지시서 업무 화면에서 홈으로 돌아가는 흐름과 일반 멤버용 `/workspace` 홈의 카드 구성을 정리한다.

## 반영 내용

- 작업지시서 홈 버튼의 이동 대상을 역할 기준으로 계산하는 유틸을 확장했다.
- 홈 버튼 라벨을 단순 `홈` 고정값이 아니라 `관리자 홈`, `내 업무 홈`처럼 이동 대상에 맞게 표시할 수 있게 했다.
- `/workspace` 카드 정의를 컴포넌트 내부에서 분리해 `lib/navigation/memberWorkspaceCards.ts`로 중앙화했다.
- `/workspace`는 업무 화면과 개인 설정 섹션을 분리해서 보여준다.

## 현재 정책

- 관리자 역할은 작업지시서 화면에서 `/admin`으로 돌아간다.
- 디자이너/검수 등 일반 멤버 역할은 작업지시서 화면에서 `/workspace`로 돌아간다.
- 시스템관리자 전용 홈은 장기적으로 `/system`을 유지할 수 있으나, 현재 작업지시서 역할 체계에는 `system_admin`이 없으므로 실제 라우팅에는 연결하지 않았다.

## 후속 작업

- 실제 사용자/권한 DB가 들어오면 `/workspace` 카드는 권한 코드 기준으로 필터링한다.
- 장기적으로 루트 단일 AppShell을 도입할 경우, 현재 route 유틸을 내부 화면 전환 유틸로 확장한다.
