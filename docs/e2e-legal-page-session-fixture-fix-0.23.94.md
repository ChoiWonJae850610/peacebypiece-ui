# 0.23.94 E2E 약관 화면 세션 fixture 수정

## 원인

0.23.92에서 일반 멤버 workspace API 세션은 `companyId`, `companyMemberId`, 현재 승인 상태, 세션 사용자 일치를 요구하도록 강화되었습니다. 기존 약관 E2E는 회사와 멤버 식별자가 모두 `null`인 가짜 일반 멤버 세션을 사용해 현재 제품 계약과 불일치했습니다.

## 수정

약관 문서 렌더링과 동의 API 동작을 검증하는 해당 시나리오만 유효한 고객사 관리자 E2E 세션을 사용합니다. 제품의 일반 멤버 검증 정책은 약화하지 않습니다. 일반 멤버 접근은 실제 승인 멤버 fixture 또는 `/dev/test-console` 역할 전환을 이용한 별도 통합 테스트로 유지합니다.
