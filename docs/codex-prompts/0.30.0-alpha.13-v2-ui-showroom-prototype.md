## Goal

Create the first WAFL v2 `/ui` showroom prototype using mock-only UI samples. The purpose is to make the v2 design direction reviewable before DB/API/workspace implementation.

The showroom must demonstrate:

1. WAFL v2 product/style header.
2. WAFL Sheet layout sample.
3. Sheet Card variants: image/sketch, base info, fabric, accessory, factory/process, PDF/share, history.
4. Assistant / next action panel.
5. Status badges using Korean labels and internal-code hints only where useful.
6. Mobile 1-column card flow sample.
7. Form fields with mobile-safe 16px input sizing.
8. Modal/drawer/bottom-sheet visual samples if feasible without behavior-heavy implementation.
9. PDF-like Sheet preview sample.
10. Do/Don’t notes showing that the moodboard is not copied literally.
