## Objective

Connect the workorder size specification UI to system-admin size standards and improve the measurement input UI so it is usable on PC, tablet, and mobile.

## Confirmed policy

- Size specs are based on system-admin standards.
- A workorder stores a snapshot of the selected template/values.
- Later changes to system standards do not automatically mutate old workorders.
- Measurement names in UI/PDF must be Korean display names when available.
- Inch input is integer plus 1/8 fraction selection.
- The UI must use canonical WAFL components, not raw square inputs styled per-screen.

## Implementation requirements

1. Confirm whether system size/POM standards can be managed from system-admin screens.
2. Confirm how product category/sex/size-set selection resolves a template.
3. Add a clear `기준 치수 불러오기` or equivalent flow if policy and schema already support it.
4. If the management UX/schema is incomplete, document `DEFERRED` blockers and do not invent destructive schema.
5. Ensure workorder size data stores a snapshot.
6. Use Korean display names for POMs.
7. Keep cm mode compact.
8. Keep inch mode compact: integer + 1/8 selector.
9. Create a shared `WaflMeasurementInput` or equivalent if current cell inputs repeat raw layout logic.
10. Reduce per-cell visual clutter.
11. PC: sticky first column and clear horizontal scroll if many sizes/POMs.
12. Tablet/mobile: use a modal/sheet/card editing pattern that is actually usable.
13. Avoid raw `<input>`, raw `<select>`, direct border/radius/shadow styling unless canonical WAFL components require it.

## PDF connection

- cm values,

- size page only when data exists.
