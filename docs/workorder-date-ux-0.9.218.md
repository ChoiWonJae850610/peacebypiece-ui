# 0.9.218 — 작업지시서 UX 보강 1차

## 목적

작업지시서 발주 정보의 날짜 입력에서 과거 날짜가 들어가는 것을 기본적으로 막는다. 현재 작업지시서 생성 모달은 납기일을 직접 받지 않으므로, 이번 버전의 실제 적용 지점은 작업지시서 상세의 발주 정보 `dueDate` 날짜 편집 필드다.

## 적용 범위

- 작업지시서 상세 > 발주 정보 > 납기일 date input
- `input type="date"` 편집 필드의 `min` 값을 오늘 날짜로 지정
- 사용자가 브라우저/키보드 입력으로 과거 날짜를 넣는 경우 오늘 날짜로 보정

## 정책

1. 기본 정책
   - 오늘 이전 날짜는 선택하지 못하게 한다.
   - 과거 날짜가 입력되면 오늘 날짜로 보정한다.

2. 관리자 예외 입력
   - 이번 버전에서는 별도 예외 모드를 만들지 않는다.
   - 기존 자료 입력/마이그레이션용 과거 날짜 허용은 별도 정책과 권한 UI가 필요하므로 후속 버전으로 분리한다.

3. 기존 데이터
   - 이미 저장된 과거 납기일을 강제로 수정하지 않는다.
   - 사용자가 해당 날짜 필드를 다시 편집할 때부터 오늘 이후 날짜 기준을 적용한다.

## 구현 기준

- 날짜 정책은 `lib/workorder/datePolicy.ts`로 분리한다.
- 상세 편집 UI는 정책 유틸만 호출한다.
- DB schema, API route, repository 저장 로직은 변경하지 않는다.

## SQL DDL 필요 여부

불필요.

## 전체 리셋 필요 여부

불필요.

## 테스트 케이스

1. 작업지시서 상세 화면에서 발주 정보의 납기일을 클릭한다.
2. 날짜 선택기의 오늘 이전 날짜가 선택 불가인지 확인한다.
3. 브라우저가 수동 과거 날짜 입력을 허용하더라도 저장값이 오늘 날짜보다 과거로 내려가지 않는지 확인한다.
4. 오늘/미래 날짜는 정상 저장되는지 확인한다.
5. 기존 자료 입력용 과거 날짜 예외 모드는 아직 제공되지 않는지 확인한다.
