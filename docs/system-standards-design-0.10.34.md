# 0.10.34 시스템관리자 기준정보 관리 화면 설계

## 목표

고객관리자 환경설정에서 단위 표준과 외주공정 유형을 시스템 표준 선택형으로 전환한 뒤, 시스템관리자가 관리할 표준 기준정보 화면의 방향을 먼저 설계한다.

## 화면

- `/system/standards` 추가
- 시스템 콘솔 카드와 바로가기에서 기준정보 관리 화면 연결

## 정책

### 단위 표준

- 시스템관리자가 한글명과 영문 코드/약어를 관리한다.
- 고객관리자는 필요한 단위만 사용/미사용으로 선택한다.
- 고객관리자 자유 입력은 통계와 발주 수량 해석을 흔들 수 있으므로 제한한다.

### 외주공정 유형

- 시스템관리자가 표준 공정명을 관리한다.
- 고객관리자는 필요한 외주공정만 사용/미사용으로 선택한다.
- 프린팅/프린트/나염처럼 같은 의미의 중복 항목을 줄이는 것이 목표다.

### 생산품 유형

- 고객사별 구조 차이가 크므로 고객관리자 직접 관리를 유지한다.
- 시스템관리자는 신규 고객사 생성 시 복사할 기본 템플릿만 관리하는 방향으로 둔다.

## 이번 버전 범위

- 설계 화면만 추가
- 실제 CRUD 없음
- DB schema 변경 없음
- 고객관리자 기준정보 저장 로직 변경 없음
- 감사 로그 흐름 변경 없음
