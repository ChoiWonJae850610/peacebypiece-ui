# 0.9.163 작업지시서 삭제 시 연결 파일/메모 휴지통 이동

## 목표

작업지시서를 삭제할 때 작업지시서만 숨기지 않고, 해당 작업지시서에 연결된 디자인 파일, 일반 첨부파일, 메모 첨부로 쓰이는 파일까지 같은 휴지통 정책으로 이동한다.

## 적용 정책

- 작업지시서는 soft delete 상태로 전환한다.
- 연결된 attachments row는 `is_active = false`, `deleted_at = now()` 기준으로 휴지통 상태로 전환한다.
- 연결 파일은 `attachment_trash_items`에 등록한다.
- R2 원본과 썸네일은 즉시 삭제하지 않는다.
- 삭제된 파일은 30일 복원 가능 정책을 따른다.
- 30일 경과 후 시스템관리자 `/system/storage-usage` purge 후보가 된다.
- 메모 본문은 작업지시서 삭제 상태와 함께 숨기기 위해 `memos` row를 soft delete 처리한다.

## 구현 범위

- `deleteDbWorkOrder`에서 작업지시서 삭제 성공 후 연결 attachments/memos를 soft delete 처리한다.
- 기존 개별 첨부 삭제 API와 동일하게 R2 물리 삭제는 수행하지 않는다.
- DB schema는 변경하지 않는다.

## 제외 범위

- 작업지시서 복원 기능
- 고객관리자 저장소 탭 구조 개편
- R2 purge 자동화
- 리오더 시 파일 복사 정책
