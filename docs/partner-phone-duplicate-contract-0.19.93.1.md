# 0.19.93.1 업체 연락처 필수/중복 기준 보정

## 목적

협력업체 등록/수정 시 같은 회사 안에서 업체명만 같은 경우는 허용하되, 업체명과 연락처가 모두 같은 경우는 중복으로 차단한다.

## 적용 기준

- 연락처는 필수 입력값이다.
- 연락처는 저장 전 숫자만 남긴 값으로 정규화한다.
- 같은 `company_id` 안에서 `업체명 + 정규화 연락처` 조합이 이미 있으면 등록/수정을 차단한다.
- 기존 업체 수정 시 자기 자신은 중복 비교에서 제외한다.
- 이름이 같아도 연락처가 다르면 별도 업체로 등록할 수 있다.

## 적용 범위

- 클라이언트 모달 1차 검증
- `/api/admin/partners` POST/PATCH 서버 검증
- `scripts/smoke-db-api.mjs` 파트너 이름/연락처 계약 smoke 검증

## 제외 범위

- DB schema 변경 없음
- `full_reset.sql` 변경 없음
- 고유 인덱스 추가 없음

DB 레벨 unique index는 기존 데이터 정리와 연락처 정규화 컬럼 설계가 필요하므로 이번 버전에서는 API/테스트 계약으로 먼저 고정한다.
