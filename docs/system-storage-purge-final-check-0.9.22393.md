# 0.9.22393 — 시스템관리자 실제 삭제 후보 최종 점검

## 목적

`/system/storage-usage` 실제 삭제 후보 화면을 운영자가 현실적으로 확인할 수 있는 범위로 정리한다. 정상 삭제 흐름은 화면에서 직접 확인하고, R2 object not found·Worker 실패·DB 업데이트 실패 같은 예외는 실제 운영 데이터로 강제 재현하지 않고 코드 처리 기준으로 점검한다.

## 이번 버전 반영 내용

- 시스템관리자 실제 삭제 후보 화면 안내 문구를 `정상 흐름 화면 확인 + 예외 로직 코드 확인` 기준으로 정리했다.
- R2 Worker 삭제 응답에서 404 또는 not found 계열 응답이 오면 이미 삭제된 객체로 보고 성공 처리한다.
- Worker 인증 실패, 권한 실패, 네트워크 실패, timeout, DB 업데이트 실패는 실패 상태로 남겨 재시도 후보가 되도록 기존 실패 흐름을 유지한다.
- 작업지시서 후보 row의 보조 문구를 실패 항목만 재시도한다는 기준으로 정리했다.
- 시스템관리자 삭제 후보 표 헤더에서 `예정일` 정렬 버튼이 중복 표시되던 부분을 제거했다.

## 실제 화면에서 확인할 항목

1. 고객관리자 `/admin/files`에서 첨부파일을 선택 삭제한다.
2. 시스템관리자 `/system/storage-usage`에 해당 첨부파일이 실제 삭제 후보로 표시되는지 확인한다.
3. 시스템관리자에서 개별 후보를 선택 삭제한다.
4. 정상 처리 후 후보 목록에서 사라지는지 확인한다.
5. 고객관리자가 아직 선택 삭제하지 않은 pending 항목은 시스템관리자 후보에 표시되지 않는지 확인한다.
6. 작업지시서를 선택 삭제한 뒤 시스템관리자 후보에 작업지시서 대표 row가 표시되는지 확인한다.
7. 작업지시서 대표 row에 첨부파일 개수, 메모 개수, 총 용량이 표시되는지 확인한다.
8. 전체삭제 버튼이 삭제 예정일 도래 또는 삭제 요청 후보만 처리하는지 확인한다.

## 코드상으로 확인할 항목

- R2 삭제 성공 응답이면 DB purge 완료 처리로 이어진다.
- R2 object not found 계열 응답은 이미 삭제된 객체로 보고 성공 처리한다.
- Worker 인증 실패, 권한 실패, 네트워크 실패, timeout은 실패 처리되어 재시도 후보로 남는다.
- R2 삭제 후 DB purge 업데이트가 실패하면 후보가 정상 완료로 사라지지 않는다.
- 작업지시서 묶음 첨부파일 중 정상 처리된 항목은 다시 후보로 노출되지 않고, 실패한 항목만 재시도 후보로 남는다.
