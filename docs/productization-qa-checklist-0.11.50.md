# 제품화 전 1차 QA 체크리스트 — 0.11.50

## 목적

PeaceByPiece 0.11.x 라인에서 정리된 관리자/시스템관리자/작업지시서 핵심 기능을 기준으로 제품화 전 1차 QA 범위를 고정한다.

이번 문서는 테스트 기준을 정리하는 문서이며, 화면 동작·DB schema·API·R2 처리 흐름은 변경하지 않는다.

## 공통 QA 원칙

- `npm run build`는 로컬에서 실행한다.
- `.env.local`은 로컬/배포 환경에서만 관리한다.
- 실제 DB URL, R2 secret, Worker token은 코드와 문서에 기록하지 않는다.
- 테스트 전 repo-state가 clean인지 확인한다.
- 테스트 전 `APP_VERSION`이 `0.11.50`으로 표시되는지 확인한다.
- DB mode 화면에서 `DB 드라이버 없음`이 다시 표시되면 `lib/db/client.ts`와 dependency 설치 상태를 먼저 확인한다.
- 화면 오류와 데이터 오류를 분리해서 기록한다.
- 모바일/태블릿 테스트는 PC 기능 확인 후 진행한다.

## 1. 빌드 및 실행

### 확인 항목

- `npm install` 또는 기존 dependency 설치 상태 확인
- `npm run build` 성공 여부
- Next.js static generation 중 runtime warning과 build failure 분리
- `npm run dev` 실행 후 `/`, `/workspace`, `/admin`, `/system` 접근 가능 여부
- Vercel 배포 시 환경변수 누락 여부

### 실패 시 우선 확인

- `pg` dependency 로딩 실패 여부
- Neon DB 연결 문자열 SSL mode 경고와 실제 연결 실패 구분
- R2 Worker URL/env key 누락 여부
- i18n resource export 누락 여부
- client/server component import 오류 여부

## 12. 배포 전 차단 기준

아래 항목이 있으면 제품화 전 수정 대상이다.

- `npm run build` 실패
- 작업지시서 목록 DB 로딩 실패
- `DB 드라이버 없음` 재발
- 작업지시서 저장 실패
- 첨부 업로드/다운로드 실패
- R2 삭제/purge 실패
- 고객관리자 영구삭제 요청이 실제 삭제로 처리되는 문제
- 시스템관리자 purge 후보가 표시되지 않는 문제
- 모바일에서 주요 action button 접근 불가
- 고객 노출 화면에 내부 개발용 문구 노출
- i18n resource 누락으로 key가 그대로 노출
