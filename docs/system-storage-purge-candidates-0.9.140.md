# 0.9.140 시스템관리자 R2 실제 삭제 후보 화면 1차

## 목적

전 고객 공통 30일 휴지통 정책을 기준으로, 시스템관리자가 R2 실제 삭제 후보를 확인할 수 있는 화면을 추가한다.

이번 버전은 후보 확인 1차이며 실제 R2 삭제 실행 버튼은 연결하지 않는다.

## 정책 기준

- 고객사 관리자가 파일을 삭제하면 R2 원본과 썸네일은 즉시 삭제하지 않는다.
- 삭제 파일은 휴지통 상태로 전환된다.
- 삭제일로부터 30일이 지나면 R2 실제 삭제 후보가 된다.
- 고객사 관리자가 영구삭제 요청을 한 항목도 R2 실제 삭제 후보에 포함한다.
- 실제 R2 삭제 시에는 원본 `storage_key`와 썸네일 `thumbnail_key`를 함께 삭제해야 한다.
- R2 실제 삭제는 Cloudflare Worker 기반 흐름을 유지한다.

## 추가 화면

### `/system/storage-usage`

시스템관리자 전용 스토리지 정리 화면이다.

표시 항목:

- 삭제 후보 수
- 후보가 있는 고객사 수
- 원본 용량 합계
- 썸네일 key 보유 파일 수
- 고객사명
- 작업지시서명
- 파일명
- 삭제일
- 삭제 예정일
- 원본 storage key
- 썸네일 key
- purge 상태
- 실패 사유

## 0.9.140에서 하지 않은 것

- R2 실제 삭제 실행 버튼 연결
- 선택 삭제 처리
- 전체 삭제 처리
- 실패 항목 재시도 처리
- DB schema 변경
- 자동 purge cron 연결

## 다음 단계

0.9.141에서 다음을 검토한다.

1. 후보 목록에서 선택 항목 삭제 확인 모달 추가
2. 전체 도래 항목 삭제 확인 모달 추가
3. Worker purge 실행 API 연결
4. 삭제 성공/실패 결과 표시
5. 실패 항목 재시도 UX 정리

## 테스트 기준

1. `/system/storage-usage`에 진입한다.
2. 화면이 runtime error 없이 표시되는지 확인한다.
3. 30일 경과 또는 영구삭제 요청된 휴지통 파일이 있으면 목록에 표시되는지 확인한다.
4. 원본 key와 썸네일 key가 함께 표시되는지 확인한다.
5. 삭제 실행 버튼이 아직 비활성 상태인지 확인한다.
6. `/system` 콘솔의 스토리지 영역에서 후보 목록 화면으로 이동되는지 확인한다.
