# 0.9.224321 — 관리자 통계 리오더 TOP5 기준 보정

## 목적

`0.9.22432`에서 정리한 리오더 TOP5 기준을 리오더 작업지시서 수가 아니라 리오더 차수 기준으로 보정한다.

## 변경 기준

- 기존: 2차 이상 리오더 작업지시서 수 기준
- 변경: `reorder_round`의 최댓값 기준
- 동일 차수 보조 정렬: 해당 리오더 묶음의 발주수량 합계 DESC
- 표시 단위: `건`이 아니라 `차`

## SQL 기준

리오더 묶음 key는 아래 우선순위로 계산한다.

1. `reorder_group_id`
2. `parent_spec_sheet_id`
3. `id`

리오더 차수는 아래 기준으로 계산한다.

1. `reorder_round`
2. `parent_spec_sheet_id`가 있으면 최소 2차로 보정

`work_order_kind = 'rework'`는 리오더에서 제외한다.

## DB 변경

없음.

## 후속 확인

- `/admin/dashboard`에서 기간 요약의 리오더 카드를 클릭한다.
- 동적 TOP5 카드 제목이 리오더 차수 기준으로 표시되는지 확인한다.
- 값 단위가 `건`이 아니라 `차`로 표시되는지 확인한다.
- 같은 리오더 차수에서는 발주수량이 큰 항목이 우선 표시되는지 확인한다.
