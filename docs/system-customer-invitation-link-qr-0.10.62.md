# 0.10.62 시스템관리자 고객사 초대 링크/QR 생성 화면

## 목표

시스템관리자가 신규 고객사 담당자에게 초대 링크와 QR을 전달할 수 있도록 `/system/invites` 화면의 입력, 미리보기, 결과 액션, 승인 정책 구조를 정리한다.

이번 버전은 실제 발송 기능을 만들지 않는다. 이메일, SMS, 카카오 자동 발송은 후순위이며, 1차 방식은 링크 복사와 QR을 사용자가 직접 공유하는 방식이다.

## 적용 범위

- `/system/invites` 화면
- 시스템관리자 고객사 초대 정책 presentation
- 고객사 초대 QR preview 문구
- `APP_VERSION` 0.10.62 반영

## 정책

1. 시스템관리자 초대는 `system_to_company_admin` 범위다.
2. 고객사 담당자는 초대 링크로 `/invite/company/[token]`에 접근한다.
3. 가입 신청만으로 `companies`를 생성하지 않는다.
4. 시스템관리자가 신청을 검토하고 승인할 때 고객사와 고객관리자 멤버십을 생성한다.
5. 승인 시 `company_members`와 `member_permissions`를 함께 확정한다.
6. role은 기본 권한 묶음 또는 표시용이고 실제 접근 제어는 `permission_code` 기준이다.
7. raw token은 DB에 저장하지 않고 `token_hash`만 저장한다.
8. 고객사 생성 시 0.10.51에서 설계한 초기 기준정보 복사 흐름과 연결한다.

## 화면 구성

- 상단 타이틀 카드
- 초대 유형/역할/만료/토큰 저장 정책 요약 카드
- 고객사명, 사업자명, 담당자명, 담당자 이메일, 요금제, 저장공간 입력 자리
- 초대 생성 흐름 카드
- 초대 링크/QR preview
- 초대 링크 생성/링크 복사/가입 신청 화면 미리보기 액션 자리
- 승인 기준 카드
- 정책 메모

## 후속 작업

- `/invite/company/[token]` 고객사 가입 신청 화면
- 초대 생성 API 연결
- token_hash DB 조회
- join_requests 저장
- 시스템관리자 고객사 승인 화면
- 고객사 생성 시 초기 기준정보 복사 실행 연결
