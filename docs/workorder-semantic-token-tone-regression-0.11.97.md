# 작업지시서 semantic token 강도 회귀 보정 — 0.11.97

## 목적

0.11.91~0.11.96에서 작업지시서 화면의 입력 가능/선택 가능/계산/선택 카드/empty state 의미 토큰을 1차 적용했다. 0.11.97에서는 기능이나 구조를 바꾸지 않고, 실제 화면에서 과하게 보이는 색상 강도만 낮춘다.

## 보정 기준

- 선택된 작업지시서 카드는 검은 면 강조가 아니라 옅은 selected surface와 약한 border로 표현한다.
- 입력 가능 필드는 버튼처럼 보이지 않도록 배경과 inset border 강도를 낮춘다.
- 선택 가능 필드는 입력 필드와 구분하되, 행 전체를 과하게 나누지 않도록 border 강도를 낮춘다.
- 계산 필드는 읽기 전용 의미가 유지되도록 neutral tone을 유지하되, 테두리 강도는 낮춘다.
- 모바일 생산구성 카드는 카드 배경과 내부 field tone이 서로 과하게 충돌하지 않도록 바깥 카드 배경을 더 조용하게 둔다.

## 변경 범위

- app/globals.css의 semantic token 기본값과 semantic class 강도만 조정했다.
- 작업지시서 데이터 처리, 저장, 삭제, 첨부, 메모, R2 흐름은 변경하지 않았다.
- 실제 테마 선택 UI는 변경하지 않았다.

## 확인 항목

- 좌측 작업지시서 선택 카드가 과하게 검은 박스처럼 보이지 않는지 확인한다.
- 생산구성 입력/선택 field가 버튼처럼 과하게 보이지 않는지 확인한다.
- 계산 field는 여전히 입력 불가/계산 값으로 인지되는지 확인한다.
- 모바일 생산구성 카드에서 field tone이 과하게 복잡해 보이지 않는지 확인한다.
