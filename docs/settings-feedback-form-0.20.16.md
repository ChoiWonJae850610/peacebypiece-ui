# 0.20.16 환경설정 서비스 건의 UI 재정리

## 목적

환경설정의 서비스 건의 탭을 설명 카드 중심에서 실제 접수 폼 중심으로 정리한다.

기존 화면은 기능 건의, 오류 제보, 개선 요청을 각각 카드로 보여주고 운영 기준을 함께 노출해 실제 사용자가 무엇을 해야 하는지 명확하지 않았다. 0.20.16에서는 고객사 관리자가 바로 내용을 작성하고 이메일 접수로 넘어갈 수 있도록 화면 밀도를 낮춘다.

## 변경 기준

- 서비스 건의 탭은 폼 + 접수 이력 구조로 구성한다.
- 문의 유형은 기능 건의 / 오류 제보 / 개선 요청으로 선택한다.
- 제목과 내용을 입력한 뒤 이메일 작성 화면으로 연결한다.
- 정식 접수 API가 생기기 전까지는 이메일 접수 모드로 유지한다.
- 접수 이력 영역은 미연결 상태를 명확히 표시하고, 향후 API 연결 위치로 남긴다.

## 화면 구조

```txt
서비스 건의

[접수 내용]
문의 유형  [기능 건의 v]
제목       [                    ]
내용       [                    ]
           [                    ]

[초기화] [이메일 작성하기]

[접수 이력]
아직 화면에 표시할 접수 이력이 없습니다.
현재는 이메일 접수만 지원합니다.
```

## 제거 또는 축소한 요소

- 기능 건의 / 오류 제보 / 개선 요청 반복 카드
- 적용 예정 카드
- 운영 기준형 긴 안내
- 버튼 하나만 담은 카드형 액션

## 유지한 요소

- 접수 이메일 주소 표시
- 이메일 작성 연결
- 향후 정식 접수 API 확장 가능성
- 접수 이력 표시 위치

## 후속 작업

- 서비스 건의 접수 DB/API가 필요해지면 접수 이력 영역에 연결한다.
- 시스템관리자 운영 화면에서 고객사 건의 목록을 처리하는 기능은 별도 버전으로 분리한다.
- 첨부 파일 접수는 이메일 첨부 우선으로 두고, 앱 내부 업로드는 저장공간 정책과 함께 검토한다.
