# 0.23.11 원단·부자재 발주 저장 피드백 통합

## 작업 목적

원단·부자재 발주 화면의 자재 종류, 공급처, 납기일 저장 흐름이 서로 다른 문구와 토스트 호출 방식을 사용하던 문제를 정리합니다.

## 변경 내용

- 자재 종류, 공급처, 납기일 저장 피드백을 `WAFL_SAVE_TARGET` 공통 메시지 정책으로 통일했습니다.
- 세 항목 모두 저장 시작 시 loading 토스트를 표시하고, 같은 토스트 ID에서 success 또는 danger 상태로 교체되도록 정리했습니다.
- 자재 종류 저장 성공 시 사용하던 별도 하드코딩 문구를 제거했습니다.
- 납기일 저장에 누락되어 있던 loading 토스트를 추가했습니다.
- 화면에 실제로 렌더링되지 않던 `headerSaveStatus`, `headerSaveMessage` 상태와 props를 제거했습니다.
- 저장 실패 시 API 오류 메시지가 있으면 그대로 사용하고, 없으면 공통 저장 실패 문구를 사용합니다.

## 영향 범위

- 원단·부자재 발주 화면의 헤더 저장 UX
- DB 스키마 및 API 계약 변경 없음
