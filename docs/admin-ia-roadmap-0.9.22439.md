# 0.9.22439 — 관리자 화면 구조 개편 로드맵

## 목적

작업지시서, 저장소/휴지통, 시스템관리자 실제 삭제 흐름이 1차 안정 기준점에 도달한 뒤 진행할 관리자 화면 구조 개편 순서를 정리한다.

이 문서는 구현 문서가 아니라 다음 버전대 설계/작업 순서 기준이다.

## 기본 방향

### 고객관리자

고객관리자는 업무 진입이 쉬운 구조를 우선한다.

1. `/admin`은 오늘 처리할 일과 주요 기능 카드 중심의 허브로 정리한다.
2. `/admin/dashboard`는 통계정보 화면으로 분리 유지한다.
3. `/admin/files`는 저장소/휴지통 운영 관리 화면으로 유지한다.
4. `/admin/settings`는 언어, 파일정책, 저장소정책, 알림정책, 화면설정으로 재정리한다.
5. 히스토리 메뉴는 고객관리자 좌측 메뉴에서 제거하는 방향을 우선 검토한다.
6. 고객관리자에게 필요한 최소 변경 이력은 각 업무 화면 안의 보조 정보로 제한한다.

### 시스템관리자

시스템관리자는 운영, 감사, 요금제, 용량 관리 중심으로 유지한다.

1. 시스템관리자 좌측 패널은 유지한다.
2. 고객관리자 히스토리 메뉴에서 빠지는 내용은 시스템관리자 로그/감사 로그로 이동한다.
3. 저장소 실제 삭제, 요금제, 용량 override, 고객별 운영 상태를 시스템관리자 영역에서 관리한다.
4. 감사 로그는 고객관리자용 안내 이력과 분리한다.

### 작업지시서 업무 화면

작업지시서 화면은 업무 집중형 화면으로 유지한다.

1. 작업지시서 상세 작업 중 좌측 패널 의존도를 낮춘다.
2. PC에서는 현재 업무 흐름을 깨지 않는 범위에서 화면 폭과 패널 구조를 정리한다.
3. 모바일/태블릿에서는 대시보드 카드형 진입과 상단 이동을 우선 검토한다.

## 권장 작업 순서

### 0.9.224391 — 관리자 IA 재정리 설계

목표:
고객관리자, 시스템관리자, 작업자 화면의 메뉴 구조와 화면 이동 기준을 확정한다.

검토 항목:

1. 고객관리자 좌측 패널 유지/축소/제거 여부
2. 고객관리자 대시보드 카드형 이동 구조
3. `/admin`과 `/admin/dashboard` 역할 분리
4. 저장소 관리 접근 위치
5. 환경설정 접근 위치
6. 작업지시서 업무 화면의 독립 화면 유지 여부
7. 모바일/태블릿 진입 구조

완료 기준:
메뉴 중심으로 갈지, 대시보드 카드 중심으로 갈지 결정한다.

### 0.9.224392 — 히스토리 메뉴 제거 및 시스템 로그 설계

목표:
고객관리자 좌측의 히스토리 메뉴를 제거하고 시스템관리자 감사 로그 구조를 설계한다.

검토 항목:

1. 기존 히스토리 화면 사용처
2. 고객관리자에게 남길 최소 이력 범위
3. 시스템관리자 로그 화면 경로
4. 감사 로그 대상 이벤트
5. audit log DB 테이블 필요 여부
6. 권한/초대/요금제 변경 로그 포함 여부

완료 기준:
히스토리를 고객관리자 메뉴가 아니라 시스템관리자 감사 로그로 이동하는 방향을 확정한다.

### 0.9.224393 — 환경설정 화면 구조 재정리 설계

목표:
환경설정 화면을 고객관리자가 이해하기 쉬운 섹션 또는 탭 구조로 재정리한다.

검토 항목:

1. 언어 설정
2. 파일 업로드 정책
3. 저장소 정책
4. 알림 정책
5. 화면 표시 설정
6. 고객관리자 설정과 시스템관리자 설정의 경계
7. i18n key 구조

완료 기준:
긴 설정 화면을 분리할 기준을 확정한다.

### 0.9.224394 — 고객관리자 좌측 패널 제거/축소 설계

목표:
고객관리자 화면에서 좌측 패널을 유지할지, 축소 rail로 바꿀지, 대시보드 카드형 이동으로 대체할지 결정한다.

추천안:

1. PC: 좌측 패널을 완전 제거하지 않고 축소형 rail 또는 약한 메뉴로 전환한다.
2. 모바일/태블릿: 좌측 패널 숨김, 대시보드 카드/상단 메뉴 중심으로 전환한다.
3. 작업지시서 상세: 업무 집중형 독립 화면으로 유지한다.
4. 시스템관리자: 좌측 패널 유지한다.

완료 기준:
고객관리자 IA 방향을 확정한다.

### 0.9.224395 — 고객관리자 대시보드 카드형 이동 1차 설계

목표:
`/admin`을 좌측 메뉴 의존도를 줄이는 카드형 진입 허브로 설계한다.

카드 후보:

1. 오늘 처리할 작업지시서
2. 검토 요청
3. 발주 요청
4. 검수 대기
5. 저장소 관리
6. 통계정보
7. 협력업체 관리
8. 환경설정

완료 기준:
고객관리자가 좌측 메뉴 없이도 주요 기능으로 이동할 수 있는 구조를 확정한다.

### 0.9.22440 — 시스템관리자 요금제/용량 관리 설계

목표:
저장소 용량 기준을 하드코딩이 아니라 plan/company 기준으로 관리할 구조를 설계한다.

검토 항목:

1. `companies`, `company_settings`, plan 관련 DB 구조
2. 요금제별 저장 용량 기준
3. 고객별 plan override
4. 저장소 화면 5GB 하드코딩 제거
5. 시스템관리자 고객별 요금제 변경 화면
6. SQL 변경 필요 여부

완료 기준:
저장소 용량 기준이 plan/company 기반으로 이동할 수 있어야 한다.

## 설계 원칙

1. 고객관리자 화면은 업무 진입과 이해도를 우선한다.
2. 시스템관리자 화면은 운영, 감사, 정책 관리를 우선한다.
3. 통계정보는 고객관리자의 경영/운영 판단 화면으로 유지한다.
4. 히스토리는 고객관리자 메뉴가 아니라 시스템관리자 감사 로그로 이동한다.
5. 화면 구조 변경과 DB schema 변경은 한 패치에서 무리하게 섞지 않는다.
6. i18n key와 공통 표시 함수는 구조 변경 전에 기준을 먼저 정리한다.
