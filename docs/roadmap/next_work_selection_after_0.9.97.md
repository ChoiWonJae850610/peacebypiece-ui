# 0.9.97 이후 다음 작업 선택 기준

Version: 0.9.97

## 기본 선택

다음 작업은 아래가 적절하다.

```text
0.9.98 — 파일 관리 화면 read-only 복원
```

## 이유

0.9.93에서 `/admin/files`는 회귀 점검 화면으로 대체되었다.  
파일 관리는 실제 운영에서 바로 필요한 화면이고, 첨부 업로드/삭제/표시 흐름을 건드리지 않고도 read-only 화면부터 복원할 수 있다.

## 0.9.98 요청문

```text
깃허브 읽고 0.9.98 작업해줘.
목표:
관리자 파일 관리 화면 read-only 복원.
0.9.93에서 안정화 화면으로 대체한 /admin/files를 기준으로, components/admin/files 하위 컴포넌트의 JSX 손상 여부를 확인하고 build 가능한 read-only 파일 관리 화면으로 복원해줘.
첨부 업로드/삭제/다운로드/R2/DB 저장 흐름은 건드리지마.
package.json, package-lock.json, .env.local은 수정하지마.
결과는 압축파일로 줘.
```

## build 실패가 있으면

0.9.98로 기능 복원을 하지 말고 아래처럼 요청한다.

```text
0.9.96 또는 0.9.97 적용 후 npm run build 에러가 난다.
에러 캡처 기준으로 0.9.98 빌드 수정 작업해줘.
```


### 요금제/용량 화면을 먼저 확인하고 싶을 때


### 거래처/공장관리 복원이 급할 때

