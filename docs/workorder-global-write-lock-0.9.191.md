# 작업지시서 전역 쓰기 잠금 기준 (0.9.191)

## 목적

작업지시서 상태 전환 중에는 DB 저장, hydrate, workflow 재계산이 동시에 진행될 수 있다.
이 구간에서 업로드, 메모 등록, 삭제, 새 작업지시서 생성, 새로고침 같은 쓰기 액션이 동시에 실행되면 화면 상태와 DB 상태가 어긋날 수 있다.

0.9.191에서는 작업지시서 화면부터 상태 전환 중 전역 쓰기 잠금 기준을 적용한다.

## 0.9.191 적용 범위

상태 전환 처리 중 잠금 대상:

- 작업지시서 생성 버튼
- 새로고침 버튼
- 설정 버튼
- 작업지시서 목록 액션: 삭제, 리오더, 재작업
- 상세 header 쓰기 액션: 제목 변경, 기본정보 수정, 담당자 변경, 재고 편집
- 상세 inline edit 및 row 추가/삭제
- 디자인/첨부 추가 메뉴
- 디자인/첨부 drag-and-drop 업로드
- 디자인/첨부 삭제
- 대표 이미지 지정
- 메모/댓글 등록, 수정, 삭제
- 숨겨진 파일 input change 처리

읽기 액션은 유지한다.

- 화면 스크롤
- 작업지시서 내용 확인
- 파일 preview
- 섹션 접기/펼치기

## 확장 기준

이 기준은 추후 저장/업로드/삭제가 있는 다른 화면에도 같은 방식으로 확장한다.

확장 후보:

- 관리자 저장소 휴지통
- 관리자 협력업체 관리
- 관리자 설정 모달
- 시스템관리자 저장소 purge 화면
- 초대/권한 관리 화면

## 구현 원칙

- 화면마다 별도 조건을 만들지 않고 `writeLocked` 성격의 단일 플래그를 전달한다.
- 쓰기 액션 버튼은 disabled 처리한다.
- drag-and-drop, hidden file input 등 버튼 외 진입점도 같이 차단한다.
- 상태 전환 완료 후 자동 해제한다.
- 패키지 추가 없이 기존 Tailwind/React 상태로 처리한다.
