# 저장소/휴지통 actionFlow 책임 분리 2차 — 0.9.22396

## 목표

0.9.22395에서 중앙화한 휴지통 결과 메시지 흐름을 기준으로, `/admin/files` 화면의 선택 항목 해석 책임을 `page.tsx`에서 `adminFiles.actionFlow.ts`로 이동한다.

## 변경 내용

- `runTrashSelectionActionFlow`를 추가해 복원/선택 삭제가 같은 actionFlow를 타도록 정리했다.
- `runPurgeAllTrashItemsFlow`를 추가해 휴지통 비우기도 같은 summary/message formatter 기준을 사용하도록 정리했다.
- `page.tsx`는 선택 ID와 클릭 이벤트만 넘기고, 실제 파일 대상 필터링과 작업지시서 ID 정규화는 actionFlow에서 처리하도록 축소했다.
- 상세 모달의 단일 파일 복원/선택 삭제가 현재 선택된 작업지시서와 섞이지 않도록 단일 항목 액션의 대상 범위를 명시했다.
- 고객관리자 저장소 화면에 남아 있던 `연결 첨부` 표현을 문서·디자인·메모 기준 문구로 정리했다.

## DB 변경

없음.

## 확인 기준

1. 상단 복원/선택 삭제 버튼이 선택된 파일과 작업지시서를 함께 처리한다.
2. 상세 모달에서 파일 단독 복원/선택 삭제를 실행해도 기존 선택된 작업지시서가 같이 처리되지 않는다.
3. 휴지통 비우기가 작업지시서와 파일을 같은 summary formatter 기준으로 표시한다.
4. 고객관리자 저장소 화면에 `연결 첨부` 표현이 남지 않는다.
5. 결과 메시지는 작업지시서 / 문서 / 디자인 / 메모 단위로 표시된다.
