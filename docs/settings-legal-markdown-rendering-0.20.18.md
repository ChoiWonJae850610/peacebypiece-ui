# 0.20.18 환경설정 약관·정책 Markdown 원문 렌더링 1차

## 목적
환경설정의 약관·정책 문서 보기 모달을 고정 샘플 섹션이 아니라 `docs/정책문서/고객공개` 하위 Markdown 원문을 읽어 보여주는 구조로 전환한다.

## 적용 범위
- 환경설정 > 약관·정책 탭
- 고객 공개 대표 문서 5개
- 문서 보기 모달
- Playwright 환경설정 smoke 테스트

## 고객 공개 대표 문서 기준
현재 화면에는 고객이 직접 확인해야 하는 대표 문서 5개를 유지한다.

1. 이용약관
2. 개인정보처리방침
3. 요금·환불정책
4. 데이터 보관·삭제정책
5. 서비스 운영정책

`docs/정책문서/고객공개` 하위의 나머지 보조 문서는 내부 세부 기준으로 관리한다. 화면의 공개 문서 수와 파일 수를 반드시 1:1로 맞추지 않는다.

## 구현 기준
- 문서 목록은 기존 `CUSTOMER_POLICY_DOCUMENTS`를 유지한다.
- 각 문서는 `sourceFileName`으로 Markdown 파일명을 가진다.
- `GET /api/policies/customer-documents/[documentId]`가 서버에서 Markdown 원문을 읽어 반환한다.
- 클라이언트 모달은 API 응답의 `markdown`을 문서형 레이아웃으로 렌더링한다.
- Markdown 원문을 읽지 못하면 모달 안에 실패 안내를 표시한다.

## 보안 기준
- API는 `documentId`로 허용된 문서만 조회한다.
- 파일명은 서버의 정적 매핑에서만 가져온다.
- 임의 경로 접근을 막기 위해 resolve된 경로가 고객 공개 정책 문서 폴더 하위인지 확인한다.

## 후속 작업
- 정책 문서 DB 연결 시점에는 현재 파일 기반 API를 DB/API 기반으로 교체한다.
- 필요하면 고객 공개 문서 5개와 내부 보조 문서 3개의 관계를 별도 관리 화면에서 정리한다.
