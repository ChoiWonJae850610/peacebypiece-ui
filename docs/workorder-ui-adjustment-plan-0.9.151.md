# 작업지시서 UI 수정사항 수집/분류 0.9.151

## 목적

0.9.148~0.9.150에서 작업지시서 detail의 container, presentation, editor 경계를 먼저 정리했다. 0.9.151은 실제 UI 변경을 바로 크게 적용하지 않고, PC 기준 수정 후보를 화면 영역별로 분류해 이후 버전에서 안전하게 적용하기 위한 기준을 만든다.

## 현재 원칙

- 0.9.77 복구 라인의 살아 있는 작업지시서 UI를 기준으로 유지한다.
- PC 화면을 먼저 정리한다.
- 모바일/태블릿은 큰 구조 변경을 보류하고 깨지지 않는 수준만 유지한다.
- 작업지시서 상태 변경, 첨부, 메모, R2 purge, 관리자 설정 안정화 기능은 변경하지 않는다.
- UI 변경과 도메인 로직 변경을 한 버전에 섞지 않는다.

## 영역별 수정 후보

### 1. 작업지시서 workspace shell

즉시 수정 가능 후보:

- 좌측 목록 패널과 중앙 상세 패널의 하단 라인 높이 맞춤
- 좌측/중앙 패널의 상단 여백과 카드 간격 통일
- PC 기준 패널 폭 기준 문서화
- 다른 관리자 화면과 비교해 과하게 튀는 border, shadow, padding 정리

주의:

- 목록 선택, 필터, 상태 이동 로직은 변경하지 않는다.
- 모바일/태블릿 전환 조건은 건드리지 않는다.

### 2. 작업지시서 상세 header/action 영역

즉시 수정 가능 후보:

- 작업지시서명, 상태 배지, 담당자 표시 간격 정리
- 주요 action 버튼 높이와 간격 통일
- 상태 버튼의 disabled/loading 표시 일관화
- PC 기준 header line-height와 버튼 영역 높이 정리

구조 정리 필요 후보:

- 상태별 action 노출 조건을 UI에서 직접 판단하는 부분이 남아 있는지 확인
- action 버튼 tooltip/보조 설명을 presentation 기준으로 통일할지 검토

주의:

- workflowPolicy/actionFlow 로직은 UI 정리 버전에서 변경하지 않는다.

### 3. 발주정보 section

즉시 수정 가능 후보:

- 셀 편집 input 높이와 일반 표시 셀 높이 맞춤
- 숫자/날짜/텍스트 입력의 padding 통일
- 저장/수정 상태 표시 문구 위치 정리
- 빈 값 placeholder 문구 통일

구조 정리 필요 후보:

- `editingCell`, `editingValue`가 필요한 section에만 전달되도록 추가 축소 가능성 검토
- 발주정보 셀 표시 format helper 분리 여부 검토

주의:

- 0.9.150에서 분리한 editing session hook 계약을 흔들지 않는다.

### 4. 생산구성 section

즉시 수정 가능 후보:

- 생산구성 통합 토글 버튼 위치와 높이 통일
- 원단/부자재/외주공정 block 간격 정리
- 접힘/펼침 상태 표시 문구 통일
- 소계/합계 카드의 높이와 정렬 기준 정리

구조 정리 필요 후보:

- section props builder에서 생산구성 표시 값이 과하게 커지면 하위 builder 추가 검토

주의:

- 원부자재/외주공정 계산식은 변경하지 않는다.

### 5. 비용/요약 영역

즉시 수정 가능 후보:

- 원가/공임/손실/합계 카드의 숫자 정렬 통일
- 단위 표기와 원화 표기 일관화
- 카드 내부 padding과 제목/값 간격 정리

구조 정리 필요 후보:

- 비용 계산 결과와 표시 format 책임 분리 여부 검토

주의:

- 계산식과 저장 대상은 변경하지 않는다.

### 6. 우측 첨부/메모 panel

즉시 수정 가능 후보:

- 첨부 카드의 썸네일, 파일명, 버튼 위치 정리
- 대표 이미지 뱃지 위치와 크기 정리
- 다운로드/미리보기/삭제 버튼 크기 통일
- 메모 입력창과 메모 목록 간격 정리
- 메모 첨부 표시와 일반 첨부 표시 구분 강화

다음 버전 이후 후보:

- 디자인 첨부 최초 업로드 시 자동 대표 지정
- 대표 이미지 삭제 시 남은 디자인 첨부 자동 대표 승계
- 대표 이미지 없음 안내 문구 표시

주의:

- 첨부 업로드/다운로드/삭제, 썸네일 표시, 메모 저장 로직은 정상 동작 중이므로 UI 정리 중 변경하지 않는다.

### 7. 모달 UX

즉시 수정 가능 후보:

- 기본정보 모달 footer 버튼 높이/정렬 확인
- 삭제 확인 모달의 경고 문구와 버튼 배치 통일
- 첨부 미리보기 모달의 닫기 버튼 위치 확인

구조 정리 필요 후보:

- 관리자 모달에서 정리한 footer/action row 패턴을 작업지시서 모달에도 가져올지 검토

주의:

- 배경 스크롤 차단, ESC 닫기, focus trap, 모바일 상단 닫기 버튼 기준은 유지한다.

## 우선순위

### 0.9.152 적용 후보

- PC 좌측/중앙 패널 높이와 하단 라인 정리
- header/action 버튼 높이와 간격 정리
- 발주정보 셀 input 높이 통일
- 생산구성 section spacing 정리

### 0.9.153 적용 후보

- 첨부/메모 panel UI 정리
- 대표 이미지 뱃지/버튼 위치 정리
- 메모 입력/목록 간격 정리

### 0.9.154 적용 후보

- 작업지시서 모달 footer/action 정리
- 삭제 확인/첨부 미리보기 모달 UX 점검

### 0.9.155 이후 후보

- 대표 이미지 자동 지정/삭제 후 승계 로직
- 모바일/태블릿 전용 UI 보정
- 통계/관리자 화면과의 공통 UI token 정리

## 테스트 기준

UI 변경을 시작하기 전후로 아래 항목을 반복 확인한다.

1. 작업지시서 선택 후 상세 화면이 PC/태블릿/모바일 폭에서 깨지지 않는지 확인한다.
2. 발주정보 셀 편집, 저장, 취소가 정상인지 확인한다.
3. 생산구성 접힘/펼침과 원부자재/외주공정 입력이 정상인지 확인한다.
4. 첨부 업로드, 썸네일 표시, 다운로드, 삭제가 정상인지 확인한다.
5. 메모 저장 후 상태 변경/새로고침 시 유지되는지 확인한다.
6. 검토요청/검토완료/발주요청 등 상태 버튼이 기존처럼 표시되고 동작하는지 확인한다.

## 금지 기준

- UI 정리 중 repository, DB schema, R2 Worker, purge API를 수정하지 않는다.
- UI 정리 중 workflowPolicy/actionFlow 조건을 변경하지 않는다.
- 한 버전에서 PC/태블릿/모바일을 동시에 크게 변경하지 않는다.
- 대규모 파일 이동과 시각적 리디자인을 한 번에 수행하지 않는다.
