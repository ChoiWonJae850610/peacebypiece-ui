# 0.10.54 멤버관리 화면 IA 재정리

## 목적

`/admin/members`를 단순 placeholder에서 고객관리자가 실제 멤버 운영 흐름을 이해할 수 있는 1차 IA 화면으로 바꾼다.

이번 버전은 실제 초대 생성, 가입 신청 승인, 권한 저장 API를 연결하지 않는다. 화면 구조와 데이터 표시 기준만 먼저 고정한다.

## 화면의 1차 구성

1. 상단 설명 영역
   - 멤버 초대
   - 가입 신청 승인
   - 권한 부여
   - 조직 설정 이동

2. 요약 카드
   - 전체 멤버
   - 초대 대기
   - 승인 대기
   - 권한 템플릿

3. 멤버 목록
   - 승인된 멤버와 정지된 멤버를 표시할 영역
   - 현재는 DB 연결 전 empty 상태

4. 역할 기본값
   - 관리자
   - 디자이너
   - 검수 담당
   - 발주/재고 담당
   - 조회 전용 후보

5. 초대 대기 목록
   - 초대 링크/QR 생성 후 활성/만료/취소 상태를 표시할 영역
   - 현재는 DB 연결 전 empty 상태

6. 가입 신청/승인 대기 목록
   - 초대 링크로 가입 신청한 사용자를 승인/거절할 영역
   - 현재는 DB 연결 전 empty 상태

7. 메인화면 카드 권한
   - 권한 코드에 따라 멤버 홈 화면 카드 노출을 제어할 후보

8. 권한 그룹
   - 권한 카탈로그를 화면에서 이해할 수 있도록 그룹 기준을 표시

## 정책

- role은 기본 권한 묶음 또는 표시용으로만 사용한다.
- 실제 접근 제어는 permission_code 기준으로 진행한다.
- 고객관리자 환경설정이 아니라 멤버관리에서 초대/승인/권한 부여를 처리한다.
- 초대는 자동 이메일/SMS 발송보다 링크 생성, QR 표시, 링크 복사를 우선한다.
- 승인 전 사용자는 작업지시서, 저장소, 통계, 협력업체, 멤버관리 화면에 접근하지 못한다.
- 프론트 카드 숨김은 UX 보조일 뿐이고 API 권한 검증은 후속 버전에서 별도로 처리한다.

## 이번 버전에서 하지 않는 것

- 실제 DB 조회 연결
- 초대 링크 생성 API 연결
- QR 생성 모달 연결
- 가입 신청 승인/거절 action 연결
- member_permissions 저장 연결
- API 권한 검증 적용
- Google OAuth 실제 연결
