# 환경설정 IA 재설계 기준 0.20.10

## 1. 배경

0.20.09 화면 리뷰에서 환경설정 화면은 기능 구현 범위가 넓어진 만큼 화면 밀도가 과도하게 높아진 상태로 확인되었다. 회사 정보, 기준정보, 요금제·저장공간, 약관·정책, 서비스 건의 영역이 모두 카드형 안내와 뱃지, 운영 기준, 요청 흐름 설명을 동시에 노출하면서 실제 운영자가 무엇을 확인하고 무엇을 눌러야 하는지 흐려졌다.

0.20.10의 목적은 기능을 추가하는 것이 아니라, 환경설정의 정보 구조와 화면 표현 원칙을 재정의하는 것이다. 이후 0.20.11부터 화면을 단계적으로 정리한다.

## 2. 핵심 판단

현재 환경설정은 “사용자가 설정을 조작하는 화면”이라기보다 “설계 문서를 카드로 펼쳐놓은 화면”에 가깝다. 고객사 관리자는 기능 설명보다 현재 상태, 등록값, 검토 결과, 변경 가능 액션, 요청 이력을 먼저 봐야 한다.

따라서 다음 원칙을 적용한다.

- 상단 5개 영역은 큰 설명 카드가 아니라 탭 또는 세그먼트 내비게이션으로 정리한다.
- 각 탭 내부는 실제 데이터와 액션 중심으로 구성한다.
- 운영 기준과 요청 흐름은 상시 노출하지 않고 필요 시 접기/모달/보조 설명으로 둔다.
- 뱃지는 실제 상태값에만 사용한다.
- 파일 업로드는 버튼 카드가 아니라 미리보기와 등록/변경 액션으로 표현한다.
- 약관 문서는 요약 카드가 아니라 문서 목록과 문서 열람 모달로 제공한다.

## 3. 전체 IA

환경설정은 다음 5개 탭 기준으로 유지한다.

1. 회사 정보
2. 기준정보
3. 요금제·저장공간
4. 약관·정책
5. 서비스 건의

상단 영역은 다음처럼 작동한다.

- 현재 선택 탭은 명확히 강조한다.
- 각 탭은 1줄 설명 정도만 가진다.
- `관리 가능`, `조회 가능`, `읽기 전용`, `이메일 접수`, `회사 기준`, `후속 연결` 같은 설명성 뱃지는 제거한다.
- 실제 상태가 있는 경우에만 뱃지를 유지한다.

유지할 수 있는 뱃지 예시는 다음과 같다.

- 승인됨
- 검토 대기
- 반려됨
- 무료체험 중
- 정상 사용 중
- 결제 실패
- 저장공간 초과
- 동의 필요

## 4. 회사 정보 탭

### 4.1 목표 구조

회사 정보 탭은 현재 등록된 회사 정보와 파일 상태, 요청 이력을 확인하는 화면이다. 설명 카드보다 실제 등록값을 우선 표시한다.

권장 구조는 다음과 같다.

```txt
회사 정보

[회사 기본 정보]
회사명        TEST A 고객사        [변경 요청]
사업자명      TEST A 사업자        [변경 요청]
회사 주소     ...                  [변경 요청]
대표 이메일   whoi850610@gmail.com  읽기 전용
계정 상태     승인됨

[대표 이미지]
미리보기 또는 빈 상태
[등록] 또는 [변경] [삭제]

[사업자등록증]
파일명 / 미리보기 / 검토 상태
[등록] 또는 [변경] [보기]

[요청 이력]
요청 유형 / 접수일 / 처리상태 / 결과
```

### 4.2 대표 이미지

대표 이미지는 등록 버튼만 있는 카드로 두지 않는다. 현재 이미지가 있으면 미리보기를 먼저 보여준다.

등록 전:

```txt
대표 이미지
등록된 대표 이미지가 없습니다.
[등록]
```

등록 후:

```txt
대표 이미지
[이미지 미리보기]
파일명: company-profile.png
등록일: 2026.06.07
[변경] [삭제]
```

### 4.3 사업자등록증

사업자등록증은 파일 상태와 검토 상태가 핵심이다.

등록 전:

```txt
사업자등록증
등록된 사업자등록증이 없습니다.
[등록]
```

등록 후:

```txt
사업자등록증
사업자등록증.pdf
검토 상태: 승인 대기
등록일: 2026.06.07
[보기] [변경]
```

반려 상태:

```txt
사업자등록증
검토 상태: 반려됨
반려 사유: 파일이 흐릿하여 확인이 어렵습니다.
[다시 등록]
```

### 4.4 탈퇴/비활성화 요청

고객사 관리자 화면에서는 `계정 비활성화 요청`보다 `탈퇴 요청`이 더 명확하다. 회사 정보 탭의 보조 액션으로 배치한다.

탈퇴 요청 버튼을 누르면 확인 모달을 띄운다.

```txt
탈퇴를 요청하시겠습니까?

탈퇴가 승인되면 서비스 이용이 제한됩니다.
필요한 파일은 미리 내려받아 주세요.
요청 후 시스템관리자 검토가 진행됩니다.

[취소] [탈퇴 요청]
```

시스템관리자 승인 후 로그인 시에는 별도 안내 모달 또는 차단 화면으로 안내한다.

```txt
탈퇴 요청이 승인되었습니다.

YYYY.MM.DD까지 필요한 자료를 내려받을 수 있습니다.
이후 계정 접근이 제한됩니다.

[내 파일 확인] [로그아웃]
```

개인설정 이동 카드는 제거한다. 개인설정은 상단 사람 아이콘을 통해 접근한다.

## 5. 요금제·저장공간 탭

요금제 탭은 운영 기준 카드보다 현재 상태와 다음 액션이 중요하다.

권장 구조는 다음과 같다.

```txt
요금제·저장공간

[현재 요금제]
Trial
상태: 무료체험 중
요금제 종료일: 2026.06.13
[요금제 변경]

[저장공간]
0B / 100MB
0%
[저장공간 추가]

[멤버]
5명 / 10명
[멤버 관리]

[결제 정보]
현재 결제수단 없음
[결제수단 등록]
```

문구는 `무료체험 종료일`보다 `요금제 종료일`을 우선한다. 상태값으로 무료체험 여부를 표현한다.

- 무료체험 중
- 정상 사용 중
- 결제 실패
- 해지 예정
- 사용 정지

`요청 흐름`, `운영 기준`은 상시 카드로 두지 않는다. 필요 시 `운영 기준 보기` 접기 영역 또는 모달로 제공한다.

## 6. 약관·정책 탭

약관·정책 탭은 중간 설명 화면을 줄이고 문서 목록과 동의 상태를 바로 보여준다.

권장 구조는 다음과 같다.

```txt
약관·정책

[필수 약관 동의 상태]
동의 필요 4건 / 동의 완료 0건
[필수 약관 전체 동의]

[이용약관]              v1.0  동의 필요   [보기]
[개인정보처리방침]       v1.0  동의 필요   [보기]
[요금·환불정책]          v1.0  동의 필요   [보기]
[데이터 보관·삭제정책]   v1.0  동의 필요   [보기]
[서비스 운영정책]        v1.0  선택 문서   [보기]
```

`보기` 버튼을 누르면 모달을 띄운다.

```txt
이용약관

[문서 내용 전체 스크롤]

[닫기] [동의]
```

약관 내용은 카드형 요약보다 실제 문서처럼 읽히는 모달이 적합하다.

## 7. 기준정보 설정 탭

기준정보 설정은 카드 설명보다 관리 UI가 우선이다. 내부 탭으로 나눈다.

```txt
기준정보 설정

[생산품 유형] [단위 표준] [외주공정 유형]

생산품 유형
검색 [          ]      [추가]

이름        사용 여부        순서        관리
티셔츠      사용              1          [수정]
팬츠        사용              2          [수정]
```

카드형 설명은 제거하고, 목록/테이블/추가/수정/사용중지 액션 중심으로 정리한다.

## 8. 서비스 건의 탭

서비스 건의는 카드형 설명보다 접수 폼과 이력이 중요하다.

```txt
서비스 건의

문의 유형
[기능 건의 v]

제목
[                ]

내용
[                ]

첨부
[파일 선택]

[접수하기]

접수 이력
- 2026.06.07 기능 건의 / 접수됨
- 2026.06.05 오류 제보 / 답변 완료
```

## 9. 헬프메시지 기준

상시 노출 헬프메시지는 1~2줄만 둔다. 긴 설명은 접기 또는 모달로 이동한다.

좋은 예:

```txt
대표 이미지와 사업자등록증은 고객사 정보 확인에 사용됩니다.
```

피해야 할 예:

```txt
대표 이미지는 고객 프로필, 시스템관리자 검토 화면, 향후 공개 표시 회사 대표 이미지로 활용되며...
```

## 10. 카드 사용 기준

카드는 다음 경우에만 사용한다.

- 현재 상태 요약
- 파일 미리보기
- 요청 이력 한 건
- 요금제 상태 요약
- 중요한 경고/차단 안내

다음 용도로는 카드 사용을 피한다.

- 버튼 하나만 들어간 카드
- 설명문만 들어간 카드
- 요청 흐름 설명 카드
- 운영 기준 설명 카드
- 실제 액션과 무관한 안내 카드

## 12. 비변경 범위

0.20.10은 설계 문서화 버전이다. 다음은 변경하지 않는다.

- 기능 코드

- API route
- R2 upload flow
- 작업지시서 첨부/디자인
- 메모
- 휴지통
