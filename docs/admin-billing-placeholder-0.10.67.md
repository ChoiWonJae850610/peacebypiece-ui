# 0.10.67 고객관리자 요금제·결제 placeholder 보정

## 목표

고객관리자 환경설정의 `요금제·결제` 항목을 단순 준비중 안내가 아니라 현재 요금제, 저장공간 한도, 멤버 한도, 변경 요청 정책을 읽기 전용으로 확인하는 placeholder로 정리한다.

## 적용 범위

- `/admin/settings`
- `요금제·결제` 카드 상태값
- 요금제·결제 안내 모달
- 요금제 placeholder presentation 데이터

## 정책

1. 고객관리자는 요금제와 저장공간 한도를 직접 수정하지 않는다.
2. 실제 수정은 시스템관리자 요금제/용량 관리 화면에서 처리한다.
3. 정식 결제 API, 카드 등록, 자동 청구는 후순위다.
4. 현재 단계에서는 기본 plan policy를 읽기 전용 placeholder로 표시한다.
5. 이후 실제 데이터 연결 시 `companies.plan_code`, `company_plan_assignments`, override 정책을 우선한다.

## 이번 버전에서 하지 않은 것

- 결제 API 연동
- 카드 등록 또는 자동 청구
- 고객관리자 직접 요금제 변경
- DB schema 변경
- 저장소 사용량 실시간 집계 변경
