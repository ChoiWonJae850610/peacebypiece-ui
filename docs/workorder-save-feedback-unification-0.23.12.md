# 0.23.12 작업지시서 저장 피드백 통합

## 작업 목적

작업지시서 명시적 저장과 공장 전달사항 저장이 서로 다른 저장 상태 표현을 사용하던 문제를 정리합니다.

## 변경 내용

- 작업지시서 저장 시작 시 loading 토스트를 표시합니다.
- 저장 성공 또는 실패 결과가 동일한 토스트 ID에서 교체되도록 정리했습니다.
- 작업지시서 저장 성공 시 기존 별도 성공 토스트 호출을 제거해 중복 표시를 방지했습니다.
- 저장 실패 시 편집 상태를 `dirty`로 복원하고 API 오류 메시지를 우선 표시합니다.
- 공장 전달사항 저장에도 loading → success/danger 단일 토스트 교체 구조를 적용했습니다.
- 작업지시서 저장 진행 문구를 한국어와 영어 i18n에 추가했습니다.

## 영향 범위

- 작업지시서 본문 명시적 저장 UX
- 작업지시서 공장 전달사항 저장 UX
- DB 스키마 및 API 계약 변경 없음
