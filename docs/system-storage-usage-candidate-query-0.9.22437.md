# 0.9.22437 — 시스템관리자 storage-usage 후보 조회 검증

## 목적

`/system/storage-usage`에서 고객관리자가 삭제 요청했거나 보관 기간이 도래한 실제 삭제 후보가 정확히 표시되는지 확인한다.

## 이번 기준

- 작업지시서 후보는 대표 row로 표시한다.
- 작업지시서 후보의 묶음 항목은 `문서 n개, 디자인 n개, 메모 n개`로 구분해 표시한다.
- 문서/디자인 수는 `attachment_trash_items`와 `attachments.type`을 기준으로 계산한다.
- 메모 수는 `memos`의 삭제 상태를 기준으로 계산한다.
- R2 실제 삭제 용량은 문서/디자인 첨부의 `size_bytes` 합산값을 사용한다.
- 고객관리자가 삭제 요청하지 않은 단순 휴지통 pending 항목은 기본 후보 목록에 나오지 않아야 한다.
- 작업지시서에 묶인 문서/디자인은 작업지시서 대표 row에 집계하고, 개별 파일 후보와 중복 노출하지 않는다.

## 확인 항목

1. `/system/storage-usage` 진입
2. 삭제 후보 총 개수 확인
3. 파일 후보와 작업지시서 후보 개수 확인
4. 작업지시서 대표 row에서 문서/디자인/메모 수 확인
5. 원본 용량 합산 확인
6. 고객관리자가 삭제 요청하지 않은 항목이 후보에 나오지 않는지 확인
7. 작업지시서 후보와 해당 묶음 파일이 중복 표시되지 않는지 확인
8. 이전 `attachmentTrashItems undefined` 계열 런타임 오류가 재발하지 않는지 확인
