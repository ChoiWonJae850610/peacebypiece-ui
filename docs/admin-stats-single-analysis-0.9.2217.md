# 0.9.2217 관리자 통계 화면 단일 분석 구조 전환

## 목적

관리자 통계 화면에서 Basic / Standard / Growth / Premium 요금제별 선택 구조를 제거하고, 고객관리자가 한 화면에서 생산/협력업체/리오더 중심 분석을 확인하도록 정리한다.

## 화면 기준

- 대상 화면: `/admin/dashboard`
- 대시보드와 저장소 메뉴에서 이미 보여주는 단순 요약 정보는 축소한다.
- 통계 화면은 생산 흐름, 협력업체 성과, 리오더, 품질/납기 준비 상태 중심으로 구성한다.

## 반영 내용

1. 요금제 선택 UI 제거
2. 상단 KPI 4개를 전체 분석 기준으로 재정의
   - 작업지시서
   - 완료율
   - 리오더
   - 저장소 요약
3. 작업 흐름 분석을 첫 번째 메인 차트로 배치
4. 생산 분석 / 협력업체 분석 / 리오더 분석을 별도 영역으로 분리
5. 저장소는 상세 통계가 아니라 요약 카드로만 유지
6. 품질/납기 영역은 실제 데이터 구조 확정 전까지 준비 상태로 표시
7. 운영/개발 기준은 `DEBUG_FLAGS.adminStatsDevSections`가 true일 때만 표시

## SQL DDL 필요 여부

불필요.

## 전체 리셋 필요 여부

불필요.

## 테스트 케이스

1. `/admin/dashboard` 접속
2. Basic / Standard / Growth / Premium 요금제 선택 버튼이 사라졌는지 확인
3. 상단 KPI가 작업지시서 / 완료율 / 리오더 / 저장소 요약으로 표시되는지 확인
4. 작업 흐름 분석, 생산 분석, 협력업체 분석, 리오더 분석 순서로 화면이 구성되는지 확인
5. 저장소 상세 정보가 과하게 노출되지 않고 요약으로만 표시되는지 확인
6. 운영/개발 기준은 runtime flag가 true일 때만 표시되는지 확인
