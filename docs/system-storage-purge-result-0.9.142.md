# 0.9.142 — 시스템관리자 R2 purge 결과 반영 및 후보 썸네일 표시

## 목적

`/system/storage-usage`에서 R2 실제 삭제 후보를 확인할 때 파일명과 key만으로 판단하지 않도록 삭제 후보 썸네일을 함께 표시한다. 또한 선택 삭제 또는 전체 도래 항목 삭제 후 후보 목록과 요약 카드가 갱신되도록 한다.

## 적용 범위

- 시스템관리자 저장소 정리 화면
- R2 purge 후보 목록 표시
- purge 실행 후 화면 갱신
- 선택 상태 초기화
- 실패 항목 유지

## 정책

1. 삭제 후보 목록의 미리보기는 `thumbnail_key`를 우선 사용한다.
2. `thumbnail_key`가 없고 이미지 파일이면 원본 `storage_key`를 축소 표시용 fallback으로 사용한다.
3. PDF/비이미지 파일은 이미지 대신 파일 타입 뱃지로 표시한다.
4. 미리보기 이미지는 삭제 판단 보조용이며 다운로드/복구 기능을 제공하지 않는다.
5. purge 실행은 계속 Worker 기반 삭제만 사용한다.
6. R2 직접 SDK 삭제 방식은 사용하지 않는다.
7. 삭제 성공 후 `router.refresh()`로 서버 snapshot을 다시 읽어 후보 목록과 요약 카드를 갱신한다.
8. 삭제 성공 후 선택 상태는 초기화한다.
9. 삭제 실패 항목은 실패 사유와 함께 후보 목록에 남겨 재시도 대상으로 유지한다.

## 미비점 보완

0.9.141의 `selected` purge 후보 조회 SQL에 불필요한 닫는 괄호가 남아 있을 수 있는 부분을 보완했다. 선택 삭제 실행 시 SQL 문법 오류가 발생하지 않도록 후보 조회 조건을 정리했다.

## 테스트 기준

1. `/system/storage-usage` 진입 시 후보 목록에 미리보기 영역이 보이는지 확인한다.
2. 이미지 후보에 `thumbnail_key`가 있으면 Network에서 `thumbnails/...` key 요청이 발생하는지 확인한다.
3. PDF/비이미지 후보는 파일 타입 뱃지로 표시되는지 확인한다.
4. 선택 삭제 실행 후 후보 목록에서 삭제 성공 항목이 사라지는지 확인한다.
5. 상단 요약 카드의 후보 수/용량/썸네일 수가 갱신되는지 확인한다.
6. 삭제 실패 항목은 실패 사유와 함께 남아 있는지 확인한다.
7. `/admin/files` 삭제/복구, 작업지시서 첨부/메모 흐름이 기존처럼 유지되는지 확인한다.
