Status: `CONFIRMED_AUDIT`
Target version: `0.24.34.6`
Baseline: `0.24.34.5` policy-docs commit `57411acb`
Purpose: record customer workspace and customer-admin gaps before `0.24.35 — Company-wide Export Execution` starts.

## 1. Audit rule

This audit records incomplete or weak productization areas. It does not claim that implementation is complete.

Every customer workspace screen must be judged by:

- actual DB/API integration versus mock/design-only data,
- customer-facing Korean copy,
- role/permission behavior,
- WAFL common component usage,
- desktop/mobile/tablet usability,
- evidence required for Product Verified,
- connection to PDF/R2/storage/billing/signup policy,
- whether the work must finish in `0.24.34.5` or can move to a later sprint.

## 3. `/workspace` main dashboard

Current problem:

- the large `PLAN AND STORAGE` section is too prominent on the main work dashboard,
- tablet landscape shows a heavy right-side storage/plan box and pushes work queues down,
- real work priorities should appear before subscription/storage details,
- internal or policy language can distract customers from daily work.

Confirmed product direction:

- remove or collapse the large `PLAN AND STORAGE` box from the main dashboard,
- replace it with a compact summary for plan, storage, and member count,
- move subscription/storage details to dedicated settings/storage/subscription screens,
- prioritize work queues and recent workorders,
- avoid internal double-scroll areas,
- provide Galaxy Tab landscape evidence.

Recommended layout:

1. compact header summary:
   - company name,
   - plan/trial status,
   - storage usage,
   - member count,
   - small links to storage/subscription/settings.
2. work queue:
   - 검토 대기,
   - 발주 대기,
   - 검수 대기,
   - 입고 지연,
   - 확인 필요.
3. recent workorders compact list.
4. secondary notices only when actionable.

Status: `TODO`
Priority: `HIGH`
Target: `0.24.34.5 continuation E`.

## 4. `/workspace/workorders`

Current gap:

- existing workorder selection, factory save, PDF creation/view, and size editing need live dev/test evidence after recent fixes,
- workorder PDF must switch from temporary technical layout to final factory-delivery table layout,
- factory-delivery PDF must exclude internal costs,
- generated incomplete/final PDFs must appear in attachments/generated documents,
- size input must be audited for WAFL component usage and mobile/tablet usability.

Confirmed PDF policy:

- use existing table-style workorder form as the basis,
- include representative design image,
- include factory instruction memo,
- include fabric/accessory/process production info,
- exclude unit price, amount, internal labor, loss cost, margin, and internal costing,
- add Korean size spec page only when values or a selected template exist,
- incomplete and final PDFs share one template and differ only by status/marking,
- generated PDF appears in attachment/generated document area.

Confirmed size policy:

- system-admin standards provide size/POM templates,
- workorder stores a snapshot,
- later standards changes do not mutate existing workorders,
- inch input uses integer plus 1/8 selection,
- use WAFL common input/table/modal components,
- mobile/tablet editing must be verified.

Status: `TODO`
Priority: `HIGH`
Target: `0.24.34.5 continuation A/B/C`.
