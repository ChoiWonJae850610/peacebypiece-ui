# Repository / Source / DB Cleanup Audit 0.24.03

## 1. Executive Summary

This audit inspected the PeaceByPiece / WAFL repository at commit `7ab4c4fc5b3268f4af9be1e2561197e11da07fcd` on branch `master`, with `origin/master` synchronized before work began. The audit did not delete files, move files, change DB schema, create or apply migrations, run Seed/Reset/Cleanup, or touch R2.

The main findings are:

- Generated outputs are mostly ignored correctly: `.next`, `artifacts`, `test-results`, `.tmp`, and Playwright output should remain uncommitted.
- `cloudflare/pdf-generator-worker/node_modules` is tracked and large, about 3,992 files. This is the highest repository hygiene finding, but it is a `DELETE-REVIEW` item until Cloudflare worker deploy expectations are confirmed.
- DB metadata on the approved dev/test target shows 60 public tables, 60 primary keys, 120 foreign keys, 17 unique constraints, and 270 indexes.
- Simulator Seed is safe-guarded and idempotent, but remains slow at about 600 seconds for full seed. The likely cause is row-by-row upsert/delete work inside a single transaction.
- PowerShell menu 9 `Reset Database Schema` is still the highest safety follow-up because it should receive Simulator-grade runtime/fingerprint/host/database guards before use.

## 10. Query/Seed Performance Audit

The full Simulator Seed currently takes about 600 seconds. This audit did not run Seed. Evidence is from 0.24.02 validation and source inspection of `tools/simulator/commands/db-data.mjs`.

Likely bottlenecks:

- Row-by-row `await client.query` loops for users, members, permissions, categories, partners, partner items, spec sheets, orders, material orders, lines, and allocations.
- Category rows are deleted and recreated per company even when unchanged.
- Member permissions are deleted and reinserted per member.
- Company H large fixture amplifies every row-by-row path.
- No table-level timing logs are emitted, making exact stage attribution difficult.

Recommended performance targets:

| Seed mode | Target | Candidate change | Target version |
|---|---:|---|---|
| Quick | under 1 minute | Subset company profile or unchanged fixture skip | 0.24.06 |
| Permissions only | under 20 seconds | Isolate member/permission fixture path | 0.24.06 |
| Storage only | a few seconds | Keep storage snapshot path separate | 0.24.06 |
| Full | under 3 minutes | Batch upserts and per-table timing logs | 0.24.06 |

Do not optimize by weakening guards or skipping integrity checks.
