# 0.24.34.7 Source ZIP Packaging Audit

Status: CONFIRMED PATCH SCOPE  
Baseline: 0.24.34.6  
Result version: 0.24.34.7  
Scope: Download watcher / PowerShell pipeline source ZIP packaging, repo-state APP_VERSION recording, QA evidence separation

## 1. Why this audit exists

The 0.24.34.6 source ZIP still contained generated or test-output directories that should not be part of the source handoff.

Observed excluded-output candidates:

- `artifacts/`
- `playwright-report/`
- `test-results/`
- `reports/`
- `tsconfig.tsbuildinfo`
- generated PDF/PNG evidence under `artifacts/pdf-qa/*`
- generated Playwright screenshots/manifests under `artifacts/ui-qa/*`

These files are useful QA evidence, but they must be delivered as a separate evidence ZIP, not inside the source ZIP.

## 2. Correct packaging policy

### 2.1 Source ZIP

The source ZIP must contain:

- application source
- committed docs
- tests
- pipeline scripts
- configuration files that are safe to commit
- `.env.example`

The source ZIP must not contain:

- `.git/`
- `node_modules/`
- `.next/`
- `.wrangler/`
- `.tmp/`
- `artifacts/`
- `playwright-report/`
- `test-results/`
- `reports/`
- `blob-report/`
- `coverage/`
- `.nyc_output/`
- `.env`, `.env.*` except `.env.example`
- generated `*.zip`
- `repo-state-*.txt`
- `build-result-*.txt`
- `*.tsbuildinfo`
- Playwright `storageState*.json`
- Playwright `*.har`
- Playwright `*.webm`
- logs/backups/temp/copy files

### 2.2 QA evidence ZIP

Screenshots, PDF PNGs, Playwright traces, generated PDFs, manifests, and QA reports must be packaged separately.

The evidence ZIP must be outside:

- Git
- source ZIP
- `4.Newest` source package

## 3. Pipeline files reviewed

### 3.1 `tools/pipeline/download-watcher.ps1`

This file is the watcher entry point. It should remain small.

Actual patch application and source backup creation flow:

```text
download-watcher.ps1
-> pipeline-common.ps1
-> pipeline-patch-processing.ps1
-> ProcessOnePatchIfReady
-> BackupProjectToZip
```

Therefore source ZIP exclude correction belongs mainly in `pipeline-patch-processing.ps1`.

### 3.2 `tools/pipeline/pipeline-patch-processing.ps1`

This file previously used a short tar exclude list:

```powershell
tar -a -c -f $zipPath --exclude=node_modules --exclude=.next --exclude=.git --exclude=.tmp --exclude=.env.local --exclude=*.log .
```

0.24.34.7 expands this list to block QA artifacts, reports, generated ZIPs, tsbuildinfo, env files, backup/temp files, and common Playwright evidence files.

### 3.3 `tools/pipeline/peacebypiece-auto-pipeline.ps1`

Menu 7 already uses a stronger ZIP contract function. 0.24.34.7 aligns that contract with the watcher backup policy by adding:

- `blob-report`
- `coverage`
- `.nyc_output`
- `storageState*.json`
- `*.har`
- `*.webm`

## 4. Repo-state APP_VERSION bug

The repo-state generator printed:

```text
APP_VERSION:
export { APP_VERSION } from "./version";
```

That happened because `pipeline-patch-processing.ps1` read the re-export file:

```text
lib/constants/app.ts
```

The actual source of truth is:

```text
lib/constants/version.ts
```

0.24.34.7 changes the patch pipeline repo-state reader to parse:

```ts
export const APP_VERSION = "0.24.34.7";
```

Expected future repo-state output:

```text
APP_VERSION:
0.24.34.7
```
