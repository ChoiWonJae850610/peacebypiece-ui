## File-list classification rule

Use the sections as follows:

- `수정 파일 목록 :`
  - Existing repository files that are overwritten.
- `추가 파일 목록 :`
  - New repository files that did not exist before the patch.
- `삭제 파일 목록 :`
  - Existing repository files that must be removed.

Keep non-file notes, manual QA items, and pending-test lists outside `삭제 파일 목록 :`.

## Source ZIP exclude contract

- `.git/`

- `.next/`
- `artifacts/`
- `playwright-report/`
- `test-results/`
- `reports/`
- `coverage/`
- `blob-report/`
- `.nyc_output/`
- `*.tsbuildinfo`
- `*.zip`
- `storageState*.json`
- `*.har`
- `*.webm`
- QA evidence ZIPs and screenshots/traces

QA evidence must remain separate from source ZIPs.

## repo-state APP_VERSION contract

