# 0.20.31 원단·부자재 발주 모바일 구조 2차

## 적용 범위

- 원단·부자재 발주 모바일 상세 화면의 상태 요약 영역을 추가했다.
- 상태 전환 버튼은 기존 `MaterialOrderStatusFlow` 로직을 그대로 사용하되 모바일에서는 compact density로 표시한다.
- 모바일 하단 고정 액션바는 `목록 / 자재 / PDF·납기` 기준 위치로 정리했다.
- PDF·납기 버튼은 후속 기능 연결 위치를 고정하기 위한 비활성 안내 액션이며, 실제 생성·저장 로직은 변경하지 않았다.
- 자재 선택 sheet의 후보 카드와 자재 행은 모바일에서 진행 수량과 남은 수량을 더 명확히 볼 수 있도록 보정했다.

## 유지한 범위

- PC 3패널 구조는 유지했다.
- 태블릿 2패널 + 자재 선택 sheet 구조는 유지했다.
- 상태전환 로직, 권한 로직, API, DB, R2, 첨부, 메모, 휴지통, purge 흐름은 변경하지 않았다.
- 발주 PDF 생성과 납기 입력 기능은 새 로직을 추가하지 않고 후속 기능 연결 위치만 예약했다.
