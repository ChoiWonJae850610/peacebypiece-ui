# 0.18.96 WAFL A-Type 보관문서 점검

## 목적

테스트 불가 상태에서 `docs/보관문서/wafl-a-type/` 문서 묶음을 삭제하거나 이동하지 않고, 현재 개발 기준과의 관계를 정리한다.

## 점검 결과

- `docs/보관문서/wafl-a-type/`는 과거 WAFL A-Type 설계·점검·UI 기준 누적 문서 묶음이다.
- 업로드된 프로젝트 기준 해당 폴더에는 약 99개의 문서가 포함되어 있다.
- 파일 수가 많고 일부 항목은 현재 `docs/현재기준/` 문서와 내용상 겹칠 수 있으므로, 현재 개발 기준으로 직접 사용하기 전에 최신 기준 문서와 대조해야 한다.
- 현재 테스트 불가 상태이므로 문서 삭제, 이동, 이름 변경은 하지 않는다.

## 보관 기준

- 과거 판단 근거와 디자인·정책 변천 기록으로 유지한다.
- 현재 개발 기준 문서가 필요한 경우 `docs/현재기준/`을 우선한다.
- 정책·약관·고객 공개 문서의 최신 기준은 `docs/정책문서/`를 우선한다.
- WAFL A-Type 문서를 기능 개발 근거로 사용할 때는 적용 대상 화면과 현재 코드 구조를 별도로 확인한다.

## 후속 정리 후보

테스트 가능 상태가 된 뒤 다음 순서로 재점검한다.

1. 현재 UI/IA와 직접 충돌하는 문서 선별
2. 정책문서로 승격할 항목과 보관 유지 항목 분리
3. 현재기준 문서로 반영할 핵심 결정사항 추출
4. 단순 중복 문서의 보관 위치 유지 또는 압축 정리 여부 결정

## 이번 패치에서 하지 않은 것

- 문서 삭제 없음
- 폴더 이동 없음
- 파일명 변경 없음
- DB/API/R2/권한/작업지시서/원단부자재 흐름 변경 없음
- UI 동작 변경 없음
