# 0.9.22407 저장소/휴지통 i18n·하드코딩 정리 1차

## 목적

저장소/휴지통 리팩토링의 기능 흐름 안정화 이후, 시스템관리자 실제 삭제 후보 화면에 남아 있던 사용자 노출 문구와 중복 메시지 조립을 presentation 계층으로 이동했다.

## 반영 내용

- `/system/storage-usage` 페이지의 주요 문구를 `lib/system/storagePurgePresentation.ts`로 이동했다.
- 시스템관리자 실제 삭제 후보 client 컴포넌트의 confirm/result/list 문구를 presentation formatter로 이동했다.
- 작업지시서 후보 설명에서 `연결 첨부파일` 표현을 제거하고 `문서, 디자인, 메모` 기준으로 정리했다.
- 시스템관리자 통계 문구의 `영구삭제` 표현을 `실제 삭제`로 정리했다.
- 고객관리자 설정/DB 점검 설명에 남아 있던 `복구`, `영구삭제`, `작지` 표현을 `복원`, `삭제 요청`, `작업지시서`로 정리했다.
- `delete_reason` 제거 이후 active code/schema 범위에서 `deleteReason`/`delete_reason` 참조가 남지 않도록 확인했다.

## 설계 기준

- page.tsx와 TSX 컴포넌트에는 긴 사용자 노출 문장을 직접 두지 않는다.
- 시스템관리자 실제 삭제 후보 화면의 문구는 `storagePurgePresentation.ts`를 통해 조립한다.
- 고객관리자 화면에서는 `영구삭제` 대신 `선택 삭제` 또는 `삭제 요청`을 사용한다.
- 시스템관리자 화면에서는 실제 R2/DB purge 의미가 필요할 때만 `실제 삭제`로 표시한다.
- 작업지시서에 종속된 항목은 `연결 첨부`가 아니라 `문서, 디자인, 메모` 기준으로 표시한다.

2. 상단 설명과 후보 목록 설명에서 `연결 첨부파일`이 보이지 않는지 확인
3. 선택 삭제/전체삭제 confirm 문구가 `문서, 디자인, 메모` 기준으로 표시되는지 확인
4. 시스템 통계 문구에서 `영구삭제` 대신 `실제 삭제`가 표시되는지 확인
5. 고객관리자 관련 저장소 문구에서 `복구`, `영구삭제`, `작지` 표현이 남지 않는지 확인
6. `npm run build` 로컬 실행 후 타입/프리렌더 오류 확인
