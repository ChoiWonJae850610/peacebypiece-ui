# WAFL UI 제품화 1차 체크포인트 — 0.18.00

## 목적

0.17.76부터 0.17.99까지 진행한 작업지시서/원단·부자재 화면의 UI 경량화, 공통 UI 래퍼 도입, 모바일·태블릿 반응형 분기, Sheet/Toast/Select 기반 정리를 1차 제품화 기준으로 묶어 점검한다.

이번 버전은 기능 로직을 추가하지 않고, 현재까지 적용된 UI 기반과 앞으로 정리해야 할 잔여 범위를 문서화한다.

## 현재까지 완료된 UI 기반

### 작업지시서 화면

- 상세 상단 요약 카드 추가
- 공정 정보와 제품 구성 영역의 표 느낌 완화
- 목록 카드 밀도 완화
- 모바일/태블릿 상세 탭 구조 적용
- 모바일 첨부·메모 패널 Sheet 전환
- 일부 목록/모달 select를 AppSelect로 전환
- PC 화면은 기존 3분할 업무 구조 유지

### 원단·부자재 화면

- 발주서 목록, 상세, 작업지시서 후보 패널의 카드/입력표 밀도 완화
- 우측 작업지시서 후보 카드의 자재 반복 노출 축소
- 자재 선택 영역을 모바일 Sheet로 전환
- 태블릿/모바일 탭 전환 구조 적용
- 종류/상태/구분/공급처/단위 select를 AppSelect로 전환
- PC 화면은 기존 3분할 구조 유지

## 현재 구조 판단

### PC

PC는 아직 작업지시서와 원단·부자재의 주요 업무 입력 흐름을 가장 안정적으로 제공한다. 지금 단계에서는 PC 화면을 기준으로 UI 제품화 작업을 계속 진행하되, 공통 섹션 수정 시 tablet/mobile 영향 여부를 계속 확인해야 한다.

### 태블릿

태블릿은 실제 기기 확인 전이므로 최종 보정 전 단계다. 현재는 분기 구조와 탭/Sheet 기반이 들어간 상태이며, 실기기 캡처 후 다음 항목을 확인해야 한다.

- 목록과 상세의 폭 균형
- 탭 위치와 sticky 필요 여부
- Sheet 높이와 스크롤 충돌
- 입력 필드 터치 영역
- 첨부파일 업로드 버튼 접근성

### 모바일

모바일은 1화면 1역할 원칙으로 이동 중이다. 현재 보조 패널을 Sheet로 옮기기 시작했으므로, 실제 모바일 확인 후 하단 버튼, 키보드, Sheet 스크롤을 별도 보정해야 한다.

## 남은 UI 리팩토링 우선순위

### 1순위: build 안정성 확인

매 버전 적용 후 build fail 로그가 있으면 다음 작업과 함께 수정한다. fail 로그가 없으면 예정된 UI 정리 작업을 진행한다.

### 2순위: PC 화면 잔여 Admin UI 의존성 축소

작업지시서와 원단·부자재 핵심 화면에서는 AdminButton/AdminCard 의존을 계속 줄인다. 단, 관리자/시스템관리자 화면은 아직 Admin 계열 컴포넌트를 유지해도 된다.

### 3순위: AppSelect 적용 확대

다음 후보:

- 업체관리 필터
- 멤버관리 필터/역할 선택
- 저장소 정렬 필터
- 히스토리 필터
- 시스템관리자 고객사/상태 필터

### 4순위: AppTooltip 적용 후보

다음 후보:

- 비활성 버튼 사유
- 권한 제한 안내
- 자재 발주 상태 설명
- 저장소 purge/복원/삭제 버튼 설명
- 통계 지표 설명

### 5순위: TanStack Table 후보 점검

작업지시서 내부 입력표에는 바로 적용하지 않는다. 우선 적용 후보는 관리형 목록이다.

- 저장소 파일 목록
- 휴지통 목록
- 멤버 목록
- 시스템 고객사 목록
- 감사로그 목록
- 통계 상세 테이블

### 6순위: React Hook Form + Zod 후보 점검

신규/복잡 폼부터 후보로 둔다.

- 고객사 온보딩
- 회사 설정
- 멤버 초대
- 업체 등록/수정
- 권한/역할 설정

## 주의할 경계

- 작업지시서/R2/첨부/메모/휴지통/purge 정상 흐름은 직접 목표가 아니면 건드리지 않는다.
- UI 라이브러리는 화면 파일에 직접 흩뿌리지 않고 App* 래퍼를 통해 적용한다.
- PC/tablet/mobile 레이아웃 분리는 유지한다.
- 공통 섹션 파일 수정 시 모바일/태블릿 영향 가능성을 먼저 확인한다.
- commit-meta.md에는 누적 테스트 항목을 넣지 않는다.
