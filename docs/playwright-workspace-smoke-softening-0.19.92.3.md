# 0.19.92.3 Playwright workspace smoke softening

## 목적

0.19.92.2의 정책/환경설정 E2E가 실제 보호 화면 UI보다 강한 selector와 필수 클릭 조건을 사용해 실패하던 문제를 보정한다.

## 변경 기준

- 보호 라우트 진입 여부를 우선 확인한다.
- heading/button 문구 하나에 강하게 의존하지 않는다.
- 조건부 UI는 있으면 검증하고 없으면 기본 화면 진입 검증으로 통과한다.
- 로그아웃은 버튼 노출을 기본 검증으로 두고, 클릭 가능한 경우에만 확인 모달 문구를 확인한다.

## 제외

- 작업지시서 생성 Enter UX는 이미 정상 확인되어 수정하지 않는다.
- DB schema/API/R2 흐름은 변경하지 않는다.
