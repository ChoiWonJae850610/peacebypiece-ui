# 작업지시서 PC UI 1차 정리 - 0.9.152

## 목적

0.9.151에서 정리한 작업지시서 PC UI 수정 후보 중 기능 로직 변경 없이 적용 가능한 항목을 우선 반영한다.

## 반영 범위

### 1. 발주정보 상태 표시

- 발주정보 표 안에 있던 검수여부 컬럼을 제거했다.
- 발주/검수 상태는 표 위 요약 배지 영역으로 이동했다.
- 공장이 여러 개인 경우 상태별 건수를 집계해 `발주대기 1`, `검수대기 2`처럼 표시한다.
- 표 안에서는 구분, 공장, 납기일, 수량, 공임비, 로스비, 관리 영역을 더 넓게 사용한다.

### 2. 원단/부자재 테이블 컬럼 순서

- 원단/부자재 테이블의 컬럼 순서를 `구분 → 거래처 → 자재명 → 수량 → 단위 → 단가 → 금액`으로 변경했다.
- 기능 로직과 저장 필드는 변경하지 않았다.

### 3. 메모/댓글 작성자 및 시간 표시

- 메모와 댓글의 작성자 영역에서 역할/권한 텍스트를 숨겼다.
- 관리자 역할은 `대표`로 표시한다.
- 그 외 사용자는 기존 authorName 값을 표시한다.
- 작성 시간은 `YY.MM.DD HH:mm` 형식으로 표시한다.
- DB schema 변경 없이 UI 표시만 변경했다.

### 4. 첨부/디자인 삭제 문구

- 삭제 모달의 위험 문구를 30일 휴지통 정책에 맞게 수정했다.
- 새 문구: `삭제한 파일은 휴지통으로 이동하며, 30일 이내 복원할 수 있습니다.`

## 제외 범위

- 작업지시서 삭제 confirm을 앱 모달로 교체하는 작업은 0.9.153 이후로 분리한다.
- 발주요청 확인 모달의 대표이미지/요청사항 박스 높이 통일은 0.9.153 이후로 분리한다.
- PDF 출력 하단 테두리 보정은 별도 print 전용 버전으로 분리한다.
- 사용자 이름 DB 컬럼 추가는 가입/초대 플로우 정비 시점에 별도 검토한다.

## 테스트 기준

1. 발주정보 표에서 검수여부 컬럼이 사라졌는지 확인한다.
2. 발주 상태 요약 배지가 표 위에 표시되는지 확인한다.
3. 납기일 컬럼이 이전보다 덜 잘리는지 확인한다.
4. 원단/부자재 표가 거래처 → 자재명 순서로 표시되는지 확인한다.
5. 메모/댓글 시간이 `26.05.03 18:11` 형식으로 표시되는지 확인한다.
6. 메모/댓글 작성자 영역에서 역할/권한 텍스트가 보이지 않는지 확인한다.
7. 첨부/디자인 삭제 모달 문구가 휴지통/30일 복원 기준으로 표시되는지 확인한다.
