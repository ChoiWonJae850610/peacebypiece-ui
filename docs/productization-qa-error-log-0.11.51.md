# 제품화 전 QA 오류 수정 라인 시작 — 0.11.51

## 목적

0.11.50에서 작성한 제품화 전 1차 QA 체크리스트를 실제 테스트에 사용할 수 있도록, QA 오류 수집 기준과 수정 우선순위를 정리한다.

이번 버전은 첨부된 build log, runtime error, 화면 캡처가 없는 상태에서 진행한 QA 오류 수정 라인 시작 버전이다. 실제 오류 재현 자료가 없으므로 기능 코드는 변경하지 않고, 다음 패치부터 오류를 작게 나누어 수정하기 위한 기록 형식을 추가한다.

## 이번 버전의 적용 범위

- APP_VERSION을 0.11.51로 갱신
- QA 오류 수집/분류 기준 문서 추가
- 작업지시서, 저장소, 통계, 멤버관리, 시스템관리자 화면별 오류 기록 항목 정의
- build log와 runtime error가 첨부된 경우 우선 처리 기준 정의

## 이번 버전에서 하지 않은 것

- 작업지시서 저장 정책 변경 없음
- 작업지시서 workflow 상태 변경 없음
- 첨부/R2 upload/download/delete/purge 흐름 변경 없음
- DB schema 변경 없음
- API 응답 포맷 변경 없음
- i18n 리소스 추가 변경 없음
- UI/CSS 직접 수정 없음

## QA 오류 기록 형식

다음 0.11.52 이후 오류 수정 요청 시 아래 항목을 기준으로 기록한다.

```text
버전:
화면:
기기/폭:
오류 유형:
재현 절차:
예상 동작:
실제 동작:
첨부 자료:
영향 범위:
수정 우선순위:
```

## 오류 유형 분류

### 1. build/type 오류

우선순위가 가장 높다. 첨부 build log의 첫 번째 TypeScript 오류를 기준으로 수정한다.

확인 항목:
- 누락 import
- prop type 불일치
- union type 누락
- server/client component 경계 오류
- i18n key typing 오류

### 2. runtime/API 오류

사용자가 캡처한 브라우저 화면, console message, network response를 기준으로 수정한다.

확인 항목:
- DB 연결 실패
- API 500
- JSON parse 실패
- 권한/상태 mismatch
- 빈 데이터 fallback 오류

### 3. 작업지시서 업무 오류

기능 안정화 우선순위가 가장 높다.

확인 항목:
- 목록 조회
- 상세 조회
- 제목/담당자 즉시 저장
- 발주정보 저장
- 생산구성 저장
- workflow action
- 디자인/문서 첨부
- 메모 작성/수정/삭제
- 삭제/복원

### 4. 저장소/R2 오류

실제 파일 손실 가능성이 있으므로 작게 나누어 수정한다.

확인 항목:
- 업로드 complete
- 다운로드 signed url
- 미리보기
- 고객관리자 삭제 요청
- 시스템관리자 purge
- R2 object key 생성
- soft delete metadata

### 5. 모바일/태블릿 표시 오류

기능이 깨지지 않는 범위에서 CSS/class 보정만 한다.

확인 항목:
- 화면 밖 overflow
- 버튼 접근 불가
- modal footer 잘림
- table 가로 스크롤 누락
- 긴 파일명/문구 줄바꿈 실패

### 6. i18n/문구 오류

사용자에게 보이는 문구부터 정리한다.

확인 항목:
- 고객관리자 문구
- 시스템관리자 문구
- 작업지시서 문구
- 저장소/휴지통 결과 메시지
- role 기반 item label

## 0.11.52 이후 권장 처리 방식

1. build log가 있으면 build error부터 수정한다.
2. runtime error 캡처가 있으면 재현 화면과 관련 API를 먼저 확인한다.
3. 화면 깨짐은 기능 로직과 분리해서 CSS/class만 수정한다.
4. 작업지시서, 저장소, R2 purge 정상 동작 흐름은 직접 오류가 아니면 건드리지 않는다.
5. 한 버전에는 하나의 오류 또는 같은 원인의 작은 오류 묶음만 수정한다.

## 다음 권장 버전
