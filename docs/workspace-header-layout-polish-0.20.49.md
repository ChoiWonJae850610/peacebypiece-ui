# 0.20.49 WAFL 공통 헤더·PC 폭·세로형 액션 UI 보정

## 범위
- 작업지시서/원단·부자재 발주 PC 업무 화면의 공통 헤더 위치와 workspace padding 기준을 맞췄다.
- 모바일/태블릿 세로 목록 메뉴 버튼은 텍스트 없이 아이콘만 표시하도록 정리했다.
- 작업지시서 세로 화면 FAB는 첨부 진입으로 명확화했다.
- 원단·부자재 발주 세로 화면 FAB는 작지·자재 선택 성격이 드러나도록 변경했다.
- 태블릿 가로라도 3패널 폭이 부족하면 드로어 기반 본문 집중 구조로 전환하도록 기준을 추가했다.
- 작업지시서 모바일/태블릿 제목·담당자·재고·분류 수정 affordance를 텍스트 버튼보다 작은 아이콘형으로 정리했다.

## 유지한 것
- PC/태블릿 가로의 기존 상태전환, 권한, API, DB, R2 흐름은 변경하지 않았다.
- 모바일 진행단계는 세로형, 태블릿/PC 진행단계는 가로형 정책을 유지했다.
- 직접그리기 최상단 modal layer 수정 흐름은 건드리지 않았다.

## 테스트 포인트
1. 작업지시서 PC와 원단·부자재 PC를 전환했을 때 헤더 top/padding/폭이 흔들리지 않는지 확인한다.
2. 협력업체/저장소 등 다른 workspace 화면과 원단·부자재 화면의 상단 폭 기준이 과도하게 다르지 않은지 확인한다.
3. 모바일/태블릿 세로에서 목록 메뉴 버튼에 텍스트가 붙지 않고 아이콘만 보이는지 확인한다.
4. 작업지시서 세로 FAB가 첨부로 표시되는지 확인한다.
5. 원단·부자재 세로 FAB가 작지·자재로 표시되고 돋보기 아이콘이 보이는지 확인한다.
6. iPad mini 등 폭이 좁은 태블릿 가로에서 원단·부자재 3패널이 무리하게 나오지 않는지 확인한다.
