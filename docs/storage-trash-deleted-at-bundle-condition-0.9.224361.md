# 0.9.224361 — 저장소 휴지통 삭제일시 표시 조건 보정

## 목적

작업지시서와 함께 삭제된 문서/디자인/메모만 작업지시서 삭제일시를 표시하고, 작업지시서 삭제 전에 이미 단독 삭제되어 있던 파일은 자기 자신의 삭제일시를 유지한다.

## 문제

0.9.22436에서는 부모 작업지시서가 삭제된 상태이면 첨부파일의 `deleted_at` 대신 작업지시서의 `deleted_at`을 우선 표시했다.

이 조건은 너무 넓어서, 예전에 단독 삭제된 첨부파일도 나중에 작업지시서를 삭제하면 화면에서 같은 삭제일시로 보이는 문제가 있었다.

## 보정 기준

작업지시서 삭제일시를 표시하는 조건을 아래로 제한한다.

- 부모 작업지시서가 삭제 상태일 것
- 첨부 휴지통 row가 작업지시서 묶음 삭제 metadata를 가질 것
- `delete_parent_id` 또는 `delete_batch_id`가 현재 `order_id`와 일치할 것

위 조건을 만족하지 않으면 첨부 자체의 `deleted_at`을 표시한다.

## 기대 동작

- 작업지시서와 함께 삭제된 첨부: 작업지시서 삭제일시와 동일하게 표시
- 작업지시서 삭제 전에 이미 단독 삭제된 첨부: 기존 첨부 삭제일시 유지
- 복원/삭제 요청 정책은 기존 정책을 유지

## 확인 항목

1. 예전에 단독 삭제한 디자인 파일의 삭제일시가 유지되는지 확인
2. 방금 작업지시서와 함께 삭제한 문서/디자인/메모 삭제일시가 작업지시서와 동일한지 확인
3. 휴지통 상세 모달의 삭제일시가 목록과 같은지 확인
4. `/system/storage-usage` 후보 조회에 영향이 없는지 확인
