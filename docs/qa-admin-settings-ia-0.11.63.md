# 0.11.63 관리자 환경설정 IA 재구성 1차

## 목표

고객관리자 환경설정 화면에서 개인 설정과 회사 설정이 섞여 보이던 구조를 정리하고, 모달 중심이던 요금제·계정·개발건의 정보를 같은 화면 하단 패널로 전환한다.

## 반영 내용

- `회사·화면 설정` 카드를 제거했다.
- 화면 테마와 언어 설정은 환경설정 화면에서 다루지 않고 우측 상단 개인 설정 영역에서 관리하는 방향으로 안내한다.
- 환경설정 메뉴는 4개 탭형 카드로 정리했다.
  - 기준정보 설정
  - 요금제·저장공간
  - 계정 정보
  - 개발 건의
- 요금제·저장공간, 계정 정보, 개발 건의는 더 이상 모달로 열지 않고 하단 패널에서 직접 표시한다.
- 기준정보 설정은 기존 `AdminStandardsSection` 흐름을 유지한다.

## 유지한 정책

- 실제 요금제 변경은 고객관리자가 직접 처리하지 않는다.
- 요금제·저장공간은 읽기 전용 확인 중심으로 유지한다.
- 계정 정보는 조직 설정과 개인 설정의 분리 기준을 안내하는 수준으로 유지한다.
- 개발 건의는 현재 이메일 접수 흐름을 유지한다.

## 변경하지 않은 것

- 기준정보 저장 API
- 회사 설정 API
- 결제/요금제 실제 변경 API
- 멤버 권한/승인 API
- 작업지시서/저장소/R2 흐름
- DB schema

## 확인 항목

1. `/admin/settings` 진입 시 기준정보 설정 탭이 기본으로 선택되는지 확인한다.
2. `회사·화면 설정` 카드가 사라졌는지 확인한다.
3. 화면 테마/언어 설정 블록이 보이지 않는지 확인한다.
4. 요금제·저장공간 카드를 누르면 하단 패널에서 현재 요금제와 저장공간 정보가 표시되는지 확인한다.
5. 계정 정보 카드를 누르면 하단 패널에서 계정/개인 설정 분리 안내가 표시되는지 확인한다.
6. 개발 건의 카드를 누르면 하단 패널에서 이메일 접수 정보와 이메일 작성 버튼이 표시되는지 확인한다.
7. 기준정보 설정 탭의 기존 기준정보 관리 동작이 유지되는지 확인한다.
