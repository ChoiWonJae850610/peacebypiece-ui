# 작업지시서 직접 그리기 native canvas 보완 (0.12.22)

## 목적

0.12.21에서 추가한 작업지시서 직접 그리기 PoC를 tablet 테스트 기준으로 한 단계 보완한다. 이번 버전은 `tldraw` 도입 전 native canvas 방식의 사용성을 먼저 검증하는 단계이며, 기존 디자인 첨부 업로드 흐름을 유지한다.

## 이번 버전 반영 범위

- 펜 색상 선택 추가
  - 검정
  - 빨강
  - 파랑
  - 초록
- 펜 굵기 선택 추가
  - 얇게
  - 기본
  - 굵게
  - 넓게
- undo / redo 추가
- 전체 지우기 후에도 undo로 이전 그림 복귀 가능
- mobile 안내 문구를 간단 스케치/가로 모드 중심으로 정리
- 저장 방식은 기존과 동일하게 PNG `File` 생성 후 기존 디자인 첨부 업로드 흐름 재사용

## 유지한 범위

- package.json / package-lock.json 변경 없음
- DB schema 변경 없음
- R2 Worker / 첨부 API / 휴지통 / purge 흐름 변경 없음
- 별도 drawing table 추가 없음
- tldraw 도입 없음

## 구현 판단

### native canvas를 한 번 더 보완한 이유

0.12.21 기준 tablet 입력이 확인되었으므로, 바로 외부 라이브러리를 도입하기보다 현재 PoC에서 실제 필요한 최소 편집 기능을 먼저 검증하는 편이 안전하다. 색상, 굵기, undo/redo는 사용성 판단에 직접 영향을 주는 기능이다.

### tldraw 도입 보류

`tldraw`는 여전히 1차 full editor 후보지만, 현재 단계에서는 dependency 추가 없이 다음 항목을 먼저 확인한다.

- tablet 펜 입력 감도
- modal scroll 충돌 여부
- 저장 이미지 품질
- 기존 디자인 첨부 업로드 흐름과의 연결 안정성
- mobile 제한 모드 필요성

## 테스트 체크리스트

### PC

- 마우스로 선 그리기
- 색상 변경 후 선 색상 반영
- 굵기 변경 후 선 굵기 반영
- 지우개 동작
- undo / redo 동작
- 전체 지우기 후 undo 동작
- 디자인으로 저장 후 첨부 목록 표시

### Tablet

- 펜 입력 확인
- 손가락 입력 확인
- 손바닥/스크롤 충돌 여부 확인
- 굵은 선과 얇은 선의 차이 확인
- undo / redo 버튼 터치 가능성 확인
- 저장 버튼 위치와 모달 닫힘 확인

### Mobile

- 세로 화면에서 모달이 과하게 좁지 않은지 확인
- 가로 모드 사용 안내가 자연스러운지 확인
- 최소 기능으로 저장까지 가능한지 확인

## 다음 판단

0.12.22 적용 후 아래 기준으로 다음 방향을 선택한다.

### A안: native canvas 유지 보완

- 현재 수준이 충분히 가볍고 빠른 경우
- 추가 후보: 텍스트 삽입, 도형, 저장 전 미리보기, 캔버스 비율 선택

### B안: tldraw PoC 전환

- 도형/텍스트/선택/이동/확대축소 같은 full editor 기능이 필요하다고 판단되는 경우
- package.json / package-lock.json 변경 필요
- 기존 native canvas modal과 비교 테스트 필요

### C안: hybrid

- PC/tablet은 tldraw
- mobile은 native simple canvas

현재 판단은 `A안으로 한 번 더 확인 후 B안 여부 결정`이 가장 안전하다.
