# 시스템 카테고리 규칙 화면 공통 UI 적용 — 0.11.12

## 목적

시스템관리자 카테고리 규칙 편집 화면에서 화면별로 직접 작성하던 버튼, 상태 라벨, empty state 일부를 관리자 공통 UI 컴포넌트 기준으로 전환한다.

## 반영 내용

- `/system/category-rules` 헤더의 버전 라벨을 `AdminStatusBadge`로 전환
- 헤더의 시스템 콘솔/초기화/저장/분류값 설정 액션을 `AdminLinkButton` 또는 `AdminButton`으로 전환
- 규칙 목록의 추가 버튼과 활성/비활성 라벨을 공통 컴포넌트로 전환
- 규칙 편집 영역의 복제/삭제 버튼과 활성 상태 라벨을 공통 컴포넌트로 전환
- 모바일 규칙 drawer의 닫기/추가 버튼과 상태/키워드 라벨을 공통 컴포넌트로 전환
- 분류값 모달의 추가/삭제/저장/초기화 버튼을 공통 버튼 기준으로 정리

## 제외

- 카테고리 규칙 저장 방식 변경 없음
- localStorage 기반 category rules persistence 변경 없음
- 카테고리 tree 구조 변경 없음
- 드래그/순서 변경 동작 변경 없음
- 모바일/태블릿 레이아웃 정식 대응은 별도 단계로 분리
