# 0.9.2236 — 저장소 휴지통 실제 복구/영구삭제 동작 점검

## 목적

저장소 관리 휴지통에서 파일/작업지시서 복구와 영구삭제 요청이 실제 처리 건수를 기준으로 표시되도록 보정한다.

## 변경 요약

- 파일 복구/영구삭제 API가 실제 affectedCount가 0인 경우 실패로 응답한다.
- 파일 복구/영구삭제 action flow가 requestedCount/affectedCount를 보존한다.
- 선택 복구/영구삭제 메시지가 선택 개수 대신 실제 처리 개수를 기준으로 표시된다.
- 작업지시서 복구/영구삭제 응답에 연결 첨부/메모 처리 개수를 포함한다.
- 작업지시서 복구/영구삭제 성공 메시지에 연결 첨부/메모 처리 개수를 반영한다.

## 확인 방법

1. `/admin/files`에서 휴지통 파일 1개를 복구한다.
2. 목록에서 해당 항목이 사라지고 메시지에 실제 복구 개수가 표시되는지 확인한다.
3. 이미 처리된 항목을 다시 처리하려고 할 때 실패 메시지가 나오는지 확인한다.
4. 작업지시서 휴지통 항목을 복구한다.
5. 작업지시서와 연결 첨부/메모 개수가 메시지에 반영되는지 확인한다.
6. 작업지시서 영구삭제는 R2 직접 삭제가 아니라 purge 요청 상태로 표시되는지 확인한다.

## DB/스토리지 정책

- DB schema 변경 없음.
- R2 직접 삭제 없음.
- 파일 영구삭제 버튼은 attachment trash row를 purge 요청 상태로 표시한다.
- 실제 R2 purge는 Worker 기반 purge 흐름에서 처리한다.
