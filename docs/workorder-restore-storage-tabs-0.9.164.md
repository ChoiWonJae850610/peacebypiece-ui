# 0.9.164 작업지시서 복원 정책 및 고객관리자 저장소 탭 구조

## 목적

0.9.163에서 작업지시서 삭제 시 연결된 디자인, 첨부파일, 메모가 함께 휴지통 상태로 이동되는 흐름을 적용했다. 0.9.164에서는 이 삭제 묶음을 어떻게 복원하고, 고객관리자 저장소에서 어떻게 보여줄지 정책과 화면 구조를 확정한다.

이번 버전은 문서화 전용이다. 실제 복원 API, 저장소 UI 개편, DB schema 변경, R2 삭제 흐름 변경은 하지 않는다.

## 확정 정책

### 1. 작업지시서 삭제 단위

작업지시서 삭제는 단일 row 삭제가 아니라 작업지시서 묶음 삭제로 본다.

삭제 대상 묶음:

- 작업지시서 본문
- 디자인 파일
- 일반 첨부파일
- 메모
- 메모에 연결된 첨부파일

삭제 시 R2 원본과 썸네일은 즉시 삭제하지 않는다. 기존 30일 휴지통 정책에 따라 복원 가능 상태로 유지한다.

### 2. 작업지시서 복원 단위

작업지시서를 복원할 때는 삭제 당시 함께 휴지통으로 이동된 항목을 같이 복원하는 것을 기본 정책으로 한다.

복원 대상:

- 작업지시서 `is_active=true`
- 연결 디자인/첨부파일 `is_active=true`, `deleted_at=null`
- 연결 메모 `is_active=true`, `deleted_at=null`
- 해당 attachment trash item은 `restored` 또는 목록 제외 상태로 전환

작업지시서 삭제 이후 사용자가 개별 첨부를 별도로 영구 삭제 요청했거나 시스템관리자 purge가 완료된 파일은 복원할 수 없다. 이 경우 작업지시서는 복원되더라도 해당 파일은 복원 불가로 표시한다.

### 3. 개별 파일 삭제와 작업지시서 묶음 삭제 구분

저장소 휴지통에서는 삭제 원인을 구분해야 한다.

권장 구분:

- 개별 파일 삭제: 사용자가 파일 카드에서 삭제
- 작업지시서 삭제 연동: 작업지시서 삭제로 연결 파일이 함께 휴지통 이동
- 메모 첨부 삭제: 메모 또는 작업지시서 삭제와 함께 숨김

현재 DB schema를 변경하지 않는 단계에서는 `delete_reason` 또는 표시용 매핑으로 우선 구분한다. schema 확장은 별도 버전에서 검토한다.

## 고객관리자 저장소 탭 구조

기존 구조:

```text
첨부파일목록 / 휴지통
```

권장 구조:

```text
작업지시서 / 첨부파일목록 / 휴지통
```

### 1. 작업지시서 탭

목적: 작업지시서 단위의 저장소 영향과 삭제/복원 상태를 확인한다.

표시 후보:

- 작업지시서명
- 현재 상태
- 디자인/첨부 파일 수
- 원본 용량
- 썸네일 용량
- 메모 수
- 삭제일
- 복원 가능 기한
- 복원 상태

초기 1차 화면은 read-only 또는 복원 준비 버튼 비활성 상태로 시작하는 것이 안전하다.

### 2. 첨부파일목록 탭

목적: 현재 활성 상태의 디자인/첨부/메모첨부 파일을 파일 단위로 관리한다.

표시 후보:

- 파일명
- 작업지시서명
- 파일 종류
- 원본 용량
- 썸네일 용량
- 등록일
- 삭제 버튼

개별 파일 삭제는 기존처럼 파일 단위 휴지통 이동으로 처리한다.

### 3. 휴지통 탭

목적: 30일 이내 복원 가능한 작업지시서 묶음과 파일 항목을 함께 관리한다.

필터 후보:

- 전체
- 작업지시서
- 첨부파일
- 메모첨부
- 삭제 예정 도래

표시 후보:

- 유형
- 작업지시서명
- 파일명 또는 묶음명
- 삭제일
- 복원 가능 기한
- 삭제 원인
- 원본/썸네일 용량
- 복원 버튼

## 복원 정책 상세

### 작업지시서 단위 복원

작업지시서 단위 복원은 묶음 복원이다.

```text
작업지시서 복원
→ 작업지시서 active 처리
→ 함께 삭제된 attachments active 처리
→ 함께 삭제된 memos active 처리
→ trash item restored 처리
```

### 파일 단위 복원

개별 첨부파일 복원은 파일만 복원한다. 작업지시서가 삭제 상태라면 파일만 복원하지 않는다.

권장 방어 정책:

```text
부모 작업지시서가 삭제 상태이면 개별 파일 복원 버튼 비활성
안내: 작업지시서를 먼저 복원해야 합니다.
```

### purge 완료 항목

R2 purge 완료 항목은 복원할 수 없다.

```text
purge_status = purged
→ 복원 불가
→ UI에서 복원 버튼 숨김 또는 비활성
```

## 용량 정책

작업지시서 삭제 시 active 용량은 줄고 trash 용량은 증가한다. 총 R2 저장량은 purge 전까지 변하지 않는다.

```text
삭제 전: active 용량
삭제 후: trash 용량
purge 후: 총 저장량 감소
```

용량 통계는 원본 `storage_key`와 썸네일 `thumbnail_key`를 함께 고려해야 한다.
