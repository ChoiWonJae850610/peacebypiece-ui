# /functions 기능 카탈로그 기반 — 0.23.62

## 목적

WAFL의 화면별 기능을 정상·예외·권한·DB 무결성·회사 격리·반응형·성능·PDF 시나리오로 관리하고, 이후 자동 테스트 파일과 연결할 수 있는 내부 기준 화면을 만든다.

## 적용 내용

- 내부 전용 `/functions` route 추가
- production 및 미허용 runtime에서 404 처리
- 기능 ID, 표시 순서, 영역, route, 역할, 자동화 상태, 출시 차단 여부 정의
- 전제 조건, 예상 UI/API 결과, 변경 DB 필드, 불변 DB 필드 구조 정의
- 검색과 영역·분류·자동화 상태 필터 제공
- 기능 상세 및 카탈로그 요약 제공
- 작업지시서, 자재 발주, 고객사 관리자, 시스템관리자, 회사 격리, 개인 설정, PDF, 성능의 1차 대표 기능 등록

## 범위 제한

0.23.62에서는 테스트 실행 버튼이나 DB 연결을 추가하지 않는다. 카탈로그 데이터는 코드와 함께 버전 관리하며, 0.23.63에서 시나리오 스키마를 확장하고 자동화 파일 연결 계약을 정의한다.

## 접근 정책

`NEXT_PUBLIC_APP_RUNTIME_MODE`가 development, dev, local, test, demo 중 하나일 때만 접근 가능하다. 값이 없거나 production 또는 알 수 없는 값이면 `/functions`는 404를 반환한다.

## 위험 요소

- 등록된 기능은 1차 대표 목록이며 전체 기능 수를 의미하지 않는다.
- 자동화 상태는 실제 테스트 통과 여부가 아니라 구현 연결 상태를 나타낸다.
- PDF와 마지막 관리자 탈퇴 정책은 사용자 결정이 필요하므로 `정책 결정 필요`로 표시한다.
