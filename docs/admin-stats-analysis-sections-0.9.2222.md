# 0.9.2222 — 통계 분석 섹션 재구성

## 목적

고객관리자 통계정보 화면을 현재 시점 요약과 기간별 분석으로 분리한 0.9.2221 기준을 이어서, 분석 섹션의 실제 표시 기준을 더 명확히 정리한다.

## 반영 내용

- 기간별 리오더 TOP5를 생산 차수 요약이 아니라 작업지시서/제품명 기준으로 표시한다.
- 생산품 유형 분석은 1차 / 2차 / 3차 이상 선택값에 따라 도넛 그래프와 TOP5가 함께 바뀌도록 한다.
- 업체 성과 테이블은 제작 건수와 납기 지연율, 검수/불량률을 한 화면에서 확인하는 구조로 정리한다.
- 현재 DB에서 계산 가능한 지표는 실제 집계값으로 표시하고, 데이터가 부족한 경우에는 준비중 또는 빈 상태로 표시한다.

## 화면 기준

대상 화면은 `/admin/dashboard`이다.

## 데이터 기준

### 리오더 TOP5

`spec_sheets` 기준으로 리오더 또는 재작업 후보를 집계한다.

- `reorder_round > 1`
- 또는 `is_rework = true`
- 제품명 기준은 `title`을 우선 사용한다.

### 생산품 유형 분석

`spec_sheets` 기준으로 생산 차수별 카테고리를 집계한다.

- 1차: `reorder_round <= 1`
- 2차: `reorder_round = 2`
- 3차 이상: `reorder_round >= 3`

### 업체 성과

`orders` 기준으로 업체별 제작 건수와 납기 지연 후보를 계산한다.
검수/불량률은 연결된 `spec_sheets`의 반려/재작업 후보를 기준으로 1차 계산한다.

## 제외한 내용

- DB schema 변경 없음
- package 변경 없음
- 저장소 상세 통계 재추가 없음
- 요금제별 통계 분기 재도입 없음
