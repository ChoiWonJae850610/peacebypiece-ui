# 0.9.224371 — 작업지시서 복원/휴지통 메모 묶음 기준 보정

## 목표
작업지시서 복원 시 작업지시서 삭제 전에 사용자가 이미 단독 삭제한 메모가 같이 복원되는 문제를 막는다.

## 정책
작업지시서 복원, 휴지통 메모 수, 시스템관리자 실제 삭제 후보 메모 수는 모두 같은 기준을 사용한다.

포함 대상:
- 작업지시서 삭제 당시 살아 있던 메모
- 작업지시서 삭제 처리로 함께 삭제된 메모
- `delete_source = workorder_bundle` 또는 `delete_scope = bundle` / `delete_parent_type = workorder` 메타데이터가 있고
- `delete_parent_id` 또는 `delete_batch_id`가 현재 작업지시서 id와 일치하는 메모

제외 대상:
- 작업지시서 삭제 전에 사용자가 단독 삭제한 메모
- 댓글 때문에 원글만 삭제 표시로 남은 메모
- 작업지시서 삭제 시점과 시간이 가까울 뿐 묶음 삭제 메타데이터가 없는 메모

## 변경 내용
- `restoreWorkOrderTrashBundle()`의 `restored_memos`에서 삭제 시각 10초 근접 fallback 제거
- 메모 복원 조건을 작업지시서 묶음 삭제 메타데이터 기준으로 제한
- `/admin/files` 작업지시서 휴지통 row의 `메모 n개` 계산에서 묶음 삭제 메모만 포함
- `/system/storage-usage` 작업지시서 실제 삭제 후보의 `메모 n개` 계산에서 묶음 삭제 메모만 포함

## 검증
1. 메모 C를 먼저 단독 삭제한다.
2. 작업지시서를 삭제한다.
3. 휴지통에서 작업지시서 row의 메모 수를 확인한다.
4. 작업지시서를 복원한다.
5. 메모 C가 복원되지 않고 계속 삭제 상태인지 확인한다.
6. 작업지시서 삭제 당시 살아 있던 메모만 복원되었는지 확인한다.
