# 0.9.22401 저장소 관리 요약/요금제 용량 표시 보정

## 목적

저장소 관리 화면 새로고침 직후 아주 짧게 `삭제 요청 0개` 위치에 `30일` 값이 빨간색으로 표시되는 초기 렌더링 깜빡임을 제거한다.

또한 저장소 용량 한도는 고객 정보의 요금제/파일 정책에서 내려오는 값만 사용하도록 화면의 초기 placeholder를 정리한다.

## 변경 내용

- `getAdminFileManagementSnapshot()`의 placeholder usage card 네 번째 항목을 `복원 가능 기간 30일`에서 `삭제 요청 0개`로 변경했다.
- 초기 placeholder의 저장소 한도값을 `5.0GB`로 고정하지 않고 `요금제 확인 중` 상태로 표시하도록 변경했다.
- 실제 `/api/admin/files/snapshot` 응답은 기존처럼 `company_settings.storage_limit_gb` 기반 `settings.filePolicy.storageLimitGb`를 사용한다.
- `FileStorageSummary`의 요금제 카드가 `limitBytes <= 0`인 초기 상태에서는 가짜 용량 대신 로딩성 문구를 표시하도록 보정했다.

## 기대 효과

- 새로고침 직후 `삭제 요청` 카드에 `30일`이 잠깐 보이는 현상이 사라진다.
- 실제 데이터 수신 전에는 5.0GB 같은 하드코딩 용량이 보이지 않는다.
- 시스템관리자에서 고객 요금제 용량을 다르게 설정하면 snapshot API 응답 기준으로 저장소 관리 화면에 반영된다.

## DB 변경

없음.

## 확인 항목

1. `/admin/files` 새로고침 직후 삭제 요청 카드가 `0개`로 유지되는지 확인한다.
2. 새로고침 직후 `30일`이 삭제 요청 위치에 나타나지 않는지 확인한다.
3. 실제 snapshot 로드 후 저장소 한도가 고객 파일 정책의 `storage_limit_gb` 값 기준으로 표시되는지 확인한다.
4. 시스템관리자에서 요금제/용량 변경 테스트 후 `/admin/files`에 반영되는지 확인한다.
