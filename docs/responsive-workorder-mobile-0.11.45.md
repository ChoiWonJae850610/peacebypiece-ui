# 작업지시서 화면 모바일 보정 2차 (0.11.45)

## 목적

0.11.44에서 작업지시서 모바일 레이아웃의 기본 wrapper와 상세 카드 overflow를 1차 보정했다. 0.11.45에서는 첨부, 메모, 디자인, 발주정보, 생산구성 영역 중 좁은 폭에서 실제 입력·읽기 동선에 영향을 주기 쉬운 부분만 추가 보정한다.

## 적용 범위

- 작업지시서 생산구성 모바일 섹션
- 원단/부자재 카드형 입력 영역
- 외주공정 카드형 입력 영역
- 작업지시서 첨부/디자인 패널
- 작업지시서 메모 패널
- 공통 인라인 편집 필드와 모바일 섹션 헤더

## 변경 내용

### 1. 생산구성 모바일 카드 폭 보정

원단/부자재와 외주공정 모바일 카드에 `min-w-0`을 보강하고, 작은 폭에서는 수량·단가·금액 필드가 1열로 내려가도록 조정했다. 380px 이상에서는 기존처럼 2열 사용이 가능하도록 했다.

### 2. 섹션 헤더 요약 줄바꿈 보정

생산구성처럼 요약 문자열이 긴 영역은 모바일에서 말줄임으로 정보가 과하게 잘릴 수 있다. 모바일에서는 요약이 줄바꿈되고, `sm` 이상에서는 기존처럼 한 줄 말줄임으로 유지되도록 했다.

### 3. 첨부/디자인 패널 모바일 보정

첨부 파일명, 소유자 라벨, 업로드 안내 문구가 모바일에서 잘리지 않도록 `break-words`와 `min-w-0`을 보강했다. 모바일에서는 첨부 패널 padding을 줄이고, 빈 상태와 업로드 안내 영역의 폭 안정성을 높였다.

### 4. 드로잉 placeholder 모달 모바일 보정

직접 그리기 placeholder 모달은 아직 실제 드로잉 기능이 아니지만, 모바일에서 버튼과 본문이 화면 밖으로 밀리지 않도록 padding과 footer 버튼 폭을 보정했다.

### 5. 메모 패널 모바일 보정

메모 작성 action row가 작은 폭에서 줄바꿈될 수 있도록 하고, 작성자명/본문/답글이 긴 경우 화면 밖으로 밀리지 않도록 `break-words`를 적용했다.

## 변경하지 않은 것

- 작업지시서 저장 정책
- 제목/담당자 즉시 저장 정책
- 발주정보/생산구성 버튼 저장 정책
- 첨부 업로드/download/delete/R2 흐름
- 메모 생성/수정/삭제 로직
- 직접 그리기 기능의 실제 저장 구조
- DB schema
- API 응답 포맷

## 테스트 권장 항목

2. 생산구성 섹션 열기/닫기
3. 원단/부자재 항목 추가, 삭제, 수량/단가 입력
4. 외주공정 항목 추가, 삭제, 수량/단가 입력
5. 디자인 첨부 목록의 긴 파일명 표시 확인
6. 일반 첨부 목록의 긴 파일명 표시 확인
7. 직접 그리기 placeholder 모달 열기/닫기
8. 메모 작성, 답글 작성, 긴 문장 표시 확인
9. 기존 첨부 업로드/미리보기/삭제 동작 확인
