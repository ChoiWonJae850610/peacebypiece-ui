# 0.11.54 QA 수정 — 협력업체 외주 공정 저장 보정

## 배경

협력업체 수정 모달에서 외주 공정을 추가한 뒤 저장하면 화면상 선택은 되었지만 DB 저장이 실패하는 문제가 확인되었다.

## 원인

협력업체 관리 화면의 외주 공정 선택지는 시스템 기준정보(`system_outsourcing_process_standards`)의 ID를 사용한다. 반면 협력업체 품목 연결 테이블(`partner_items`)의 `outsourcing_process_id`는 고객사 외주 공정 테이블(`outsourcing_processes`)을 참조한다.

따라서 시스템 기준정보 ID를 그대로 `partner_items.outsourcing_process_id`에 저장하면 FK 제약에 걸릴 수 있다.

## 수정 내용

- 협력업체 역할 품목 저장 시 외주 공정 ID를 고객사 외주 공정 ID로 해석하도록 보정했다.
- 시스템 기준정보 ID가 들어오면 같은 이름의 고객사 외주 공정을 찾아 연결한다.
- 고객사 외주 공정이 없으면 해당 시스템 기준정보 이름으로 고객사 외주 공정을 생성한 뒤 연결한다.
- 협력업체 목록을 다시 읽을 때는 고객사 외주 공정 이름을 시스템 기준정보와 매칭해 화면 선택 상태가 유지되도록 보정했다.

## 유지한 사항

- 협력업체 생성/수정 API 경로는 유지했다.
- 외주 공정 기준정보 관리 흐름은 변경하지 않았다.
- 작업지시서 저장, 첨부, R2, 휴지통 흐름은 변경하지 않았다.
- DB schema 변경은 하지 않았다.

## 확인 항목

1. `/admin/partners`에서 협력업체 수정 모달을 연다.
2. 유형에서 `외주`를 선택한다.
3. 외주 공정을 1개 이상 오른쪽 목록에 추가한다.
4. 저장한다.
5. 모달을 다시 열어 선택한 외주 공정이 유지되는지 확인한다.
6. 협력업체 목록의 유형/공정 표시가 외주 공정명으로 표시되는지 확인한다.
7. 작업지시서 발주/생산구성에서 외주 업체 옵션이 유지되는지 확인한다.
