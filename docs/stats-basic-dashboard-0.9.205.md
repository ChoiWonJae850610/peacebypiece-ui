# 0.9.205 — 고객 관리자 Basic 통계 1차

## 목적

0.9.204에서 도입한 Recharts를 고객 관리자 통계 화면에 실제로 연결한다. 이번 버전은 통계 API를 새로 만들지 않고 기존 `getAdminStatsSnapshot` 결과를 사용해 Basic 플랜에서 바로 확인 가능한 차트 영역을 안정화한다.

## 반영 범위

- 작업지시서 상태별 현황 차트를 Recharts BarChart 기반으로 표시한다.
- 협력업체 분포, 파일 사용량, 생산 단계 비율을 Recharts PieChart 기반 도넛 차트로 표시한다.
- 기존 통계 repository/API/SQL schema는 변경하지 않는다.
- 고급 통계, 요금제 잠금, export, TanStack Query 캐싱은 이번 범위에서 제외한다.

## Basic 통계 1차 기준

Basic 통계 화면에서 우선 노출하는 항목은 다음과 같다.

1. 전체 작업지시서 수
2. 협력업체 수
3. 파일 사용량
4. 완료된 작업지시서 수
5. 검토/검수/지연/리오더성 지표 카드
6. 작업지시서 상태별 현황 차트
7. 협력업체 분포 차트
8. 파일 사용량 차트
9. 생산 단계 비율 차트

## 구현 기준

- 차트 컴포넌트는 `components/admin/dashboard/AdminBasicStatsCharts.tsx`에 둔다.
- 기존 `AdminStatsDashboard`는 데이터 전달과 화면 조립만 담당한다.
- 통계 계산은 기존 selector/presentation/repository 계층을 유지한다.
- 차트 색상은 기존 관리자 테마 변수와 stone 계열을 사용한다.
- 데이터가 0건인 경우 기존 빈 상태 메시지를 유지한다.

## SQL DDL 필요 여부

불필요.

0.9.203에서 통계용 schema/index 1차 반영을 완료했으며, 이번 버전은 UI 차트 연결만 수행한다.

## 전체 리셋 필요 여부

불필요.

DB schema 변경이 없으므로 `full_reset.sql` 실행이 필요하지 않다.

## 테스트 케이스

1. `/admin/dashboard` 진입 시 페이지가 렌더링되는지 확인한다.
2. 기간 필터 7일/15일/30일/월별/누적 링크가 기존처럼 동작하는지 확인한다.
3. 작업지시서 현황 영역이 Recharts 막대 차트로 표시되는지 확인한다.
4. 협력업체 분포, 파일 사용량, 생산 단계 비율이 도넛 차트로 표시되는지 확인한다.
5. DB 데이터가 없는 경우에도 빈 상태 메시지와 0값 차트가 깨지지 않는지 확인한다.
6. `npm run build`를 실행해 Recharts import/type 오류가 없는지 확인한다.
