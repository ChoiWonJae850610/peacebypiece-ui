# 0.9.22378 시스템 저장소 선삭제 첨부 분리 보정

## 목적

작업지시서에 연결된 첨부파일 중 일부가 작업지시서 삭제 전에 이미 개별 삭제되어 고객관리자 휴지통에 남아 있던 경우를 분리한다.

## 정책

- 작업지시서 실제 삭제 대표 row에는 작업지시서 삭제 시 함께 휴지통으로 이동한 첨부파일만 포함한다.
- 작업지시서보다 먼저 개별 삭제된 첨부파일은 작업지시서 묶음 purge 대상이 아니다.
- 선삭제 첨부파일은 부모 작업지시서가 시스템관리자에 의해 purged 처리되어도 고객관리자 휴지통 목록에 계속 표시한다.
- 선삭제 첨부파일의 R2 객체가 남아 있는 것은 정상이다.
- 선삭제 첨부파일은 고객관리자가 별도로 영구삭제 요청하거나 보관 기간이 도래했을 때 시스템관리자 실제 삭제 후보로 이동한다.

## 수정 내용

- `/admin/files` 휴지통 조회에서 부모 작업지시서가 purged 상태여도 선삭제 첨부파일은 숨기지 않도록 조건을 보정했다.
- 시스템관리자 파일 purge 후보 조회에서도 부모 작업지시서가 purged 상태인 선삭제 첨부파일은 별도 파일 후보로 처리될 수 있도록 조건을 보정했다.
- 작업지시서 대표 row의 첨부 개수와 용량은 `delete_reason = '작업지시서 삭제로 함께 휴지통 이동'`인 bundle trash item만 기준으로 집계하도록 수정했다.

## 확인 시나리오

1. 작업지시서에 첨부 2개를 등록한다.
2. 첨부 1개를 먼저 개별 삭제한다.
3. 작업지시서를 삭제한다.
4. 작업지시서를 영구삭제 요청한다.
5. 시스템관리자가 작업지시서 대표 row를 실제 삭제한다.
6. 작업지시서 삭제와 함께 묶인 첨부만 R2 purge 처리되는지 확인한다.
7. 먼저 삭제한 첨부는 `/admin/files` 휴지통 목록에 계속 보이는지 확인한다.
8. 먼저 삭제한 첨부의 R2 객체가 남아 있는지 확인한다.
