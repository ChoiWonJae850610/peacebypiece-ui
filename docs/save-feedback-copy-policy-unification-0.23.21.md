# 저장 피드백 문구 정책 통일 0.23.21

## 목적
작업지시서, 자재 발주서, 공장 전달사항에서 서로 다르게 사용하던 `변경`, `저장`, `수정 완료` 표현을 저장 중심 문장 구조로 통일한다.

## 정책
- 필드 즉시 저장: `항목 저장 중입니다.` → `항목이 저장되었습니다.`
- 문서 전체 저장: `변경사항 저장 중입니다.` → `변경사항이 저장되었습니다.`
- 저장 실패: API 오류 메시지를 우선 사용하고, 오류 메시지가 없으면 `항목을 저장하지 못했습니다.`를 사용한다.
- loading/success/danger 상태는 동일한 operation toast id에서 교체한다.

## 적용 범위
- 자재 발주서 자재 종류, 공급처, 납기일 저장
- 작업지시서 전체 저장
- 작업지시서 공장 전달사항 저장

## 제외 범위
생성, 삭제, 상태 전환, PDF 생성처럼 저장과 다른 업무 동작의 문구는 이번 버전에서 변경하지 않는다.
