# 0.10.66 개발 건의 이메일 기반 1차 처리

## 목적

고객관리자 환경설정의 `개발 건의` 메뉴를 준비중 안내에서 이메일 접수 방식으로 전환한다.

이번 버전은 DB 저장이나 시스템관리자 검토 화면을 만들지 않고, 기본 메일 앱을 여는 `mailto:` 링크로 개선 요청, 오류 제보, 기능 제안을 접수하는 1차 방식이다.

## 변경 범위

- `/admin/settings` 개발 건의 카드 상태를 `이메일 접수`로 변경
- 개발 건의 모달에 접수 이메일과 안내 문구 표시
- `이메일 작성하기` 버튼 추가
- `mailto:` 제목/본문 템플릿 생성 유틸 추가

## 접수 이메일

현재 1차 접수 이메일은 중앙 상수로만 관리한다.

```ts
ADMIN_FEEDBACK_CONTACT_EMAIL
```

실제 운영 이메일이 확정되면 `lib/admin/settings/adminFeedbackContact.ts`의 값만 교체한다.

## 메일 템플릿

메일 제목은 기본적으로 다음 형식을 사용한다.

```text
PeaceByPiece 개선 요청
```

본문에는 다음 항목을 포함한다.

- 요청 유형
- 사용 중인 화면
- 현재 불편한 점 또는 오류 상황
- 원하는 개선 방향
- 첨부/캡처 여부

## 후속 확장 후보

- `feature_requests` DB 테이블 추가
- 고객관리자 화면 내 직접 접수 폼 추가
- 시스템관리자 개발 건의 검토 화면 추가
- 상태값 관리: `received`, `reviewing`, `planned`, `done`, `rejected`
- 감사 로그 연결

## 제외 범위

- DB 저장 없음
- 첨부파일 업로드 없음
- 자동 이메일 발송 없음
- 시스템관리자 검토 화면 없음
