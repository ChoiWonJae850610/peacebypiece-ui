# 0.10.44 — 고객관리자 기준정보 저장 오류 수정 및 업무 화면 선택지 연결

## 목적

고객관리자 기준정보 정책을 실제 업무 화면 선택지와 연결한다.

## 반영 내용

- 생산품 유형은 고객사별 직접 관리 구조를 유지한다.
- 생산품 유형 기본값 복원 후 저장 시 회사 전용 `item_categories` 기준으로 저장/조회되도록 보정한다.
- 단위 표준은 시스템 표준 원장과 고객사별 사용 여부를 기준으로 작업지시서 자재 단위 선택지에 반영한다.
- 작업지시서 외주 단가 기준은 고객사가 사용 처리한 단위에서 `{단위명}당` 형태로 파생한다.
- 외주공정 유형은 고객사별 사용 여부가 반영된 `processDefinitions`를 협력업체 등록/수정 모달의 외주공정 선택지로 사용한다.

## 변경하지 않은 것

- DB schema 변경 없음
- 시스템 기준정보 원장 CRUD 변경 없음
- 고객관리자 생산품 유형의 1차 → 2차 → 3차 직접 관리 정책 변경 없음
- 감사 로그 흐름 변경 없음
