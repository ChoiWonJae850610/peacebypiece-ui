# 작업지시서 UI 공통화 1차 — 0.19.46

## 목적

0.19.45에서 정리한 적용 범위를 기준으로 작업지시서 화면의 기능 흐름을 건드리지 않고 WAFL 공통 버튼 규격을 부분 적용한다.

## 적용 범위

- 작업지시서 홈 버튼
- 작업지시서 목록 카드 우측 추가 작업 버튼
- 우측 첨부/디자인 패널의 추가 작업 메뉴 버튼
- 자재 발주 PDF 생성 버튼
- 메모 수정/삭제 icon-only 버튼

## 유지 범위

- PC 3패널 구조
- 태블릿 split 구조
- 모바일 stack/sheet 구조
- 작업지시서 DB 저장 흐름
- R2 첨부/디자인 업로드 흐름
- 메모 작성/수정/삭제 흐름
- PDF 생성 요청 흐름
- workflow state 변경 흐름

## 구현 기준

- `components/workorder/common/WorkOrderActionButton.tsx`를 추가해 작업지시서 전용 adapter를 둔다.
- 기본 icon-only 버튼은 `WaflActionButton`을 내부 사용한다.
- 목록 카드 active 상태는 작업지시서 선택 카드의 어두운 배경과 충돌하지 않도록 active class를 adapter에서 별도 처리한다.
- 메모 수정/삭제처럼 기존 UI 밀도가 작은 버튼은 `WorkOrderMiniActionButton`으로 크기만 제한하고 tone은 WAFL ActionButton 기준을 따른다.

## 테스트 기준

- 작업지시서 홈 버튼 이동이 기존과 동일해야 한다.
- 목록 카드 선택/상세 진입이 기존과 동일해야 한다.
- 목록 카드 우측 추가 작업 메뉴 열기/닫기, Escape/외부 클릭 닫기가 유지되어야 한다.
- 삭제 메뉴의 권한/상태 제한이 기존과 동일해야 한다.
- 첨부/디자인 메뉴에서 파일 추가, 기본 디자인 지정, 삭제, 미리보기 흐름이 기존과 동일해야 한다.
- 자재 발주 PDF 생성 버튼 동작이 기존과 동일해야 한다.
- 메모 수정/삭제 버튼 동작이 기존과 동일해야 한다.
- PC/태블릿/모바일 layout과 breakpoint가 변하면 안 된다.
