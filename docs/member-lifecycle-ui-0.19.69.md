# 0.19.69 멤버 탈퇴/비활성 UI 1차

## 목적

멤버관리 상세 모달에서 멤버의 생애주기 상태를 직접 확인하고 변경할 수 있도록 UI를 추가했다. 이번 변경은 이미 0.19.67에서 반영된 DB/API 상태값을 화면에서 선택할 수 있게 연결하는 1차 작업이다.

## 반영 범위

- 멤버 상세 모달에 `멤버 상태` 섹션 추가
- 상태 선택 select 추가
- 재직중/비활성/탈퇴 요청/탈퇴 완료 빠른 선택 버튼 추가
- 상태 변경 기준 안내 문구 추가
- ko/en i18n 문구 추가

## 상태 기준

- `approved`: 재직중
- `suspended`: 비활성
- `withdrawal_requested`: 탈퇴 요청
- `withdrawn`: 탈퇴 완료

## 의도적으로 제외한 범위

- 개인 프로필의 본인 탈퇴 요청 버튼
- 탈퇴 요청 전용 승인/거절 별도 API
- 멤버관리 목록 행별 즉시 처리 버튼
- 권한 판정 로직 변경
- 담당자 후보 조회 로직 변경
- DB schema 변경

## 테스트 포인트

1. 멤버관리 목록에서 기존 멤버 row를 클릭한다.
2. 상세 모달에 `멤버 상태` 섹션이 표시되는지 확인한다.
3. 상태 select에서 재직중/비활성/탈퇴 요청/탈퇴 완료를 선택할 수 있는지 확인한다.
4. 빠른 선택 버튼을 눌렀을 때 상태 select 값이 같이 바뀌는지 확인한다.
5. 저장 버튼을 눌렀을 때 기존 `/api/admin/members/{memberId}` PATCH 흐름으로 저장되는지 확인한다.
6. 본인 계정을 비활성/탈퇴 완료로 변경하려고 할 때 기존 self status 방어가 유지되는지 확인한다.
7. 권한 체크 저장, 역할 변경, 초대/승인 목록이 기존과 동일하게 동작하는지 확인한다.
