# 0.9.22392 — 저장소 관리 최종 회귀 테스트 및 안정화 확인

## 기준

- 기준 원본: 0.9.22391
- 결과 버전: 0.9.22392
- 범위: 고객관리자 `/admin/files` 저장소 관리 화면의 휴지통 문구, 선택 삭제/삭제 요청 정책, 작업지시서 묶음 row 표시 기준 최종 점검

## 이번 점검에서 정리한 문구 기준

- 고객관리자 휴지통 화면의 되돌리기 동작은 `복원`으로 표시한다.
- 고객관리자가 실행하는 삭제 계열 동작은 실제 R2 삭제가 아니라 `선택 삭제` 또는 `삭제 요청`으로 표시한다.
- 고객관리자 화면에서는 `영구삭제`라는 표현을 노출하지 않는다.
- 축약 표현 `작지`는 사용하지 않고, 짧은 배지는 `작업`으로 표시한다.
- 실제 삭제는 시스템관리자 `/system/storage-usage`의 Worker 기반 purge 흐름에서만 처리한다.

## 회귀 확인 항목

1. `/admin/files` 휴지통 목록이 표시되는지 확인한다.
2. 첨부파일 단독 삭제 후 `복원`이 가능한지 확인한다.
3. 첨부파일 단독 삭제 후 `선택 삭제` 실행 시 고객관리자 목록에서 제외되고 시스템관리자 실제 삭제 후보로 이동하는지 확인한다.
4. `휴지통 비우기` 실행 시 “휴지통의 모든 항목을 삭제 요청하시겠습니까?” 확인 모달이 표시되는지 확인한다.
5. 작업지시서 삭제 후 `복원` 시 작업지시서와 작업지시서 삭제로 함께 휴지통 이동한 첨부/메모가 함께 복원되는지 확인한다.
6. 작업지시서 삭제 후 `선택 삭제` 시 고객관리자 휴지통 기본 목록에서 제외되고 시스템관리자 실제 삭제 후보로 이동하는지 확인한다.
7. 작업지시서 삭제 전 먼저 개별 삭제된 첨부파일이 작업지시서 묶음 purge 대상에서 제외되고 고객관리자 휴지통에 남는지 확인한다.
8. 작업지시서 묶음 첨부파일이 선택 불가 row 톤으로 표시되는지 확인한다.
9. 리오더 작업지시서명이 휴지통 목록, 상세 모달, 연결 첨부 row에서 동일하게 표시되는지 확인한다.
10. 컬럼 정렬이 대상, 삭제일시, 작업지시서, 유형, 크기 기준으로 동작하는지 확인한다.
11. 선택 가능 row와 선택 불가 묶음 row 톤이 구분되는지 확인한다.

## 정책 고정

- 고객관리자 선택 삭제는 hard delete가 아니다.
- 고객관리자 휴지통의 `purge_requested` 항목은 기본 목록에서 숨긴다.
- 시스템관리자가 최종 실제 삭제를 처리하기 전까지 R2 직접 SDK 삭제 방식으로 되돌리지 않는다.
- R2 삭제는 Worker 기반 purge 흐름을 유지한다.
- DB schema 변경 없이 기존 상태값과 중앙 정책 함수를 기준으로 처리한다.
