# 정책 재동의 안내 UI 1차 (0.20.02)

## 목적

`0.20.01`에서 추가한 정책 재동의 DB/API 계약을 `/workspace/legal` 화면에 연결한다. 중요 정책 변경이 있는 경우 사용자가 업무 차단 화면으로 이동하기 전에 정책 열람 화면에서 재동의 필요 상태를 확인하고 재동의를 저장할 수 있게 한다.

## 적용 화면

- `/workspace/legal`
- 고객사 관리자와 일반 멤버가 함께 접근하는 고객 공개 약관·정책 화면
- 시스템관리자 운영 화면은 이번 버전의 적용 대상이 아니다.

## UI 구성

### 정책 재동의 필요 상태 패널

`PolicyReagreementStatusPanel`을 추가하고 기존 `PolicyAgreementStatusPanel`보다 위에 배치한다.

표시 항목:

- 재동의 대상 건수
- 재동의 완료 건수
- 남은 재동의 건수
- 재동의 필요 문서 목록
- 문서별 카테고리, 버전, 재동의 상태, 동의 시각
- `필수 정책 전체 재동의` 버튼

재동의 대상이 없으면 `재동의 필요 없음` 상태를 표시한다.

## API 연결

- 조회: `GET /api/policies/reagreement`
- 저장: `POST /api/policies/reagreement`

저장 성공 후에는 API가 반환한 최신 상태로 화면을 갱신한다.

## 자동테스트 기준

`tests/e2e/workspace-policy-settings.spec.mjs`에 다음 기준을 추가한다.

- `/workspace/legal` 진입 시 정책 재동의 필요 상태 문구 확인
- 재동의 필요 mock 응답 확인
- `필수 정책 전체 재동의` 클릭
- 재동의 완료 문구 확인

## 제외 범위

이번 버전에서는 다음 작업을 하지 않는다.

- 업무 화면 접근 차단
- 라우트 가드 또는 middleware 차단
- 정책 미동의 사용자의 예외 라우팅
- DB schema 변경
- full_reset.sql 변경
- 시스템관리자 정책 별도 적용

위 항목은 `0.20.03` 정책 미동의 업무 접근 차단 1차에서 다룬다.
