# 0.20.23 작업지시서 화면 정리 1차 기준

## 목적

환경설정 UI 정리 이후 작업지시서 화면을 바로 대규모로 바꾸기 전에, 현재 화면에서 우선 손볼 범위를 정리한다. 작업지시서 화면은 목록, 상세, 우측 첨부/디자인/메모 패널이 동시에 동작하므로 한 번에 구조를 크게 바꾸지 않고 공통 UI 기준을 먼저 맞춘다.

## 현재 판단

- PC 화면은 3패널 구조가 유지되어야 한다.
- 좌측 목록, 중앙 상세, 우측 첨부/디자인/메모 패널의 역할은 분리되어야 한다.
- 기능 흐름 자체보다 화면 밀도, 헤더 구조, 버튼 톤, 패널 간 간격, 상태 표현의 일관성이 먼저 필요하다.
- 환경설정에서 정리한 WAFL 공통 탭/헤더/토큰 기준을 작업지시서에도 단계적으로 확장한다.
- 작업지시서 저장, 첨부, 디자인, 메모, R2, 휴지통, purge 흐름은 직접 목표가 아니면 변경하지 않는다.

## 1차 정리 범위

### 1. 좌측 작업지시서 목록

- 회사명/버전/툴바 영역의 높이와 버튼 톤 점검
- 검색/상태필터/정렬 영역의 밀도 점검
- 생성 버튼의 상태와 비활성 안내 점검
- 목록 카드의 선택 상태 색상과 WAFL selected token 연동 점검
- 목록 카드 안의 상태, 담당자, 납기, 카테고리 표시 우선순위 정리

### 2. 중앙 상세 영역

- 상세 상단 헤더의 제목/상태/액션 배치 점검
- 진행 단계 표시가 너무 무겁거나 어긋나지 않는지 확인
- 발주정보, 생산구성, 금액요약, 액션 영역의 섹션 헤더 기준 통일
- 설명성 문구와 실제 입력/액션 영역 분리
- 저장 버튼/요청 버튼/검토 버튼의 CTA 우선순위 점검

### 3. 우측 사이드 패널

- 첨부/디자인/메모 섹션 헤더 높이 통일
- 등록된 항목 없음 empty state 통일
- 업로드/메모 저장 버튼의 위치와 톤 통일
- 작업지시서 파일 흐름은 기존 DB/R2 구조를 유지한다.

## WAFL 공통화 후보

### 공통 컴포넌트 후보

- `WaflWorkspaceShell`
- `WaflWorkspacePanel`
- `WaflWorkspacePanelHeader`
- `WaflWorkspaceSection`
- `WaflWorkspaceActionBar`
- `WaflSelectedListItem`
- `WaflEmptyState`
- `WaflStatusPill`

기존 컴포넌트를 즉시 대체하지 않고, 작업지시서 화면에서 반복되는 구조부터 점진 적용한다.

## 금지 범위

이번 단계에서는 다음을 건드리지 않는다.

- 작업지시서 생성/수정/삭제 API
- 첨부/디자인/메모 저장 API
- R2 Worker 업로드 흐름
- 휴지통/복원/purge 흐름
- 원단·부자재 발주 상태 전환
- DB schema 변경

## 결론

작업지시서 화면은 기능 흐름을 건드리기 전에, 3패널 운영 화면의 공통 UI 기준을 먼저 잡는다. 환경설정에서 정리한 WAFL 토큰과 공통 헤더/상태/empty 패턴을 작업지시서 목록, 상세, 우측 패널에 순차 적용한다.
