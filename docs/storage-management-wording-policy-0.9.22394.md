# 0.9.22394 — 저장소 관리 문구/정책 문서 정리

## 목적

저장소 관리에서 고객관리자와 시스템관리자가 보는 삭제·복원·실제 삭제 문구를 역할 기준으로 분리한다. 사용자가 화면에서 직접 테스트해야 하는 항목과 코드상 확인으로 충분한 예외 처리 항목도 분리해 운영 회귀 테스트 범위를 현실적으로 고정한다.

## 역할 기준

### 고객관리자 `/admin/files`

고객관리자는 복원 가능한 휴지통 항목을 관리하고, 실제 삭제가 필요한 항목을 시스템관리자에게 넘기는 역할이다.

- 고객관리자 화면의 되돌리기 동작은 `복원`으로 표시한다.
- 고객관리자 화면의 삭제 동작은 `선택 삭제` 또는 `삭제 요청`으로 표시한다.
- 고객관리자 화면에서는 `영구삭제`라는 표현을 사용하지 않는다.
- 고객관리자 선택 삭제는 R2 object를 직접 삭제하지 않는다.
- 고객관리자 선택 삭제 후 항목은 `purge_requested` 계열 상태가 되고 기본 휴지통 목록에서 숨긴다.
- 고객관리자 파일 운영 요약의 `삭제 요청`은 첨부파일 개수와 용량만 집계한다.
- 작업지시서 자체 개수는 고객관리자 삭제 요청 요약에 포함하지 않는다.

### 시스템관리자 `/system/storage-usage`

시스템관리자는 고객관리자가 선택 삭제 또는 비우기로 넘긴 항목을 실제 삭제 후보로 확인하고 최종 purge 처리하는 역할이다.

- 시스템관리자 화면에서는 실제 삭제 후보를 표시한다.
- 시스템관리자 `선택 삭제` 또는 `전체삭제`는 Worker 기반 R2 purge와 DB 상태 완료 처리를 실행한다.
- R2 직접 SDK 삭제 방식으로 되돌리지 않는다.
- R2 object not found 계열 응답은 이미 삭제된 객체로 보고 성공 처리한다.
- Worker 인증 실패, 권한 실패, 네트워크 실패, timeout은 실패로 남겨 재시도 후보로 유지한다.
- R2 삭제 성공 후 DB 업데이트가 실패하면 정상 완료로 간주하지 않는다.

## 용어 기준

| 상황 | 고객관리자 문구 | 시스템관리자 문구 | 의미 |
| --- | --- | --- | --- |
| 휴지통에서 되돌리기 | 복원 | 해당 없음 | 복원 가능한 상태를 원래 목록으로 되돌림 |
| 항목을 삭제 후보로 넘김 | 선택 삭제 | 해당 없음 | 실제 삭제가 아니라 삭제 요청 처리 |
| 휴지통 일괄 삭제 요청 | 비우기 | 해당 없음 | 고객관리자 휴지통 항목을 삭제 요청 상태로 전환 |
| 실제 삭제 후보 처리 | 해당 없음 | 선택 삭제 | R2 Worker purge와 DB 상태 완료 처리 |
| 실제 삭제 후보 일괄 처리 | 해당 없음 | 전체삭제 | 시스템관리자가 처리 가능한 후보를 일괄 purge |
| 축약된 작업지시서 배지 | 작업 | 작업 | 좁은 영역에서만 사용하는 짧은 배지 |
| 일반 화면 문구 | 작업지시서 | 작업지시서 | 축약 표현 `작지` 사용 금지 |

## 작업지시서 묶음 삭제 정책

1. 작업지시서 삭제 시 함께 휴지통으로 이동한 첨부파일과 메모는 작업지시서 묶음 항목으로 취급한다.
2. 작업지시서 묶음 첨부파일 row는 고객관리자 휴지통에서 선택 불가 톤으로 표시한다.
3. 작업지시서보다 먼저 개별 삭제된 첨부파일은 작업지시서 묶음 purge 대상에서 제외한다.
4. 선삭제 첨부파일은 고객관리자 휴지통에 독립 항목으로 남아야 한다.
5. 작업지시서 묶음 purge 후 정상 처리된 첨부파일은 시스템관리자 후보에 다시 노출하지 않는다.
6. R2 Worker 또는 DB 처리 실패 항목만 재시도 후보로 남긴다.

## 휴지통 비우기 정책

고객관리자 휴지통의 `비우기`는 실제 삭제가 아니다.

- 확인 모달 문구는 `휴지통의 모든 항목을 삭제 요청하시겠습니까?`로 유지한다.
- 예를 누르면 복원 가능한 항목들이 삭제 요청 상태로 전환된다.
- 삭제 요청 상태가 된 항목은 고객관리자 휴지통 기본 목록에서 숨긴다.
- 실제 R2 삭제는 시스템관리자 실제 삭제 후보 화면에서만 처리한다.

## 화면에서 직접 확인할 항목

1. `/admin/files` 휴지통 목록이 정상 표시되는지 확인한다.
2. 고객관리자 화면에 `복구`, `영구삭제`, `작지` 표현이 남아 있지 않은지 확인한다.
3. 첨부파일 단독 삭제 후 `복원`이 가능한지 확인한다.
4. 첨부파일 단독 삭제 후 `선택 삭제` 시 고객관리자 목록에서 숨겨지고 시스템관리자 후보로 이동하는지 확인한다.
5. `비우기` 실행 시 확인 모달이 표시되고 삭제 요청 상태로 전환되는지 확인한다.
6. 작업지시서 삭제 후 `복원`이 가능한지 확인한다.
7. 작업지시서 삭제 후 `선택 삭제` 시 시스템관리자 후보에 작업지시서 대표 row가 표시되는지 확인한다.
8. 작업지시서 삭제 전 선삭제한 첨부파일이 고객관리자 휴지통에 독립 항목으로 남는지 확인한다.
9. 작업지시서 묶음 첨부파일이 선택 불가 row 톤으로 표시되는지 확인한다.
10. 리오더 작업지시서명이 목록, 상세 모달, 연결 첨부 row에서 동일하게 표시되는지 확인한다.
11. 시스템관리자 실제 삭제 성공 후 후보에서 사라지는지 확인한다.
12. 고객관리자가 아직 선택 삭제하지 않은 pending 항목은 시스템관리자 후보에 표시되지 않는지 확인한다.

## 코드상 확인으로 충분한 항목

아래 항목은 실제 운영 화면에서 매번 강제 재현하지 않는다.

- R2 object not found 계열 응답은 성공 처리되는지 확인한다.
- Worker 인증 실패는 실패 처리되어 재시도 후보로 남는지 확인한다.
- Worker 권한 실패는 실패 처리되어 재시도 후보로 남는지 확인한다.
- 네트워크 실패 또는 timeout은 실패 처리되어 재시도 후보로 남는지 확인한다.
- R2 삭제 성공 후 DB 업데이트 실패 시 후보가 완료 처리로 사라지지 않는지 확인한다.

## SQL / DB

- DB schema 변경 없음.
- SQL DDL 실행 필요 없음.
- seed 데이터 변경 없음.
- full reset 필요 없음.

## 금지 기준

- 고객관리자 화면에서 실제 삭제처럼 보이는 `영구삭제` 문구를 되살리지 않는다.
- `복구`와 `복원`을 혼용하지 않는다.
- `작지` 축약 표현을 되살리지 않는다.
- R2 직접 SDK 삭제 방식으로 되돌리지 않는다.
- 삭제 요청 상태의 항목을 고객관리자 휴지통 기본 목록에 다시 노출하지 않는다.
