# 0.9.22383 저장소 관리 서버 로직 리팩토링 4차

## 목적

저장소 휴지통과 시스템관리자 실제 삭제 후보 로직에서 남아 있던 상태 문자열 직접 비교를 줄이고, 작업지시서 묶음 R2 삭제 실패 집계가 중복 반영되지 않도록 보정한다.

## 변경 내용

- 고객관리자 휴지통 작업지시서 조회 조건에서 `delete_status`, `purge_status` 직접 문자열 비교를 중앙 정책 SQL 목록으로 대체했다.
- 고객관리자 휴지통에 노출하지 않을 작업지시서 delete/purge 상태 목록을 `trashPolicy.ts`로 분리했다.
- 작업지시서 묶음 첨부 R2 삭제 실패 시 `failedCount`가 한 항목당 한 번만 증가하도록 보정했다.

## 정책

- DB schema는 변경하지 않는다.
- `delete_reason` 컬럼은 기존 구조를 유지한다.
- reason code 전환은 별도 DB schema 변경 작업에서 검토한다.
- R2 삭제는 Worker 기반 흐름을 유지한다.
