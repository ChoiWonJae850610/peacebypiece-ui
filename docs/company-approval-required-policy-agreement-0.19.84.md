# 0.19.84 고객사 승인 요청 전 필수 정책 동의 연결 1차

## 목적

고객사 관리자 온보딩의 승인 요청 제출 전에 필수 약관·정책 동의 이력이 저장되도록 연결한다.

## 적용 범위

- 회사 정보 입력 모달에 필수 약관·정책 동의 확인 섹션을 추가했다.
- 사용자가 동의 체크를 하지 않으면 승인 요청 제출을 막는다.
- 승인 요청 API는 서버에서도 필수 정책 동의 여부를 확인한다.
- 동의 체크가 함께 제출되면 현재 필수 정책에 대한 동의 이력을 저장한 뒤 승인 요청 흐름을 진행한다.

## 유지한 범위

- 정책 강제 차단 화면은 아직 추가하지 않는다.
- 이미 승인된 고객사의 업무 화면 접근 흐름은 변경하지 않는다.
- 정책 문서 DB schema는 0.19.82 기준을 그대로 사용한다.
- 회사 승인 처리, 시스템관리자 검토 화면, R2 업로드 흐름은 기존 구조를 유지한다.

## 테스트 포인트

1. 회사 정보 입력 모달에서 동의 체크 없이 승인 요청 제출 시 차단되는지 확인한다.
2. 동의 체크 후 승인 요청 제출 시 정책 동의 이력이 저장되는지 확인한다.
3. 승인 요청 후 회사 상태가 기존처럼 승인 대기로 전환되는지 확인한다.
4. 선택 파일 업로드가 동의 이후 기존과 동일하게 처리되는지 확인한다.
