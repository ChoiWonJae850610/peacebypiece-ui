# 작업지시서·발주서 최종 표시 정책 감사 — 0.23.29

## 목적

0.23.30 통합 테스트 전에 작업지시서와 발주서에서 남아 있던 저장/변경 표현 호환 계층과 태블릿 목록 드로어 높이 하드코딩을 정리한다.

## 적용 내용

- 공장 전달사항 변경 경로를 `WAFL_CHANGE_TARGET`과 `getWaflChangeFeedbackMessage`로 직접 이관했다.
- 더 이상 사용하는 호출부가 없는 `WAFL_SAVE_TARGET`, `getWaflSaveFeedbackMessage` 호환 API를 제거했다.
- 문서 필드의 기본 인라인 상태 문구를 저장 중심 표현에서 변경 중심 표현으로 통일했다.
- 발주서 모바일 목록 드로어의 `72dvh` 최소 높이 하드코딩을 제거하고 가용 높이를 사용하도록 정리했다.
- 작업지시서·발주서 변경 토스트의 성공/실패 기본 문구를 공통 생성기에서만 관리하도록 유지했다.

## 정책

- 진행 중: 변경 대상을 구체적으로 표시한다.
- 성공: `정보가 변경되었습니다.`
- 실패: `정보를 변경하지 못했습니다.` 또는 API가 제공한 구체적인 오류
- 생성·삭제·발주요청 등 정보 변경과 성격이 다른 작업은 별도 명시 문구를 유지한다.
