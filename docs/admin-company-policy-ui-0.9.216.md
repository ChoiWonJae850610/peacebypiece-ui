# 0.9.216 — 고객사 관리자 UI 정책 정리

## 목적

고객사 관리자 환경설정 화면에서 삭제방식, 저장소 용량 한도, 휴지통 용량 포함 여부, 용량 주의 기준, 개발중 기능 상태를 한 번에 확인할 수 있도록 정책 영역을 추가한다.

이번 버전은 실제 권한 저장 UI나 API 차단을 연결하지 않는다. 0.9.215에서 정리한 permission code와 feature flag 체계를 기준으로, 고객사 관리자 UI에서 어떤 정책을 어디에 노출할지 먼저 고정한다.

## 반영 범위

- `/admin/settings`에 `고객사 관리자 정책` 영역 추가
- 파일/용량 정책 카드 추가
- 운영 정책 정리 카드 추가
- 개발중 기능 카드 추가
- 후속 연결 기준 문서화

## 파일/용량 정책

| 항목 | 기준 |
| --- | --- |
| 삭제 방식 | 휴지통 보관 후 영구삭제 |
| 기본 용량 한도 | company settings의 storageLimitGb 값 사용 |
| 휴지통 용량 포함 | company settings의 includeTrashInUsage 값 사용 |
| 용량 주의 기준 | company settings의 warningThresholdPercent 값 사용 |

삭제방식은 즉시 R2 삭제가 아니라 DB soft delete와 휴지통 상태를 먼저 거치는 정책을 유지한다. 휴지통 파일도 실제 R2 객체가 남아 있으므로 기본적으로 저장소 사용량에 포함한다.

## 운영 정책

| 항목 | 기준 |
| --- | --- |
| 정책 표시 위치 | 환경설정 메인 |
| 로그/히스토리 노출 | 업무 화면과 분리 |
| 권한 설정 UI | DB seed/API 차단 이후 연결 |

고객관리자 메인 화면은 업무 요약을 우선하고, 로그성 이벤트는 추적/시스템 화면으로 분리한다.

## 개발중 기능

- 알림 정책
- 알림 서비스
- 작업지시서 이름 추천 AI

위 항목들은 초기 제품 핵심 기능이 아니므로 개발중 상태로 유지한다. 실제 활성화는 워크플로우 이벤트, 권한, 로그, 발송 채널이 안정화된 뒤 진행한다.

## SQL DDL 필요 여부

불필요.

이번 버전은 UI와 presentation 정책 파일 추가만 포함한다. DB table, column, index, constraint는 변경하지 않는다.

## 전체 리셋 필요 여부

불필요.

`full_reset.sql`, `full_reset_smoke_test.sql`, seed SQL 실행은 필요하지 않다.
