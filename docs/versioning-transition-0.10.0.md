# PeaceByPiece 0.10.0 버전 체계 전환 기준

## 목적

0.10.0은 기존 0.9.224xxx 형태의 긴 보정 버전 체계를 마무리하고, 관리자 IA 개편과 권한 기반 workspace 구조를 시작하기 위한 새 기준 버전이다.

이번 버전에서는 기능 로직, DB schema, R2 흐름을 변경하지 않는다. 버전 체계만 0.10.x로 전환하고, 이후 작업의 기준을 문서화한다.

## 전환 전 기준

- 마지막 긴 보정 버전: 0.9.224394
- 해당 버전의 성격:
  - 작업지시서 lazy load 안정화
  - 저장소/휴지통 1차 안정화
  - 시스템관리자 실제 삭제 후보 흐름 1차 안정화
  - 고객관리자 좌측 패널 제거 방향 확정
  - 카드형 홈 1차 구현
  - 작업지시서 홈 버튼 및 멤버 홈 진입점 추가
  - 모바일 홈 버튼 타입 오류 수정

## 전환 후 기준

- 새 기준 버전: 0.10.0
- 이후 패치 예시:
  - 0.10.1
  - 0.10.2
  - 0.10.3

0.10.x는 관리자 IA 개편, workspace 홈, 개인 설정, 조직 설정, 멤버/권한 설계를 다루는 버전 라인으로 사용한다.

## 버전 관리 규칙

1. APP_VERSION은 `lib/constants/app.ts`에서 관리한다.
2. patch zip 이름은 `peacebypiece-patch-{version}-files.zip` 형식을 유지한다.
3. commit-meta.md의 Version 값과 APP_VERSION은 항상 일치해야 한다.
4. 문서 파일명에 버전을 포함할 경우 실제 패치 버전과 일치시킨다.
5. 긴 보정 번호가 필요한 경우가 아니면 0.10.x 이후부터는 짧은 semver형 패치 번호를 사용한다.

## 이후 작업 권장 순서

### 0.10.1 — 개인 설정 /me/settings 설계

- 언어 설정
- 색상/테마 설정
- 알림 선호
- 기본 진입 화면 후보
- 조직 설정과 개인 설정 분리 기준 정리

### 0.10.2 — 조직 설정 /admin/settings 재정리 설계

- 파일 정책
- 저장소 정책
- 협력업체 정책
- 권한 정책
- 회사/조직 단위 설정 구조 정리

### 0.10.3 — 멤버/권한 관리 설계

- 멤버관리 화면 후보
- 역할과 권한 코드 구분
- 권한 기반 카드 노출 원칙
- 실제 DB schema 변경 전 설계 확정

### 0.10.4 이후 — 권한 기반 workspace 구현 준비

- 카드 registry와 permission code 연결
- 일반 멤버 홈 확장
- 작업지시서 업무 화면의 홈 복귀 구조 안정화
- 권한 DB와 초대 시스템은 별도 라인에서 진행

## 이번 버전의 변경 범위

- APP_VERSION을 0.10.0으로 변경
- 버전 체계 전환 문서 추가

## 이번 버전에서 하지 않는 일

- DB schema 변경 없음
- package.json/package-lock.json 변경 없음
- R2 흐름 변경 없음
- 권한 DB 구현 없음
- 멤버 초대 구현 없음
- 기존 작업지시서/저장소/휴지통 로직 변경 없음
