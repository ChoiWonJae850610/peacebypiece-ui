# 통계정보 기간 컨트롤 소스 정리 1차 (0.18.84)

## 목적

통계정보 대시보드 컴포넌트에 직접 들어 있던 기간 선택 상태와 날짜 검증 로직을 별도 hook으로 분리하여, 대시보드 렌더링 책임과 기간 컨트롤 상태 책임을 분리한다.

## 변경 내용

- `useAdminStatsPeriodControls` hook 추가
- 7일/30일 프리셋 선택 상태 관리 분리
- 사용자 지정 시작일/종료일 변경 handler 분리
- 미래 날짜 차단, 종료일이 시작일보다 앞서는 경우 차단, 적용 가능 여부 계산 로직 분리
- 기간 컨트롤 label 구성과 section 유지 href 구성 로직 분리
- `AdminStatsDashboard`는 기간 상태를 hook 반환값으로 받아 렌더링에 집중하도록 정리

## 범위

- UI 결과 변경 없음
- 통계 계산 변경 없음
- DB/API 변경 없음
- R2/첨부/메모/휴지통/purge 흐름 변경 없음

## 확인 필요

- 통계정보 화면에서 7일/30일 버튼 선택 및 재클릭 초기화
- 사용자 지정 기간 시작일/종료일 선택
- 미래 날짜 선택 차단
- 종료일이 시작일보다 빠른 경우 차단
- 기간 적용 후 `section=period`와 `topMode` 유지
