# 0.9.147 관리자 설정 모달 버튼/행 레이아웃 통일 기준

## 목적

관리자 환경설정 화면의 여러 모달에서 `기본값 복원`, `저장`, `닫기`, 추가 입력 row, 사용/미사용 토글의 크기와 위치가 서로 달라지는 문제를 줄인다.

## 적용 범위

- 로그 이벤트 모달
- 파일 정책 모달
- 알림 정책 모달
- 생산품 유형 모달
- 단위 표준 모달
- 외주 공정 유형 모달

## 이번 버전 기준

1. 모달 footer는 `AdminModalFooterActions`를 통해 렌더링한다.
2. 에러/상태 메시지는 footer 왼쪽 영역에 표시한다.
3. 기본값 복원/저장 버튼은 footer 오른쪽에 동일 높이와 간격으로 배치한다.
4. 저장 중에는 입력 추가, 토글, 저장, 기본값 복원 동작을 비활성화한다.
5. 생산품 유형/단위 표준/외주 공정 모달은 기존 데이터 구조와 저장 API를 변경하지 않는다.

## 보류 항목

- 모달 전체 visual redesign
- 테이블/목록 row 전체 공통 컴포넌트화
- 모바일 전용 모달 레이아웃 재설계
- 저장 API 응답 구조 변경

## 테스트 기준

- 각 모달이 runtime error 없이 열린다.
- 저장 중 버튼 중복 클릭이 막힌다.
- 에러 메시지가 footer 영역에 표시된다.
- 기존 저장/복원 동작이 유지된다.
