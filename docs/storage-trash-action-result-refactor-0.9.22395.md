# 저장소/휴지통 액션 결과 중앙화 1차 - 0.9.22395

## 목표

고객관리자 `/admin/files` 휴지통에서 상세 모달, 상단 복원, 상단 선택 삭제, 비우기 버튼이 서로 다른 메시지 조립 흐름을 타던 문제를 줄이고, 같은 actionFlow와 같은 presentation formatter를 통해 결과 메시지를 표시하도록 정리했다.

## 변경 범위

- `app/admin/files/page.tsx`의 복원/삭제 요청 메시지 직접 조립 함수를 제거했다.
- 상세 모달의 작업지시서 복원/삭제 요청도 상단 버튼과 같은 selection flow를 사용하도록 변경했다.
- 파일 단독, 작업지시서 단독, 파일+작업지시서 혼합 선택을 `AdminTrashActionResultSummary` 기준으로 합산한다.
- 결과 문구는 `createAdminTrashActionMessage` formatter를 통해 생성한다.
- 고객관리자 문구에서 `연결 첨부`, `복구`, `영구삭제`, `작지` 표현을 쓰지 않도록 정리했다.

## 메시지 기준

작업지시서가 포함된 경우:

```text
작업지시서 n건과 문서 n개, 디자인 n개, 메모 n개를 복원하였습니다.
작업지시서 n건과 문서 n개, 디자인 n개, 메모 n개를 삭제 요청하였습니다.
```

파일 단독인 경우:

```text
문서 n개를 복원하였습니다.
디자인 n개를 삭제 요청하였습니다.
문서 n개, 디자인 n개를 복원하였습니다.
```

처리 불가 항목이 섞인 경우:

```text
... 삭제 요청하였습니다. 삭제 요청할 수 없는 항목 n개는 제외했습니다.
```

## 데이터 구조

이번 버전은 DB schema를 변경하지 않는다.

다만 UI/actionFlow에서 문서와 디자인 개수를 구분할 수 있도록 `AdminTrashFileItem`과 `AdminManagedFileItem`에 `fileKind`를 추가했다.

- `document`: 고객관리자 UI의 `문서`
- `design`: 고객관리자 UI의 `디자인`

작업지시서 묶음 복원/삭제 요청 API는 DB 조회 결과에서 문서/디자인/메모 개수를 응답에 포함한다.
