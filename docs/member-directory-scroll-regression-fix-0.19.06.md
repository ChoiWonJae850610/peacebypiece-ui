# 0.19.06 멤버관리 row/scroll 회귀 보정

## 목적

0.19.04~0.19.05 공통 상태/피드백 컴포넌트 적용 후 `/workspace/members`에서 확인된 목록 하단 row 짤림과 화면 진입 직후 마우스 휠 스크롤이 둔하게 동작하는 문제를 먼저 보정한다.

## 변경 범위

- 멤버관리 대시보드 내부의 중첩 `2xl:overflow-y-auto`를 제거해 `WorkspaceShell`의 페이지 스크롤을 우선 사용한다.
- 멤버 목록 패널의 고정 높이(`2xl:h-[452px]`)를 제거하고 최소 높이 기준으로 완화한다.
- 멤버 목록 패널과 responsive row wrapper 하단에 소량의 여백을 추가해 마지막 row가 카드 경계에 걸리지 않도록 한다.
- 협력업체, 저장소, 통계, 작업지시서, 원단·부자재, 시스템관리자 화면은 변경하지 않는다.

## 테스트 위치

- `/workspace/members`

## 확인할 것

- 화면 진입 직후 마우스 휠 스크롤이 자연스럽게 동작하는지 확인한다.
- 멤버 관리 탭의 마지막 row 하단이 잘리지 않는지 확인한다.
- 검색어 입력 후에도 마지막 row가 잘리지 않는지 확인한다.
- 상태 필터와 역할 필터를 변경해도 목록 하단 여백과 스크롤이 유지되는지 확인한다.
- 멤버 row 클릭 시 상세/권한 모달이 기존처럼 열린다.

## 바뀌면 안 되는 것

- 멤버 목록 데이터 개수
- 멤버 상태/역할 표시
- 검색·상태·역할 필터 결과
- 멤버 초대 버튼
- 멤버 상세/권한 모달
- 권한 저장 흐름
- 승인/거절 흐름

## 다음 작업

0.19.07에서는 토스트/피드백 전체 소스 전수 감사를 진행한다. Floating toast와 inline feedback box의 역할을 분리하고, 작업지시서·원단부자재·저장소관리 등에서 네모형 기본 메시지가 뜨는 지점을 먼저 분류한다.
