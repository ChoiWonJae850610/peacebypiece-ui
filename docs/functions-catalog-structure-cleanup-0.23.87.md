# Functions catalog structure cleanup — 0.23.87

## 목표
`/functions`에서 제품 기능 명세, 테스트 시나리오, 자동화 연결, dev/test 도구를 한 화면에 혼합해 표시하던 구조를 목적별 보기로 분리한다.

## 적용
- 기능 명세 / 테스트 시나리오 / 자동화 현황 / 개발 도구 보기 추가
- 정상·자동화 완료 항목의 상태 배지 기본 숨김
- 미자동화, 부분 자동화, 수동 확인, 정책 미확정, 실패, 차단만 강조
- 기능 상세에 화면·경로, 사용자 역할, 자동화 방식, 최근 결과를 고정 메타데이터로 표시
- 테스트 보기에서 실행 절차, 예외, 권한, tenant, 반응형, 성능, PDF 계약 표시
- 자동화 연결을 표 형태로 분리
- Seed·Reset·Cleanup 관련 테스트 환경 항목을 개발 도구 보기로 분리
- 기존 catalog 데이터와 route 접근 정책은 유지

## 미포함
- 전체 route와 모든 버튼 자동 수집
- PowerShell 명령 직접 실행
- 최근 실행 로그 자동 수집
- OK/Failed 결과 영속화
- 기능 명세 DB 저장

이 항목들은 이후 각 제품 화면 정리와 함께 확장한다.
