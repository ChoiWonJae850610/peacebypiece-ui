# 0.9.143 시스템관리자 R2 purge 표시/재시도 UX 보완

## 목적

`/system/storage-usage`의 R2 실제 삭제 후보 목록에서 썸네일이 없는 과거 이미지 파일의 표시 상태를 명확히 하고, purge 실행 결과 메시지를 운영자가 해석하기 쉽게 정리한다.

## 확정된 표시 정책

- `thumbnail_key`가 있는 이미지 후보는 썸네일을 표시한다.
- `thumbnail_key`가 없는 이미지 후보는 원본 key를 축소 표시용 fallback으로 사용한다.
- 원본 fallback을 쓰는 경우 반드시 `썸네일 없음 · 원본 표시` 문구를 표시한다.
- PDF/비이미지 파일은 이미지를 요청하지 않고 파일 타입 뱃지를 표시한다.
- purge 실행 시 삭제 대상은 원본 `storage_key`와 `thumbnail_key`를 모두 포함한다.

## 결과 메시지 정책

- 전체 성공: 삭제 완료 개수와 목록/요약 갱신 안내를 표시한다.
- 일부 실패: 성공 개수, 실패 개수, 실패 항목은 목록에 남아 재시도 가능하다는 안내를 표시한다.
- 요청 실패: 에러 메시지를 붉은 톤으로 표시한다.

## 재시도 기준

현재 단계에서는 별도 “실패 항목만 재시도” 버튼을 추가하지 않는다. 실패 항목은 `last_purge_error`가 남아 후보 목록에 계속 표시되므로 사용자는 해당 항목을 다시 선택해 수동 재시도할 수 있다.

향후 별도 재시도 버튼을 추가할 경우 다음 조건을 지킨다.

1. `last_purge_error IS NOT NULL` 항목만 필터링한다.
2. 원본 key와 썸네일 key를 다시 Worker purge 대상으로 보낸다.
3. 성공 시 purge 완료 상태로 전환한다.
4. 실패 시 최신 실패 사유로 갱신한다.
5. R2 직접 SDK 삭제 방식은 사용하지 않는다.

## 이번 버전에서 하지 않은 일

- DB schema 변경 없음
- 자동 purge 스케줄러 없음
- R2 직접 SDK 삭제 없음
- 별도 실패 항목 전용 필터/버튼 없음
