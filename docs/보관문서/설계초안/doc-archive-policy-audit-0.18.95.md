# 0.18.95 문서 분류 기준 점검

## 목적

테스트 불가 상태에서 기능·DB·권한·R2 흐름을 건드리지 않고, 문서 폴더의 현재 사용 기준을 명확히 정리한다.

## 점검 범위

- `docs/현재기준/`
- `docs/정책문서/`
- `docs/보관문서/`
- 루트 단위 버전별 작업 기록 문서

## 분류 기준

### 현재 기준 문서

`docs/현재기준/`은 지금 개발·검수·리팩토링의 우선 기준으로 본다. 같은 주제의 과거 문서와 충돌하면 현재 기준 문서를 우선한다.

현재 기준 문서 후보:

- 리팩토링 규칙
- 소스 구조
- 라우팅 구조
- 데이터베이스 구조
- 데이터베이스 쿼리·인덱스 정책
- 작업지시서 상태 구조
- 워크스페이스 경계
- 고객사 통계 지표
- 시스템 통계 지표
- 통계 저장소 구조
- 초대 정책 설계
- 요금·저장소 정책 설계
- 원단·부자재 데이터베이스 설계
- 원단·부자재 발주 설계
- full-reset 검증

### 정책 문서

`docs/정책문서/`은 고객 공개 문서, 서비스 운영 정책, 요금·환불·저장소·개인정보·약관의 근거 문서로 본다.

고객 공개 문서에서는 서비스명을 WAFL 기준으로 유지하고, PeaceByPiece 명칭은 개발 프로젝트명이나 내부 기록에서만 사용한다.

### 보관 문서

`docs/보관문서/`은 삭제 대상이 아니라 과거 판단 근거다. 특히 `wafl-a-type/` 누적 문서는 현재 구현과 다른 내용이 있어도 즉시 삭제하지 않는다.

보관 문서를 현재 작업에 사용할 때는 아래 순서로 확인한다.

1. 같은 주제가 `docs/현재기준/`에 있는지 확인한다.
2. 현재 기준 문서와 충돌하면 현재 기준을 우선한다.
3. 보관 문서의 내용은 과거 설계·검토 근거로만 사용한다.
4. 실제 구현 변경은 테스트 가능 상태 또는 명확한 작업 목표가 있을 때만 진행한다.

## 삭제하지 않은 이유

테스트 불가 상태에서는 문서 삭제도 위험할 수 있다. 특히 정책·DB·권한·작업지시서·원단부자재 관련 문서는 과거 결정 근거가 될 수 있으므로 즉시 삭제하지 않고 분류와 인덱스 정리를 우선한다.
