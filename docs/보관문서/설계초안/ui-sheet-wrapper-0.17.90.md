# 0.17.90 UI Sheet Wrapper 설계 및 1차 추가

## 목적

WAFL의 태블릿/모바일 화면에서 목록, 상세, 보조 패널을 한 화면에 모두 노출하지 않고 필요한 순간에 열리는 보조 패널 구조로 전환하기 위한 내부 공통 래퍼를 추가한다.

## 추가 컴포넌트

- `components/common/ui/AppSheet.tsx`

## 설계 기준

- 화면 파일에서 Radix Dialog를 직접 사용하지 않는다.
- 화면 파일은 `AppSheet`만 사용한다.
- 실제 구현체는 내부에서 `@radix-ui/react-dialog`를 사용한다.
- PC 화면의 3분할 구조는 유지한다.
- 태블릿/모바일의 보조 패널만 Sheet 전환 후보로 둔다.
- 작업지시서 첨부/메모, 원단·부자재 자재 선택 패널이 1차 적용 후보이다.

## 지원 옵션

- `side`: `right` | `left` | `bottom`
- `size`: `sm` | `md` | `lg` | `full`
- `title`
- `description`
- `footer`
- `contentClassName`

## 적용 순서

1. AppSheet 컴포넌트만 추가한다.
2. 기존 화면에는 바로 대량 적용하지 않는다.
3. 모바일/태블릿 캡처를 보고 Sheet가 필요한 보조 패널부터 적용한다.
4. 첫 적용 후보는 원단·부자재 모바일의 자재 선택 패널이다.
5. 두 번째 후보는 작업지시서 모바일의 첨부/메모 패널이다.

## 주의 사항

- 저장, 상태 전환, 자재 계산, DB/API/R2 흐름과 분리한다.
- PC 화면에서 이미 잘 작동하는 3분할 구조를 Sheet로 바꾸지 않는다.
- Sheet 내부에 과도한 테이블을 그대로 넣지 말고, 목록/요약/상세를 분리해서 넣는다.
