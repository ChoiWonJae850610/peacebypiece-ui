# 0.19.74 시스템관리자 운영 화면 정리 설계 1차

## 목적

0.19.74는 정책 문서에서 도출된 운영 기능을 실제 개발 순서로 연결하기 전, 시스템관리자 화면이 맡아야 할 운영 범위를 고정한다. 이번 버전은 설계 문서와 버전 기준선 정리만 포함하며, DB/API/권한/결제/저장소 로직은 변경하지 않는다.

## 현재까지 반영된 운영 흐름

- 고객사 관리자 환경설정에서 회사 정보 변경 요청과 계정 비활성화 요청을 접수할 수 있다.
- 고객사 관리자 환경설정에서 최근 회사 계정 요청 이력을 조회할 수 있다.
- 멤버관리에서 일반 멤버의 탈퇴 요청, 비활성, 탈퇴 완료 상태를 다룰 수 있다.
- 시스템관리자에는 고객사 관리, 감사로그, 저장소 삭제 후보, 요금제/용량, 기준정보, 카테고리 규칙 화면이 이미 분리되어 있다.

## 시스템관리자 운영 메뉴 재정의

### 1. 고객사 운영

시스템관리자가 가장 먼저 확인해야 하는 고객사 단위 운영 흐름이다.

- 고객사 목록
- 신규 고객사 승인 요청
- 고객사 관리자 초대
- 회사 정보 변경 요청
- 계정 비활성화 요청
- 사업자등록증 수동 확인
- 고객사 상태 변경
- 승인/거절/반려 이력

### 2. 정책·약관 운영

정책 문서와 실제 고객 공개 화면을 연결하는 운영 영역이다.

- 이용약관
- 개인정보처리방침
- 요금·환불정책
- 요금제·저장소정책
- 데이터 보관·삭제정책
- 데이터 내보내기정책
- 서비스운영정책
- 정책 버전
- 필수 동의 여부
- 재동의 필요 여부

### 3. 요금·결제·증빙 운영

초기에는 카드결제 중심으로 설계한다. 실제 PG 연동은 별도 단계로 분리한다.

- 요금제 관리
- 무료체험 7일
- 저장소 100MB 제한
- 멤버 3명 제한
- 고객사 구독 상태
- 결제수단 상태
- 결제 실패·미납
- 해지 예정
- 환불 요청
- 영수증 URL
- 세금계산서 요청 상태

저장하지 않아야 하는 데이터:

- 전체 카드번호
- CVC
- 카드 비밀번호
- 주민등록번호
- 카드 유효기간 원문

### 4. 저장소·데이터 운영

R2와 DB 상태가 맞물리는 고위험 영역이므로 실제 구현은 작은 단위로 분리한다.

- 저장소 사용량
- 80% 안내
- 100% 업로드 제한
- 고객사 휴지통
- 30일 자동 영구삭제
- R2 삭제 후보
- 데이터 내보내기 요청
- export job 상태
- 다운로드 링크 7일 만료

### 5. 감사·접근 로그 운영

시스템관리자의 권한 남용을 막기 위한 운영 기록 영역이다.

- 고객사 승인/거절 로그
- 회사 정보 변경 처리 로그
- 계정 비활성화 처리 로그
- 정책 버전 변경 로그
- 요금제 변경 로그
- 저장소 삭제 로그
- 시스템관리자 파일 열람/다운로드 로그
- 파일 접근 사유 입력

### 6. 기준정보·분류 운영

서비스 전체 기준값과 추천 규칙을 관리하는 영역이다.

- 단위 기준
- 공정 기준
- 생산품 유형 템플릿
- 고객사 초기 기준정보 seed 상태
- 카테고리 추천 규칙
- 분류 체계 관리

## 고객사 관리자 화면과 시스템관리자 화면의 경계

### 고객사 관리자 화면에 남길 것

- 회사 정보 변경 요청 작성
- 계정 비활성화 요청 작성
- 요청 이력 조회
- 멤버관리
- 기준정보 설정
- 요금제/저장공간 조회
- 약관·정책 열람
- 데이터 내보내기 요청

### 시스템관리자 화면에서 처리할 것

- 회사 정보 변경 요청 승인/반려
- 계정 비활성화 요청 승인/반려
- 고객사 승인/거절
- 사업자등록증 확인
- 요금제/저장공간 한도 관리
- 정책 버전 발행
- 정책 재동의 필요 여부 설정
- 결제 실패/미납/해지 운영
- 저장소 영구삭제 후보 처리
- 데이터 내보내기 job 처리 상태 확인
- 감사로그와 파일 접근 로그 조회

## 이번 버전에서 하지 않는 것

- DB schema 변경
- API route 추가
- 결제/PG 연동
- 이메일 발송 연동
- R2 삭제/업로드 흐름 변경
- 정책 동의 이력 저장
- 시스템관리자 승인/반려 실제 처리
- 고객사 상태 변경 로직 변경

## 테스트 기준

이번 버전은 설계 문서 추가와 APP_VERSION 변경만 포함한다. 기능 테스트는 다음 항목만 확인한다.

1. npm run build
2. 앱 헤더 또는 버전 표시가 0.19.74로 보이는지 확인
3. docs/system-admin-operations-design-0.19.74.md 파일이 포함되었는지 확인
4. 고객사 관리자 환경설정, 멤버관리, 시스템관리자 홈 기존 동작 회귀 여부 확인
