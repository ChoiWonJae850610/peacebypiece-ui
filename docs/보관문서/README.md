# 보관문서 README

- 기준 앱 버전: `0.19.94.10`
- 위치: `docs/보관문서/`
- 목적: 현재 개발 기준으로 직접 사용하지 않는 과거 설계·점검·테스트·DB 기록을 보관한다.

## 분류

- `wafl-a-type/`: WAFL A-Type 누적 작업 기록
- `점검기록/`: 과거 점검·감사·정리 기록
- `테스트기록/`: 과거 테스트·회귀 확인 기록
- `설계초안/`: 현재 기준으로 확정되지 않았거나 과거 흐름에 가까운 설계 초안
- `DB기록/`: 과거 DB 설계·마이그레이션·검증 기록

## 사용 기준

보관 문서는 삭제 대상이 아니다. 다만 현재 기능 구현 기준으로 바로 적용하지 않는다. 과거 문서를 다시 기준으로 사용할 때는 다음 순서로 확인한다.

1. `docs/현재기준/` 문서와 충돌 여부 확인
2. 현재 DB schema/API/권한 정책과 일치 여부 확인
3. 필요한 경우 새 버전 문서로 승격하거나 현재 기준 문서에 반영

## 삭제 기준

폴더 단위 삭제는 금지한다. 파일 삭제가 필요할 때는 실제 삭제 대상 파일을 patch `commit-meta.md`의 삭제 목록에 명시하고, 생성 산출물과 실제 문서를 구분한다.
