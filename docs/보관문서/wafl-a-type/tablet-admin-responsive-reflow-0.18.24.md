# 0.18.24 태블릿 관리자 화면 반응형 재보정

## 목표

태블릿에서 저장소관리, 협력업체관리, 멤버관리 화면이 내부 패널 스크롤에 갇히거나 터치 스크롤 지점이 불명확해지는 문제를 줄인다.

## 적용 원칙

- 태블릿 구간에서는 페이지 전체 스크롤을 우선한다.
- PC에서 필요한 고정 높이/내부 스크롤 구조는 `lg` 이상에서만 강하게 적용한다.
- 표 헤더는 태블릿에서 과도한 가로 공간을 차지하지 않도록 `lg` 이상에서 노출한다.
- 태블릿에서 넓은 표는 가로 스크롤을 허용하되, 세로 방향은 페이지 스크롤로 흘러가게 한다.

## 주요 변경

- `WorkspaceShell` 스크롤 영역에 `touch-pan-y`를 추가했다.
- `AdminTable`은 태블릿 이하에서 내부 세로 스크롤보다 페이지 흐름을 우선하도록 조정했다.
- `AdminPanelSection`은 태블릿 이하에서 `overflow-visible`을 사용하도록 조정했다.
- 저장소관리 휴지통 목록은 태블릿에서 내부 고정 높이를 줄이고, PC에서만 고정 패널처럼 동작하도록 조정했다.
- 멤버관리의 초대/멤버관리 패널은 태블릿 이하에서 고정 높이를 풀고, PC에서만 기존 높이 기준을 유지하도록 조정했다.
- 협력업체 목록은 태블릿에서 넓은 row가 잘리지 않도록 최소 폭 기준을 명시했다.

## 테스트 포인트

1. 태블릿 저장소관리
   - 가로/세로에서 하단까지 페이지 스크롤 가능
   - 휴지통 목록에서 행 선택/복원/삭제 버튼 터치 가능
   - 가로 화면에서 표가 잘려도 가로 스크롤로 접근 가능

2. 태블릿 협력업체관리
   - 가로/세로에서 업체 목록 하단까지 접근 가능
   - row 터치로 수정 모달 진입 가능
   - 등록 버튼 접근 가능

3. 태블릿 멤버관리
   - 멤버 초대 탭 세로 화면 하단이 잘리지 않음
   - 멤버 관리 탭 가로 화면에서 목록 터치 가능
   - 탭 전환 후 페이지 스크롤 위치와 내부 스크롤 충돌 확인

## 변경하지 않은 것

- 통계정보 화면
- DB/API/R2 흐름
- 첨부/메모/휴지통/purge 로직
- 멤버 초대/승인/권한 저장 로직
