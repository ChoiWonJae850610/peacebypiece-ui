# 04. 기기별 화면 규칙

## 1. DeviceKind

```ts
type DeviceKind = "pc" | "tablet-landscape" | "tablet-portrait" | "mobile";
```

현재 작업지시서의 `desktop/tablet/mobile` 분기는 확장 대상이다.

## 2. PC

```txt
1280px 이상
기준 설계 폭 1440px
좌측 고정 사이드바
상단 글로벌 헤더
카드/테이블 중심
```

작업지시서:

```txt
좌측 목록 / 중앙 상세 / 우측 디자인·첨부·메모
```

## 3. 태블릿 가로

```txt
900px ~ 1279px 또는 tablet + landscape
좌측 아이콘 레일
2열 중심
테이블 축약
```

작업지시서:

```txt
좌측 목록 / 우측 상세
디자인·첨부·메모는 상세 내부 탭 또는 섹션
```

중요 정책:

```txt
작업지시서 직접 그리기 기능은 태블릿 가로모드에서 사용을 제한한다.
가로모드에서는 안내 메시지를 보여주고 입력을 막는다.
세로 전환 후 사용할 수 있게 안내한다.
```

문구 예시:

```txt
디자인 직접 그리기는 태블릿 세로 화면에서 사용할 수 있습니다.
기기를 세로로 돌린 뒤 다시 시도해 주세요.
```

## 4. 태블릿 세로

```txt
768px ~ 899px 또는 tablet + portrait
1열 스택
테이블 금지
카드 리스트 전환
시트/전체화면 폼 사용
```

직접 그리기:

```txt
태블릿 세로에서만 허용하는 방향을 기본으로 한다.
화면 회전 시 그리기 내용이 사라지지 않도록 canvas state 보존이 필요하다.
```

## 5. 모바일

```txt
767px 이하
하단 탭
1열 카드 흐름
테이블 금지
BottomSheet / FullscreenSheet 사용
```

## 6. Device 판정 예시

```ts
export function getDeviceKind(width: number, height: number): DeviceKind {
  const isPortrait = height >= width;
  if (width < 768) return "mobile";
  if (width >= 1280) return "pc";
  if (isPortrait) return "tablet-portrait";
  return "tablet-landscape";
}
```

## 7. 화면별 변환표

| 화면 | PC | 태블릿 가로 | 태블릿 세로 | 모바일 |
|---|---|---|---|---|
| 운영대시보드 | 카드+2열 | 2열 | 1열 | 1열 |
| 작업지시서 | 3열 | 2열 | 목록/상세 분리 | 목록/상세 분리 |
| 협력업체 | 테이블 | 축약 테이블 | 카드 리스트 | 카드 리스트 |
| 저장소 | 카드+테이블 | 축약 테이블 | 카드 리스트 | 카드 리스트 |
| 통계 | 2열 차트 | 2열 차트 | 1열 차트 | 1열 차트 |
| 멤버관리 | 폼+목록 | 폼+목록 | 세로 섹션 | 단계형/시트 |
| 환경설정 | 카드 그리드 | 2열 | 리스트 | 리스트 |
| 로그인 | 2분할 | 2분할 | 세로 | 세로 |
| 초대 | 폼+목록 | 폼+목록 | 세로 | 단계형 |
| 에러 | 가로 카드 | 가로/중앙 카드 | 중앙 카드 | 중앙 카드 |

## 8. 회전 처리

```txt
- 입력 중인 form state 보존
- 선택된 item id 보존
- 열린 modal/sheet 상태 보존 또는 안전하게 닫기
- canvas/drawing state는 반드시 보존
- 사용 불가 orientation이면 안내 메시지 표시
```

## 9. 모바일/태블릿 웹 공유 정책

앱 개발 전 단계에서는 모바일/태블릿 웹에서 기기 기본 공유창을 우선 사용한다.

### 9.1 기본 정책

```txt
초대:
- 초대 링크 생성
- Web Share API로 공유
- 미지원 시 링크 복사 fallback
- QR 코드 표시 가능

작업지시서:
- PDF 생성
- PDF 링크 공유
- 파일 공유 지원 시 PDF 파일 공유
- 미지원 시 링크 복사 또는 다운로드 fallback
```

### 9.2 태블릿/모바일 차이

```txt
태블릿 가로:
- 공유 버튼은 상단 액션 또는 상세 카드 우측에 둔다.
- PDF 링크 공유와 다운로드를 함께 제공한다.

태블릿 세로:
- 공유 버튼은 섹션 하단 또는 하단 고정 액션으로 둔다.
- 공유 실패 시 링크 복사 안내를 띄운다.

모바일:
- 하단 액션 또는 더보기 메뉴에서 공유한다.
- 카카오톡/문자/메일은 기기 공유창에서 사용자가 직접 선택한다.
```

### 9.3 앱 필요성 판단

다음 조건 전까지는 앱 개발을 후순위로 둔다.

```txt
- 안정적인 푸시 알림이 필요함
- 오프라인 입력/동기화가 필요함
- 네이티브 카메라/파일 권한이 더 깊게 필요함
- 앱스토어/플레이스토어 검색 설치가 필요함
```
