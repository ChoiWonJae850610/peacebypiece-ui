# 0.19.23 멤버 초대 목록 responsive 기준 보정

## 목적

멤버관리의 초대 링크 목록이 태블릿에서 카드형으로 전환되며 공유/복사/취소 버튼이 중복 노출되는 문제를 줄인다. 초대 링크 목록은 표시 정보가 적으므로 모바일 세로에서만 카드형을 유지하고, 그 외 화면에서는 테이블형을 유지한다.

## 적용 기준

- PC: 테이블
- 태블릿 가로/세로: 테이블
- 모바일 가로: 테이블
- 모바일 세로: 카드

프로젝트의 기존 Tailwind breakpoint 기준을 유지하되, 초대 링크 목록은 `640px` 이상에서 테이블형으로 고정한다. 새 기기 전용 분기나 임의 user-agent 판별은 추가하지 않는다.

## 작업 버튼 기준

- PC: 복사 disabled, 취소 조건부 활성
- 공유 가능 화면: 공유, 복사, 취소 1세트만 표시
- 동일 row에서 복사 버튼이 중복 노출되지 않아야 한다.
- 취소됨/만료됨/사용됨 상태는 기존 disabled/조건부 규칙을 유지한다.

## 변경 금지

- 멤버 초대 생성 API
- 초대 취소 API
- 링크 복사/공유 처리 함수
- 가입 신청 승인/거절 API
- 멤버 권한 저장 API
- DB/API/R2 흐름
