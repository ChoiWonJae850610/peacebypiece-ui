# 0.18.25 태블릿 가로 관리자 화면 재보정

## 목적

0.18.24 적용 후 태블릿 세로 화면은 저장소관리, 협력업체관리, 멤버관리에서 정상화되었으나 태블릿 가로 화면은 여전히 PC 고정 패널 구조와 태블릿 페이지 스크롤 구조가 섞여 터치/스크롤 지점이 불명확했다.

이번 버전은 태블릿 가로를 PC 레이아웃으로 보지 않고, xl 이상에서만 PC 고정 레이아웃을 사용하도록 관리자 화면의 주요 breakpoint를 조정한다.

## 적용 원칙

- 태블릿 가로는 page scroll 중심으로 유지한다.
- 내부 세로 스크롤 컨테이너는 xl 이상 PC에서만 강하게 적용한다.
- table header/grid 고정형 표시도 xl 이상으로 미룬다.
- 협력업체관리, 저장소관리, 멤버관리만 보정한다.
- 통계정보 화면은 변경하지 않는다.
- DB/API/R2/첨부/메모/휴지통/purge 흐름은 변경하지 않는다.

## 주요 변경

- `WorkspaceShell`의 fixed-md 고정 레이아웃 기준을 xl 이상으로 이동.
- `AdminPanelSection`, `AdminTable`의 고정 높이/내부 스크롤 기준을 xl 이상으로 이동.
- 저장소관리 휴지통 목록의 PC식 스크롤/높이 기준을 xl 이상으로 이동.
- 협력업체관리 필터/목록의 PC식 가로 배치 기준을 xl 이상으로 이동.
- 멤버관리 멤버 탭/초대 탭의 2열과 고정 높이 기준을 xl 이상으로 이동.

## 확인 항목

1. 태블릿 가로 저장소관리
   - 전체 페이지가 자연스럽게 세로 스크롤되는지 확인한다.
   - 휴지통 row 선택/복원/삭제 터치가 가능한지 확인한다.

2. 태블릿 가로 협력업체관리
   - 목록 row를 터치해 수정 모달에 진입할 수 있는지 확인한다.
   - 필터와 등록 버튼이 가려지지 않는지 확인한다.

3. 태블릿 가로 멤버관리
   - 멤버 관리 탭과 멤버 초대 탭에서 하단이 잘리지 않는지 확인한다.
   - 탭 전환 후 터치/스크롤 충돌이 없는지 확인한다.

4. PC
   - xl 이상 화면에서 기존 고정 패널형 레이아웃이 유지되는지 확인한다.

5. 모바일
   - 이번 보정으로 새 잘림이나 터치 불량이 생기지 않았는지 확인한다.
