# 0.17.97 Select 전환 후보 점검

## 목적

0.17.96에서 원단·부자재 화면의 주요 선택 UI를 `AppSelect`로 전환한 뒤, 다음 단계에서 작업지시서와 관리자 화면의 select 전환 후보를 점검한다.

이번 버전은 다음 두 가지를 함께 처리한다.

1. 0.17.96 build fail 원인 보정
2. 작업지시서/관리자 화면의 select 전환 후보 정리

## build fail 보정

### 원인

`features/material-orders/MaterialOrderListPanel.tsx`에서 기존 입력 필드 class helper인 `filterFieldClassName()` 호출이 남아 있었지만, 해당 helper가 더 이상 정의되어 있지 않았다.

### 보정

이미 import되어 있던 `MATERIAL_ORDER_PANEL_FILTER_FIELD_CLASS` 상수를 검색 input의 `className`에 직접 연결했다.

```tsx
className={MATERIAL_ORDER_PANEL_FILTER_FIELD_CLASS}
```

## AppSelect 전환 원칙

### 전환 대상

- 사용자가 하나의 값을 고르는 필터
- 상태/역할/담당자/공급처처럼 option 목록이 명확한 입력
- 반복적으로 동일 스타일이 필요한 select
- 모바일/태블릿에서도 같은 선택 UI가 필요한 영역

### 보류 대상

- 브라우저 기본 select가 기능상 더 안전한 임시 개발 UI
- 파일 업로드, 날짜 입력, 숫자 입력과 섞인 복합 row editor
- 현재 편집/저장 흐름이 불안정한 화면
- 권한/상태 로직과 강하게 결합되어 있어 리팩토링이 필요한 select

## 1차 전환 후보

### 작업지시서 화면

- 좌측 목록 상태 필터
- 좌측 목록 정렬 필터
- 담당자 선택
- 공정 업체 선택
- 제품 구성 단위 선택
- 카테고리/품목/세부 형태 선택

### 원단·부자재 화면

0.17.96에서 다음 항목은 이미 1차 전환했다.

- 발주서 목록 종류 필터
- 발주서 목록 상태 필터
- 발주 상세 구분 선택
- 발주 상세 공급처 선택
- 발주 품목 단위 선택

추가 후보는 다음과 같다.

- 상태 전환 선택 UI가 추가될 경우
- 자재 선택 패널의 필터가 늘어날 경우
- 공급처 검색/선택을 combobox로 확장할 경우

### 관리자 화면

- 멤버 역할 선택
- 권한 템플릿 선택
- 승인 상태 필터
- 저장소 유형 필터
- 통계 기간/분류 선택
- 환경설정의 언어/테마/옵션 선택

### 시스템관리자 화면

- 고객사 상태 필터
- 요금제 선택
- 사용량/저장소 필터
- 감사로그 대상/행위 필터

## 후속 작업 권장

### 0.17.98

작업지시서 좌측 목록의 상태/정렬 필터부터 `AppSelect`로 전환한다.

이유:

- 작업지시서 핵심 화면에 바로 보이는 영역이다.
- 저장 로직과 직접 연결되지 않아 비교적 안전하다.
- PC/tablet/mobile에서 동일한 필터 UI 기준을 적용할 수 있다.

### 0.17.99

작업지시서 상세 내부의 단위/업체/담당자 선택 후보를 점검한다.

### 0.18.00

관리자/시스템 화면의 select 전환 후보를 별도 묶음으로 정리한다.
