# 0.19.21 멤버관리 초대 관리 하단 compact 구조 정리

## 목적

멤버관리 화면의 중심을 승인 대기와 멤버 목록으로 두고, 초대 링크 생성과 초대 링크 목록은 하단 보조 영역으로 축소한다.

## 변경 기준

- 멤버관리 단일 화면 구조는 유지한다.
- 승인 대기/멤버 목록을 초대 관리보다 먼저 배치한다.
- 초대 만료 선택, 링크 생성 버튼, 초대 링크 목록을 하나의 compact 초대 관리 카드로 합친다.
- 초대 링크 URL과 suffix 컬럼은 계속 노출하지 않는다.
- 초대 링크 목록은 상태, 생성일, 만료일, 작업만 표시한다.
- 초대 작업 버튼은 기존 복사/취소 동작을 유지한다.

## 변경 금지

- 멤버 초대 생성 API 변경 금지
- 멤버 초대 취소 API 변경 금지
- 링크 복사 실제 동작 변경 금지
- 가입 신청 승인/거절 API 변경 금지
- 멤버 권한 저장 API 변경 금지
- DB/API/R2 흐름 변경 금지

## 테스트 기준

- `/workspace/members`에서 승인 대기/멤버 목록이 초대 관리보다 먼저 보이는지 확인한다.
- 초대 관리는 화면 하단의 작은 카드 하나로 표시되는지 확인한다.
- 초대 만료 select와 링크 생성 버튼이 같은 compact 영역에 있는지 확인한다.
- 초대 목록은 상태/생성일/만료일/작업만 표시되는지 확인한다.
- 복사/취소 실제 동작이 기존과 동일한지 확인한다.
