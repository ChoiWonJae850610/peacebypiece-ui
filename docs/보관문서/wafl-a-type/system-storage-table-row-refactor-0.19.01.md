# 0.19.01 시스템 저장소 삭제 후보 row/table 구조 보정

## 목적

`/system/storage-usage`의 삭제 후보 목록에서 PC 폭임에도 row 내용이 첫 번째 영역에 세로로 몰려 보이는 문제를 보정한다.

## 변경 범위

- `SystemStoragePurgeCandidatesClient`의 `AdminTable` 호출부만 조정
- header와 row의 grid 전환 기준을 `lg`부터 동일하게 맞춤
- PC 폭에서는 테이블 컬럼 기준으로 row가 가로 정렬되도록 보정
- 좁은 폭에서는 기존처럼 1열 카드형 흐름을 유지

## 변경하지 않은 것

- 삭제 후보 조회 로직
- 선택 체크박스 상태 로직
- 새로고침
- 선택 삭제
- 전체삭제
- `/api/system/storage-usage/purge` 호출
- R2 purge 처리 흐름
- 용량 계산식
- 회사/고객사 범위 조회

## 테스트 위치

- `/system/storage-usage`

## 확인할 것

### PC 넓은 화면

- 삭제 후보 목록의 header와 row가 같은 컬럼 기준으로 정렬되는지
- row 내용이 왼쪽 한 영역에 세로로 몰리지 않는지
- 체크박스, 구분, 고객사/작업지시서, 대상, 날짜, 용량/상태, 저장소 경로가 각 컬럼 아래에 보이는지

### PC 창 폭 줄임

- 일정 폭 이하에서는 가로 스크롤 또는 1열 카드형 흐름으로 자연스럽게 전환되는지
- 정보가 사라지지 않는지

### 바뀌면 안 되는 것

- 삭제 후보 개수
- 체크박스 선택/해제
- 새로고침 버튼
- 선택 삭제 버튼
- 전체삭제 버튼
- 삭제 확인 confirm
- 실제 purge 요청 결과
