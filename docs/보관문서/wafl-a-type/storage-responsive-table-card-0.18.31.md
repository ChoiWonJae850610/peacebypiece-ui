# 0.18.31 저장소관리 responsive table/card 재정리

## 목적

0.18.30에서 저장소관리 휴지통 목록이 PC에서도 카드처럼 보이고, 태블릿 폭에 따라 컬럼 정보가 사라지는 문제를 보정한다.

## 기준

- WorkspaceShell 스크롤 구조는 변경하지 않는다.
- 넓은 화면은 table/header/grid 구조를 사용한다.
- 좁은 태블릿 세로, iPad mini, 모바일은 compact label-card 구조를 사용한다.
- 저장소 요약 카드에서 도넛 카드가 태블릿 세로 2열의 왼쪽만 차지하지 않도록 2칸 전체 span을 적용한다.

## 변경 요약

- 파일 유형 도넛 카드를 md 이상 2열 구간에서는 col-span-2로 처리하고 2xl 이상 PC에서는 다시 1칸으로 복귀한다.
- 휴지통 목록은 1180px 이상에서 table-like grid/header를 표시한다.
- 1180px 미만에서는 선택 + 삭제 대상 + 주요 메타 정보가 compact card/list로 표시된다.
- PC 및 넓은 태블릿 가로에서 삭제 대상, 삭제 일시, 작업지시서, 유형, 크기 컬럼이 정렬되어 보이도록 보정한다.

## 비변경

- DB/API/R2/첨부/메모/휴지통/purge 흐름 변경 없음.
- WorkspaceShell 스크롤 복구 구조 변경 없음.
