# 0.19.14 Toast icon-first 규격과 멤버 초대 목록 보정

## 목적

0.19.12~0.19.13에서 중앙 하단 WAFL Toast와 loading toast 정책을 잡았지만, 실제 화면 검수에서 success/warning/error 라벨과 색상 면적이 과하게 드러나고 멤버 초대 목록의 action button과 초대 링크 표시가 공통 UI처럼 보이지 않는 문제가 확인되었다.

이번 버전은 다음 기준을 좁게 보정한다.

- Toast는 텍스트 라벨 대신 상태 아이콘을 사용한다.
- Toast 배경은 테마 surface를 유지하고, 상태 구분은 아이콘과 약한 border tone으로만 처리한다.
- 멤버 초대 목록의 초대 링크는 전체 URL이 아니라 끝 8자리만 표시한다.
- 멤버 초대 목록의 복사/취소 action은 저장소관리 계열의 공통 compact action button처럼 보이도록 테두리와 아이콘을 명확히 한다.

## Toast 표시 기준

| tone | 표시 예시 | 원칙 |
| --- | --- | --- |
| success | ✅ 상태가 변경되었습니다. | 완료 결과 |
| loading | ⏳ 검토 요청 중입니다. | DB/API/화면전환 처리중 |
| warning | ⚠️ 삭제 요청 중입니다. | 주의 또는 위험 전 단계 |
| danger | ❌ 처리하지 못했습니다. | 실패/오류 |
| info | ℹ️ 링크가 복사되었습니다. | 일반 안내 |

`안내`, `완료`, `오류`, `주의`, `처리중` 같은 텍스트 라벨은 toast 내부에 직접 표시하지 않는다. 상태 의미는 icon으로 전달하고, 본문에는 실제 사용자 메시지만 남긴다.

## 멤버 초대 목록 기준

- 전체 초대 URL은 테이블에 노출하지 않는다.
- 링크 컬럼에는 초대 URL의 마지막 8자리만 `…xxxxxxxx` 형태로 표시한다.
- 전체 URL은 `title`에만 보조로 남기고, 실제 사용은 복사 버튼으로 처리한다.
- 가입 대기/활성 초대만 복사 가능하다.
- 취소됨/만료됨/사용됨 초대는 복사/취소 action을 비활성화한다.
- 복사 action은 outline icon button, 취소 action은 destructive icon button 기준으로 보인다.

## 변경 금지

- 멤버 초대 생성/취소 API 흐름은 변경하지 않는다.
- 초대 상태 판정 로직은 UI 표시 기준만 보정한다.
- DB/API/R2/권한/작업지시서/저장소 흐름은 변경하지 않는다.
