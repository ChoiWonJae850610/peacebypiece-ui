# 0.19.07 멤버관리 테이블 공통화 2차

## 목적

멤버관리 화면의 멤버 목록과 초대 링크 목록을 저장소관리·협력업체관리에서 사용하는 목록 기준에 더 가깝게 정리한다.

## 반영 범위

- `AdminTable`에 공통 정렬 header 버튼을 연결할 수 있는 구조를 추가한다.
- `AdminTable` body의 scroll 방식을 `internal` / `page`로 분리한다.
- 초대 링크 목록은 empty 상태에서 내부 scroll container가 마우스 휠을 먹지 않도록 page scroll 기준으로 전환한다.
- 멤버 관리 목록의 PC table header에 정렬 버튼을 연결한다.
- 초대 링크 목록의 상태, 초대 링크, 만료일, 생성일 컬럼에 정렬 버튼을 연결한다.

## 의도적으로 변경하지 않은 범위

- 멤버 승인/거절 API
- 멤버 상세/권한 저장 API
- 초대 링크 생성/복사/취소 API
- DB schema
- R2/첨부/메모/휴지통/purge 흐름
- 토스트 전체 공통화

## 테스트 위치

- `/workspace/members`

## 확인할 것

- 멤버 초대 탭의 empty table 영역에 마우스를 올려도 페이지 스크롤이 되는지 확인한다.
- 멤버 관리 탭의 마지막 row가 잘리지 않는지 확인한다.
- 멤버 관리 탭의 이름, 이메일, 연락처, 역할, 상태, 신청일, 승인일, 마지막 접속 header 클릭 시 정렬 방향이 바뀌는지 확인한다.
- 멤버 초대 탭의 상태, 초대 링크, 만료일, 생성일 header 클릭 시 정렬 방향이 바뀌는지 확인한다.
- 멤버 초대 생성, 복사, 취소 버튼은 기존처럼 동작해야 한다.
- 멤버 상세/권한 모달 진입과 저장 흐름은 기존처럼 유지되어야 한다.

## 바뀌면 안 되는 것

- 멤버 목록 데이터 개수
- 초대 링크 목록 데이터 개수
- 승인/거절 흐름
- 멤버 권한 저장 흐름
- 검색/필터 결과
- 고객사 관리자 권한 기준
