# 0.19.22 멤버 초대 관리 PC/모바일 역할 분리

## 목적

멤버관리 화면에서 초대 관리를 보조 영역으로 유지하되, PC와 모바일/태블릿의 사용 맥락을 분리한다.

## 반영 기준

- PC 화면에서는 초대 만료 선택과 초대 생성 버튼을 노출하지 않는다.
- PC 화면에서는 초대 링크 목록을 상태/만료 확인 및 취소 관리 용도로만 둔다.
- PC 화면의 링크 복사 버튼은 disabled 상태로 유지한다.
- 모바일/태블릿에서는 초대 만료 선택, 초대 생성, 공유, 복사, 취소를 제공한다.
- 공유는 Web Share API를 우선 사용하고, 미지원 또는 실패 시 클립보드 복사로 fallback한다.
- 기존 DB/API 흐름, 초대 생성/취소 API, 승인/권한 저장 흐름은 변경하지 않는다.

## 화면 검수 포인트

- PC에서 초대 생성 UI가 보이지 않아야 한다.
- PC에서 초대 목록의 복사 버튼은 disabled이고 취소 버튼만 조건부 활성이어야 한다.
- 모바일/태블릿에서 초대 생성 UI와 공유 버튼이 보여야 한다.
- 모바일/태블릿에서 공유 버튼 클릭 시 기기 공유창이 열리거나, 미지원 시 링크가 복사되어야 한다.
- 멤버 목록/승인 대기 영역이 화면 중심으로 유지되어야 한다.
