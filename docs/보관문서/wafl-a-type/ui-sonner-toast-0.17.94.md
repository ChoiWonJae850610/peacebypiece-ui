# 0.17.94 Sonner toast 1차 도입

## 목적

화면별로 직접 렌더링하던 `ToastMessage`를 Sonner 기반 알림으로 연결해 저장/상태 변경/오류 피드백을 점진적으로 통일한다.

## 반영 범위

- `AppToaster`를 루트 레이아웃에 1회 등록했다.
- 기존 `ToastMessage` API는 유지했다.
- 기존 화면의 `message` state와 자동 초기화 로직은 그대로 둔다.
- `ToastMessage`는 이제 DOM을 직접 렌더링하지 않고 Sonner toast를 호출한다.

## 의도

기존 화면 코드를 대량 수정하지 않고, 먼저 알림 표시 레이어만 Sonner로 바꾼다. 이후 저장/상태변경/오류 메시지를 직접 `toast` 호출로 옮길 수 있다.

## 비변경 사항

- DB/API/R2 흐름 변경 없음
- 작업지시서/원단·부자재 상태 전환 로직 변경 없음
- 첨부/메모 업로드 로직 변경 없음
- 모바일/태블릿 레이아웃 구조 변경 없음
