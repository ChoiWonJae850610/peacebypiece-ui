# 발주서 저장 mutation 직렬화 — 0.23.38

## 목적

발주서 자재 종류·공급처·납기일·상태·취소 작업을 WAFL 공통 mutation lifecycle과 문서 단위 lock으로 통합한다.

## 정책

- 동일 발주서의 저장 요청은 동시에 실행하지 않는다.
- DB/API 작업은 실제 완료까지 `await`한다.
- loading → success/error는 동일 operation ID에서 교체한다.
- 저장 실패 시 낙관적 화면 상태를 이전 snapshot으로 복원한다.
- 저장 중 발주서 선택과 추가 편집을 차단한다.
- 자재 종류 변경은 헤더 변경, 공급처·금액 초기화, 품목·할당 삭제를 하나의 DB transaction에서 처리한다.

## 적용 범위

- 자재 종류 변경
- 공급처 변경
- 납기일 변경
- 상태 변경
- 발주서 취소
- 목록 선택 잠금
- 품목·할당 편집 잠금

## DB

기존 transaction과 컬럼을 사용한다. 신규 migration은 없다.
