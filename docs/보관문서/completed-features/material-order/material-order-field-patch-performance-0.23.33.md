# 발주서 단일 필드 PATCH 및 응답 경량화 — 0.23.33

## 목적

발주서 기본정보와 상태 변경 시 전체 발주서 목록·품목·할당 정보를 매번 다시 조회하던 경로를 줄이고, 요청에 포함된 필드만 SQL UPDATE에 반영한다.

## 적용 범위

- 자재 종류
- 공급처
- 납기일
- 발주서 상태

## 변경 사항

1. 발주서 헤더 UPDATE는 요청에 실제 포함된 컬럼만 SET 절에 추가한다.
2. 자재 종류가 실제로 바뀐 경우에만 공급처·금액·품목·할당을 초기화한다.
3. 헤더 및 상태 변경 API는 변경된 발주서 한 건만 반환한다.
4. 클라이언트는 반환된 발주서 한 건을 기존 목록에 병합한다.
5. 상태 변경 전 검증은 경량 SELECT로 수행하고, 중복 전체 상세 조회를 제거한다.
6. 발주서 삭제 후에는 전체 목록 재조회 대신 현재 목록에서 해당 항목을 제거한다.

## 유지되는 정책

- 회사 범위와 담당자 가시성 검증
- 관리자 여부와 상태별 편집 잠금
- 상태 전환 검증
- 자재 종류 변경 시 품목·할당 초기화
- 상세 품목 저장 시 트랜잭션 처리

## DB Migration

없음. 기존 테이블과 컬럼을 그대로 사용한다.
