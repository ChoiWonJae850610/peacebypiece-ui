# 발주서 품목 선택 즉시 저장 — 0.23.47

## 문제

작업지시서 자재를 발주품목에 추가하거나 해제해도 `lines` 로컬 상태만 변경되고 DB 저장은 상태 변경 시점에만 실행되었습니다. 그 결과 새로고침, 공급처 변경, 납기일 변경, 발주서 재선택 시 선택한 품목이 사라졌습니다.

## 변경

- 작업지시서 자재 선택 확정 시 발주서 상세 collection mutation을 즉시 실행합니다.
- 이미 선택된 자재를 다시 누르거나 발주품목에서 삭제하면 즉시 DB에서 제거합니다.
- 품목 편집 모달의 품명·단위·수량·단가 적용 시 즉시 저장합니다.
- 저장은 기존 품목·할당 transaction을 사용하므로 lines와 allocations가 함께 원자적으로 교체됩니다.
- 성공 응답의 서버 발급 line ID와 allocation 상태를 현재 발주서와 로컬 lines에 다시 반영합니다.
- 실패하면 해당 조작 직전 lines로 rollback하고 오류 토스트를 표시합니다.
- 저장 중에는 같은 발주서의 다른 mutation을 문서 잠금으로 차단합니다.

## 저장 규칙

작업지시서 자재 선택은 임시 UI 상태가 아니라 발주서 품목 collection 변경입니다. 별도 저장 버튼 없이 다음 동작 자체가 저장 트리거입니다.

- 자재 선택 확정
- 선택 해제
- 발주품목 수정 적용
- 발주품목 삭제

## DB

스키마 변경과 migration은 없습니다. 기존 material-order detail transaction과 WAFL collection patch 결과 계약을 사용합니다.
