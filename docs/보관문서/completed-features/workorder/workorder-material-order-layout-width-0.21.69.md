# 작업지시서·발주서 레이아웃 폭 기준 재정리 (0.21.69)

## 목표

- 발주서 목록 패널을 기준으로 작업지시서 목록의 상단 컨트롤, 생성 버튼, 구분선, 목록 카드 폭을 정렬한다.
- 작업지시서 중앙 상세 패널의 과한 바깥 여백을 줄이되 내부 카드 자체 폭은 넓히지 않는다.
- 우측 패널 폭은 유지해 디자인·첨부·메모 영역의 기존 밀도를 보존한다.

## 변경

- 작업지시서 좌측 목록 패널 폭을 데스크톱 296px, XL 312px로 확대했다.
- 중앙 상세 콘텐츠 최대 폭을 780px로 제한해 진행단계·비용·공정·자재 카드가 과도하게 넓어지지 않도록 했다.
- 작업지시서 목록 shell의 3.5 패딩을 패널 전체에 한 번만 적용하도록 정리했다.
- 스크롤 목록에 별도로 적용되던 좌측 패딩을 제거해 생성 버튼과 목록 카드의 좌우 기준선을 일치시켰다.
- 우측 패널 폭은 기존 302px, XL 316px 기준을 유지했다.

## 영향 범위

- 데스크톱 및 가로형 3패널 작업지시서 화면
- 작업지시서 목록 드로어 내부 정렬
- 발주서 기능과 데이터 흐름에는 변경 없음

## 확인 항목

- 작업지시서 생성 버튼과 첫 목록 카드의 좌우 폭 일치
- 작업지시서/발주서 생성 버튼 및 하단 구분선 높이 비교
- 1480px 이하 데스크톱에서 3패널 overflow 여부
- 태블릿 가로 3패널에서 중앙 상세 카드 줄바꿈 여부
