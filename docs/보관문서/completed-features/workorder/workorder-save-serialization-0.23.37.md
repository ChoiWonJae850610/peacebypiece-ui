# Workorder save serialization — 0.23.37

## 목표

작업지시서 기본정보 저장을 실제 DB 완료까지 기다리는 직렬 저장 흐름으로 정리하고, 저장 중 다른 필드 변경이 오래된 snapshot을 덮어쓰는 경쟁 상태를 줄인다.

## 변경 사항

- 담당자 변경 함수가 DB PATCH 완료까지 `await`하도록 변경
- 재고 변경과 검수 완료의 공유 저장 함수가 DB 완료까지 `await`하도록 변경
- 저장 실패 시 낙관적 화면 상태를 이전 snapshot으로 복원
- 수량을 즉시 저장 DB 필드로 전환
- 기본정보 변경 시 `workOrdersRef`를 즉시 동기화하고 성공 시 서버 응답 상태로 다시 동기화
- 작업지시서 문서 단위 lock key를 사용해 기본정보·담당자·재고·제목 변경을 직렬화
- 성공 토스트는 실제 mutation 완료 후 표시

## 유지 정책

- DB schema 및 migration 변경 없음
- Full Reset 불필요
- 요청에 포함된 필드만 PATCH
- 단일 작업지시서 저장 중 동일 문서의 추가 저장 차단
