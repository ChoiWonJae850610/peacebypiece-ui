# 작업지시서 변경 피드백 정비 0.23.27

## 목적
작업지시서에서 작업지시서명, 담당자, 재고, 전체 저장이 서로 다른 진행·성공 문구를 사용하던 문제를 정리한다.

## 적용 정책
- 진행 문구는 변경 대상을 구체적으로 표시한다.
- 성공 문구는 `정보가 변경되었습니다.`로 통일한다.
- 전체 저장은 `작업지시서 정보 변경 중입니다.`를 사용한다.
- 작업지시서명, 담당자, 재고는 각각 기본정보 항목명을 포함한다.
- 공장 전달사항은 기존 `작업지시서 공장 전달사항 변경 중입니다.` 정책을 유지한다.

## 영향 범위
- 작업지시서 변경 피드백 presentation 상수
- 작업지시서 lifecycle/action-flow 다국어 문구
- 작업지시서 workspace write-lock 진행 문구
