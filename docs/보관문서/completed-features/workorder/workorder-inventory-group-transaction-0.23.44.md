# Workorder inventory group transaction — 0.23.44

## 목적

리오더 그룹 재고 변경을 여러 개의 개별 PATCH로 저장하지 않고 하나의 서버 요청과 DB transaction으로 처리한다.

## 정책

- 재고 수량과 재고 상태는 작업 단계와 무관하게 권한이 있으면 변경할 수 있다.
- 같은 리오더 그룹의 작업지시서만 한 요청으로 묶는다.
- 모든 대상 행이 성공해야 commit한다.
- 하나라도 없거나 업데이트에 실패하면 전체 rollback한다.
- 응답은 각 작업지시서의 재고 필드만 포함하는 `WaflPatchResult[]`이다.
- 클라이언트는 기존 작업지시서 객체에 재고 필드만 병합한다.

## 변경 흐름

1. 클라이언트가 그룹 작업지시서 ID와 재고 값을 한 번에 전송한다.
2. 서버가 회사 범위, 권한, 존재 여부, 리오더 그룹 일치 여부를 검증한다.
3. DB transaction 안에서 각 행의 재고 수량·상태·저장 시각만 UPDATE한다.
4. 전부 성공하면 commit 후 PATCH 결과 배열을 반환한다.
5. 실패하면 rollback하고 공통 mutation 실패 처리를 사용한다.
