# 저장소 사용량 제한 설계 1차 0.20.07

## 목적

`0.20.05`에서 추가한 `company_subscriptions`와 `0.20.06`에서 환경설정에 연결한 요금제·저장공간 정보를 기준으로, 파일 업로드 전에 저장공간 한도를 확인하는 1차 제한 정책을 정의한다.

이번 버전은 설계 문서와 버전 갱신만 포함한다. 실제 업로드 차단, R2 Worker 호출 변경, DB schema 변경, API 구현 변경은 `0.20.08` 이후로 분리한다.

## 기본 원칙

- 저장공간 제한은 고객사 단위로 적용한다.
- 제한 대상은 파일 업로드 계열 기능이다.
- 텍스트 기반 업무 입력, 조회, 수정, 삭제, 복원은 저장공간 초과 상태에서도 가능한 범위로 둔다.
- 80% 이상은 경고, 100% 이상은 신규 업로드 차단을 기본값으로 한다.
- 저장공간 계산 기준은 1차로 `company_files`의 활성 파일 사용량을 사용한다.
- 추후 작업지시서 첨부/디자인/문서 파일이 별도 테이블에 남아 있다면 사용량 집계 함수를 공통화하면서 포함 범위를 확장한다.
- R2 직접 업로드 직전과 업로드 메타데이터 저장 직전에 모두 검증한다.

## 사용량 기준

### 1차 포함 대상

`0.20.05` 기준에서는 `company_files` 테이블의 활성 파일을 기준으로 사용량을 계산한다.

- `company_files.deleted_at IS NULL`
- `company_files.company_id = 현재 회사 ID`
- `size_bytes`가 있는 파일

대표 이미지와 사업자등록증도 회사 저장공간 사용량에 포함한다.

### 후속 포함 후보

후속 버전에서 아래 항목이 회사 저장공간에 포함되어야 한다.

- 작업지시서 첨부 문서
- 작업지시서 디자인 이미지
- 작업지시서 메모 첨부 파일이 생기는 경우 해당 파일
- 회사 문서함 파일
- 시스템관리자 검토용으로 고객사가 업로드한 회사 증빙 파일

단, 이미 삭제되었거나 휴지통에서 영구삭제된 파일은 사용량에서 제외한다. 휴지통 보관 중 파일은 운영 정책에 따라 포함 여부를 정해야 하나, 1차 기준은 “R2에 남아 있고 복원 가능한 파일은 사용량에 포함”으로 둔다.

## 상태 구간

### 정상

조건:

```txt
usage_ratio < 80%
```

동작:

- 업로드 허용
- 환경설정에 정상 상태 표시
- 별도 경고 없음

### 경고

조건:

```txt
80% <= usage_ratio < 100%
```

동작:

- 업로드 허용
- 업로드 화면과 환경설정에 저장공간 부족 안내 표시
- 고객사 관리자에게 요금제 변경 또는 불필요 파일 삭제 안내
- 일반 멤버에게는 “관리자에게 문의” 중심으로 안내

권장 문구:

```txt
저장공간 사용량이 80%를 넘었습니다. 새 파일 업로드는 가능하지만, 한도에 도달하면 업로드가 제한됩니다.
```

### 차단

조건:

```txt
usage_ratio >= 100%
```

동작:

- 신규 파일 업로드 차단
- 기존 데이터 조회 허용
- 텍스트 기반 작업지시서 작성/수정 허용
- 메모 텍스트 입력 허용
- 기존 파일 다운로드/미리보기 허용
- 파일 삭제/휴지통 이동/영구삭제 허용
- 저장공간을 줄인 뒤 재시도할 수 있게 안내

권장 문구:

```txt
저장공간 한도에 도달하여 파일 업로드가 제한되었습니다. 기존 파일을 삭제하거나 요금제를 변경한 뒤 다시 시도해주세요.
```

## 업로드 guard 위치

업로드 제한은 UI 안내만으로 처리하면 안 된다. 서버/API 계층에서 반드시 차단해야 한다.

### 1차 guard 위치

1. 업로드 준비 API
   - R2 presigned URL 또는 R2 Worker 업로드 URL을 발급하기 전 검증
   - 파일 크기와 현재 사용량을 더했을 때 한도 초과 여부 확인

2. 메타데이터 저장 API
   - 클라이언트가 업로드 후 파일 메타데이터를 저장하기 전 재검증
   - 업로드 준비 시점과 저장 시점 사이에 다른 파일이 올라간 경우를 방지

3. 파일 교체 API
   - 기존 파일을 대체하는 경우 단순히 새 파일 크기만 더하지 않는다.
   - 계산식은 `현재 사용량 - 교체 대상 파일 크기 + 새 파일 크기`를 기준으로 한다.

### API 응답 코드

저장공간 부족은 인증/권한 실패가 아니므로 아래 형태를 권장한다.

```txt
HTTP 409 Conflict
code: STORAGE_QUOTA_EXCEEDED
```

응답 예시:

```json
{
  "ok": false,
  "code": "STORAGE_QUOTA_EXCEEDED",
  "message": "저장공간 한도에 도달하여 파일 업로드가 제한되었습니다.",
  "usageBytes": 104857600,
  "limitBytes": 104857600,
  "remainingBytes": 0
}
```

## 화면별 적용 기준

### 환경설정 > 요금제·저장공간

표시 항목:

- 현재 요금제
- 구독 상태
- 저장공간 사용량
- 저장공간 한도
- 사용률
- 80% 이상 경고
- 100% 이상 업로드 제한 안내
- 멤버 사용량/한도

### 회사 파일 업로드

대상:

- 대표 이미지
- 사업자등록증

동작:

- 파일 선택 전에도 현재 저장공간 상태를 표시한다.
- 100% 이상이면 파일 선택/업로드 버튼을 비활성화한다.
- 80% 이상이면 버튼은 유지하되 경고 문구를 표시한다.
- 선택한 파일 크기를 더하면 한도를 넘는 경우 업로드 준비 API에서 차단한다.

### 작업지시서 첨부/디자인

후속 적용 대상이다. 기존 정상 흐름을 깨지 않기 위해 `0.20.08`에서 공통 업로드 guard를 만든 뒤 연결한다.

동작 기준:

- 첨부 문서 업로드 전 guard
- 디자인 이미지 업로드 전 guard
- 업로드 완료 후 메타데이터 저장 전 guard
- 실패 시 기존 첨부/디자인 목록은 그대로 유지

### 메모

텍스트 메모는 저장공간 한도와 무관하게 허용한다.

메모에 파일 첨부 기능이 추가되면 해당 파일만 저장공간 제한 대상으로 본다.

## 권한별 안내 기준

### 고객사 관리자

고객사 관리자는 저장공간 상태와 해결 방법을 모두 볼 수 있다.

- 요금제 변경 요청
- 불필요 파일 삭제
- 저장소 관리 화면 이동
- 고객지원 문의

### 일반 멤버

일반 멤버는 저장공간 수치와 차단 사유는 볼 수 있으나 요금제 변경 액션은 노출하지 않는다.

- 관리자에게 문의 안내
- 파일 삭제 권한이 있는 화면에서는 삭제 안내
- 작업 입력 자체는 계속 가능하다는 안내

### 시스템관리자

시스템관리자는 고객사별 저장공간 상태를 조회할 수 있어야 한다. 단, 고객사 파일 원본 열람은 별도 권한/감사 로그 기준을 따른다.

## 공통 함수 후보

후속 구현에서는 아래 유틸/함수를 분리하는 방향이 적절하다.

```txt
lib/billing/storageQuotaPolicy.ts
lib/billing/storageQuotaRepository.ts
```

후보 함수:

```txt
calculateStorageUsageRatio(usageBytes, limitBytes)
getStorageQuotaTone(usageBytes, limitBytes)
assertCanUploadFile({ companyId, fileSizeBytes, replacingFileId })
getStorageQuotaSnapshot(companyId)
```

문자열 literal 비교를 줄이기 위해 상태 코드는 상수화한다.

```txt
STORAGE_QUOTA_STATUS_NORMAL
STORAGE_QUOTA_STATUS_WARNING
STORAGE_QUOTA_STATUS_BLOCKED
STORAGE_QUOTA_ERROR_EXCEEDED
```

## 자동테스트 기준

### DB/API smoke

`0.20.08` 구현 시 smoke test에 아래 contract를 추가한다.

- 한도 미만 파일 업로드 가능 판정
- 80% 이상 warning 판정
- 100% 이상 blocked 판정
- 교체 업로드 계산식 검증
- 삭제된 파일 사용량 제외
- 같은 회사 범위만 집계

### Playwright E2E

화면 테스트는 mock 기반으로 아래를 확인한다.

- 환경설정 요금제·저장공간 카드에서 80% 이상 경고 표시
- 100% 이상일 때 회사 파일 업로드 버튼 비활성화
- 일반 멤버에게 요금제 변경 액션 미노출
- 고객사 관리자에게 해결 안내 표시

## 0.20.08 구현 범위 제안

`0.20.08`은 한 번에 모든 업로드 경로를 바꾸지 않고 다음 순서로 진행한다.

1. 저장공간 snapshot repository 추가
2. 회사 파일 업로드 API에만 guard 적용
3. 환경설정 회사 파일 UI에 warning/blocked 상태 표시
4. smoke test 추가
5. Playwright mock 테스트 추가

작업지시서 첨부/디자인, 문서함, 기타 업로드 경로는 0.20.08 이후 작은 단위로 확장하는 편이 안전하다.

## 제외 범위

이번 설계와 0.20.08 1차 구현에서 아래는 제외한다.

- 실제 PG 연동
- 결제수단 저장
- 자동 요금제 변경
- 저장공간 자동 증설
- 시스템관리자 강제 한도 변경 UI
- R2 오브젝트 실제 삭제 정책 변경
- 휴지통/purge 동작 변경

## 결론

저장소 사용량 제한은 고객사 단위 파일 업로드 guard로 시작한다. `80% 안내`, `100% 업로드 차단`, `텍스트 업무 계속 허용`, `삭제/정리 허용`을 기본 원칙으로 두고, 첫 구현은 회사 파일 업로드 경로에만 적용한다. 작업지시서 첨부/디자인 등 기존 고위험 업로드 흐름은 공통 guard가 안정화된 뒤 단계적으로 연결한다.
