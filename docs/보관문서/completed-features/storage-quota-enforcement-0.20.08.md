# 저장소 사용량 제한 1차 (0.20.08)

## 목적

`0.20.07` 설계 기준에 따라 회사 파일 업로드 경로부터 저장공간 한도 guard를 적용한다.

이번 버전은 기존 R2 업로드 구조를 유지하면서 업로드 준비 API와 메타데이터 저장 API 양쪽에 같은 저장공간 검증을 추가한다.

## 적용 범위

- 회사 파일 업로드 준비 API: `POST /api/admin/company-files/upload`
- 회사 파일 메타데이터 저장 API: `POST /api/admin/company-files`
- 대상 파일:
  - 대표 이미지
  - 사업자등록증

작업지시서 첨부, 디자인 첨부, 일반 저장소 문서 업로드 제한은 후속 버전에서 확장한다.

## 판정 기준

현재 사용량은 `GET /api/admin/subscription`과 같은 기준을 사용한다.

- `company_files.deleted_at IS NULL`인 활성 회사 파일의 `size_bytes` 합계
- 현재 교체하려는 같은 `file_type`의 기존 활성 파일 크기는 차감
- 신규 파일 크기를 더한 예상 사용량을 계산

| 상태 | 조건 | 처리 |
| --- | --- | --- |
| 정상 | 예상 사용량 < 80% | 업로드 허용 |
| 주의 | 예상 사용량 >= 80% and <= 100% | 업로드 허용 + 경고 |
| 차단 | 예상 사용량 > 100% | 업로드 차단 |

## API 응답

### 80% 이상

업로드는 허용하되 응답에 `quota.status = "warning"`을 포함한다.

```json
{
  "ok": true,
  "quota": {
    "status": "warning",
    "message": "저장공간 사용량이 80% 이상입니다."
  }
}
```

### 100% 초과

업로드 준비 단계와 메타데이터 저장 단계 모두 `409`를 반환한다.

```json
{
  "ok": false,
  "error": "STORAGE_QUOTA_EXCEEDED",
  "message": "저장공간 한도 100MB를 초과하여 파일을 업로드할 수 없습니다."
}
```

## 이중 guard 이유

R2 업로드는 준비 URL 발급 후 실제 PUT, 이후 메타데이터 저장 순서로 진행된다.
따라서 업로드 준비 시점과 저장 시점 사이에 다른 파일이 추가될 수 있다.

이번 버전은 두 API 모두 같은 quota 검사를 수행해 race condition을 줄인다.

## UI 처리

- 저장공간 80% 이상 경고가 오면 업로드 성공 토스트에 경고 문구를 함께 표시한다.
- 저장공간 초과 응답이 오면 R2 업로드를 시작하지 않고 차단 메시지를 표시한다.
- 환경설정 전체 화면은 차단하지 않는다.

## 자동테스트 기준

- Playwright 설정 화면 mock 업로드 응답에 quota warning을 추가했다.
- 회사 파일 업로드 성공과 함께 80% 이상 경고 문구가 표시되는지 확인한다.
- DB/API smoke는 기존 `company_subscriptions`, `company_files` 계약을 유지한다.

## 제외 범위

- 실제 PG/결제 연동 없음
- 요금제 변경 요청 없음
- 작업지시서 첨부/디자인 업로드 제한 없음
- R2 Worker 기존 흐름 변경 없음
- 휴지통/purge 흐름 변경 없음
