# 0.20.52 태블릿 2패널 전환 및 모달 상태 안정화

## 목적

- iPad mini 가로처럼 폭이 부족한 태블릿 가로 화면에서 목록을 드로어로 숨기고 본문과 우측 보조 패널을 유지한다.
- 분류/담당자/재고 모달의 모바일 터치, 중복 open, 선택/적용 흐름을 안정화한다.

## 적용 정책

### 태블릿 가로 작업지시서

- 충분히 넓은 태블릿 가로: 목록 / 본문 / 첨부·디자인·메모 3패널.
- 폭이 부족한 태블릿 가로: 목록은 드로어, 본문과 첨부·디자인·메모는 2패널로 유지.
- 태블릿 세로와 모바일은 기존처럼 목록 드로어 중심 구조를 유지한다.

### 분류 모달

- 상위 분류 클릭: 하위 목록 펼침.
- 같은 상위 분류 재클릭: 하위 목록 접힘.
- 중분류도 같은 방식으로 펼침/접힘.
- 최종 하위 항목 클릭은 임시 선택만 변경한다.
- 실제 반영은 하단 우측 적용 버튼을 눌렀을 때만 수행한다.

### 담당자 변경 모달

- 담당자 후보 클릭은 임시 선택만 변경한다.
- 적용 버튼을 눌러야 실제 담당자가 변경된다.
- 이미 모달이 열려 있으면 추가 open 요청을 무시한다.

## 변경 제외

- 상태전환 로직 변경 없음.
- 권한 로직 변경 없음.
- API/DB/R2/첨부/메모/휴지통/purge 흐름 변경 없음.
