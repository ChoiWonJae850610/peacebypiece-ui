# Simulator category tree normalization — 0.24.01

## 문제

0.23.99의 Simulator 품목 fixture는 각 3단계 경로마다 1차·2차·3차 row를 모두 새로 생성했다. 그 결과 환경설정에서 `상의`, `하의`, `아우터`가 여러 번 반복되고 동일한 `티셔츠`, `맨투맨` 부모도 세부 품목마다 중복됐다.

## 수정

- 회사별 기존 Simulator `item_categories`를 Seed transaction 안에서 제거한다.
- 1차는 이름 경로, 2차는 `1차::2차`, 3차는 `1차::2차::3차`를 고유 경로로 사용한다.
- 동일 경로 노드는 Map에서 재사용한다.
- 작업지시서는 정규화된 category id로 다시 연결한다.
- Seed 계획의 `itemCategories` 집계도 실제 고유 노드 수를 표시한다.

## 기대 구조

- 1차: 상의, 하의, 아우터, 원피스, 셋업, 잡화, 홈웨어
- 상의 선택 시: 맨투맨, 티셔츠, 셔츠, 후드 등
- 맨투맨 선택 시: 기본 맨투맨, 오버핏 맨투맨, 기모 맨투맨 등

Seed를 반복 실행해도 동일 부모가 추가 생성되지 않아야 한다.
