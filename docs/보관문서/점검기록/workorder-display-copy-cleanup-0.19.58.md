# 0.19.58 작업지시서 표시/문구 정리 1차

## 목적

작업지시서 화면의 기능 로직을 변경하지 않고, 사용자가 빈 상태·로딩 상태·첨부/디자인/메모 패널을 이해하기 쉽게 문구를 정리한다.

## 적용 범위

- 작업지시서 empty workspace 문구
- 상세 패널/우측 패널 로딩 문구
- 첨부·디자인·메모 빈 상태 문구
- 발주서 PDF 재생성 안내 문구
- 홈 이동 title/aria-label
- 작업지시서 목록 카드의 추가 작업 aria-label
- 한국어/영어 i18n 동시 정리

## 제외 범위

- 작업지시서 상태 변경 로직
- DB 저장 로직
- R2 업로드/삭제/복원 흐름
- PDF 생성 API
- 첨부/디자인/메모 CRUD 로직
- 3패널 layout / 모바일 sheet / 태블릿 split 구조
- 권한 정책

## 변경 기준

### Empty state

기존의 단순한 “없습니다” 문구를 다음 기준으로 바꾼다.

- 왜 비어 있는지 설명한다.
- 사용자가 다음에 무엇을 해야 하는지 암시한다.
- 권한 범위 때문에 보이지 않을 수 있음을 과하게 기술하지 않는다.

### Loading state

DB라는 내부 구현 단어가 사용자에게 직접 노출되는 문구는 줄이고, 화면 기준 설명으로 바꾼다.

- 상세 정보와 연결 데이터
- 첨부·디자인·메모
- 오른쪽 패널 표시 데이터

### 첨부/디자인/메모

첨부와 디자인의 역할을 분리해서 안내한다.

- 첨부: PDF, 참고 이미지, 전달 문서
- 디자인: 디자인 이미지, 직접 그리기 저장 결과
- 메모: 확인사항, 진행 메모

## QA

1. 작업지시서가 없는 계정 또는 검색 결과 없음 상태에서 문구 확인
2. 작업지시서 선택 전 가운데/오른쪽 패널 empty state 확인
3. 첨부/디자인/메모가 없는 작업지시서에서 빈 상태 문구 확인
4. 발주서 PDF가 없는 발주요청 작업에서 재생성 안내 문구 확인
5. 홈 버튼 title/aria-label로 인한 UI 폭 변화가 없는지 확인
6. 한국어/영어 전환 시 문구가 깨지지 않는지 확인
