# Workspace / System Integration Checkpoint

기준 버전: 0.16.49

목적:
- `/workspace/materials` 추가 이후 기존 저장소, 통계, 시스템 감사로그 흐름과 충돌하지 않는지 점검한다.
- R2 삭제/복원/purge 흐름은 직접 수정하지 않고, 경계와 점검 기준만 고정한다.

## 1. 라우트 경계

### workspace

고객사 업무 화면은 다음 경로를 기준으로 둔다.

- `/workspace`
- `/workspace/workorders`
- `/workspace/materials`
- `/workspace/partners`
- `/workspace/files`
- `/workspace/stats`
- `/workspace/members`
- `/workspace/settings`

`/workspace/materials`는 기준정보 성격의 원단·부자재 화면이다. 작업지시서 상세의 원단·부자재 연결 패널은 이 기준정보를 참조하되, 작업지시서의 기존 생산구성 편집 흐름을 대체하지 않는다.

### system

시스템관리자 화면은 다음 경로를 유지한다.

- `/system/storage-usage`
- `/system/audit-logs`
- `/system/customers`
- `/system/billing`
- `/system/policies`

시스템관리자 저장소 화면은 전체 고객사 기준 R2 purge 후보를 다룬다. 고객사 업무 저장소 화면(`/workspace/files`)의 휴지통/복원/삭제 요청 흐름과 분리한다.

## 2. 저장소 충돌 점검

원단·부자재 1차 기능은 파일 업로드를 포함하지 않는다. 따라서 0.16.23 기준으로 다음을 원칙으로 둔다.

- `materials` 기준정보는 저장소 사용량에 포함하지 않는다.
- `workorder_material_lines` 연결 정보는 저장소 사용량에 포함하지 않는다.
- R2 key 생성, 첨부 업로드, 메모 첨부, 휴지통, restore, purge 흐름은 변경하지 않는다.
- 원단·부자재에 이미지/거래명세서/샘플 사진 첨부가 붙는 경우에는 별도 버전에서 storage domain에 명시적으로 편입한다.

점검 대상:

- `/workspace/files` snapshot API가 원단·부자재 테이블을 잘못 조회하지 않아야 한다.
- `/system/storage-usage` purge 후보가 원단·부자재 연결 row를 파일 후보로 오인하지 않아야 한다.
- 작업지시서 purge 시 문서, 디자인, 메모 처리 범위가 기존과 동일해야 한다.

## 3. 통계 충돌 점검

0.16.23 기준 원단·부자재는 통계에 직접 포함하지 않는다. 통계 반영은 다음 단계 이후 별도 정책으로 처리한다.

후보 통계:

- 원단·부자재 사용 빈도
- 작업지시서별 원단·부자재 연결 수
- 발주 상태별 원단·부자재 항목 수
- 공급처별 원단·부자재 사용량

주의:

- 기존 `/workspace/stats`의 생산 수량, 지연율, 검수/불량률, 저장소 사용량 계산을 원단·부자재 테이블과 즉시 섞지 않는다.
- 통계가 필요해지면 `workorder_material_lines.company_id`, `workorder_id`, `material_id`, `order_status` 인덱스를 기준으로 repository query를 추가한다.
- 통계 성능 때문에 중복 컬럼을 추가하기 전에는 실제 쿼리 병목과 인덱스 적용 여부를 먼저 확인한다.

## 4. 감사로그 충돌 점검

시스템 감사로그의 target type에 `material`을 허용한다. 다만 0.16.23에서는 원단·부자재 생성/수정/삭제 감사로그를 실제로 기록하지 않는다.

후속 기록 후보:

- `material.create`
- `material.update`
- `material.delete`
- `material.line.attach`
- `material.line.detach`
- `material.line.order_status_update`

감사로그 기록 시 원칙:

- `companyId`를 반드시 포함한다.
- 원단·부자재 이름, 거래처명 등 현재 조회 가능한 값은 metadata에 과도하게 복사하지 않는다.
- 당시 기록 보존이 필요한 경우에만 snapshot 필드를 제한적으로 사용한다.
- R2 key, 실제 URL, token, secret 등은 metadata에 넣지 않는다.

## 5. 0.16.23 결론

- `/workspace/materials`는 workspace 업무 경계에 편입한다.
- 저장소/R2 사용량에는 아직 편입하지 않는다.
- 기존 통계 계산에는 아직 편입하지 않는다.
- 시스템 감사로그 target type에는 `material`을 추가해 후속 연결 여지를 둔다.
- 실제 감사로그 기록, 통계 집계, 저장소 첨부 편입은 후속 버전에서 별도로 진행한다.
