# 0.19.65 기능 추가 전 안정화 QA 기준

## 목적

0.19.65는 테스트 불가 상태에서 실제 기능 로직을 건드리지 않고, 0.19.59부터 0.19.64까지 누적된 표시/문구/버튼/상태 보정 작업의 QA 기준을 정리한다.

## 이번 버전에서 변경하지 않는 범위

- DB schema, migration, full_reset.sql
- API route와 repository 저장 로직
- R2 upload/delete/restore/purge 흐름
- PDF 생성 route/template/worker 흐름
- 작업지시서 상태 전환 조건
- 원단·부자재 발주 생성/상태 변경/자재 할당 계산식
- 첨부/디자인/메모 저장·삭제·복원 로직
- package.json, package-lock.json

## 누적 QA 우선순위

### 1. 작업지시서 상세

- 작업지시서 상세 진입 시 중앙 패널과 우측 패널이 정상 표시되는지 확인한다.
- 상단 요약 카드가 생산, 자재, 원가 기준으로 표시되는지 확인한다.
- 원가 권한이 없는 계정에서 원가 정보가 노출되지 않는지 확인한다.
- 진행 단계 버튼이 상태별로 정상 비활성/활성 처리되는지 확인한다.
- 현재 단계 또는 처리 중 단계에서 안내 문구가 과도하게 중복 표시되지 않는지 확인한다.

### 2. 작업지시서 첨부·디자인·메모

- 첨부 파일과 디자인 파일 제목 옆 개수 배지가 실제 개수와 맞는지 확인한다.
- 첨부/디자인 파일명 아래 작성자와 파일 유형 보조 정보가 화면을 밀어내지 않는지 확인한다.
- 대표 디자인 지정, 미리보기, 삭제 버튼의 기존 동작이 유지되는지 확인한다.
- 메모 작성, 수정, 삭제, 댓글 열기/닫기 동작이 기존과 동일한지 확인한다.
- 메모 입력 글자 수와 작성자 라벨이 모바일/태블릿에서 깨지지 않는지 확인한다.

### 3. 작업지시서 PDF 표시

- 발주서 PDF가 없는 발주요청 완료 작업지시서에서 누락 안내와 재저장 버튼이 표시되는지 확인한다.
- 발주서 PDF가 이미 첨부된 작업지시서에서 저장 완료 안내가 표시되는지 확인한다.
- PDF 미리보기와 다운로드 동작이 기존과 동일한지 확인한다.
- PDF 재저장 버튼 클릭 후 기존 PDF 생성/첨부 흐름이 유지되는지 확인한다.

### 4. 원단·부자재 발주 목록/상세

- 원단·부자재 발주 화면 진입 시 페이지 설명, 검색 placeholder, 필터 문구가 자연스러운지 확인한다.
- 발주서 없음, 검색 결과 없음, loading, error 상태 문구가 정상 표시되는지 확인한다.
- 상세 패널의 발주 기본정보, 자재 종류, 공급처, 발주 품목 문구가 일관되는지 확인한다.
- 발주 품목 empty state가 기존 기능 흐름을 방해하지 않는지 확인한다.

### 5. 원단·부자재 발주 버튼/상태

- 새 발주서 생성 버튼 클릭 시 기존 생성 흐름이 유지되는지 확인한다.
- 발주 진행 단계 버튼 클릭 중 처리 중 표시가 보이는지 확인한다.
- 작성중이 아닌 발주서에서 품목 수정/삭제 잠금 안내가 표시되는지 확인한다.
- 모바일/태블릿 자재 선택 버튼이 기존처럼 선택 패널을 여는지 확인한다.
- 선택됨, 발주 완료, 진행중 자재 버튼이 기존처럼 비활성 유지되는지 확인한다.
- PDF 생성, 미리보기, 발주 완료 흐름이 기존과 동일한지 확인한다.

## 디바이스별 확인 기준

- PC Chrome: 3패널 레이아웃, 버튼 정렬, 테이블 밀도 확인
- iPad mini: 좌우 패널 폭, 버튼 줄바꿈, 메모 입력 영역 확인
- iPad Pro 13: 넓은 화면에서 카드/테이블 여백 과다 여부 확인
- Galaxy Tab: 가로/세로 회전 후 stale width 현상 재발 여부 확인
- iPhone: 모바일 액션 섹션 버튼 순서와 줄바꿈 확인

## 기능 추가 전 통과 기준

- 화면 진입 자체를 막는 런타임 오류가 없어야 한다.
- 상태 변경, PDF, R2, 첨부, 메모, 자재 선택의 기존 동작이 유지되어야 한다.
- 표시 문구 변경으로 인해 사용자가 내부 용어를 보거나 기능 의미를 오해하지 않아야 한다.
- 모바일/태블릿에서 버튼이 화면 밖으로 밀려나지 않아야 한다.
- 테스트 불가 상태에서 발견하지 못한 회귀는 다음 QA 보정 버전에서 별도 처리한다.
