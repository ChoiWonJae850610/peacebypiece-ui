# 0.11.71 작업지시서 목록 필터/정렬 회귀 보정

## 목적

0.11.70에서 추가한 작업지시서 목록 상태 필터와 정렬의 1차 회귀 위험을 줄인다.

## 반영 내용

- 기본 `진행 중` 목록에서 `completed`뿐 아니라 legacy `완료` 상태도 제외한다.
- `완료` 필터에서 `completed`, `완료` 상태를 함께 조회한다.
- 검토요청, 검토완료, 검수/생산, 작성중, 반려 필터도 legacy 상태값을 함께 조회한다.
- 목록 summary 조회에서 `deleted_at IS NULL` 조건을 추가해 휴지통 이동 항목이 섞이는 위험을 줄인다.
- 선택 적용용 DB index SQL을 추가했다.

## 추가 SQL

- `db/schema/patch_0_11_71_workorder_list_indexes.sql`

## 테스트 체크포인트

1. `/worker` 기본 진입 시 완료 작업지시서가 목록에서 빠지는지 확인한다.
2. legacy 상태값 `완료`가 남아 있는 데이터도 기본 목록에서 빠지는지 확인한다.
3. 상태 필터를 `완료`로 변경하면 완료 작업지시서만 보이는지 확인한다.
4. 상태 필터를 `전체`로 변경하면 완료 작업지시서까지 포함되는지 확인한다.
5. 검토요청, 검토완료, 검수/생산, 작성중, 반려 필터가 각각 정상 조회되는지 확인한다.
6. 이름순, 업체명순, 납기일 빠른순 정렬이 화면 목록에 반영되는지 확인한다.
7. 휴지통으로 이동한 작업지시서가 `/worker` 목록에 섞이지 않는지 확인한다.

## 주의

- SQL은 자동 적용하지 않는다.
- Neon에서 직접 적용하기 전 `EXPLAIN ANALYZE`로 실제 병목이 summary query인지 확인하는 것이 좋다.
- 이번 버전은 상세 hydrate, 첨부, 메모, R2 흐름을 변경하지 않는다.
