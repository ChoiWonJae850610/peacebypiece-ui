# 0.9.210 — 시스템 관리자 통계 1차

## 목적

시스템관리자가 고객사별 운영 상태를 한 화면에서 확인할 수 있도록 `/system` 콘솔에 시스템 통계 1차 상황판을 추가한다.

이번 버전은 실제 API 구현이 아니라, 시스템 관리자 화면에서 확인해야 할 통계 기준과 UI 위치를 먼저 고정한다.

## 반영 범위

- `/system` 콘솔에 시스템 관리자 통계 섹션 추가
- 고객사별 작업지시서 수 표시
- 고객사별 저장 용량 사용률 표시
- 요금제별 고객사 수 표시
- 용량 초과 위험 고객 표시
- 고객사별 최근 활동일 표시
- 시스템 콘솔의 통계 탭 상태를 API 준비에서 화면 반영으로 변경

## 1차 지표

| 지표 | 설명 | 0.9.210 상태 |
| --- | --- | --- |
| 고객사 수 | 시스템에 등록된 고객사 수 | sample 기반 표시 |
| 활성 고객사 수 | 최근 활동 또는 운영중 고객사 | sample 기반 표시 |
| 고객사별 작업지시서 수 | 고객사별 누적 작업지시서 수 | sample 기반 표시 |
| 저장 용량 사용률 | 사용량 / 한도 비율 | sample 기반 표시 |
| 요금제 분포 | Basic / Standard / Premium 고객사 수 | sample 기반 표시 |
| 최근 활동일 | 고객사별 마지막 업무 활동 시각 | sample 기반 표시 |
| 용량 위험 고객 | 용량 사용률 70% 이상 후보 | sample 기반 표시 |

## 향후 DB/API 연결 기준

향후 실제 DB 연결 시 아래 데이터가 필요하다.

- companies
- company_users
- spec_sheets
- attachments
- company plan 또는 billing plan 테이블
- storage usage summary
- audit_logs 또는 activity_logs

## SQL DDL 필요 여부

불필요.

이번 버전은 UI와 상수 데이터 구조만 추가한다. 테이블, 컬럼, index, constraint는 변경하지 않는다.

## 전체 리셋 필요 여부

불필요.

0.9.2071 seed 데이터는 그대로 사용할 수 있다.

## package 변경 여부

없음.

`package.json`과 `package-lock.json`은 수정하지 않는다.

## 테스트 케이스

1. `/system` 접속
2. 시스템 관리자 통계 1차 섹션 표시 확인
3. 고객사별 사용 현황 카드 표시 확인
4. 저장 용량 사용률 progress bar 표시 확인
5. 요금제 분포 표시 확인
6. 운영 위험 신호 표시 확인
7. 시스템 콘솔 통계 탭이 화면 반영 상태로 표시되는지 확인
8. `APP_VERSION`이 `0.9.210`인지 확인
