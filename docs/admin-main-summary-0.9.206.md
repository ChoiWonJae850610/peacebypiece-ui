# 0.9.206 — 고객 관리자 메인 요약/최근 작업 진행표

## 목적

고객관리자 메인을 단순 메뉴 허브가 아니라 실제 업무 상황판으로 전환한다. 통계 화면은 분석용 화면으로 유지하고, `/admin` 메인은 오늘 확인해야 하는 작업과 주요 운영 메뉴를 빠르게 확인하는 진입점으로 정리한다.

## 반영 범위

- `/admin` 메인에 기존 `AdminOperationsDashboard`를 연결한다.
- 최근 작업 진행표, 검토대기, 검수대기, 입고 지연 요약을 고객관리자 메인에서 확인할 수 있게 한다.
- 기존 메뉴 카드와 API 준비 카드는 유지하되, 메인 화면의 우선순위를 작업 진행 요약으로 변경한다.
- 통계 메뉴는 `/admin/dashboard` 화면 연결로 정리한다.
- 협력업체 메뉴의 사용자 표시명을 `협력업체 관리`로 정리한다.
- 히스토리는 메인 핵심 지표가 아니라 추적용 화면으로 분리한다.

## 화면 역할 분리

| 화면 | 역할 |
| --- | --- |
| `/admin` | 고객관리자 메인 상황판. 최근 작업, 검토 필요 항목, 운영 메뉴 진입 |
| `/admin/dashboard` | 통계 분석 화면. 기간 필터와 차트 중심 |
| `/admin/history` | 상태 변경과 주요 작업 기록 추적용 화면 |

## 데이터 기준

이번 버전은 기존 `getAdminOperationalDashboardSnapshots` repository를 재사용한다.

- DB 설정이 있으면 `spec_sheets`, `orders`, `attachments` 기준으로 작업 진행 요약을 계산한다.
- DB 설정이 없으면 0건 상태로 안전하게 표시한다.
- 조회 실패 시 화면이 깨지지 않고 error source state로 0건 표시한다.

## SQL DDL 필요 여부

불필요.

이번 버전은 기존 repository와 기존 DB schema를 재사용한다. 테이블, 컬럼, index, constraint를 추가하지 않는다.

## 전체 리셋 필요 여부

불필요.

`full_reset.sql` 실행이 필요하지 않다.

## package 변경 여부

없음.

`package.json`과 `package-lock.json`은 수정하지 않는다.

## 테스트 케이스

1. `/admin` 접속 시 고객관리자 메인 title이 표시되는지 확인한다.
2. `/admin`에서 운영 대시보드가 표시되는지 확인한다.
3. DB 데이터가 없는 상태에서도 검토대기/검수대기 작업 없음 메시지가 표시되는지 확인한다.
4. 오늘/이번주/이번달 기간 버튼이 동작하는지 확인한다.
5. 통계 카드가 `/admin/dashboard`로 이동하는지 확인한다.
6. 협력업체 관리 카드가 `/admin/partners`로 이동하는지 확인한다.
7. 운영 히스토리 카드가 `/admin/history`로 이동하는지 확인한다.
8. `APP_VERSION`이 `0.9.206`인지 확인한다.
