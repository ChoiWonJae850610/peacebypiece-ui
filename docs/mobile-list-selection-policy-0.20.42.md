# 0.20.42 작업지시서/원단·부자재 목록 선택 정책 및 모바일 드로어 정리

## 목적

작업지시서와 원단·부자재 발주 화면에서 검색/필터/정렬 조작만으로 본문 상세가 자동 변경되지 않도록 선택 정책을 통일했다.

## 선택 정책

- 검색어 입력은 목록 표시만 변경한다.
- 검색어 삭제는 목록 표시만 복원한다.
- 필터/정렬 변경은 목록 표시만 변경한다.
- 본문 상세는 사용자가 목록 항목을 직접 선택한 경우에만 변경한다.
- 현재 선택 항목이 검색/필터 결과에 없어도 본문 상세는 유지한다.
- 현재 선택 항목이 실제 데이터에서 삭제되거나 접근 불가한 경우에만 fallback 선택을 허용한다.

## 작업지시서

- 필터링된 목록의 첫 항목을 자동 선택하던 흐름을 제거했다.
- 선택 유지 기준을 필터 결과가 아니라 실제 작업지시서 데이터 기준으로 변경했다.
- 검색/필터/정렬 변경 시 URL의 workOrderId를 삭제하지 않도록 보정했다.
- PC/태블릿/모바일 모두 같은 선택 정책을 사용한다.

## 원단·부자재 발주

- 목록 필터는 기존처럼 목록 표시만 변경하고 선택 상태를 변경하지 않는다.
- 모바일 발주서 목록 드로어는 작업지시서와 같은 WaflMobileListDrawer 기반 구조를 유지한다.
- 모바일 드로어 내부에서는 중복 제목/건수 뱃지를 제거하고, 검색/필터/목록만 보이도록 정리했다.

## 영향 제한

- 상태전환/권한/API/DB/R2 흐름은 변경하지 않았다.
- PC 3패널과 태블릿 구조는 유지했다.
- 직접그리기 0.20.41 최상단 레이어 수정 흐름은 변경하지 않았다.
