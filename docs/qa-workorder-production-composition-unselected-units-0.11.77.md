# 0.11.77 작업지시서 생산구성 선택 안함/기본 단위 보정

## 목적

작업지시서 생산구성에서 상위 선택값이 비어 있는 경우 하위 거래처/외주처 후보를 노출하지 않도록 정리한다.

## 반영 내용

- 원단/부자재 구분이 `선택 안함` 또는 기타 미지원 값이면 거래처 후보를 빈 목록으로 처리한다.
- 원단/부자재 구분이 `원단`이면 원단 거래처만 노출한다.
- 원단/부자재 구분이 `부자재`이면 부자재 거래처만 노출한다.
- 외주공정 공정 select에 `선택 안함`을 항상 포함한다.
- 외주공정 공정이 `선택 안함`이면 외주처 후보를 빈 목록으로 처리한다.
- 외주공정 공정이 `선택 안함`일 때 표시값은 `-`로 보이도록 정리한다.
- 원단으로 변경하면 단위를 `야드`로 자동 설정한다.
- 부자재로 변경하면 단위를 `개`로 자동 설정한다.
- 신규 외주공정의 단가 기준 기본값을 `장당`으로 변경한다.
- 시스템 단위 표준에 `개` 단위를 추가하는 patch SQL을 추가했다.
- `full_reset.sql`과 `system_standards_seed.sql`에도 `개` 단위 표준을 반영했다.

## 확인 항목

1. 원단/부자재 구분을 `선택 안함`으로 바꾼 뒤 거래처 select에 업체가 나오지 않는지 확인한다.
2. 원단을 선택하면 거래처 후보가 원단 거래처로 제한되고 단위가 `야드`가 되는지 확인한다.
3. 부자재를 선택하면 거래처 후보가 부자재 거래처로 제한되고 단위가 `개`가 되는지 확인한다.
4. 외주공정 공정 목록 첫 항목에 `선택 안함`이 있는지 확인한다.
5. 외주공정을 `선택 안함`으로 바꾸면 외주처 후보가 나오지 않고 표시값이 `-`로 보이는지 확인한다.
6. 신규 외주공정 추가 시 단가 기준이 `장당`인지 확인한다.
7. 운영 DB에 `db/schema/patch_0_11_77_unit_each_standard.sql`을 적용하면 기준정보 단위 표준에 `개`가 추가되는지 확인한다.
