# 0.10.1 개인 설정 1차 진입점

## 목적

0.10.1은 고객관리자/일반 멤버 공통으로 사용할 개인 설정 진입점을 추가한다. 조직 설정과 개인 설정을 분리하기 위한 1차 구조이며, DB schema와 권한 시스템은 아직 변경하지 않는다.

## 이번 버전에서 확정한 구분

### 개인 설정

개인 설정은 사용자 개인 선호에 가까운 항목만 다룬다.

- 언어
- 색상
- 화면 밀도
- 기본 진입 화면 후보

현재는 브라우저 localStorage 기반으로 저장한다. 로그인/멤버 DB가 고정된 뒤 사용자별 DB 저장으로 옮길 수 있도록 `lib/me/personalSettings.ts`에 정규화와 저장 키를 분리했다.

### 조직 설정

조직 설정은 고객사 운영 정책에 해당한다.

- 저장소 정책
- 파일 정책
- 알림 정책
- 협력업체 정책
- 권한 정책
- 요금제/용량 정책

이 항목은 `/admin/settings`에서 유지한다.

## 화면 구조

- `/me/settings`: 개인 환경설정
- `/workspace`: 일반 멤버용 공통 홈
- `/admin/settings`: 고객사 조직 설정

## 이번 버전에서 한 일

1. `/me/settings` 페이지 추가
2. 개인 설정 컴포넌트 추가
3. 개인 설정 정규화/저장 유틸 추가
4. `/workspace`의 개인 설정 카드를 사용 가능 상태로 변경
5. 고객관리자 상단바에서 개인 설정으로 이동할 수 있는 버튼 추가
6. 한국어/영어 i18n 문구 추가

## 이번 버전에서 하지 않은 일

- 사용자별 DB 저장
- 로그인 사용자 식별
- 권한별 개인 설정 제한
- 조직 설정 DB schema 변경
- `/admin/settings` 재구성
- 멤버 초대/권한 관리 UI

## 후속 작업 후보

1. `/admin/settings`를 조직 설정 중심으로 재정리
2. `/me/settings` 저장소를 사용자 DB로 이전
3. 기본 진입 화면 설정을 로그인 후 redirect 정책과 연결
4. 화면 밀도 설정을 실제 목록/카드 UI에 반영
5. 권한 기반 카드 노출과 멤버 관리 DB 설계
