# 0.9.22361 — 작업지시서 삭제 후보 시스템관리자 연결 보정

## 목적

작업지시서가 삭제되었을 때 시스템관리자 `/system/storage-usage`의 실제 삭제 후보 화면에서 파일 후보만 보이고 작업지시서 후보가 보이지 않는 문제를 보정한다.

## 변경 기준

- 파일 후보는 기존처럼 `attachment_trash_items` 기준으로 유지한다.
- 작업지시서 후보는 `spec_sheets`의 삭제 상태를 기준으로 별도 후보로 표시한다.
- 작업지시서 후보는 R2 object를 직접 삭제하지 않는다.
- 작업지시서 후보 처리 시 DB의 작업지시서를 삭제 완료 상태로 전환하고, 연결된 첨부파일은 R2 purge 후보로 이어지도록 `attachment_trash_items.purge_status = 'purge_requested'`로 표시한다.

## 화면 표시

시스템관리자 실제 삭제 후보 목록에는 두 종류가 같이 표시된다.

- 파일 후보
- 작업지시서 후보

작업지시서 후보는 `workorder:{id}` 형태의 내부 후보 ID를 사용한다.

## 처리 정책

### 선택 처리

선택 항목에 파일 후보와 작업지시서 후보가 섞여 있어도 각각 분리해 처리한다.

- 파일 후보: Worker 기반 R2 삭제 후 trash item purged 처리
- 작업지시서 후보: DB 삭제 완료 상태 전환 + 연결 첨부파일 purge 요청 상태 전환

### 전체 도래 후보 처리

전체 처리에서는 기존 정책처럼 삭제 예정일이 도래했거나 purge 요청된 후보만 처리한다. 다만 화면에는 작업지시서가 방금 삭제된 직후에도 확인 가능하도록 pending 후보를 표시한다.

## 검증

1. 작업지시서를 삭제한다.
2. `/system/storage-usage`에 접속한다.
3. 삭제된 작업지시서가 작업지시서 후보로 보이는지 확인한다.
4. 선택 처리 시 작업지시서가 삭제 완료 상태로 전환되는지 확인한다.
5. 연결 첨부파일이 purge 요청 후보로 이어지는지 확인한다.
