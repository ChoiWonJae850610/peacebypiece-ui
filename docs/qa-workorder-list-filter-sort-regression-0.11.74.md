# 0.11.74 작업지시서 목록 필터/정렬 실제 사용 회귀 보정

## 목표

작업지시서 업무 화면에서 검색어, 상태 필터, 정렬을 바꾼 뒤 왼쪽 목록은 바뀌었지만 가운데 상세/오른쪽 첨부·메모 패널의 선택 상태가 비는 문제를 줄인다.

## 반영 내용

- 현재 선택된 작업지시서가 검색/필터 결과에서 사라지면 첫 번째 표시 작업지시서를 자동 선택한다.
- 선택값이 비어 있고 표시 가능한 작업지시서가 있으면 첫 번째 표시 작업지시서를 자동 선택한다.
- 왼쪽 목록에서 작업지시서를 선택하면 URL에 `workOrderId` query를 반영한다.
- 같은 작업지시서를 다시 클릭해도 선택을 해제하지 않고 현재 상세를 유지한다.

## 유지한 정책

- 기본 목록에서 완료건을 제외하는 정책은 유지한다.
- 완료/전체 필터의 SQL 조회 정책은 변경하지 않는다.
- 작업지시서 상세 hydrate 구조는 변경하지 않는다.
- 첨부/메모/R2 저장 흐름은 변경하지 않는다.

## 확인 항목

- `/worker`에서 검색어 입력 후 표시 목록의 첫 항목 상세가 자동으로 보이는지 확인한다.
- 상태 필터를 `완료`, `전체`, `진행 중`으로 바꿔도 가운데/오른쪽 패널이 비어 있지 않은지 확인한다.
- 정렬 변경 후 표시 목록과 선택 상세가 자연스럽게 유지되는지 확인한다.
- 왼쪽 목록에서 항목 클릭 시 URL에 `workOrderId`가 반영되는지 확인한다.
