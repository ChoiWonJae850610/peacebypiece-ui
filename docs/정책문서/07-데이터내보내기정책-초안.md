# 데이터 내보내기 정책 초안

## 문서 상태
- Policy Version: 0.16.49
- Based App Version: 0.16.49
- 문서 성격: 법무 검토 전 초안

## 1. 목적
고객사가 서비스 이용기간 중 자신의 업무자료를 내려받을 수 있는 데이터 내보내기 기능의 범위, 방식, 제한사항을 정의한다.

## 2. 버튼명
서비스 화면의 기능명은 “데이터 내보내기”로 한다.

## 3. 내보내기 요청 방식
고객사 관리자는 서비스 이용기간 중 “데이터 내보내기” 버튼을 통해 데이터 내보내기를 요청할 수 있다.

요청 전 안내 팝업에는 다음 내용을 표시한다.
- 백업 파일 생성에는 시간이 걸릴 수 있음
- 생성 완료 후 고객사 관리자 이메일로 다운로드 링크가 발송됨
- 다운로드 링크는 생성 완료 후 7일 동안 유효함
- 7일 경과 후 백업 파일은 삭제될 수 있음
- 통계정보와 시스템 내부 로그는 내보내기 대상에서 제외됨

## 4. 제공 방식
- export zip은 서버에서 생성한다.
- 생성된 export zip은 임시 저장소에 저장한다.
- 고객사 관리자에게 이메일로 다운로드 링크를 발송한다.
- 이메일에는 zip 파일을 직접 첨부하지 않는다.
- 다운로드 링크는 생성 완료 후 7일 동안만 유효하다.
- 7일 경과 후 export zip은 삭제될 수 있다.

## 5. 내보내기 포함 대상
- 작업지시서
- 디자인 파일
- 일반 첨부파일
- 메모
- 업체정보
- 멤버정보

## 6. 내보내기 제외 대상
- 통계정보
- 시스템 운영 로그
- 감사 로그
- 서비스 내부 분석 데이터
- 결제대행사 내부 데이터
- 인증정보
- 초대 토큰
- 권한 상세값
- 계정 상태

## 7. 멤버정보 제공 범위
포함:
- 이름
- 전화번호
- 역할

제외:
- 이메일
- 권한 상세값
- 계정 상태
- 생년월일
- 로그인 기록
- 내부 user id
- 인증 provider 정보
- 초대 토큰
- 시스템 로그

## 8. export zip 구조 초안
```text
company-export-YYYYMMDD.zip
├─ README.txt
├─ workorders.xlsx
├─ partners.xlsx
├─ members.xlsx
├─ memos.xlsx
└─ workorders/
   ├─ 작업지시서명-001/
   │  ├─ designs/
   │  ├─ attachments/
   │  └─ memos.txt
   └─ 작업지시서명-002/
      ├─ designs/
      ├─ attachments/
      └─ memos.txt
```

## 9. 해지 신청 상태에서의 안내
고객사가 해지를 신청한 경우 서비스는 이용 종료 예정일 전까지 필요한 데이터를 내보내도록 안내할 수 있다.

해지 효력이 발생한 이후에는 서비스 접근 및 데이터 다운로드가 제한될 수 있다.

## 10. 개발 필요사항
- 데이터 내보내기 버튼
- 안내 팝업
- export_jobs 테이블
- export zip 생성
- 임시 저장소 저장
- 다운로드 링크 생성
- 다운로드 가능 기한 7일
- 이메일 발송
- 만료 후 삭제
- 다운로드 이력

## 11. 이메일 발송 방식
데이터 내보내기 완료 알림은 초기에는 Google Workspace 기반 이메일 발송으로 처리한다.

대표/운영 이메일은 hello@wafl.co.kr을 기준으로 한다.

정책 문서에는 Gmail API 또는 SMTP 같은 구현 방식을 명시하지 않고, 구체적인 발송 방식은 구현 시점에 기술 검토 후 정한다.

## 12. 다운로드 링크 보안
다운로드 링크는 로그인 필요 방식으로 한다.

- 고객사 관리자 로그인 필요
- 고객사 관리자 권한 확인 필요
- export job의 회사 범위 확인 필요
- 다운로드 가능 기간 7일 확인 필요
- 만료된 export zip은 다운로드 불가
- 다운로드 이력 기록 검토
