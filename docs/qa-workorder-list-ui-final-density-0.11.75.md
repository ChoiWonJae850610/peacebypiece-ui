# 0.11.75 작업지시서 목록 필터/정렬 UI 최종 밀도 보정

## 목적

작업지시서 업무 화면 좌측 목록의 검색/필터/정렬 컨트롤이 목록 영역을 과하게 차지하지 않도록 밀도를 조정하고, 검색/필터/정렬을 함께 초기화할 수 있는 단일 액션을 추가한다.

## 반영 내용

- PC 좌측 목록 검색 입력 높이와 필터/정렬 select 높이를 한 단계 축소했다.
- 모바일 drawer의 검색/필터/정렬 컨트롤도 같은 기준으로 정리했다.
- 현재 상태 필터, 정렬, 표시 건수 요약은 유지하되 우측에 `초기화/Reset` 버튼을 조건부로 표시한다.
- 검색어/상태/정렬이 기본값이 아닐 때만 초기화 버튼을 표시한다.
- 초기화 액션은 URL query와 내부 상태를 한 번에 기본값으로 되돌리도록 `WorkOrderWorkspace`에서 통합 처리한다.
- 상태/정렬 option 생성 로직을 `lib/workorder/list/workOrderListControls.ts`로 모아 PC sidebar와 mobile drawer의 중복 option 정의를 줄였다.

## 변경하지 않은 내용

- 작업지시서 summary SQL 조회 정책은 변경하지 않았다.
- 완료건 기본 제외 정책은 유지했다.
- 검색은 현재 조회된 목록 안에서 client-side 검색으로 유지했다.
- 상세 hydrate, 첨부, 메모, R2 흐름은 변경하지 않았다.
- DB schema 변경은 없다.

## 확인 항목

1. `/worker` 진입 후 검색/필터/정렬 컨트롤이 이전보다 과하게 크지 않은지 확인한다.
2. 검색어를 입력하면 `초기화` 버튼이 표시되는지 확인한다.
3. 상태 필터나 정렬을 기본값이 아닌 값으로 바꾸면 `초기화` 버튼이 표시되는지 확인한다.
4. `초기화` 클릭 시 검색어가 지워지고 상태는 `진행 중`, 정렬은 `최근 수정순`으로 돌아오는지 확인한다.
5. URL에서 `q`, `status`, `sort` query가 기본값 기준으로 정리되는지 확인한다.
6. 모바일 drawer에서도 같은 초기화 동작이 적용되는지 확인한다.
