# 0.18.91 로컬 산출물 ignore 기준 점검

## 목적

테스트 불가 상태에서 앱 동작 흐름을 건드리지 않고, 프로젝트 루트와 패치 자동화 과정에서 생성될 수 있는 로컬 산출물이 소스 관리 대상에 섞이지 않도록 `.gitignore` 기준을 보강한다.

## 기준 상태

- 기준 버전: `0.18.90`
- 결과 버전: `0.18.91`
- 작업 범위: `.gitignore`, 루트/문서 README, `APP_VERSION`
- 테스트 불가 상태 유지

## 보강한 ignore 대상

```text
repo-state-*.txt
FAIL_*_build*.txt
build-*.txt
patch_result/
```

## 판단

- `repo-state-*.txt`는 패치 적용 후 상태 확인용 로컬 산출물이다. 프로젝트 코드가 아니므로 Git 관리 대상에서 제외한다.
- `FAIL_*_build*.txt`는 로컬 빌드 실패 로그다. 다음 패치의 원인 분석에는 필요하지만, 프로젝트 소스에 누적 보관할 파일은 아니다.
- `build-*.txt`는 빌드 로그 계열 산출물이다. 실제 배포 소스와 무관하다.
- `patch_result/`는 사용자의 패치 자동화 결과 폴더 후보이므로 프로젝트 소스에 포함하지 않는다.

## 이번에 건드리지 않은 항목

- DB schema 변경 없음
- API 변경 없음
- R2/첨부/메모/휴지통/purge 흐름 변경 없음
- 권한/멤버/작업지시서 상태 흐름 변경 없음
- UI 화면 동작 변경 없음
- package.json/package-lock.json 변경 없음

## 후속 점검 후보

테스트 가능 상태가 되기 전까지는 다음 수준의 저위험 점검을 이어갈 수 있다.

1. `docs/보관문서`와 `docs/현재기준`의 충돌 후보 문서화
2. `scripts/` 폴더의 현재 사용 여부 문서화
3. 루트에 반복 생성되는 패치 산출물이 자동화 과정에서 프로젝트 폴더로 복사되는지 확인
4. 개발용 라우트 접근 제한 정책 문서화
