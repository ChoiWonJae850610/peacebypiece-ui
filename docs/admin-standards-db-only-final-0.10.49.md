# 0.10.49 기준정보 DB-only 최종 정리

## 목표

0.10.46 이후 기준정보는 DB 결과만 사용한다는 원칙을 세웠지만, 일부 화면과 레거시 유틸에 화면용 기본 목록이 남아 있어 새로고침 시 기준정보 수량이 흔들리는 문제가 있었다.

## 반영 내용

- 시스템 단위 표준, 외주공정 유형, 생산품 유형 기본 템플릿의 화면용 fallback 상수 배열을 빈 배열로 정리했다.
- 시스템 기준정보 화면의 조회 요청에 최신 요청만 화면에 반영하는 guard를 추가했다.
- 고객관리자 기준관리 화면도 기준정보 조회 요청의 최신 응답만 반영하도록 보정했다.
- 협력업체 관리 컨트롤러의 외주공정 초기값을 레거시 기본값이 아닌 빈 배열로 시작하도록 변경했다.
- 레거시 파트너 persistence와 actionFlow의 외주공정 기본값 복원도 빈 배열 기준으로 정리했다.
- 고객관리자 생산품 유형 기본값 복원은 시스템관리자가 `is_default=true`로 지정한 활성 템플릿만 사용한다. 기본 템플릿이 없으면 복원하지 않는다.

## 유지한 사항

- DB schema 변경 없음
- 시스템 기준정보 CRUD 변경 없음
- 고객관리자 기준정보 저장 API 경로 유지
- 감사 로그 흐름 변경 없음

## 확인 기준

1. `/system/standards/units` 새로고침 반복 시 단위 표준 개수가 흔들리지 않아야 한다.
2. `/system/standards/processes` 새로고침 반복 시 외주공정 유형 개수가 흔들리지 않아야 한다.
3. `/system/standards/product-templates` 새로고침 반복 시 템플릿 개수가 흔들리지 않아야 한다.
4. `/admin/settings`에서 생산품 유형/단위 표준/외주공정 유형 숫자가 새로고침마다 바뀌지 않아야 한다.
5. 생산품 유형 기본값 복원은 시스템관리자 기본 템플릿과 같은 구조로 복원되어야 한다.
