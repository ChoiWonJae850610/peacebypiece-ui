# 0.18.89 프로젝트 루트 README 최신화

## 작업 목적

Next.js 기본 템플릿 상태로 남아 있던 최상위 `README.md`를 WAFL 프로젝트 기준 문서로 교체했다.

테스트가 불가능한 기간이므로 실행 흐름, DB/API/R2, 권한, 작업지시서 상태 변경에는 손대지 않고 문서 정리만 반영했다.

## 변경 내용

- 최상위 `README.md`의 기본 Next.js 안내 문구를 제거했다.
- WAFL/PieceByPiece UI의 현재 프로젝트 성격을 간단히 정리했다.
- 로컬 개발 실행 명령과 빌드 확인 명령을 분리해 명시했다.
- ChatGPT/container에서 `npm run build`를 실행하지 않는 원칙을 명시했다.
- 주요 문서 위치를 `docs/README.md`, `docs/현재기준/`, `docs/정책문서/`, `docs/보관문서/`, `pending-tests.md`로 정리했다.
- 테스트 불가 기간에는 문서 최신화, 소스 점검, 로컬 생성물 정리, 명백한 타입 오류 수정처럼 저위험 작업을 우선한다는 기준을 명시했다.

## 영향 범위

- UI 렌더링 변경 없음
- DB 변경 없음
- API 변경 없음
- R2/첨부/메모/휴지통/purge 변경 없음
- 권한/상태 흐름 변경 없음
- package.json/package-lock.json 변경 없음

## 후속 후보

테스트 불가 상태가 유지되면 다음 작업도 기능 변경이 아니라 문서·구조 점검 중심으로 진행한다.

- 현재기준 문서와 실제 라우트 구조의 차이 점검
- 보관문서와 현재기준 문서의 충돌 후보 목록화
- pending-tests.md 누락 항목 정리
- 루트/문서/스크립트 생성물 추적 여부 점검
