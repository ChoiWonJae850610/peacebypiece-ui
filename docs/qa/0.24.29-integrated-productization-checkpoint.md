# 0.24.29 Integrated Productization Checkpoint QA

Status: `PENDING_USER_QA`

Use this checklist after the 0.24.29 commit is deployed to Vercel. Do not use production customer data for destructive or fixture work.

## PC Chrome

1. Public signup
   - Path: `/pending?type=signup` or public signup entry.
   - Data: verified Google account prepared for signup QA.
   - Action: confirm identity, enter company/business data, agree to required policies, upload PNG/JPEG/PDF business certificate, submit.
   - Expected: pending/submitted status is shown, raw Google sub, raw R2 key, signed URL, token, or DB error is not visible.
   - Must not happen: unverified email proceeds, missing consent submits, missing certificate submits in production path.

2. System signup review
   - Path: `/system/signup-applications`.
   - Role: actual active system-admin.
   - Action: open list, filter queue, open detail, view consent evidence, open certificate inline viewer, move submitted to reviewing, request changes, reject in a fixture/test case.
   - Expected: list/detail text wraps, certificate opens inline, download link is not provided, reasons are required for changes/reject.
   - Must not happen: applicant/customer account can access system routes, raw storage key or signed URL appears.

3. Approval provisioning
   - Path: `/system/signup-applications/[applicationId]`.
   - Action: check approval plan and approval gate for an eligible test application.
   - Expected: company, user/member, company-admin permission, Trial 7 days/100MB/3 members, and catalog provisioning are created only by server-approved flow.
   - Must not happen: duplicate company/member/subscription on repeated approval, existing user tenant fields overwritten incorrectly.

4. Workspace and catalog
   - Path: `/workspace`, `/workspace/settings/catalog`.
   - Role: approved company-admin.
   - Action: enter workspace, inspect catalog, enable/disable optional category.
   - Expected: apparel default active, underwear/accessory inactive until enabled, sizes and POM visible.
   - Must not happen: existing companies backfilled unexpectedly, inactive categories appear in create combobox as active choices.

5. Workorder PDF
   - Path: `/workspace/workorders`.
   - Action: create/open workorder, confirm due date, generate PDF, open inline viewer, regenerate.
   - Expected: due-date-missing vendor/share PDF is blocked, viewer uses app server proxy, latest regenerated PDF remains active.
   - Must not happen: vendor/share PDF exposes cost/internal history/permissions, raw Worker URL or raw R2 key appears.
