# 작업지시서 모바일 구조 1차 (0.20.28)

## 목적

작업지시서 모바일 화면에서 PC 3패널 축소 구조를 기본 흐름으로 쓰지 않고, 모바일 전용 흐름을 `목록 → 상세` 구조로 분리한다.

## 반영 범위

- 기존 `WaflMobileShell` 위에 작업지시서 모바일 목록 패널을 연결했다.
- 모바일 진입 기본 화면을 작업지시서 목록으로 두고, 목록 카드 선택 시 상세 화면으로 전환한다.
- 상세 화면 하단 고정 액션바에는 `목록` 복귀 버튼과 `첨부·디자인·메모` 진입 버튼을 함께 배치했다.
- 첨부·디자인·메모는 기존 `AppSheet` 기반 full-size bottom sheet 흐름을 유지한다.
- PC 3패널과 태블릿 2패널 구조는 변경하지 않았다.

## 비변경 범위

- 상태전환 로직 변경 없음
- 권한 로직 변경 없음
- API 변경 없음
- DB schema 변경 없음
- R2/첨부/메모/휴지통/purge 흐름 변경 없음

## 테스트 포인트

- 모바일에서 작업지시서 화면 진입 시 목록이 먼저 보이는지 확인한다.
- 목록 카드 선택 시 상세 화면으로 전환되는지 확인한다.
- 상세 화면 하단의 `목록` 버튼으로 목록에 복귀되는지 확인한다.
- 상세 화면 하단의 `첨부·디자인·메모` 버튼으로 기존 패널이 열리는지 확인한다.
- PC에서는 기존 좌/중/우 3패널이 유지되는지 확인한다.
- 태블릿에서는 기존 2패널 구조가 유지되는지 확인한다.
