# 0.17.26 발주 정보 단일 테이블 기준

## 목적
작업지시서 발주 정보에서 봉제와 외주를 별도 블록으로 나누지 않고, 하나의 발주 항목 테이블 안에서 구분해 관리한다.

## 반영 기준
- 봉제와 외주는 같은 테이블에 표시한다.
- 구분 컬럼은 `봉제` 또는 `외주`를 표시한다.
- 봉제 행의 항목은 샘플/메인/재작업을 사용한다.
- 외주 행의 항목은 나염/자수/워싱 등 등록된 외주공정 목록을 사용한다.
- 봉제 행의 업체는 생산 공장 목록을 사용한다.
- 외주 행의 업체는 선택한 외주공정에 맞는 외주처 목록을 사용한다.
- 납기일은 이번 PC 발주 정보 테이블에서 제거한다.
- 외주 행에는 로스비를 입력하지 않고 `-`로 표시한다.

## 유지 사항
- 생산 공장은 작업지시서당 1개 기준을 유지한다.
- 외주 행은 여러 개 추가할 수 있다.
- 기존 `orderEntries`와 `outsourcing` 데이터 구조는 유지한다.
- DB schema는 변경하지 않는다.
- 작업지시서 PDF 양식은 변경하지 않는다.

## 후속 검토
- 외주공정 마스터와 외주처 마스터의 연결 기준을 기준정보/업체관리에서 더 명확히 정리한다.
- 발주 PDF 생성 시 봉제 행과 외주 행을 문서 목적에 맞게 분리 출력할지 검토한다.
