# 0.17.16 원단·부자재 발주 readiness 정리

## 목적

작업지시서 화면의 큰 진행 단계 UI는 유지하고, 원단·부자재 발주 후보 판단은 내부 readiness 상태로 분리한다.

## 표시 단계 원칙

작업지시서 상세 화면에서 사용자가 보는 큰 단계는 기존 흐름을 유지한다.

```txt
작성중 → 검토요청 → 검토완료 → 발주요청 → 검수 → 완료
```

자재 발주 대기는 별도 큰 단계로 끼워 넣지 않는다. 발주요청 흐름 안에서 보조 상태로 표시한다.

```txt
발주요청 · 자재 발주 대기
```

## 내부 readiness 1차 기준

0.17.16에서는 DB 스키마 변경 없이 후보 계산 계층만 분리한다.

```txt
not_ready
material_order_pending
material_order_in_progress
material_order_completed
```

- `material_order_pending`: 필요 자재가 있고, 작업지시서가 검토완료 상태인 경우
- `material_order_in_progress`: 기존 데이터 호환을 위해 필요 자재가 있고, 검수 단계에 진입한 경우
- `material_order_completed`: 완료 상태
- `not_ready`: 필요 자재가 없거나 아직 발주 후보 단계가 아닌 경우

## 후속 작업

0.17.17 이후에는 실제 발주서 line allocation 기준으로 작업지시서별 필요 자재의 전체 발주 완료 여부를 계산한다. 그 다음 모든 필요 자재가 발주 완료된 작업지시서만 검수 단계로 이동시키는 흐름을 검토한다.
