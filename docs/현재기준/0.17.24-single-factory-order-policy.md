# 0.17.24 작업지시서 단일 생산 공장 기준

## 목적
작업지시서의 발주 정보에서 복수 공장/복수 발주 추가 흐름을 제거하고, 작업지시서 하나에는 생산 공장 1개만 관리하는 방향으로 단순화한다.

## 반영 기준
- 생산 공장은 작업지시서당 1개만 사용한다.
- 발주 정보의 `+ 발주 추가` 흐름은 제거한다.
- 기존 데이터에 여러 발주 행이 남아 있더라도 상세 화면에서는 대표 생산 공장 1개 기준으로 표시한다.
- 발주 정보 안의 구분은 샘플/메인 생산/재작업 같은 생산 구분으로 유지한다.
- 공장 하나당 발주서 1개 생성 흐름을 기준으로 본다.
- 외주공정은 생산 공장과 분리된 다중 항목으로 이후 발주 정보 영역 안에 별도 블록으로 추가한다.

## 이번 버전에서 하지 않은 것
- 외주공정 발주 블록 추가는 다음 버전으로 분리한다.
- DB schema 변경은 하지 않는다.
- 작업지시서 PDF 양식은 변경하지 않는다.
- 기존 orderEntries 배열 구조 삭제/마이그레이션은 하지 않는다.

## 다음 단계
0.17.25에서 발주 정보 영역 안에 외주공정 발주 블록을 추가한다.
외주공정은 여러 개를 허용하고, 공정/외주처/납기/수량/외주비를 입력하는 방향으로 설계한다.
