# 0.17.17 자재 발주 대기 흐름 보정

## 목적

0.17.16 적용 후 작업지시서에 원단·부자재가 있어도 발주요청 시 검수 단계로 바로 이동하고 작업지시서 PDF가 생성되는 문제가 확인되었다. 0.17.17은 작업지시서 발주요청과 원단·부자재 발주 완료를 분리한다.

## 기준 흐름

```txt
작업지시서 발주요청
→ 필요 자재 있음
→ workflowState = material_order_pending
→ 화면 표시 단계 = 발주요청
→ 작업지시서 PDF 생성 안 함
→ 원단·부자재 화면 후보에 표시
```

```txt
작업지시서 발주요청
→ 필요 자재 없음
→ workflowState = inspection
→ 기존 검수/PDF 흐름 유지
```

## 이번 반영 범위

- `material_order_pending`을 실제 workflow state에 추가한다.
- 화면 진행 단계는 기존 6단계를 유지하고, `material_order_pending`은 `request_order` 단계로 매핑한다.
- 발주요청 시 필요 자재가 있으면 검수로 바로 이동하지 않는다.
- `material_order_pending` 상태에서는 작업지시서 PDF 생성을 차단한다.
- 원단·부자재 화면 우측 후보는 `material_order_pending` 상태이면서 필요 자재가 있는 작업지시서만 표시한다.
- 작업지시서 필요 자재에서 품목 추가 시 단위, 수량, 단가를 함께 반영한다.

## 다음 범위

0.17.18 이후에는 원단·부자재 발주완료 시 연결된 작업지시서의 필요 자재가 모두 발주완료되었는지 계산하고, 조건을 만족한 작업지시서만 검수 단계로 이동시키는 흐름을 추가한다.
