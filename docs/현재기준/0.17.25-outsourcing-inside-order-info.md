# 0.17.25 외주공정 발주 정보 내부 배치 기준

## 목적

작업지시서의 발주 정보는 생산 공장 1개를 기준으로 유지하고, 외주공정은 같은 발주 정보 영역 안에서 별도 다중 블록으로 관리한다.

## 기준

- 생산 공장은 작업지시서당 1개만 관리한다.
- 생산 공장 정보는 공장 1개 기준 발주서 1개 생성의 기준이 된다.
- 샘플, 메인 생산, 재작업은 생산 공장 행의 구분값으로 유지한다.
- 외주공정은 생산 공장과 별도 블록으로 관리한다.
- 외주공정은 여러 개 추가할 수 있다.
- 외주공정 행은 공정, 외주처, 납기일, 수량, 외주비를 입력한다.
- 외주공정 추가 시 생산 공장 행의 납기일과 수량을 기본값으로 가져온다.
- 가져온 수량과 납기일은 사용자가 수정할 수 있다.

## 이번 버전 범위

- PC 발주 정보 영역 안에 외주공정 발주 블록을 추가한다.
- 기존 생산 구성 영역에는 필요 원단/부자재만 유지한다.
- 기존 outsourcing 배열을 재사용하고, DB schema는 변경하지 않는다.
- 외주공정의 dueDate는 JSON 데이터의 선택 필드로 추가한다.
- 작업지시서 PDF 양식은 변경하지 않는다.

## 후속 검토

- 외주공정 마스터/외주처 마스터 선택 UX 정리
- 외주공정별 발주 PDF 분리 여부
- 생산 공장 발주서와 외주공정 발주 문서의 생성 타이밍 분리
