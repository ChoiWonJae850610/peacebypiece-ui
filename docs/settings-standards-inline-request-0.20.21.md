# 환경설정 기준정보 직접관리/요청관리 재구성 0.20.21

## 목적

환경설정 > 기준정보 설정 화면을 실제 운영 흐름에 맞게 재구성한다. 기존 화면은 탭 구조는 있었지만 선택 항목을 다시 모달로 열어야 했고, 코드명·순서·상태 뱃지가 많아 운영자가 실제로 무엇을 할 수 있는지 직관적이지 않았다.

## 변경 기준

- 생산품 유형은 고객사 관리자가 직접 추가하고 사용 여부를 변경한다.
- 단위 표준과 외주공정 유형은 회사별 임의 관리가 아니라 전체 운영 기준과 연결되므로 추가 요청 방식으로 둔다.
- 상위 관리 모달을 제거하고 탭 안에서 즉시 목록과 액션을 확인한다.
- 코드명, 순서, 녹색 상태 뱃지는 운영 화면에서 제거한다.
- 사용 여부는 WAFL 톤의 pill/toggle형 UI로 표시한다.

## 생산품 유형

생산품 유형 탭에서는 다음을 직접 처리한다.

- 신규 생산품 유형 입력
- 생산품 유형 추가
- 기존 생산품 유형 사용/미사용 전환

저장 방식은 기존 `saveAdminItemCategoriesToApi`를 재사용한다. DB schema는 변경하지 않는다.

## 단위 표준

단위 표준 탭은 목록 열람과 추가 요청을 제공한다.

- 현재 등록된 단위 표준 이름과 사용 여부 표시
- `유형 추가 요청` 버튼 제공
- 요청 항목명과 요청 사유 입력
- `/api/admin/settings/feedback`에 DB 문의로 접수

## 외주공정 유형

외주공정 유형 탭도 단위 표준과 동일하게 요청형으로 운영한다.

- 현재 등록된 외주공정 유형 이름과 사용 여부 표시
- `유형 추가 요청` 버튼 제공
- 요청 항목명과 요청 사유 입력
- `/api/admin/settings/feedback`에 DB 문의로 접수

## 제외 범위

이번 버전에서는 다음을 변경하지 않는다.

- 기준정보 DB schema
- 단위 표준 저장 API
- 외주공정 유형 저장 API
- 시스템관리자 문의 검토 UI
- 작업지시서 기준정보 사용 흐름
- R2/첨부/메모/휴지통/purge 흐름

## 후속 작업

- 시스템관리자 문의/요청 검토 화면에서 기준정보 추가 요청을 분류 표시
- 단위 표준/외주공정 유형 승인 후 실제 기준정보 추가 흐름 연결
- 환경설정 전체 E2E에 기준정보 탭 최소 확인 추가
