# 0.19.78 회사 계정 요청 승인 결과 실제 반영 1차

## 목적
시스템관리자가 회사 계정 요청을 검토한 결과를 고객사 환경설정 이력에 더 명확하게 표시하고, 계정 비활성화 요청이 승인된 경우 실제 회사 접근 상태에도 반영한다.

## 반영 범위
- 고객사 환경설정 최근 회사 계정 요청 이력에 검토자, 검토일, 검토 메모 표시
- 고객사 환경설정 요청 이력 API에서 `reviewed_by_system_user_id` 기준 시스템관리자 검토자 이름 조회
- 계정 비활성화 요청 승인 시 회사 상태 반영
  - `companies.is_active = false`
  - `companies.status = 'suspended'`
  - `companies.subscription_status = 'canceled'`

## 의도적으로 제외한 범위
- 회사 정보 변경 요청 승인 시 회사 필드 자동 변경
  - 현재 요청 화면은 구조화된 회사 정보 필드가 아니라 자유 입력 메모 중심이다.
  - 실제 회사명, 사업자명, 주소, 사업자등록번호를 자동 변경하려면 별도 구조화 입력/검증 UI가 필요하다.
- DB schema 변경
- 정책/약관 동의 이력
- 결제/PG 연동
- R2/파일/첨부 흐름

## 테스트 기준
1. 시스템관리자 `/system/account-requests`에서 계정 비활성화 요청을 승인한다.
2. 해당 고객사의 `companies.subscription_status`가 `canceled`로 변경되는지 확인한다.
3. 해당 고객사의 `companies.status`가 `suspended`, `is_active`가 `false`로 변경되는지 확인한다.
4. 고객사 환경설정 요청 이력에 검토자/검토일/검토 메모가 표시되는지 확인한다.
5. 회사 정보 변경 요청은 승인 상태만 변경되고 회사 기본 정보가 임의로 바뀌지 않는지 확인한다.
