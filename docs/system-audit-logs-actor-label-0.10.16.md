# 0.10.16 시스템 감사 로그 행위자 표시 보정

## 목적

0.10.15에서 작업지시서 상태 변경 로그가 기록되기 시작했지만, 행위자 칼럼이 실제 조작자가 아니라 작업지시서 담당자/managerId 기준으로 표시될 수 있었다.

예: 디자이너가 검토요청하고 관리자가 검토완료했을 때, 감사 로그 행위자가 `user-sample-designer`, `user-sample-admin` 같은 내부 ID로 보이거나 역할이 명확하지 않았다.

## 반영 기준

- 상태 변경 요청 시 현재 로그인/선택 사용자를 `auditActor`로 함께 전달한다.
- 서버는 `auditActor`가 있으면 그 값을 감사 로그의 실제 행위자로 우선 사용한다.
- 기존 담당자 정보는 `managerName`으로 metadata에 유지한다.
- 화면 표시에서는 내부 ID만 보여주지 않고 `이름 (역할)` 형식으로 표시한다.

## 표시 예시

- `샘플 디자이너 (디자이너)`
- `샘플 관리자 (관리자)`
- `샘플 시스템관리자 (시스템관리자)`

## 범위

- 작업지시서 상태 변경 감사 로그
- 작업지시서 전체 저장 중 상태 변경 감사 로그
- bulk 상태 변경 감사 로그
- 기존 샘플 계정 ID 표시 보정

## 비고

DB schema 변경은 없다. 기존 로그 중 actor_user_id가 샘플 계정 ID인 경우 화면에서 이름/역할을 보정한다.
