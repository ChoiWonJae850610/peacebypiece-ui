# 0.23.26 Workspace 스크롤 및 변경 피드백 통합

## 목적

태블릿 가로 화면에서 `/workspace` 운영 대시보드의 세로 스크롤이 막히는 문제를 해결하고, 작업지시서와 발주서의 저장·변경 토스트 문구를 공통 변경 피드백 정책으로 정리한다.

## WorkspaceShell 변경

`scroll` 모드는 뷰포트 높이 안에서 콘텐츠 프레임이 항상 세로 스크롤을 담당한다. 기존에는 `lg` 이상에서 바깥 스크롤을 막으면서 내부 스크롤은 `2xl`부터 활성화되어 1024~1535px 구간에서 스크롤이 사라졌다.

`fixed-md` 모드는 기존 작업지시서·발주서 패널 내부 스크롤 정책을 유지한다.

## 변경 피드백 정책

진행 문구는 변경 대상을 구체적으로 표시한다.

- 작업지시서 정보 변경 중입니다.
- 작업지시서 공장 전달사항 변경 중입니다.
- 발주서 기본정보(자재 종류) 변경 중입니다.
- 발주서 기본정보(공급처) 변경 중입니다.
- 발주서 기본정보(납기일) 변경 중입니다.

성공과 기본 실패 문구는 화면별 표현 차이를 없애기 위해 공통 문구를 사용한다.

- 정보가 변경되었습니다.
- 정보를 변경하지 못했습니다.

## 호환성

기존 `WAFL_SAVE_TARGET`과 `getWaflSaveFeedbackMessage`는 단계적 이관을 위해 유지하며 내부적으로 새 변경 피드백 정책을 사용한다.
