# 0.11.90 작업지시서 생산구성 단위 표시 정규화

## 목적

작업지시서 생산구성 화면에서 기존 데이터나 mock 데이터에 남아 있는 `yd`, `ea`, `pcs` 같은 내부/구버전 단위값이 사용자 화면에 그대로 노출되지 않도록 정리한다.

## 반영 기준

- 원단 기본 단위는 `야드`로 표시한다.
- 부자재 기본 단위는 `개`로 표시한다.
- 기존 DB 또는 mock 데이터에 `yd`, `yard`, `yards`가 남아 있어도 화면에서는 `야드`로 표시한다.
- 기존 DB 또는 mock 데이터에 `ea`, `pc`, `pcs`, `piece`, `pieces`가 남아 있어도 화면에서는 `개`로 표시한다.
- 신규 저장 시 자재 단위는 가능한 한 표준 표시값으로 정규화한다.

## 확인 범위

- PC 작업지시서 생산구성 원단/부자재 테이블
- tablet 작업지시서 생산구성 원단/부자재 카드/tablet layout
- mobile 작업지시서 생산구성 원단/부자재 카드
- DB 조회 결과에 구버전 단위값이 남아 있는 경우의 표시값
- 신규 자재 추가 시 기본 단위

## 제외 범위

- 입력 가능/계산 필드 색상 구분은 다음 단계에서 별도 보정한다.
- 모바일 모달 검색 input 포커스 이탈 문제는 다음 단계에서 별도 보정한다.
- R2/첨부/메모/휴지통/purge 흐름은 변경하지 않는다.
