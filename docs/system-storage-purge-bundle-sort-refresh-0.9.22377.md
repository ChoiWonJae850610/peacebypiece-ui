# 0.9.22377 시스템관리자 실제 삭제 후보 묶음 처리 UX 보정

## 목적

작업지시서 영구삭제 요청을 시스템관리자 화면에서 작업지시서 대표 row 1개로 유지하되, 실제 삭제 처리 시 연결 첨부파일과 메모까지 같은 흐름에서 처리하도록 보정한다.

## 변경 기준

- 고객관리자가 작업지시서를 영구삭제 요청하면 `/system/storage-usage`에는 작업지시서 대표 후보 1개가 표시된다.
- 대표 row에는 첨부파일 개수, 메모 개수, 첨부파일 총 용량을 표시한다.
- 시스템관리자가 실제 삭제를 실행하면 작업지시서, 메모, 연결 첨부파일 R2 삭제를 같은 처리 흐름에서 진행한다.
- 정상 처리된 연결 첨부파일은 다음 후보 목록에 다시 노출하지 않는다.
- R2 삭제에 실패한 첨부파일만 재시도 후보로 목록에 남긴다.

## 화면 보정

- 삭제 후보 목록에 새로고침 버튼을 추가했다.
- 컬럼 헤더 클릭 정렬을 추가했다.
- 정렬 기준은 구분, 고객사/작업지시서, 대상, 삭제일, 예정일, 용량, 상태, 첨부 수, 메모 수이다.
- 기존 `전체 도래 후보 처리` 버튼 문구를 `전체삭제`로 변경했다.
- 작업지시서 후보 설명을 `연결 파일 함께 처리`, `실패한 R2만 별도 노출` 기준으로 정리했다.

## 확인 항목

1. 고객관리자가 작업지시서 영구삭제 요청
2. `/system/storage-usage`에 작업지시서 대표 후보 1개 표시
3. 대표 row에 첨부파일 개수, 메모 개수, 첨부파일 총 용량 표시
4. 컬럼 헤더 클릭 시 오름차순/내림차순 정렬 전환
5. 새로고침 버튼 클릭 시 목록 갱신
6. 전체삭제 버튼 문구 확인
7. 실제 삭제 성공 후 연결 첨부파일이 다시 후보 목록에 노출되지 않는지 확인
8. R2 삭제 실패 항목만 삭제 실패/재시도 후보로 남는지 확인
