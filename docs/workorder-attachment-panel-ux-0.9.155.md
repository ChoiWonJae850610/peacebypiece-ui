# 0.9.155 작업지시서 디자인/첨부 영역 UX 1차

## 목적

작업지시서 PC 화면의 우측 디자인/첨부 영역에서 파일 추가 진입점이 덜 명확한 문제를 보완한다. 실제 업로드/삭제/썸네일/메모 저장 로직은 유지하고, 사용자가 어디에서 파일을 추가할 수 있는지 인지하기 쉬운 UI만 먼저 정리한다.

## 반영 내용

- 디자인/첨부 panel header의 직접 `+` 버튼을 `...` 액션 메뉴로 변경한다.
- 액션 메뉴 안에 기존 파일 추가 동작을 유지한다.
- 디자인 panel에는 향후 그리기 기능을 연결할 수 있도록 `직접 그리기` 메뉴 항목을 준비 상태로 표시한다.
- 첨부 목록 하단 또는 빈 상태 아래에 점선 안내 영역을 추가한다.
- 안내 영역은 클릭하면 기존 파일 선택 업로드를 실행한다.
- 실제 drag-and-drop 업로드 핸들러는 이번 버전에서 연결하지 않는다.

## 정책

- 디자인 파일 추가와 첨부 파일 추가는 기존 `onOpenAttachmentPicker(scope)` 흐름을 그대로 사용한다.
- 직접 그리기 메뉴는 버튼/진입점만 준비하고 비활성 상태로 둔다.
- 그리기 라이브러리 연결, 캔버스 저장, 저장 결과를 디자인 첨부로 등록하는 기능은 별도 버전에서 진행한다.

## 다음 후보 작업

- 실제 drag-and-drop 업로드 연결
- 디자인 직접 그리기 라이브러리 연결
- 그린 이미지를 디자인 첨부로 저장
- 디자인/첨부 액션 메뉴의 모바일/태블릿 배치 재점검
