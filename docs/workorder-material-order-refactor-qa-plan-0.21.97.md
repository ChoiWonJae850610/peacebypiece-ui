# 0.21.97 작지·발주서 리팩토링 후 전체 테스트 계획

## 목적
0.21.93~0.21.97 리팩토링 라인에서 변경된 Workspace, Empty State, 공정·자재 카드, 발주 품목·할당 카드, 숫자 입력/표시 primitive의 회귀를 한 번에 확인한다.

## 1. 공통 build 확인
- `pnpm install`
- `pnpm run build`
- TypeScript import 오류
- 사용하지 않는 import로 인한 lint/build 오류
- client/server component 경계 오류
- `APP_VERSION` 0.21.97 표시 확인

## 2. 작업지시서 화면 QA
### PC
- 3패널: 목록 / 상세 / 첨부·디자인·메모
- 선택 전 가운데 Empty State
- 선택 전 우측 Empty State
- 상단 카드: 작업구분 / 납기일 / 최근 변경일 정렬
- 제작공정 카드 수정/삭제 메뉴
- 추가공정 카드 수정/삭제 메뉴
- 원단·부자재 카드 수정/삭제 메뉴
- 수량/단가/로스/금액 표시

### 아이패드 미니 가로
- 목록은 드로어
- 가운데 상세 + 우측 패널 2패널
- 가운데·우측 독립 스크롤
- Empty State 위치/높이

### 태블릿 세로/모바일
- 목록 드로어
- 본문 단일 패널
- 상단 헤더와 본문 카드 폭 일치
- 입력 확대 없음
- 모달 위치 및 버튼 활성 조건

## 3. 작업지시서 입력 QA
- 제작공정: 업체 미선택 시 수량/단가/로스 비활성화
- 제작공정: 수량 0이면 적용 비활성화, 1 이상 활성화
- 추가공정: 항목 + 업체 + 수량 1 이상일 때만 적용 활성화
- 원단/부자재: 구분 + 자재명 + 수량 1 이상일 때만 적용 활성화
- 저장 후 재조회 시 수량/금액 유지

## 4. 발주서 화면 QA
### PC
- 3패널: 목록 / 발주 상세 / 발주 대상
- 선택 전 가운데 Empty State
- 선택 전 우측 Empty State
- 발주 품목 없음 문구: 발주 품목이 없습니다.
- 발주 품목 카드 메뉴가 패널 상단/하단에서 잘리지 않음
- 수량/단가 입력 콤마 표시
- 금액 계산 표시

### 아이패드 미니 가로
- 목록은 드로어
- 가운데 발주 상세 + 우측 발주 대상 2패널
- 우측 발주 대상 Empty State
- 자재 추가/빼기 버튼 동작

### 태블릿 세로/모바일
- 목록 드로어
- 본문 단일 패널
- 발주 대상은 하단 드로어/패널 정책 유지
- 상단 헤더와 본문 카드 폭 일치

## 5. 발주서 부분 할당 QA
- 필요 10 / 주문 5 → 할당 5, 잔여 5, 여유 0
- 주문 10 → 할당 10, 잔여 0, 여유 0
- 주문 15 → 할당 10, 잔여 0, 여유 5
- 우측 진행 표시 5/10, 10/10이 색상 정책에 맞게 표시
- 동일 자재 여러 작업지시서 추가 시 주문 요약 공장 행 표시 유지

## 6. 삭제 확인
- 디자인 패널에 고급 그리기 버튼 없음
- `pnpm why tldraw` 결과 없음
- `pnpm list tldraw` 결과 없음

- 부분 할당/초과 주문
