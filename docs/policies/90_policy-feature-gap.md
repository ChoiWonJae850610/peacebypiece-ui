# Policy Feature Gap

## 문서 목적
정책 문서를 작성하면서 발견되는 미구현 기능, 상태값, 화면, DB 구조, 운영 도구를 기록한다.

이 문서는 정책 문구와 실제 서비스 기능 사이의 차이를 관리하기 위한 문서다. 정책 문서 초안에 들어간 표현이 실제 기능으로 아직 구현되지 않은 경우 이곳에 반드시 기록한다.

## Gap 16. 시스템관리자 사업자등록증 수동 확인
정책 결정:
- 국세청 API는 1차에 연동하지 않는다.
- 시스템관리자가 승인 전 사업자등록증 파일을 직접 확인한다.
- “사업자등록증 확인” 버튼은 수동 확인 상태 기록용이다.
- 개업일자 입력 UI와 API 진위확인은 1차에 넣지 않는다.

필요 개발:
- 사업자등록증 파일 필수 업로드
- 사업자명/사업자등록번호/대표자명 필수 입력
- 사업자등록증 파일 열람
- 사업자등록증 확인 버튼
- 확인 완료 상태값/확인 일시/확인자/확인 메모

## Gap 17. 해지 예약, 해지 취소, 접근 차단
정책 결정:
- 해지는 다음 결제일부터 정기결제를 중단한다는 의미다.
- 해지 신청 후 다음 결제일 전까지는 정상 사용 가능하다.
- 해지 취소는 다음 결제일 전날 23:59까지 가능하다.
- 다음 결제일이 도래하면 접근이 차단된다.
- 해지 취소 시 구독 상태는 즉시 active로 복귀하고, 다음 예정 결제일에 정기결제가 정상 재개된다.

필요 개발:
- subscription status
- cancel_requested_at
- cancel_effective_at
- cancel_reversible_until
- current_period_end
- 해지 신청/해지 취소
- 해지 안내 모달
- 로그인 시 남은 이용기간 팝업
- 이용 종료 후 접근 차단

## Gap 18. 데이터 내보내기 및 export zip 생성
정책 결정:
- 버튼명은 “데이터 내보내기”로 한다.
- export zip은 서버에서 생성한다.
- 생성 완료 후 고객사 관리자 이메일로 다운로드 링크를 발송한다.
- 다운로드 링크는 7일간 유효하다.
- 이메일에는 zip 파일을 직접 첨부하지 않는다.
- 7일 경과 후 export zip은 삭제될 수 있다.

필요 개발:
- 데이터 내보내기 버튼
- 요청 전 안내 팝업
- export_jobs 테이블
- export zip 생성
- 임시 저장소 저장
- 다운로드 링크 생성
- 다운로드 가능 기한 7일
- 고객사 관리자 이메일 발송
- 다운로드 이력
- export zip 만료/삭제 처리

## Gap 19. Google Workspace 기반 이메일 발송
정책 결정:
- 이메일 발송 기능은 반드시 도입한다.
- 초기에는 Google Workspace 기반 발송을 우선 검토한다.
- 대표/운영 이메일은 hello@wafl.co.kr로 한다.
- support/no-reply/privacy/billing 주소는 alias 또는 group으로 검토한다.
- 추후 전문 발송 서비스로 전환할 수 있게 설계한다.

필요 개발:
- Gmail API 또는 SMTP 발송 방식 검토
- 이메일 발송 provider 추상화
- 백업 완료/해지 안내/정책 변경 안내 이메일 템플릿
- 발송 성공/실패 로그
- 추후 provider 교체 가능 구조

## Gap 20. 카드결제 증빙 및 세금계산서 미발행 안내

필요 개발:
