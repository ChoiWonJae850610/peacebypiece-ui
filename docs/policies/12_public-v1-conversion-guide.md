# 고객 공개 v1.0 문서 변환 기준

## 문서 상태
- Policy Version: P-0.91.19
- Based App Version: 0.15.73
- 문서 성격: 내부 정책 문서를 고객 공개 v1.0 문서로 전환하기 위한 기준
- 적용 대상: WAFL 고객 공개 약관/정책 문서 전체

## 1. 목적
이 문서는 내부 정책 문서(P-0.91.x)를 실제 고객에게 공개할 v1.0 정책 문서로 변환할 때의 기준을 정리한다.

고객 공개 문서는 내부 의사결정 자료가 아니라 실제 고객이 읽는 법적·운영 문서이므로, 내부 메모, 미정 사항, 개발 필요사항, feature gap, 법무 검토 전 주석을 제거하고 확정된 정보만 반영해야 한다.

## 2. 고객 공개 문서 기본 원칙

### 2.1 서비스명
고객 공개 문서의 서비스명은 WAFL로 통일한다.

고객 공개 문서에서는 PeaceByPiece 명칭을 사용하지 않는다.

### 2.2 버전 체계
내부 정책 문서는 P-0.91.x 버전을 사용한다.

고객 공개 문서는 v1.0부터 시작한다.

예시:

```text
내부 정책 문서:
P-0.91.19

고객 공개 문서:
v1.0
```

### 2.3 고객 공개 문서의 문체
고객 공개 문서는 고객이 이해할 수 있는 간결한 문장으로 작성한다.

내부 검토용 표현, 개발자용 표현, 의사결정 과정 표현은 사용하지 않는다.

## 3. 고객 공개 문서에서 제거할 표현
고객 공개 v1.0 문서 작성 시 다음 표현과 항목을 제거한다.

- Policy Version P-0.91.x
- Based App Version
- 문서 성격: 법무 검토 전 초안
- 개발 필요사항
- feature gap
- 내부 검토 메모
- 미정 사항
- 아직 정하지 않음
- 추후 검토
- 방향으로 한다
- 후보
- 예정
- 검토한다
- 개발 필요
- 시스템 내부 작업 메모
- 사용자와의 의사결정 이력
- 실제 운영 전 가정값
- 파일 경로
- 코드/DB 구현 지시

## 5. 고객 공개 문서 이메일 기준
고객 공개 문서의 이메일 역할은 다음 기준을 사용한다.

- 대표/일반 문의: hello@wafl.co.kr
- 고객지원: support@wafl.co.kr
- 자동 발신/알림: no-reply@wafl.co.kr
- 개인정보/정책 문의: privacy@wafl.co.kr
- 결제 문의: billing@wafl.co.kr

고객 공개 전 wafl.co.kr 도메인 소유권과 이메일 계정 설정을 완료해야 한다.

## 7. 내부 문서별 고객 공개 전환 기준

### 7.1 이용약관
고객 공개 이용약관은 다음 내용을 포함한다.

- 서비스 이용계약
- 고객사 승인
- 고객사 관리자와 멤버
- 서비스 제공 범위
- 고객사의 의무
- 결제 및 요금
- 해지
- 데이터 내보내기
- 파일 및 데이터 보관
- 서비스 변경 및 중단
- 이용 제한
- 책임 제한
- 약관 변경
- 준거법 및 분쟁 해결

제거할 항목:
- 내부 검토 필요 사항
- Policy Version
- Based App Version
- 개발 필요사항
- 미정 값

### 7.2 개인정보처리방침
고객 공개 개인정보처리방침은 실제 수집 항목, 처리 목적, 보유기간, 파기, 처리위탁, 정보주체 권리, 문의처를 포함한다.

공개 전 실제 확인 필요:
- 사업자 정보
- 개인정보 보호책임자 또는 담당자
- 처리위탁사 목록
- 수집 항목
- 보유기간
- 고객지원/개인정보 문의 이메일

### 7.3 결제 및 환불 정책
고객 공개 결제 및 환불 정책은 요금제, 무료 체험, 정기결제, 결제 실패, 장기 미납, 환불, 해지, 결제 증빙 기준을 포함한다.

공개 전 실제 확인 필요:
- PG사
- 결제수단
- 카드매출전표/영수증 제공 방식
- 요금제명과 금액
- VAT 포함 표시

### 7.4 파일 및 저장소 보관 정책
고객 공개 파일 및 저장소 보관 정책은 저장소 초과, 휴지통 30일, 영구 삭제, export zip 7일, 백업/복구 제한, 시스템관리자 파일 접근 조건을 포함한다.

고객에게 너무 내부적인 DB/R2/삭제 구현 방식은 공개하지 않는다.

### 7.5 데이터 내보내기 정책
고객 공개 데이터 내보내기 정책은 데이터 내보내기 버튼, export zip 생성, 이메일 다운로드 링크, 로그인 필요 링크, 7일 다운로드 가능 기간, 포함/제외 대상을 설명한다.

### 7.6 서비스 운영 정책
고객 공개 서비스 운영 정책은 점검, 장애, 고객지원, 부정 이용, 불법 파일, 서비스 변경/종료, 장애 보상 원칙을 설명한다.

고객에게 내부 운영 화면이나 관리자 구현 구조는 공개하지 않는다.

### 7.7 정책 변경 고지 및 동의 정책
고객 공개 정책 변경 고지 및 동의 정책은 재동의가 필요한 변경, 공지만 필요한 변경, 재동의하지 않은 경우, 동의 이력 관리 기준을 설명한다.

## 10. 고객 공개 문서 관리 방식
고객 공개 문서는 내부 정책 문서와 별도 위치에서 관리한다.

예시:

```text
docs/policies/internal/
docs/policies/public/
```

또는:

```text
docs/policies/
docs/public-policies/
```

고객 공개 문서는 실제 서비스 화면에서 열람 가능한 형태로 변환할 수 있어야 한다.
