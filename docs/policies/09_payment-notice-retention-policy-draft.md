# 결제 실패·장기 미납 안내 및 정책 변경 고지 기준 초안

## 문서 상태
- Policy Version: P-0.91.10
- Based App Version: 0.15.73
- 문서 성격: 법무 검토 전 초안
- 적용 대상: 결제 실패 안내, 장기 미납 안내, 데이터 삭제 로그, 약관 변경 고지 및 재동의

## 1. 목적
이 문서는 WAFLOW 서비스에서 결제 실패, 장기 미납, 데이터 삭제, 약관 변경이 발생했을 때 고객사와 시스템관리자에게 어떤 방식으로 안내하고 기록할지 정리한다.

## 2. 결제 실패 안내

### 2.1 안내 방식
정기결제가 실패하면 회사는 다음 방식으로 고객사 관리자에게 안내할 수 있다.

- 서비스 내 팝업
- 상단 배너
- 이메일

### 2.2 팝업 문구 초안
```text
정기결제가 완료되지 않았습니다.

등록된 결제수단으로 이번 결제가 처리되지 않아 현재 유료 업무 기능이 제한되었습니다.

결제수단을 변경하거나 결제를 다시 시도하면 서비스 이용이 즉시 정상화됩니다.

제한되는 기능:
- 작업지시서 생성/수정/삭제
- 파일 업로드
- 메모 작성/수정
- 멤버 초대
- 데이터 내보내기

기존 데이터는 삭제되지 않으며, 결제 복구 후 다시 이용할 수 있습니다.
```

### 2.3 버튼
- 결제수단 변경
- 결제 다시 시도
- 고객지원 문의

### 2.4 이메일 제목
```text
[WAFLOW] 정기결제가 완료되지 않았습니다
```

### 2.5 이메일 본문 핵심 문구
```text
등록된 결제수단으로 정기결제가 완료되지 않아 유료 업무 기능이 제한되었습니다.

결제수단을 변경하거나 결제를 다시 시도하면 서비스 이용이 정상화됩니다.

결제 실패 상태가 장기간 지속될 경우, 약관 및 결제 정책에 따라 이용계약 해지 및 데이터 삭제 절차가 진행될 수 있습니다.
```

## 3. 장기 미납 안내

### 3.1 미납 30일
미납 30일 경과 시 장기 미납 안내를 발송한다.

안내 방식:
- 이메일
- 로그인 시 팝업
- 상단 배너

문구 초안:
```text
결제 실패 상태가 30일간 지속되고 있습니다.

현재 유료 업무 기능이 제한되어 있으며, 결제를 완료하면 즉시 정상 이용할 수 있습니다.

결제 실패 상태가 장기간 지속될 경우 이용계약 해지 및 데이터 삭제 절차가 진행될 수 있습니다.
```

### 3.2 미납 60일
미납 60일 경과 시 데이터 삭제 예정 안내를 발송한다.

안내 방식:
- 이메일
- 로그인 시 팝업
- 상단 배너

문구 초안:
```text
결제 실패 상태가 60일간 지속되고 있습니다.

결제가 복구되지 않을 경우, 미납 90일 경과 시 이용계약 해지 및 고객사 업무 데이터 삭제 절차가 진행될 수 있습니다.

삭제 대상에는 작업지시서, 첨부파일, 메모, 업체정보, 멤버 업무정보 등이 포함될 수 있습니다.
```

### 3.3 미납 90일
미납 90일 도달 시 자동 삭제하지 않고 시스템관리자 화면에 삭제 가능 대상으로 표시한다.

시스템관리자가 최종 확인한 후 이용계약 해지 및 고객사 업무 데이터 삭제를 처리한다.

## 4. 데이터 삭제 처리 로그 UI

### 4.1 제공 범위
데이터 삭제 처리 로그 UI는 시스템관리자 전용으로 둔다.

고객사 관리자에게는 데이터 삭제 로그 UI를 제공하지 않는다.

고객사에는 이메일, 팝업, 배너, 공지 등 회사가 정한 방식으로 안내한다.

### 4.2 로그 항목
시스템관리자 삭제 로그에는 다음 항목을 기록한다.

- 고객사명
- 삭제 사유
- 삭제 기준일
- 삭제 요청일
- 삭제 처리일
- 처리자
- 삭제 대상 요약
- 삭제 상태
- 상세 보기

### 4.3 삭제 상태
삭제 상태는 다음과 같이 관리할 수 있다.

- 삭제 가능
- 삭제 대기
- 삭제 진행 중
- 삭제 완료
- 삭제 실패
- 삭제 보류

### 4.4 삭제 대상 요약 예시
```text
작업지시서 124건
디자인 파일 312개
일반 첨부파일 88개
메모 430개
업체정보 26건
멤버 업무정보 8건
사업자등록증 1개
```

## 5. 약관 변경 및 재동의 기준

### 5.1 재동의가 필요한 변경
다음 항목이 실질적으로 변경되는 경우 고객사 관리자 재동의를 받는다.

- 요금
- 환불
- 해지
- 데이터 삭제
- 개인정보 처리
- 데이터 내보내기 제한
- 결제 실패/미납에 따른 서비스 제한

### 5.2 재동의가 필요 없는 변경
다음 변경은 공지만 하고 재동의는 받지 않는다.

- 오탈자 수정
- 문장 표현 정리
- 조항 번호 정리
- 회사 연락처 변경
- 고객지원 이메일 변경
- 서비스명 표기 정리
- 법령명 또는 링크 정리

### 5.3 재동의하지 않은 경우
재동의가 필요한 변경에 대해 고객사 관리자가 동의하지 않는 경우 회사는 서비스 이용을 제한할 수 있다.

구체적인 제한 방식은 약관 변경 정책 또는 서비스 운영 정책에서 별도로 정한다.

## 6. 개발 필요사항
- 결제 실패 팝업/배너
- 결제 실패 이메일 템플릿
- 미납 30일 안내 템플릿
- 미납 60일 삭제 예정 안내 템플릿
- 미납 90일 삭제 가능 대상 표시
- 시스템관리자 데이터 삭제 로그 UI
- 삭제 상태 관리
- 약관 버전 관리
- 사용자별 약관 동의 이력
- 재동의 필요 여부 판단
- 재동의 미완료 시 접근 제한
