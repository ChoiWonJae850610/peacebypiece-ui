# docs archive 이동 계획 0.19.94.5

## 목적

테스트가 어려운 기간에는 기능 코드나 DB/API/R2 흐름을 건드리지 않고, 문서 정리 기준을 먼저 고정한다. 이 문서는 `docs/` 하위 문서를 실제로 이동·삭제하기 전의 분류 기준이다.

## 현재 관찰

- `docs/**/*.md` 문서 수: 391개
- `docs/` 루트의 버전별 문서가 많아 현재 기준 문서와 과거 작업 기록이 섞여 있다.
- `docs/현재기준/`, `docs/정책문서/`, `docs/보관문서/` 구조는 이미 존재한다.
- 테스트 불가 상태에서는 문서 이동 자체도 링크/참조 영향을 줄 수 있으므로 이번 버전에서는 실제 이동을 하지 않는다.

## 분류 기준

### 1. 현행 유지

아래 문서는 삭제하거나 이동하지 않는다.

```text
- docs/현재기준/**
- docs/정책문서/**
- docs/company-file-upload-design-0.19.94.md
- docs/patch-automation-delete-safety-0.19.94.4.md
- docs/project-doc-sql-script-cleanup-audit-0.19.94.3.md

- docs/db-api-smoke-test-*.md
- docs/playwright-*.md
- docs/*policy*.md
- docs/*billing*.md
- docs/*storage*.md
```

### 2. 삭제 금지

기능 구현·운영 정책·DB 초기화·테스트 기준에 직접 연결되는 문서는 삭제하지 않는다.

```text
- DB schema / full_reset / smoke test 관련 문서
- 권한 / 멤버 / 회사 승인 / 정책 동의 관련 문서
- R2 / 파일 업로드 / 데이터 내보내기 관련 문서
- 작업지시서 상태 / 원단·부자재 발주 / PDF 생성 조건 관련 문서
- 고객 공개 약관·정책 문서
```

### 3. 보관 후보

아래 문서는 즉시 삭제하지 않고 `docs/보관문서/` 하위로 이동하는 후보로 둔다.

```text
- 과거 UI 세부 보정 단일 패치 문서
- 이미 현재 기준 문서로 흡수된 A-Type 또는 WAFL UI 누적 문서
- 과거 감사/점검 문서 중 현재 개발 기준으로 직접 쓰지 않는 문서
- 버전명이 오래된 단기 작업 기록 문서
```

### 4. 삭제 후보

아래 조건에 모두 해당할 때만 삭제 후보로 볼 수 있다.

```text
- 내용이 비어 있거나 실질 정보가 없음
- 다른 현행 문서와 완전히 중복됨
- 코드/README/정책 문서에서 참조되지 않음
- 최근 로드맵이나 작업 기준에 필요하지 않음
```
