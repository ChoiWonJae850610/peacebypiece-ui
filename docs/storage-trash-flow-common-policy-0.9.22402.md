# 0.9.22402 저장소/휴지통 내부 흐름 회귀 보정

## 목표

고객관리자 저장소 관리 화면에서 파일 단독 항목과 작업지시서 묶음 항목이 같은 선택 흐름 안에서 섞일 때, 작업지시서 묶음 첨부가 별도의 실패/제외 항목처럼 집계되지 않도록 보정한다.

## 변경 사항

- 작업지시서가 선택된 경우 해당 작업지시서의 묶음 첨부는 파일 단독 처리 대상에서 제외하고 작업지시서 actionFlow에서 처리하도록 정리했다.
- 휴지통 비우기에서도 작업지시서 묶음 첨부가 `삭제 요청할 수 없는 항목`으로 잘못 집계되지 않도록 보정했다.
- 복원 가능 여부와 삭제 요청 가능 여부 판정을 `trashPolicy.ts`의 공통 함수로 이동했다.
- actionFlow의 일부 결과 메시지 생성 책임을 presentation formatter로 이동했다.
- 파일 정책 저장/실패 메시지와 purge worker 결과 메시지 생성도 presentation 계층 함수를 사용하도록 정리했다.

## 확인 기준

1. 작업지시서와 그 묶음 첨부가 같이 보이는 상태에서 휴지통 비우기 실행
2. 묶음 첨부가 `삭제 요청할 수 없는 항목 n개 제외`로 잘못 표시되지 않는지 확인
3. 작업지시서 단독 선택 삭제 메시지가 작업지시서/문서/디자인/메모 단위로 표시되는지 확인
4. 파일 단독 선택 삭제 메시지가 문서/디자인 단위로 표시되는지 확인
5. `/admin/files` 새로고침 후 휴지통 목록과 삭제 요청 요약이 기존과 동일하게 표시되는지 확인
