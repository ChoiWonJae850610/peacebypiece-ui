# 0.10.38 시스템 기준정보 DB schema 설계

## 목적

고객관리자 환경설정에서 단위 표준과 외주공정 유형을 직접 추가하지 않고, 시스템관리자가 관리하는 표준 원장 중 사용 여부만 선택하도록 구조를 분리한다.

생산품 유형은 고객사별 업무 분류 방식 차이가 크므로 고객관리자 직접 관리를 유지한다. 다만 신규 고객사를 만들 때 사용할 기본 템플릿은 시스템관리자가 관리할 수 있도록 별도 원장을 둔다.

## 추가 schema

### 단위 표준

- `system_unit_standards`
  - 시스템관리자 단위 표준 원장
  - 한글명, 영문 코드/약어, 분류, 설명, 사용 예시, 정렬 순서, 활성 상태 관리
- `company_enabled_unit_standards`
  - 고객사별 단위 표준 사용 여부
  - 고객사는 시스템 표준 중 필요한 항목만 사용/미사용으로 선택

### 외주공정 유형

- `system_outsourcing_process_standards`
  - 시스템관리자 외주공정 유형 원장
  - 공정명, 분류, 설명, 사용 예시, 정렬 순서, 활성 상태 관리
- `company_enabled_process_standards`
  - 고객사별 외주공정 유형 사용 여부
  - 고객사는 시스템 표준 중 필요한 항목만 사용/미사용으로 선택

### 생산품 유형 기본 템플릿

- `system_product_type_templates`
  - 신규 고객사 생성 시 복사할 생산품 유형 기본 템플릿 원장
- `system_product_type_template_categories`
  - 템플릿 안의 1차 → 2차 → 3차 계층 항목

## 기존 테이블과의 관계

- `units`
  - 현재 고객사별 단위 목록 테이블로 유지
  - 후속 버전에서 시스템 표준 사용 여부와 실제 작업지시서 선택값 연결 정책을 정리한다.
- `outsourcing_processes`
  - 현재 고객사별 외주공정 목록 테이블로 유지
  - 후속 버전에서 시스템 표준 사용 여부와 협력업체/작업지시서 공정 선택값 연결 정책을 정리한다.
- `item_categories`
  - 고객사별 생산품 유형 직접 관리 테이블로 유지
  - 시스템 템플릿은 신규 고객사 초기값 복사용으로만 사용한다.

## 이번 버전에서 하지 않은 것

- 실제 CRUD API 구현 없음
- 시스템관리자 기준정보 저장/수정/삭제 연결 없음
- 고객사별 사용 여부 UI 저장 연결 없음
- 신규 고객사 생성 시 생산품 유형 템플릿 복사 로직 없음
- 기존 고객관리자 기준정보 저장 로직 변경 없음
- 감사 로그 쓰기 연결 없음
