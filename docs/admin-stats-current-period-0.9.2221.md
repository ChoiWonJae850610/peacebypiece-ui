# 0.9.2221 — 통계 화면 구조 재정의 1차

## 목적

고객관리자 통계 화면을 요금제 기준이 아니라 현재 시점 요약과 기간별 분석 기준으로 재정의한다.

## 변경 기준

- 상단 4개 카드는 기간 필터를 적용하지 않는 현재 시점 요약으로 유지한다.
- 기간 선택은 작업흐름분석 이하의 기간별 분석에만 적용한다.
- 저장소 상세 요약은 통계 화면에서 제거하고 저장소 관리 화면에서 다룬다.
- 저장소 카드는 상단 현재 요약으로만 표시하고 저장소 화면으로 이동하는 링크는 제거한다.

## 상단 현재 시점 카드

1. 누적 생산
   - 전체 작업지시서 수
   - 보조 정보: 리오더 수
2. 누적 납기 지연율
   - 납기일이 있는 발주 중 현재 지연 후보 비율
   - 완료일 기준 누적 지연율은 후속 데이터 구조 확정 필요
3. 누적 검수/불량률
   - 반려 또는 리오더/재작업 후보 기준
   - 정식 검수 결과 테이블 확정 후 보강 필요
4. 현재 저장소 사용량
   - 사용률 %
   - GB 단위는 소수점 둘째 자리까지 표시
   - 보조 정보로 MB 환산값 표시

## 기간별 분석

기간 선택은 다음 3개 구조로 정리한다.

- 최근 7일
- 최근 30일
- 직접 선택: 시작일 / 종료일

직접 선택은 1차 UI만 추가한다. 실제 DB 조회 연결은 후속 버전에서 처리한다.

## 통계 화면 섹션

- 작업흐름분석
- 기간별 리오더 TOP5
- 생산품 유형 비율
- 생산품유형 TOP5
- 업체 성과
- 업체별 납기/검수 지표
- 품질/납기 후보 요약

## 제외한 항목

- 요금제 선택 UI
- 저장소 상세 요약 카드
- 저장소 카드 클릭 이동
- 운영/개발 기준
- 성능/캐시/summary table 기준

## 후속 작업

0.9.2222에서 생산품 유형 회차 필터와 업체 성과/납기/검수 지표의 실제 집계 기준을 추가 정리한다.
