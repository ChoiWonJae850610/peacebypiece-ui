# 0.10.33 — 기준정보 관리 권한 구조 재정리

## 목표

고객관리자 환경설정의 기준정보 관리에서 생산품 유형, 단위 표준, 외주공정 유형의 관리 권한을 데이터 성격에 맞게 분리한다.

## 기준

### 생산품 유형

- 고객사마다 분류 체계가 다를 수 있으므로 고객관리자 직접 관리를 유지한다.
- 1차 품목 → 2차 품목 → 3차 세부 유형의 계층 구조를 유지한다.
- 추가, 사용/미사용, 기본값 복원, 저장 흐름은 기존과 동일하다.

### 단위 표준

- 단위 표준은 시스템관리자가 제공하는 표준 목록을 기준으로 한다.
- 고객관리자는 이 고객사가 사용할 단위만 사용/미사용으로 선택한다.
- 고객관리자 화면에서는 새 단위 추가 입력 UI를 제거한다.
- 새 단위가 필요하면 개발 건의 또는 관리자 문의로 요청하는 방향으로 안내한다.

### 외주공정 유형

- 외주공정 유형은 시스템관리자가 제공하는 표준 목록을 기준으로 한다.
- 고객관리자는 이 고객사가 사용할 공정만 사용/미사용으로 선택한다.
- 고객관리자 화면에서는 새 공정 추가 입력 UI를 제거한다.
- 새 공정 유형이 필요하면 개발 건의 또는 관리자 문의로 요청하는 방향으로 안내한다.

## 이번 버전 범위

- UI 정책만 반영한다.
- 기존 저장 로직과 데이터 구조는 변경하지 않는다.
- DB schema는 변경하지 않는다.
- 시스템관리자 표준 기준정보 관리 화면은 아직 추가하지 않는다.

## 후속 작업 후보

- 시스템관리자 기준정보 관리 화면 추가
- 고객사별 표준 사용 여부 저장 구조 명확화
- 기준정보 추가 요청을 개발 건의 또는 시스템관리자 승인 흐름과 연결
