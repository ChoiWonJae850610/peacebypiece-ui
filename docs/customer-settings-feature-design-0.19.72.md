# 0.19.72 고객사 환경설정 실제 기능 보강 설계 1차

## 목적

고객사 관리자 환경설정 화면을 실제 서비스 화면처럼 보이도록 정리하고, 미구현 상태를 직접 드러내는 개발용 배지와 버전 기준 문구를 제거한다.

## 이번 버전 반영 범위

- 고객사 관리자 정책 카드의 내부 버전 배지를 제거한다.
- `개발중`처럼 고객 화면에 미완성으로 보이는 상태 표현을 운영 검토/운영 항목 표현으로 낮춘다.
- `개발 건의` 메뉴명은 `서비스 건의`로 변경한다.
- 권한 UI 설명에서 과거 permission code 버전 표현을 제거한다.
- 실제 환경설정 저장, 결제, 정책 문서 관리, 알림/AI 기능 구현은 포함하지 않는다.

## 후속 구현 기준

### 회사 정보 변경 요청

- 고객사 관리자는 환경설정에서 변경 사유를 제출한다.
- 제출된 요청은 시스템관리자 검토 대상으로 분리한다.
- 회사명, 사업자정보, 주소, 대표 계정 변경은 즉시 반영하지 않는다.

### 계정 비활성화 요청

- 고객사 단위 비활성화는 개인 멤버 탈퇴와 분리한다.
- 고객사 관리자가 요청하면 시스템관리자 검토 후 회사 상태를 변경한다.
- 비활성 회사는 업무 화면 접근을 제한하되, 데이터 보관/내보내기 정책과 충돌하지 않게 처리한다.

### 요금제와 저장공간

- 고객사 화면은 조회와 요청 접수 중심으로 둔다.
- 실제 요금제 변경, 결제수단, 청구 이력은 시스템관리자/결제 도메인에서 처리한다.
- 저장공간 사용량은 저장소관리와 같은 기준을 사용해야 한다.

### 약관·정책

- 고객 공개 문서에서는 WAFL 명칭만 사용한다.
- 고객사 화면은 최신 문서 조회만 제공한다.
- 문서 작성/배포/버전 관리는 시스템관리자 화면으로 분리한다.

## 변경하지 않은 것

- DB schema 변경 없음
- API route 변경 없음
- 회사 설정 저장 흐름 변경 없음
- 결제/요금제 기능 구현 없음
- 정책 문서 route 구현 없음
