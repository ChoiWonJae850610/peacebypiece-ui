# 0.18.90 현재 기준 문서 점검

## 작업 목적

테스트 불가 상태에서 DB/API/R2/권한/작업지시서 상태 흐름을 건드리지 않고, `docs/현재기준/` 문서와 실제 프로젝트 구조의 차이 후보를 정리한다.

## 점검 범위

- `docs/README.md`의 현재 기준 문서 인덱스
- `docs/현재기준/`에 남아 있는 기준 문서 목록
- 현재 기능 구현과 직접 연결될 가능성이 높은 문서군
- 보관문서로 이동할 가능성이 있는 과거 버전별 기록 문서

## 확인 내용

### 1. 현재 기준 문서 인덱스 보강

`docs/현재기준/`에는 통계 관련 기준 문서가 존재하지만 `docs/README.md`의 현재 기준 문서 목록에는 일부가 빠져 있었다.

이번 버전에서 인덱스에 아래 항목을 추가했다.

- `현재기준/고객사-통계-지표.md`
- `현재기준/시스템-통계-지표.md`
- `현재기준/통계-저장소-구조.md`

### 2. 이동 보류 문서

`docs/현재기준/`에는 `0.17.16`부터 `0.17.27`까지의 원단·부자재 발주 준비 기록이 남아 있다.

이 문서들은 파일명상 과거 버전 기록에 가깝지만, 아직 원단부자재발주 화면 정리와 연결될 수 있으므로 이번 패치에서는 이동하지 않았다.

후속 테스트 가능 시점 또는 원단부자재발주 작업 재개 시 아래 중 하나로 처리한다.

- 현재 구현 기준으로 필요한 내용만 `현재기준/원단-부자재-발주-설계.md`에 흡수
- 나머지는 `docs/보관문서/점검기록/` 또는 `docs/보관문서/설계초안/`으로 이동

### 3. 유지해야 할 기준 문서군

아래 문서들은 현재 코드/운영 정책과 직접 연결되므로 `docs/현재기준/`에 유지한다.

- 리팩토링 규칙
- 소스 구조
- 라우팅 구조
- 데이터베이스 구조
- 데이터베이스 쿼리·인덱스 정책
- 작업지시서 상태 구조
- 워크스페이스 경계
- 초대 정책 설계
- 요금·저장소 정책 설계
- 원단·부자재 데이터베이스 설계
- 원단·부자재 발주 설계
- full-reset 검증
- 고객사 통계 지표
- 시스템 통계 지표
- 통계 저장소 구조

## 이번 패치에서 하지 않은 일

- 문서 폴더 이동 없음
- DB schema 변경 없음
- API 변경 없음
- R2/첨부/메모/휴지통/purge 흐름 변경 없음
- 권한/상태 전환 흐름 변경 없음
- UI 동작 변경 없음

## 다음 후보

테스트 불가 상태가 유지되면 다음 작업은 문서 폴더 이동이 아니라, 소스 전체에서 명백한 임시 산출물·불필요 파일 후보를 더 점검하는 방향이 안전하다.
