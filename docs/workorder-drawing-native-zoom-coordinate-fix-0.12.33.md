# 작업지시서 직접 그리기 확대/축소 좌표계 보정 (0.12.33)

## 목적

0.12.32에서 확대/축소와 손바닥 이동을 추가한 뒤, 50% 축소 상태에서 기존 최초 표시 영역 밖의 캔버스 구석을 그리려고 하면 pointer 입력이 잡히지 않는 문제가 있었다.

## 원인

캔버스 자체에 CSS transform을 적용한 상태에서 pointer 이벤트도 canvas 요소에 직접 묶여 있었다. 축소 상태에서는 시각적으로 보이는 캔버스와 실제 pointer 이벤트를 받는 hit area가 어긋날 수 있었다.

## 변경

- pointer 이벤트 수신 대상을 canvas에서 canvas container로 이동했다.
- canvas는 `pointer-events-none`으로 두고, wrapper가 항상 pointer 이벤트를 받도록 했다.
- pointer 좌표는 transform 적용 후의 canvas bounding rect 기준으로 canvas 좌표로 변환한다.
- 변환된 좌표가 실제 canvas 범위 안에 있을 때만 그리기를 시작하거나 이어가도록 보정했다.
- 펜, 지우개, 직선, 화살표, 사각형, 원/타원 도구는 동일한 좌표 변환을 사용한다.
- 저장 PNG는 기존처럼 원본 canvas 전체 기준을 유지한다.

## 테스트 기준

1. 직접 그리기 모달을 연다.
2. 50%로 축소한다.
3. 최초 화면에서 잘 보이지 않던 캔버스 구석 영역에 선을 그린다.
4. 윈도우 기본 커서로 바뀌지 않고 drawing cursor가 유지되는지 확인한다.
5. 펜, 지우개, 직선, 화살표, 사각형, 원/타원 모두 위치가 맞는지 확인한다.
6. 저장 후 구석에 그린 내용이 PNG에 포함되는지 확인한다.
