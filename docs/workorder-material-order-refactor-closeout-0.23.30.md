# 작업지시서·발주서 리팩토링 종료 감사 — 0.23.30

## 범위

- 작업지시서와 발주서의 모바일·태블릿·데스크톱 반응형 구조
- `/workspace` 일반 콘텐츠 스크롤과 작업 화면 고정 패널 스크롤의 역할 분리
- 변경 작업 토스트의 진행·성공·실패 문구 통일
- 공통 저장 피드백 호환 계층 제거
- 발주서 대형 편집 훅의 토스트 책임 분리
- 태블릿 Topbar·목록 드로어·패널 너비 정책 공통화

## 최종 피드백 정책

단일 정보 변경은 공통 변경 대상과 상태를 사용한다.

- 진행: `{문서/영역/항목} 변경 중입니다.`
- 성공: `정보가 변경되었습니다.`
- 실패: `정보를 변경하지 못했습니다.`

작업지시서 전체 저장도 동일한 공통 생성기를 사용한다. 언어 파일에 중복 보관하던 전체 저장 토스트 문구는 제거했다.

생성·삭제·발주 요청·첨부 업로드처럼 정보 변경과 의미가 다른 작업은 고유 문구를 유지한다.

## 스크롤 정책

- `WorkspaceShell`의 `scroll` 모드: 운영 대시보드 등 일반 콘텐츠 영역이 내부 세로 스크롤을 담당한다.
- `fixed-md` 모드: 작업지시서·발주서처럼 데스크톱 이상에서 패널별 스크롤을 사용한다.
- 태블릿 가로에서도 외부와 내부 스크롤이 동시에 차단되는 breakpoint 공백이 없어야 한다.

## 종료 판단

0.23.30을 작업지시서·발주서 구조 정비의 1차 종료 기준으로 사용한다. 이후 변경은 통합 실기기 테스트에서 확인된 회귀나 기능 오류만 별도 QA 패치로 처리한다.
