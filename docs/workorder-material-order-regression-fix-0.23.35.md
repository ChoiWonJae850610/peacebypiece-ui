# 0.23.35 작업지시서 단일 PATCH 회귀 및 발주서 자재 종류 저장 수정

## 목적

0.23.32~0.23.33 단일 필드 저장 경량화 이후 확인된 두 회귀를 수정한다.

1. 작업지시서 납기일 등 일부 필드 변경 후 담당자·분류·수량 등 요청하지 않은 필드가 화면에서 덮어써지는 문제
2. 발주서 자재 종류 변경 시 `supplier_partner_id`가 SQL SET 절에 중복 추가되어 저장이 실패하는 문제

## 적용 내용

- 작업지시서 PATCH 결과 병합 기준을 서버 응답 객체의 속성 존재 여부가 아니라 실제 요청 patch의 필드 포함 여부로 변경
- 납기일 변경 시 담당자·분류·수량·재고 등 요청하지 않은 값 유지
- 작업지시서 즉시 변경 경로 전체를 공통 operation toast lifecycle로 연결
- 납기일·분류·수량 등도 필드별 진행 문구와 success/danger tone 사용
- 공통 operation 진행 중에는 기존 saveStatus 기반 processing overlay가 중복 표시되지 않도록 차단
- 발주서 자재 종류 변경 시 공급처 초기화 SQL이 한 번만 생성되도록 수정
- 자재 종류 변경 시 기존 정책대로 공급처·금액·품목·할당 초기화 유지
