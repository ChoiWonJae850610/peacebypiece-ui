# docs 수동 검토 후보 상세 분류 0.19.94.7

## 목적

0.19.94.6에서 파일명만으로 판단하기 어려워 수동 검토로 남긴 `docs/` 루트 UI 문서 9개를 실제 내용 기준으로 재분류한다. 이번 패치는 테스트 불가 기간의 저위험 정리 작업이므로 실제 이동이나 삭제는 하지 않는다.

## 결론

수동 검토 대상 9개는 모두 현행 기준 문서가 아니라 0.17대 UI 라이브러리 도입, 공통 래퍼 추가, 특정 화면의 공통 컴포넌트 전환 기록이다. 따라서 삭제하지 않고 보관 후보로 확정한다.

- 삭제 대상: 없음
- 현행 유지 승격: 없음
- 보관 후보 확정: 9개
- 즉시 이동: 없음

## 다음 이동 기준

다음에 실제 이동 패치를 만들 경우 삭제가 아니라 보관 이동을 우선한다.

- `docs/보관문서/wafl-a-type/`
  - WAFL UI 공통 컴포넌트, wrapper, toast, select, tooltip처럼 WAFL UI 체계 도입 기록
- `docs/보관문서/설계초안/`
  - 의존성 도입 배경, sheet wrapper 설계처럼 구현 전후의 설계 배경 기록
- `docs/보관문서/점검기록/`
  - 특정 화면 적용 결과, 전환 완료 기록, 과거 QA 성격의 문서

## 삭제 보류 기준

이번 9개 문서는 오래된 작업 기록이지만 삭제하지 않는다. 이유는 다음과 같다.

1. WAFL 공통 UI 시스템 도입 배경을 설명한다.
2. 과거 선택지와 현재 구조가 왜 달라졌는지 추적할 수 있다.
3. 실제 코드와 충돌할 가능성은 낮지만, 후속 리팩토링 판단 근거로 사용할 수 있다.

## 후속 작업

- 0.19.94.8에서 실제 docs 이동 여부를 결정한다.
- 테스트 불가 상태가 계속되면 실제 이동 없이 `docs/README.md` 인덱스 정리만 진행한다.
- 실제 이동을 할 경우 패치 삭제 목록에는 파일 경로만 넣고, 폴더 삭제는 포함하지 않는다.
