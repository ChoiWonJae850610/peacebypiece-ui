# 41. 카드결제/청구/증빙 정책

## 1. 목적

0.15.18은 WAFL 운영을 카드결제 중심 SaaS로 전제하고, 저장 가능한 결제 데이터와 저장하면 안 되는 민감 데이터를 구분한다.

이 문서는 PG 연동 전 결제수단, 결제내역, 영수증, 세금계산서 요청, 정산자료 출력의 데이터 책임 범위를 정의한다.

```txt
적용 성격:
- 정책 문서화
- 저장 가능/금지 데이터 구분
- 증빙 유형/상태 후보 정리

이번 버전 제외:
- PG 연동
- 결제 API 구현
- DB schema 생성
- 카드 등록 화면 구현
- 세금계산서 발행 자동화
```

## 2. 기본 결제 정책

```txt
- 초기 운영은 카드결제를 기본으로 한다.
- 계좌이체는 초기 범위에서 제외한다.
- 결제수단 저장은 PG의 customerKey/billingKey 기반으로 처리한다.
- 전체 카드번호와 보안정보는 저장하지 않는다.
- 기본 증빙은 카드매출전표다.
- 사업자 고객사는 세금계산서 요청 여부를 별도로 관리한다.
```

## 3. 저장 가능한 결제수단 데이터

```txt

customerKey
billingKey
카드사명

결제수단 상태
등록일
비활성일
삭제/만료 여부
```

### 3.1 결제수단 상태 후보

```txt
active
- 사용 가능

inactive
- 사용 중지

expired
- 만료

revoked
- 사용자가 해지 또는 삭제

failed
- PG 등록 또는 인증 실패
```

상태 비교는 화면 문자열이 아니라 enum 또는 중앙 status helper를 기준으로 처리한다.

## 6. 저장 금지 데이터

다음 데이터는 WAFL DB, 로그, 문서, commit-meta, 테스트 seed에 저장하지 않는다.

```txt
전체 카드번호
CVC
카드 비밀번호
주민등록번호
카드 유효기간 원문
```

주의:

```txt
- 카드 마지막 4자리는 저장 가능하다.
- 카드사명은 저장 가능하다.
- billingKey는 저장 가능하되 secret처럼 노출하지 않는다.
- PG secret key는 환경변수에서만 읽는다.
- 로그에 결제 인증 payload 전체를 출력하지 않는다.
```

## 7. 고객사 사업자 정보

세금계산서 요청과 정산자료 출력을 위해 고객사 사업자 정보는 별도로 관리한다.

```txt
회사명
사업자등록번호
대표자명
사업장 주소
업태
종목
세금계산서 수신 이메일
```

사업자등록번호는 카드 보안정보와 성격이 다르지만, 외부 노출을 최소화하고 권한 있는 화면에서만 표시한다.

## 8. 시스템관리자 화면 책임

```txt
/system/billing/plans
- 요금제 상품 관리

/system/billing/subscriptions
- 고객사 구독 상태 관리

/system/billing/payment-methods
- 결제수단 상태 확인

/system/billing/payments
- 결제내역 확인

/system/billing/failures
- 결제 실패/미납 관리

/system/billing/refunds
- 환불 관리

/system/billing/evidence
- 증빙 요청/발행 상태 관리

/system/billing/settlements
- 정산자료 출력
```

## 9. 고객사관리자 화면 책임

```txt
/admin/billing
- 현재 요금제 확인
- 카드 등록/변경
- 결제내역 확인
- 영수증 보기
- 증빙 요청 정보 확인/수정

/admin/settings
- 회사 정보 관리
- 사업자 정보 관리

/admin/legal
- 이용약관/개인정보처리방침/환불정책/데이터 보관정책 보기
```

고객사관리자는 본인 회사의 결제수단과 결제내역만 볼 수 있다. 다른 고객사의 결제 정보는 시스템관리자도 필요한 운영 범위에서만 접근한다.

## 10. 정산자료 출력 기준

```txt
기간별 매출
고객사별 매출
공급가액
부가세
합계
결제 상태
증빙 유형
발행 상태
영수증 URL
```

정산자료 출력 화면은 실제 세무 신고 자동화가 아니라, 신고와 확인에 필요한 데이터를 정리해서 출력하는 운영 보조 화면으로 둔다.

## 11. 로그/감사 기준

```txt
기록 대상:
- 결제수단 등록/변경/비활성화
- 결제 성공/실패
- 환불 요청/처리
- 세금계산서 요청/발행 상태 변경
- 요금제 변경

기록 금지:
- 전체 카드번호
- CVC
- 카드 비밀번호
- 주민등록번호
- 카드 유효기간 원문
- PG secret key
```

## 12. 구현 전 확인 사항

```txt
[ ] 사용할 PG사의 billingKey/customerKey 개념을 확인했는가?
[ ] 카드결제 정기결제와 단건결제 흐름을 구분했는가?
[ ] 결제수단 저장 가능 데이터와 금지 데이터를 DB schema에 반영했는가?
[ ] Vercel 환경변수에 PG secret을 저장하고 코드/문서에는 포함하지 않는가?
[ ] 로그에 인증 payload 전체가 남지 않게 했는가?
[ ] 고객사관리자와 시스템관리자 접근 범위가 분리되어 있는가?
```
