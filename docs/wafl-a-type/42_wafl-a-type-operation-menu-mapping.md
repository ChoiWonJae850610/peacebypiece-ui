# 42. 운영 IA 기반 홈/메뉴 매핑

## 1. 목적

0.15.18에서 정리한 SaaS 운영 IA를 실제 홈 카드와 메뉴 구조에 1차 반영한다.

이번 단계는 화면 진입 구조 정리이며, 결제/증빙/원단 발주 기능 구현은 포함하지 않는다.

## 2. 시스템관리자 홈 매핑

시스템관리자 홈은 다음 3개 묶음으로 분리한다.

```txt
고객사 운영:
- 고객사 관리
- 고객사 구독 관리
- 저장소 관리

요금·결제·증빙:
- 요금제 관리
- 결제수단 관리
- 결제내역
- 결제 실패·미납
- 환불 관리
- 증빙 관리

시스템 운영 기준:
- 시스템 감사로그
- 서비스 문서 관리
- 정산자료 출력
- 기준정보 관리
- 카테고리 규칙
```

아직 전용 route가 없는 메뉴는 기존 `/system/billing`, `/system/standards` 같은 준비 화면으로 연결한다.  
실제 결제 처리, PG 연동, 증빙 발행, 정산 출력은 후속 버전에서 별도 구현한다.

## 3. 고객사관리자 홈 매핑

고객사관리자 홈의 업무 바로가기는 다음 순서로 재배치한다.

```txt
1. 작업지시서 업무 화면
2. 원단/부자재 발주
3. 협력업체 관리
4. 저장소 관리
5. 통계정보
6. 멤버 관리
7. 요금·결제
8. 약관·정책
```

현재 사용 가능한 화면은 기존 route로 연결한다.  
`원단/부자재 발주`, `요금·결제`, `약관·정책`은 준비 중 카드로 표시하고 href는 연결하지 않는다.

## 4. 고객사관리자 환경설정 매핑

환경설정은 다음 카드로 구분한다.

```txt
- 계정 정보
- 기준정보 설정
- 요금제·저장공간
- 약관·정책
- 개발 건의
```

약관·정책은 정식 `/admin/legal` route 구현 전까지 환경설정 안에서 준비 상태와 후속 연결 기준만 안내한다.

## 5. 구현 제외 범위

이번 버전에서 제외한 항목:

```txt
- PG 결제 API 연동
- 카드 등록/변경 flow
- 결제내역 DB/API
- 세금계산서/영수증 발행 flow
- 원단/부자재 발주 DB schema
- 원단/부자재 발주 route 구현
- 시스템관리자 서비스 문서 CRUD
```
