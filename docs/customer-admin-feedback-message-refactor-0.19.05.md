# 0.19.05 고객사 관리자 Error/Feedback 표시 공통화 1차

## 목적

고객사 관리자 화면에서 반복되는 오류/피드백 표시를 공통 컴포넌트로 정리한다. 이번 단계는 동작 흐름을 바꾸지 않고, 멤버관리와 환경설정의 인라인 오류 메시지 표시만 좁게 공통화한다.

## 변경 범위

- `components/admin/common/AdminFeedbackMessage.tsx` 추가
- 멤버관리의 멤버 목록/승인 대기 목록 로드 오류 표시를 공통 컴포넌트로 교체
- 환경설정 저장 실패 오류 표시를 공통 컴포넌트로 교체

## 변경하지 않은 범위

- 멤버 목록 조회 API
- 승인 대기 목록 조회 API
- 멤버 상세/권한 저장 흐름
- 환경설정 저장 로직
- ToastMessage 동작
- DB/R2/첨부/메모/휴지통/purge 흐름

## 테스트 위치

1. `/workspace/members`
2. `/workspace/settings`
3. `/workspace/storage`

## 확인할 것

- 멤버관리 화면이 정상 진입되는지 확인한다.
- 멤버 목록과 승인 대기 목록이 기존처럼 표시되는지 확인한다.
- 환경설정 화면에서 테마/언어 버튼과 저장 상태 배지가 기존처럼 보이는지 확인한다.
- 저장소관리의 토스트 메시지, 삭제/복원/비우기 동작이 영향받지 않았는지 가볍게 확인한다.

## 바뀌면 안 되는 것

- 멤버 목록 데이터 개수
- 멤버 상세/권한 모달 진입
- 멤버 권한 저장
- 환경설정 저장 결과
- 저장소관리 삭제/복원/비우기 흐름
- ToastMessage 표시 방식

## 허용되는 변화

- 오류 메시지 박스의 색상/간격이 관리자 공통 톤으로 소폭 통일된다.
