# /functions 테스트 시나리오 스키마 확장 — 0.23.63

## 목적

0.23.62의 기능 목록을 단순 설명 목록에서 자동 테스트로 연결 가능한 실행 계약으로 확장한다. 사용자 직접 테스트가 어려운 기간에도 정상·예외·권한·DB·회사 격리·반응형·성능·PDF 항목을 동일 구조로 누적할 수 있게 한다.

## 추가된 계약

- 실행 단계와 단계별 기대 결과
- 예외 입력 및 실패 조건
- 권한 조건
- DB before/after snapshot
- 변경 허용 필드와 불변 필드
- 행위 회사와 대상 회사의 tenant isolation 계약
- viewport와 scroll/overflow/drawer/modal 반응형 계약
- 대량 fixture, 측정 항목, 기준선 상태의 성능 계약
- PDF 생성 조건, 검증 항목, 사용자 결정 대기 항목
- 자동화 유형, 파일 경로, 테스트 데이터 세트, 최근 실행 결과
- 정책 결정 상태

## 현재 범위

실제 테스트 실행기나 DB 연결은 추가하지 않았다. `/functions`는 시나리오 계약과 자동화 연결 상태를 표시하는 내부 카탈로그 역할만 수행한다.

## 대표 상세화

- 작업지시서 납기일 단독 변경: 변경 전 row snapshot, PATCH, 변경 후 불변조건 비교
- 일반 멤버 관리자 접근 차단: 메뉴·URL·API 권한 조건
- 회사 간 작업지시서 조회 차단: 회사 A/B tenant 계약
- 기기별 workspace: PC·iPad Mini·Galaxy Tab·모바일 viewport 및 구조 검증
- 작업지시서 PDF: 생성 조건과 금액·문서 분리·로고·서명 결정 대기

## 후속 작업

0.23.64에서 10개 테스트 회사와 역할·요금제·상태·데이터 규모를 가진 fixture/seed 기반을 연결한다. 실제 DB seed 실행과 destructive reset은 사용자 테스트 가능 시점까지 보류한다.
