# 작업지시서 저위험 기능 수정 1차 계획 — 0.19.57

## 목적

0.19.56까지 UI 공통화와 기능 추가 전 QA 기준 정리가 끝난 상태에서, 실제 기능 수정을 시작하기 전 작업지시서 화면의 저위험 기능 수정 후보를 선별한다.

이번 버전은 기능 코드를 직접 변경하지 않는다.  
이유는 아직 구체적인 기능 수정 요청 목록과 QA 결과가 확정되지 않았기 때문이다.

## 현재 판단

작업지시서 화면은 다음 흐름이 서로 연결되어 있어 기능 추가 시 회귀 위험이 높다.

- 작업지시서 목록 선택
- 상세 패널 표시
- 상태 변경
- PDF 생성
- 첨부 / 디자인 / 메모
- R2 업로드 / 삭제 / 복원
- 권한별 버튼 노출
- 모바일 / 태블릿 패널 전환

따라서 다음 기능 수정은 반드시 “표시/UI 보정”과 “데이터/상태 로직 변경”을 분리해서 진행한다.

## 0.19.58부터 우선 가능한 저위험 수정 후보

### 1. 버튼/문구/안내 보정

위험도: 낮음

가능 범위:

- 작업지시서 목록 카드 문구 조정
- 상세 패널 안내 문구 조정
- 빈 상태/로딩 상태 문구 조정
- 권한 없음 안내 문구 조정
- PDF 생성 버튼 설명 문구 조정
- 첨부/디자인/메모 패널 도움말 축소

금지 범위:

- 상태 변경 조건 수정
- 권한 판정 함수 수정
- DB 저장값 변경
- PDF 생성 route 수정

### 2. 표시 전용 badge / count 보정

위험도: 낮음~중간

가능 범위:

- 첨부/디자인/메모 개수 표시 위치 조정
- 상태 badge 문구 정리
- 담당자/회사/카테고리 표시 순서 조정
- 목록 카드의 secondary 정보 축소

주의:

- count 계산이 서버 응답값에 의존하면 repository/API 변경 없이 화면 표시만 수정한다.
- count가 DB join 기준이면 별도 중위험 작업으로 분리한다.

### 3. 목록 정렬/필터 UI 보정

위험도: 중간

가능 범위:

- 정렬 버튼 위치
- 검색 input 안내 문구
- 필터 영역 responsive 정리
- 현재 선택된 필터 badge 표시

금지 범위:

- 쿼리 조건 변경
- 담당자 scope 변경
- 회사 scope 변경
- 권한별 목록 노출 조건 변경

### 4. 첨부/디자인/메모 패널 UI 보정

위험도: 중간

가능 범위:

- action button 정렬
- empty state 문구
- 업로드 안내 문구
- 미리보기 카드 간격
- 메모 item 간격

주의:

- 업로드 완료 API
- R2 key 생성
- 삭제/복원/영구삭제
- 대표 디자인 지정
- 메모 저장/삭제 API  
위 항목은 건드리지 않는다.

### 5. PDF 관련 문구/표시 보정

위험도: 중간~높음

가능 범위:

- PDF 생성 버튼 label
- PDF 생성 전 안내 문구
- PDF 생성 완료 toast 문구
- PDF 파일명 preview 문구

주의:

- 실제 PDF route
- PDF HTML template
- R2 저장 경로
- 첨부 자동 등록  
위 항목은 별도 버전에서 다룬다.

## 0.19.58 작업 추천

다음 버전은 실제 기능 변경보다 안전한 “작업지시서 표시/문구 정리 1차”가 적절하다.

권장 범위:

- 작업지시서 상세/우측 패널 도움말 축소
- 빈 상태/로딩 상태 문구 정리
- 버튼 label 및 title 정리
- 기능 로직 변경 없음

## 0.19.59 이후 권장 순서

1. 작업지시서 표시/문구 정리 1차
2. 작업지시서 권한별 버튼 노출 QA 보정
3. 작업지시서 첨부/디자인/메모 패널 UI 보정
4. 작업지시서 PDF 관련 문구/표시 보정
5. 원단·부자재 발주 표시/문구 정리
6. 원단·부자재 발주 상태/납기/PDF 기능 보정
7. 멤버/환경설정/시스템관리자 기능 개발

## 테스트 기준

이번 버전은 기능 코드 변경이 없으므로 다음만 확인한다.

- 앱 버전이 0.19.57로 표시되는지 확인
- 문서가 추가되었는지 확인
- 기존 작업지시서 기능 동작이 변하지 않는지 확인
