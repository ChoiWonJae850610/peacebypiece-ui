# 0.9.22376 시스템관리자 실제 삭제 후보 pending 노출 보정

## 목적

고객관리자가 아직 영구삭제 요청을 하지 않은 작업지시서가 시스템관리자 실제 삭제 후보 목록에 보이는 문제를 보정한다.

## 정책

- 고객관리자 휴지통의 일반 삭제 항목은 복구 가능한 상태다.
- 복구 가능한 pending 작업지시서는 시스템관리자 실제 삭제 후보 목록에 노출하지 않는다.
- 시스템관리자 실제 삭제 후보에는 아래 항목만 표시한다.
  - 고객관리자가 영구삭제 요청한 `purge_requested` 작업지시서
  - 보관 기간이 도래한 `pending` 작업지시서
  - 삭제 실패 후 재시도가 필요한 `failed` 작업지시서
- 파일 후보도 기존처럼 `purge_requested`, 보관 기간 도래, 실패 항목만 표시한다.

## 확인 항목

1. `/worker`에서 작업지시서를 일반 삭제한다.
2. `/admin/files` 휴지통에는 복구 가능한 항목으로 표시되는지 확인한다.
3. 고객관리자가 영구삭제 요청을 누르기 전에는 `/system/storage-usage` 실제 삭제 후보에 작업지시서가 표시되지 않아야 한다.
4. 고객관리자가 영구삭제 요청을 누른 뒤에는 `/system/storage-usage` 실제 삭제 후보에 작업지시서가 표시되어야 한다.
5. 보관 기간이 도래한 pending 항목은 시스템관리자 실제 삭제 후보에 표시될 수 있다.
