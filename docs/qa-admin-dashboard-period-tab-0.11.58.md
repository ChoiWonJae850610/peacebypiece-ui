# 0.11.58 통계 기간분석 탭 동작 분리

## 목적

관리자 통계 화면에서 기간분석 탭의 기간 필터가 생산 구성 탭의 누적 데이터와 탭 위치를 흔들지 않도록 보정한다.

## 반영 내용

- `/admin/dashboard`의 `section` query parameter를 해석해 초기 통계 탭을 결정한다.
- 기간분석 탭의 7일, 30일, custom 적용, 초기화 링크에 `section=period`를 유지한다.
- 기간 필터 조작 뒤 첫 번째 탭인 생산 구성으로 돌아가지 않고 기간분석 탭을 유지한다.
- 새로고침 모양의 초기화 버튼을 텍스트 `초기화` 버튼으로 변경했다.
- 생산 구성/업체 성과용 분포 쿼리에서는 기간 필터를 제거해 누적 기준으로 유지했다.
- 기간별 완료/리오더/불량 Top5와 기간 요약은 기존대로 선택 기간 기준을 유지한다.

## 변경하지 않은 것

- 통계 API route 구조 변경 없음
- Recharts 데이터 구조 변경 없음
- 저장소/작업지시서/R2 흐름 변경 없음
- DB schema 변경 없음

## 확인 항목

1. `/admin/dashboard` 진입 후 생산 구성 탭이 기본 표시되는지 확인한다.
2. 기간 분석 탭으로 이동한다.
3. 7일, 30일, custom 적용, 초기화 버튼을 누른 뒤 기간 분석 탭이 유지되는지 확인한다.
4. 생산 구성 탭의 도너츠/품목 데이터가 기간 필터 변경 전후 동일하게 유지되는지 확인한다.
5. 기간 분석 탭의 Top5와 기간 요약은 선택 기간에 따라 바뀌는지 확인한다.
