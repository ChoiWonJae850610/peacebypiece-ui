# 0.19.66 멤버 생애주기 기능 설계 1차

## 목적

멤버관리에서 앞으로 추가할 탈퇴 요청, 비활성, 탈퇴 처리, 승인/거절 흐름을 구현하기 전에 화면·상태·데이터 영향 범위를 먼저 고정한다. 이번 버전에서는 실제 DB/API 기능을 추가하지 않고, 테스트 불가 상태에서 안전하게 진행 가능한 설계 문서와 멤버관리 요약 카드 표시 보정만 수행한다.

## 설계 대상

- 일반 멤버의 탈퇴 요청
- 고객사 관리자의 멤버 비활성 처리
- 고객사 관리자의 멤버 탈퇴 처리
- 탈퇴 요청 승인/거절
- 멤버 상태값과 담당자 후보 제외 기준
- 기존 작업지시서, 메모, 첨부 작성자 표시 이력 보존 방식
- DB schema와 full_reset.sql 변경 필요 여부

## 권장 상태 모델

| 상태 | 의미 | 업무 접근 | 담당자 후보 | 멤버관리 표시 |
| --- | --- | --- | --- | --- |
| pending | 승인 대기 | 불가 | 제외 | 승인 대기 목록 |
| approved | 사용 중 | 가능 | 포함 | 멤버 목록 |
| suspended | 비활성 | 불가 | 제외 | 멤버 목록 |
| withdrawal_requested | 탈퇴 요청 | 제한 또는 읽기 전용 | 제외 | 탈퇴 요청 목록 |
| withdrawn | 탈퇴 완료 | 불가 | 제외 | 이력성 표시 또는 숨김 |
| rejected | 승인 거절 | 불가 | 제외 | 승인 이력 |

## 일반 멤버 탈퇴 요청 흐름

1. 일반 멤버가 개인 프로필에서 탈퇴 요청을 누른다.
2. 확인 모달에서 탈퇴 요청 사유를 선택하거나 입력한다.
3. 상태는 `withdrawal_requested`로 변경한다.
4. 고객사 관리자는 멤버관리에서 탈퇴 요청을 확인한다.
5. 고객사 관리자는 승인 또는 거절한다.
6. 승인 시 상태는 `withdrawn`으로 변경한다.
7. 거절 시 상태는 기존 `approved` 또는 `suspended`로 복귀한다.

## 고객사 관리자 직접 처리 흐름

고객사 관리자는 멤버 상세에서 다음 작업을 수행할 수 있다.

- 비활성 처리: 상태를 `suspended`로 변경한다.
- 사용 재개: 상태를 `approved`로 변경한다.
- 탈퇴 처리: 상태를 `withdrawn`으로 변경한다.

직접 탈퇴 처리는 위험 작업으로 취급하고 확인 모달을 사용한다. 실제 삭제 대신 상태 변경을 기본으로 한다.

## 이력 보존 원칙

- 작업지시서 담당자 이력은 삭제하지 않는다.
- 메모 작성자명, 첨부 업로드 작성자명은 삭제하지 않는다.
- 탈퇴 완료 후에도 과거 업무 이력에는 당시 표시명을 유지한다.
- 개인정보 삭제가 필요한 경우 별도 익명화 정책을 설계한다.

## 담당자 후보 제외 기준

담당자 변경 후보에는 다음 멤버만 포함한다.

- 같은 회사 소속
- 승인 완료 상태 `approved`
- system_admin 제외
- pending, suspended, withdrawal_requested, withdrawn, rejected 제외

## DB/API 구현 시 검토 항목

- 기존 `members.status`에 신규 상태를 추가할지
- 탈퇴 요청 전용 테이블을 둘지
- 요청 사유, 요청일, 처리자, 처리일, 처리 결과를 별도 저장할지
- full_reset.sql seed 데이터와 smoke test에 신규 상태를 반영할지
- 기존 권한 판정 함수가 신규 상태를 안전하게 차단하는지

## UI 구현 시 검토 항목

- 개인 프로필에 탈퇴 요청 버튼 추가
- 멤버관리 목록에 탈퇴 요청 상태 표시
- 멤버 상세 모달에 비활성/사용 재개/탈퇴 처리 액션 추가
- 위험 작업은 공통 확인 모달 사용
- 고객사 관리자는 기존 정책대로 멤버관리 목록에 표시하지 않음

## 이번 버전에서 실제 반영한 표시 보정

- 멤버관리 hero의 역할/권한 배지를 제거했다.
- 멤버관리 요약 카드의 ready/pending/planned 상태 배지를 제거했다.
- 요약 카드에는 숫자와 설명만 남겨 실제 서비스 화면처럼 보이도록 정리했다.

## 이번 버전에서 변경하지 않은 것

- DB schema 변경 없음
- API 변경 없음
- 권한 판정 로직 변경 없음
- 멤버 상태 변경 기능 추가 없음
- 탈퇴 요청 기능 추가 없음
- 기존 초대/승인/멤버 수정 흐름 변경 없음
