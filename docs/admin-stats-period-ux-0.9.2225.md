# 0.9.2225 — 통계 직접 기간 선택 UX 정리

## 목적

고객관리자 통계정보 화면에서 직접 기간 선택 UI의 적용 조건을 명확히 하고, 생산품 유형 도넛 차트의 빈 데이터 문구가 중앙 총합 문구와 겹쳐 보이는 문제를 수정한다.

## 변경 내용

- 직접 기간 선택은 시작일과 종료일이 모두 있고 시작일이 종료일보다 늦지 않을 때만 적용 가능하다.
- 직접 선택 조건이 부족하거나 잘못된 경우 적용 버튼을 비활성 상태로 유지한다.
- 현재 기간 안내 문구를 작업흐름/리오더/생산품유형/업체성과에 적용되는 범위로 정리했다.
- 도넛 차트 데이터가 0일 때 중앙 총합 텍스트를 숨기고 빈 데이터 문구만 표시한다.

## 확인 기준

- `/admin/dashboard`에서 최근 7일/최근 30일 버튼은 기존처럼 동작한다.
- 직접 선택 시작일/종료일이 모두 입력되어야 적용 버튼이 활성화된다.
- 시작일이 종료일보다 늦으면 적용 버튼이 비활성화되고 안내 문구가 표시된다.
- 생산품 유형 데이터가 0인 경우 도넛 중앙에 문구가 하나만 보인다.

## 변경하지 않은 항목

- DB schema 변경 없음
- API route 변경 없음
- package.json / package-lock.json 변경 없음
