# WAFL Resource State Patch — 0.23.40

## 목적
작업지시서·발주서 등 WAFL 리소스가 부분 PATCH 응답을 전체 객체로 교체하지 않도록 공통 상태 병합 규칙을 정의한다.

## 공통 규칙
- 부분 PATCH는 기존 완전한 리소스 객체 위에 요청 필드만 병합한다.
- 리소스 ID는 변경할 수 없다.
- 상세 비동기 응답은 현재 기본정보를 덮어쓰지 않는다.
- 로컬 상태와 persisted 상태는 동일한 patch 결과를 사용한다.
- 전체 객체 교체는 생성·전체 재조회처럼 완전한 스냅샷이 보장되는 경우에만 허용한다.

## 0.23.40 적용 범위
- WAFL 공통 `applyWaflResourcePatch` 추가
- 작업지시서 기본정보 저장 성공 상태에 공통 patch 병합 적용
- 담당자 저장 성공 상태에 공통 patch 병합 적용
- 상세 스냅샷 병합 시 즉시 저장 기본정보 보존
- persistedWorkOrders의 전체 객체 교체 경로 축소

## 후속 범위
- 재고·리오더 그룹 저장을 patch 결과 배열로 전환
- 발주서 헤더·상태·품목·할당에 동일 계약 적용
- 프로젝트 전체의 전체 객체 교체 경로 감사
