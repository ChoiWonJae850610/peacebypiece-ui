# 환경설정 기준정보 생산품 유형 계층형 보정 0.20.21.1

## 배경

0.20.21에서 생산품 유형을 단순 목록형으로 정리했으나, 실제 작업지시서에서 사용하는 생산품 유형은 단순 기준값이 아니라 1차·2차·3차 계층형 분류 체계다.

예시:

- 1차: 상의
- 2차: 티셔츠
- 3차: 반팔 티셔츠 / 긴팔 티셔츠

따라서 생산품 유형은 단순 추가 목록이 아니라 폴더형/트리형 구조로 관리되어야 한다.

## 적용 기준

### 생산품 유형

- 고객사 관리자가 직접 관리한다.
- 1차·2차·3차 계층으로 구분한다.
- PC에서는 3열 구조로 표시한다.
- 1차 선택 시 해당 2차 목록을 표시한다.
- 2차 선택 시 해당 3차 목록을 표시한다.
- 각 단계에서 새 항목을 추가할 수 있다.
- 각 항목은 사용/미사용을 전환할 수 있다.
- 상위 항목을 미사용 처리하면 하위 항목도 함께 미사용 처리한다.

### 단위 표준

- 고객사가 직접 추가하지 않는다.
- 현재 사용 가능한 목록만 표시한다.
- 필요한 항목은 유형 추가 요청으로 접수한다.

### 외주공정 유형

- 고객사가 직접 추가하지 않는다.
- 현재 사용 가능한 목록만 표시한다.
- 필요한 항목은 유형 추가 요청으로 접수한다.

## 제거/축소한 요소

- 생산품 유형 단순 목록형 추가 구조
- 코드명 표시
- 순서 표시
- 단계 번호 표시 중심 UI
- 과한 상태 뱃지
- 선택 항목 관리 모달

## 후속 검토

- 생산품 유형 이름 수정 UI는 별도 후속 버전에서 검토한다.
- 생산품 유형 삭제는 데이터 참조 가능성이 있으므로 사용/미사용 중심으로 유지한다.
- 모바일에서는 3열 구조가 좁을 수 있으므로 이후 드릴다운 또는 세로 스택 UX를 별도로 점검한다.
