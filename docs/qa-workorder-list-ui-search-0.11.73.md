# 0.11.73 작업지시서 목록 필터/정렬 UI 밀도와 검색 연동 보정

## 목적

0.11.70~0.11.72에서 작업지시서 목록의 상태 필터와 정렬 query를 추가한 뒤, 실제 사용 화면에서 목록 상단 컨트롤이 과하게 높아지지 않도록 밀도를 정리하고 검색어도 URL query와 연동한다.

## 반영 내용

- `/worker?q=...` query를 읽어 초기 검색어로 사용한다.
- 작업지시서 검색어 변경 시 현재 URL의 `q` query를 갱신한다.
- 검색어가 비어 있으면 `q` query를 제거한다.
- 상태 필터 또는 정렬 변경 시 기존 검색어 query를 유지한다.
- 검색어 변경 시 특정 작업지시서 직접 진입용 `workOrderId` query는 제거한다.
- PC 좌측 목록의 검색/필터/정렬 컨트롤 높이를 줄였다.
- 검색어가 있을 때만 `지우기/Clear` 버튼을 표시한다.
- 현재 필터, 정렬, 표시 건수를 요약해 좌측 목록에 표시한다.
- 모바일 drawer에도 상태 필터와 정렬 select를 추가했다.

## 유지한 정책

- 상태 필터는 서버 query 기준으로 동작한다.
- 검색은 현재 조회된 목록 안에서 client-side로 동작한다.
- 기본 상태 필터는 `진행 중(active)`이다.
- 완료건 기본 제외 정책은 유지한다.
- 작업지시서 상세 hydrate, 첨부, 메모, R2 흐름은 변경하지 않는다.

## 확인 항목

- `/worker?q=셔츠` 직접 진입 시 검색어가 입력칸에 유지되는지 확인한다.
- 검색어 입력 후 새로고침해도 검색어가 유지되는지 확인한다.
- 검색어 입력 후 상태 필터를 바꿔도 검색어가 유지되는지 확인한다.
- 검색어 입력 후 정렬을 바꿔도 검색어가 유지되는지 확인한다.
- `지우기` 버튼 클릭 시 검색어와 URL `q` query가 제거되는지 확인한다.
- 모바일 drawer에서 상태 필터와 정렬이 표시되는지 확인한다.
