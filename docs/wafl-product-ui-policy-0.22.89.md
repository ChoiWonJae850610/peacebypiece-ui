# WAFL 제품 UI 구조 고정 정책 (0.22.89)

## 목적

작업지시서와 발주서는 서로 직접 결합하지 않고 동일한 WAFL 제품 UI 계층에만 의존한다. 공통 외관과 반응형 동작은 WAFL 컴포넌트 및 공통 workspace 정책에서 수정하며, 공정·자재·원가 같은 업무 로직은 각 도메인에 유지한다.

## 공식 규칙

- 제품 UI 컴포넌트 이름은 `Wafl` 접두사를 사용한다.
- 작업지시서와 발주서의 화면 모드는 `useWorkspaceLayoutMode` 결과만 사용한다.
- PC·태블릿 가로·모바일 표현은 상위 workspace가 명시적으로 하위 컴포넌트에 전달한다.
- 하위 상세·우측 패널은 viewport를 다시 판정하지 않는다.
- 패널 외곽선, 패딩, 스크롤, 빈 상태, 로딩, 저장 상태는 WAFL 공통 컴포넌트가 소유한다.
- 업무 화면은 콘텐츠 배치와 도메인 동작만 소유한다.
- 일반 HTML 입력은 캔버스 편집기나 접근성 오버레이처럼 WAFL 필드로 대체할 수 없는 경우에만 허용한다.

## 반응형 기준

기기명이나 user agent가 아니라 viewport 너비·높이·방향·짧은 변을 사용한다.

- `drawer`: 모바일, 태블릿 세로, 가로 휴대폰과 좁은 폴더블
- `tabletTwoPanel`: 충분한 짧은 변을 가진 일반 태블릿 가로
- `threePanel`: 넓은 태블릿 가로와 PC

각 breakpoint는 바로 아래·정확한 값·바로 위에서 회귀 확인한다.

## 자동 감사

`npm run audit:wafl-ui`

감사 항목:

- 레거시 App UI 컴포넌트 식별자 금지
- 레거시 작업지시서 기기 판정 훅 금지
- 작업지시서·발주서의 native form control 사용처 분류

현재 허용된 native control은 접근성 오버레이와 드로잉 캔버스 편집기뿐이다.
