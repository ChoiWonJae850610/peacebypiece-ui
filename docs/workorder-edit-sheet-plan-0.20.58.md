# 0.20.58 작업지시서 공정/구성 편집 시트 전환 검토

## 목적

작업지시서의 공정 탭과 구성 탭은 현재 입력 필드가 본문 카드 안에 계속 펼쳐져 있어 모바일·태블릿에서 카드 중첩이 깊고 화면 밀도가 높다. 다음 단계에서는 본문을 목록 카드 중심으로 단순화하고, 실제 입력은 별도 편집 시트/모달로 분리하는 방향이 적합하다.

## 기본 원칙

- 본문 탭은 현재 등록된 항목을 읽고 비교하는 영역으로 둔다.
- 항목 추가와 수정은 별도 편집 UI에서 처리한다.
- 모바일은 bottom sheet 중심으로 처리한다.
- 태블릿은 centered modal 또는 side sheet를 우선 검토한다.
- PC는 작은 centered modal 또는 오른쪽 편집 drawer를 검토한다.
- 비용 계산, 권한, 상태전환, API, DB, R2 흐름은 분리 작업 중 변경하지 않는다.

## 구성 탭 제안

### 본문 표시

- 제품 구성 제목과 필요 자재 수만 표시한다.
- 자재 항목은 카드 목록으로 표시한다.
- 카드에는 구분, 항목명, 수량, 단위만 표시한다.
- 각 카드에는 수정, 삭제 액션만 둔다.
- 구분 옵션은 원단, 부자재만 유지한다.

### 편집 시트

- 구분
- 항목명
- 수량
- 단위
- 적용 버튼

## 공정 탭 제안

### 본문 표시

- 제작 공정과 추가 공정을 카드 목록으로 표시한다.
- 카드에는 구분, 항목, 업체, 수량, 단가, 로스비만 표시한다.
- 금액 요약은 공정 탭에 표시하지 않고 요약 탭 비용 요약 카드로 단일화한다.

### 편집 시트

- 구분
- 항목
- 업체
- 수량
- 단가
- 로스비
- 적용 버튼

## 단계별 적용안

### 0.20.59 후보

- 구성 탭 자재 입력을 편집 시트 구조로 전환한다.
- 본문에는 자재 카드 목록만 남긴다.
- 기존 저장/삭제 로직은 그대로 연결한다.

### 0.20.60 후보

- 공정 탭 입력을 편집 시트 구조로 전환한다.
- 본문에는 공정 카드 목록만 남긴다.
- 비용 요약은 요약 탭에만 유지한다.

### 0.20.61 후보

- PC/태블릿/모바일 밀도와 문구를 재점검한다.
- 공정/구성 탭의 빈 상태와 추가 버튼을 공통화한다.

## 보류 사항

- DB 모델링 변경은 이번 UI 전환 이후 별도 검토한다.
- 원단·부자재 발주 권한 UI는 작업지시서 공정/구성 정리 후 진행한다.
