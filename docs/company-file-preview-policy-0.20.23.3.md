# 회사 파일 미리보기/업로드 정책 UI 1차 (0.20.23.3)

## 목적

대표 이미지와 사업자등록증 업로드 화면을 작업지시서 첨부 파일 흐름과 같은 운영 기준으로 정리한다.

## 적용 범위

- 회사 파일 카드 표시 방식
- 허용 파일 유형/용량 안내
- 업로드 전 파일 유형 1차 검증
- 회사 파일 미리보기 API
- 이미지/PDF 미리보기 모달

## 정책

### 대표 이미지

- 허용 유형: JPG, PNG, WEBP
- 최대 용량: 5MB
- 화면 기본 표시: 실제 이미지 미리보기
- 파일명은 기본 카드 전면에 표시하지 않고 미리보기 모달의 상세 정보로만 표시한다.

### 사업자등록증

- 허용 유형: JPG, PNG, WEBP, PDF
- 최대 용량: 10MB
- 화면 기본 표시: 파일 종류/용량/등록일 요약
- PDF 또는 이미지 파일은 미리보기 모달에서 확인한다.

## 구현 메모

- 작업지시서 첨부의 파일 접근 방식처럼 R2 key를 서버 API에서 회사 범위로 검증한 뒤 Worker signed file URL 또는 R2 SDK fallback으로 반환한다.
- `app/api/admin/company-files/file`은 로그인 회사와 파일 key의 companyId가 일치할 때만 파일을 반환한다.
- 업로드 정책은 `companyFileUploadPolicy.ts`로 분리하여 클라이언트 표시와 서버 검증이 같은 기준을 참조할 수 있게 했다.

## 제외 범위

- DB schema 변경 없음
- R2 업로드/Worker 업로드 정책 변경 없음
- 작업지시서 첨부/메모/휴지통/purge 흐름 변경 없음
