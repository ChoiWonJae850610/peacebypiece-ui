## 문서 정리 판단

### docs 루트

0.19.94.9 기준 docs 루트에는 README를 제외하고 44개 문서가 남아 있다. 이 문서들은 최근 운영·정책·DB/API smoke·Playwright·회사 파일 업로드 관련 활성 문서이므로 추가 이동하지 않는다.

### 보관문서

`docs/보관문서/`는 과거 설계·점검·테스트·DB 기록을 유지하는 보관 영역으로 확정한다. 삭제 영역이 아니며, 현재 기능 작업에서 직접 기준으로 사용할 때는 `docs/현재기준/` 문서와 충돌 여부를 먼저 확인한다.

### 현재기준

`docs/현재기준/`은 기능 작업 전 우선 확인해야 하는 기준 문서 영역으로 확정한다. DB/API, 권한, 라우팅, 작업지시서 상태, 원단·부자재 발주, 요금·저장소 정책처럼 기능 구현에 영향을 주는 기준 문서를 둔다.

## 자동테스트 복귀 기준

사용자가 테스트 가능 상태를 명시했으므로 다음 기능 작업부터 자동테스트를 기본 확인 기준으로 삼는다.

- DB/API 또는 SQL 변경: `npm run test:smoke:db-api`
- 화면/라우팅/권한/모달 변경: `npm run test:e2e`
- 큰 기능 묶음 또는 릴리스 후보: `npm run build`, `npm run test:smoke:db-api`, `npm run test:e2e`

ChatGPT/container에서는 `npm run build`를 실행하지 않는다. 빌드와 실제 자동테스트 실행은 사용자가 로컬 환경에서 확인한다.
