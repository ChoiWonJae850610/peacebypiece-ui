# 0.23.28 발주서 변경 피드백과 훅 책임 분리

## 목적

발주서 기본정보와 상태 변경의 진행·성공·실패 문구를 작업지시서와 같은 공통 변경 피드백 정책으로 통일하고, 대형 `useMaterialOrderDraftEditor` 훅에서 토스트 표현 책임을 분리한다.

## 적용 내용

- 자재 종류, 공급처, 납기일, 상태 변경에 `WAFL_CHANGE_TARGET`을 직접 사용한다.
- 진행 문구는 변경 대상을 명시한다.
- 성공 문구는 `정보가 변경되었습니다.`로 통일한다.
- 실패 fallback은 `정보를 변경하지 못했습니다.`로 통일한다.
- 발주서 전용 `useMaterialOrderFeedback` 훅으로 operation toast와 상태별 tone 결정을 분리한다.
- 생성·삭제는 변경 피드백과 성격이 다르므로 기존 명시적 문구를 유지한다.

## 영향 범위

API, DB, 상태 전환 정책, 자재 계산 로직은 변경하지 않는다.
