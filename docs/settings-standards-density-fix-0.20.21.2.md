# 0.20.21.2 기준정보 계층형 UI 오류 및 밀도 보정

## 목적

0.20.21.1에서 생산품 유형을 1차·2차·3차 계층형으로 보정한 뒤 발견된 HTML 구조 오류와 기준정보 화면 밀도 문제를 정리한다.

## 보정 내용

### 1. 생산품 유형 항목 구조

- 생산품 유형 행의 전체 영역을 `button`으로 감싸지 않는다.
- 항목 선택 버튼과 사용/미사용 버튼을 형제 요소로 분리한다.
- `button` 안에 `button`이 들어가는 HTML 구조를 제거해 hydration 오류를 방지한다.

### 2. 선택 상태 표현

- 선택된 생산품 유형을 검은색 반전 배경으로 표시하지 않는다.
- WAFL 테마 토큰 기반의 선택 배경과 좌측 accent line으로 표시한다.
- 사용/미사용 토글도 hardcoded black 계열 대신 WAFL action/semantic token을 사용한다.

### 3. 단위 표준 / 외주공정 유형 화면 밀도

- 단위 표준과 외주공정 유형은 직접 생성·수정 대상이 아니라 요청 기반 운영 대상이다.
- 빈 공간이 넓게 보이지 않도록 현재 목록 영역과 유형 추가 요청 패널을 좌우로 분리한다.
- 목록에는 이름, 사용처, 사용 여부를 표시한다.
- 영문명은 아직 운영 필드로 확정하지 않고 이번 보정에서는 추가하지 않는다.

## 후속 기준

- 생산품 유형은 고객사가 직접 관리하는 1차·2차·3차 계층형 기준정보로 유지한다.
- 단위 표준과 외주공정 유형은 시스템관리자 검토 기반의 추가 요청 구조로 유지한다.
- 향후 영문명/약어 필드가 실제 운영에 필요해질 경우 DB/API 설계와 함께 별도 버전에서 추가한다.
