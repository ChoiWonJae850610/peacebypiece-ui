# 0.11.76 작업지시서 생산구성 종속 선택값 초기화

## 목적

작업지시서 생산구성에서 상위 선택값을 변경했을 때 기존 하위 거래처/외주처 선택값이 그대로 남아 잘못 저장될 수 있는 문제를 방지한다.

## 반영 내용

- 원단/부자재 `구분(type)` 변경 시 기존 `vendor`를 공백으로 초기화한다.
- 원단/부자재 `구분(type)` 변경 시 기존 `vendorRef`를 `null`로 초기화한다.
- 외주공정 `process` 변경 시 기존 `vendor`를 공백으로 초기화한다.
- 같은 값으로 다시 저장하는 경우에는 기존 값을 불필요하게 초기화하지 않는다.

## 변경하지 않은 항목

- 저장 API 구조 변경 없음
- DB schema 변경 없음
- R2/첨부/메모 흐름 변경 없음
- 기본 단위 자동 설정은 다음 버전에서 별도 처리
- `lib/runtimeMode.ts` 플래그 정리는 0.11.79에서 별도 조사/정리 예정

## 확인 항목

1. 작업지시서 상세 화면에서 생산구성 영역을 연다.
2. 원단/부자재 행에서 거래처를 선택한다.
3. 구분을 원단에서 부자재 또는 부자재에서 원단으로 바꾼다.
4. 거래처 필드가 공백으로 초기화되는지 확인한다.
5. 외주공정 행에서 외주처를 선택한다.
6. 공정을 다른 공정으로 바꾼다.
7. 외주처 필드가 공백으로 초기화되는지 확인한다.
