# 0.19.92 공통 이동 피드백·로그아웃 확인 UX 1차

## 목적

업무 화면에서 홈, 환경설정, 좌측 메뉴 이동 시 즉시 반응이 보이도록 WAFL 로딩 토스트를 연결하고, 로그아웃은 즉시 실행하지 않고 확인 모달을 거치도록 보정한다.

## 적용 범위

- 고객사 관리자 상단바 홈 버튼
- 고객사 관리자 상단바 환경설정 버튼
- 고객사 관리자 좌측 메뉴 링크
- 일반 멤버 상단바 홈 버튼 기존 토스트 유지
- 고객사 관리자/일반 멤버 공통 로그아웃 확인 모달
- Playwright 정책/환경설정 E2E에 로그아웃 확인 모달 검증 추가

## UX 기준

### 확인 모달이 필요한 항목

- 로그아웃

로그아웃은 세션 종료 동작이므로 확인 모달을 표시한다. 모달에서 취소하면 현재 화면에 머무르고, 확인하면 `/api/auth/logout` POST 요청을 보낸다.

### 확인 모달이 필요하지 않은 항목

- 홈 이동
- 환경설정 이동
- 좌측 메뉴 이동

단순 화면 이동은 확인 모달 없이 로딩 토스트만 표시한다. 미저장 입력값 보호는 화면별 dirty-state 설계가 필요하므로 이번 버전에서는 전역 차단을 넣지 않는다.

## 테스트

```bash
npm run test:e2e
```

기대 기준:

- 공개 로그인 화면 smoke 통과
- 정책/환경설정 화면 smoke 통과
- 환경설정 화면 상단 로그아웃 버튼 클릭 시 확인 모달 표시

## 제외 범위

- 전역 미저장 변경 감지
- 새로고침 버튼별 dirty-state 차단
- 작업지시서 생성 Enter submit
- 개인 프로필 언어 전환 UI 숨김
