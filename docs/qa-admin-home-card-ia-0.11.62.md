# 0.11.62 고객관리자 홈 운영 카드 IA 재정리

## 목적

0.11.61에서 운영 관리 카드의 시각 밀도를 높였지만 PC 화면에서는 카드가 과도하게 커졌다. 이번 버전은 카드 크기를 단순 축소하는 대신 운영 진입 구조를 정리한다.

## 변경 기준

- 작업지시서 업무 화면은 운영 관리 카드에서 제거한다.
- 작업지시서 업무 화면 진입은 검토·발주 대기 영역의 아이콘 버튼으로 제공한다.
- 환경설정 카드는 운영 관리 카드에서 제거한다.
- 관리자 화면 우측 상단 톱니바퀴 버튼을 `/admin/settings`로 연결한다.
- 운영 관리 카드는 협력업체 관리, 저장소 관리, 통계정보, 멤버 관리 4개만 유지한다.
- PC에서는 4개 카드가 한 줄에 배치되도록 한다.
- 태블릿은 2열, 모바일은 1열 구조를 유지한다.

## 변경 파일

- `components/admin/dashboard/AdminConsoleSections.tsx`
- `components/admin/dashboard/AdminOperationsDashboard.tsx`
- `components/admin/layout/AdminTopbar.tsx`
- `lib/i18n/ko/admin.ts`
- `lib/i18n/en/admin.ts`
- `lib/constants/app.ts`

## 회귀 확인

1. `/admin` 진입
2. 검토·발주 대기 제목 옆 아이콘 버튼 클릭 시 `/worker` 이동 확인
3. 우측 상단 톱니바퀴 클릭 시 `/admin/settings` 이동 확인
4. 운영 관리 카드가 4개만 표시되는지 확인
   - 협력업체 관리
   - 저장소 관리
   - 통계정보
   - 멤버 관리
5. PC에서 4개 카드가 한 줄에 들어오는지 확인
6. 모바일에서 카드가 1열로 자연스럽게 쌓이는지 확인

## 변경하지 않은 항목

- 작업지시서 목록/상세 로직
- 고객관리자 권한 판정 로직
- 카드별 링크 목적지
- 시스템관리자 화면
- DB schema
