# 관리자 통계 화면 1차 안정 체크포인트 — 0.9.22434

## 목적

0.9.22429 이후 통계 화면 구조 재배치, 타입/SQL 보정, 레이아웃 보정, TOP5 기준 보정, i18n 정리를 거친 뒤 `/admin/dashboard`를 1차 안정 상태로 고정한다.

## 이번 보정

- 생산품 유형 비율과 선택항목 품목 TOP5에서 쓰는 비율 막대 계산 함수를 컴포넌트 외부 공통 함수로 분리했다.
- 선택 카테고리와 기간별 TOP5 리스트 계산을 `useMemo` 기반으로 정리했다.
- `useEffect` 의존성에 배열 map/join 표현식을 직접 넣지 않도록 보정했다.
- 선택 카테고리 상태가 카테고리 깊이 변경 후에도 현재 표시 가능한 항목 기준으로 안정적으로 보정되도록 유지했다.

## 확인 기준

- `/admin/dashboard` 첫 진입 시 런타임 오류가 없어야 한다.
- 7일 / 30일 기간 변경이 정상 동작해야 한다.
- 직접 기간 선택 시 날짜 키보드 입력 없이 DayPicker popover로만 선택되어야 한다.
- 기간 요약 카드 클릭 시 완료 작업지시서 / 리오더 / 불량 작업지시서 TOP5가 전환되어야 한다.
- 생산품 유형 비율에서 대분류/품목 전환과 선택항목 품목 TOP5가 정상 동작해야 한다.
- 업체 성과와 업체별 납기·검수 지표가 함께 렌더링되어야 한다.
- 한국어/영어 전환 시 통계 화면에 fallback 문구가 섞이지 않아야 한다.

## 보류

- 불량 수량 전용 필드는 0.9.22453에서 검수 데이터 구조 확정 후 실제 기준으로 정교화한다.
- 저장소 용량 기준은 아직 `ADMIN_FILE_LIMIT_BYTES` fallback을 사용한다. 요금제/용량 DB 연동은 0.9.22440 이후 작업한다.
