# 0.9.162 작업지시서 삭제 휴지통 정책

## 목적

작업지시서 삭제를 단일 row 삭제가 아니라 작업지시서와 연결 리소스 전체를 휴지통/숨김 상태로 이동하는 정책으로 확정한다.

## 확정 정책

1. 작업지시서를 삭제하면 작업지시서는 목록에서 숨김/휴지통 상태가 된다.
2. 연결된 디자인 파일은 함께 휴지통 대상으로 본다.
3. 연결된 일반 첨부파일은 함께 휴지통 대상으로 본다.
4. 메모 첨부가 있다면 함께 휴지통 대상으로 본다.
5. 메모 본문은 작업지시서 삭제 상태에서는 함께 숨김 처리한다.
6. R2 원본과 썸네일은 즉시 삭제하지 않는다.
7. 휴지통 항목은 30일 이내 복원 가능하다.
8. 30일이 지나면 시스템관리자 `/system/storage-usage`의 R2 purge 후보가 된다.
9. 시스템관리자 purge 실행 시 원본 `storage_key`와 `thumbnail_key`를 함께 삭제한다.

## 이번 버전 적용 범위

- 삭제 확인 모달 문구를 휴지통 정책에 맞게 정리한다.
- 0.9.161에서 발생한 build 오류를 수정한다.
- 실제 작업지시서 삭제 시 연결 첨부 전체 soft delete 적용은 다음 버전 이후로 분리한다.

## build 오류 원인

0.9.161에서 `WorkOrderDeleteConfirmModal`은 `WorkOrder` 타입을 요구했지만, 실제로는 목록에서 찾은 `WorkOrderListItem`이 전달됐다. `WorkOrderListItem`에는 `WorkOrder` 전체 필드가 없으므로 TypeScript build가 실패했다.

0.9.162에서는 삭제 확인 모달이 실제로 사용하는 필드가 `displayTitle`, `title` 정도임을 기준으로 prop 타입을 `WorkOrderListItem | null`로 좁힌다.

## 다음 단계

0.9.163 이후에는 작업지시서 삭제 API/repository 흐름을 점검해 연결된 디자인/첨부/메모 첨부를 함께 soft delete 하는 로직을 적용한다. 이때도 R2 직접 삭제는 금지하고, 시스템관리자 purge 후보 흐름을 유지한다.
