# 0.9.224346 — 작업지시서 lazy load 회귀 보정

## 목적

0.9.224345에서 작업지시서 목록/상세 lazy load를 1차 적용한 뒤, 선택 상세가 아직 hydrate되지 않은 상태에서 상세 영역이 빈 값으로 보이거나 저장/첨부/메모 액션이 먼저 실행될 수 있는 위험을 줄인다.

## 변경 기준

- 첫 목록 조회는 `/api/workorders/summary`를 유지한다.
- 선택된 작업지시서의 `hasDetailSnapshot`이 `false`이면 상세 영역과 우측 패널은 로딩 상태로 표시한다.
- 상세 로딩 중에는 저장/첨부/메모/상태 변경 계열 write action을 잠근다.
- 같은 작업지시서를 여러 번 클릭하거나 상태가 재렌더링되어도 동일 상세 요청이 중복 호출되지 않도록 in-flight id를 관리한다.
- 선택을 빠르게 바꾸더라도 이미 시작된 상세 요청은 완료 시 해당 작업지시서 row에 detail snapshot을 병합한다.

## 확인 항목

1. 작업지시서 화면 첫 진입 시 목록이 먼저 표시된다.
2. 선택 상세가 아직 로딩 중이면 중앙 상세와 우측 패널에 loading state가 표시된다.
3. 상세 로딩 중 저장/첨부/메모/상태 변경이 실행되지 않는다.
4. 이미 상세 로딩된 작업지시서는 다시 선택해도 불필요한 상세 호출이 반복되지 않는다.
5. Network 기준 `/api/workorders/summary` 호출 후 선택된 id에 대해서만 `/api/workorders/{id}`가 호출된다.
6. 작업지시서를 빠르게 여러 개 선택해도 도착한 상세 데이터가 각 row에 병합된다.
