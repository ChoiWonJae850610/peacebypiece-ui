# 0.19.76 시스템관리자 회사 계정 요청 승인/반려 처리 1차

## 목적

시스템관리자가 회사 계정 요청 검토 화면에서 접수된 요청을 `검토 중`, `승인됨`, `반려됨` 상태로 처리할 수 있도록 1차 API와 UI를 연결한다.

## 포함 범위

- `PATCH /api/system/company-account-requests` 추가
- 시스템관리자 권한 확인 유지
- 요청 상태 업데이트
- 검토자, 검토일시, 검토 메모 저장
- 시스템관리자 검토 화면에 검토 메모 입력과 처리 버튼 추가
- 목록/상세 화면에서 처리 후 상태 즉시 반영

## 제외 범위

- 회사 정보 실제 변경 적용
- 계정 비활성화 실제 적용
- 결제/요금제/저장공간 상태 변경
- 고객사 관리자에게 별도 알림 발송
- 이메일 발송

이번 버전의 승인/반려는 운영 검토 상태를 기록하는 단계다. 실제 회사 정보 반영과 계정 비활성화 적용은 다음 단계에서 분리한다.

## 테스트 포인트

1. 시스템관리자로 `/system/account-requests` 진입
2. 접수됨 요청 선택
3. 검토 메모 입력 후 `검토 중` 클릭
4. 상태가 `검토 중`으로 바뀌고 검토자/검토일이 표시되는지 확인
5. 검토 중 요청을 `승인` 처리
6. 승인됨 요청은 버튼이 비활성화되는지 확인
7. 새 접수 요청을 `반려` 처리
8. 반려 메모가 5자 미만일 때 차단되는지 확인
9. 고객사 관리자 환경설정 요청 이력에서 상태가 반영되는지 확인
