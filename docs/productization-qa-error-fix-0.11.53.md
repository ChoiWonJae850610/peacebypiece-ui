# 제품화 전 QA 오류 수정 2차 기준 정리 (0.11.53)

## 목적

0.11.52 이후 제품화 전 QA 오류 수정 라인의 두 번째 기준 문서다. 이번 입력에는 build log, runtime error, 화면 캡처가 첨부되지 않았으므로 실제 기능 코드는 임의로 수정하지 않는다.

## 이번 버전 판단

- 기준 원본: 0.11.52
- 결과 버전: 0.11.53
- 실제 오류 자료: 없음
- 코드 수정 범위: APP_VERSION 갱신만 수행
- 기능 로직 변경: 없음

## 0.11.53 이후 오류 수정 원칙

### P0: 즉시 수정

- build 실패
- 작업지시서 DB 로딩 실패
- 작업지시서 저장 실패
- 첨부 업로드/다운로드/삭제 실패
- 고객관리자 저장소 복원/삭제 요청 실패
- 시스템관리자 실제 삭제 처리 실패

### P1: 다음 패치에서 수정

- 모바일/태블릿에서 핵심 버튼 접근 불가
- 모달 footer 버튼이 화면 밖으로 밀림
- table/list overflow로 데이터 확인 불가
- i18n 전환 시 주요 문구 누락
- 권한 있음/없음 표시가 실제 동작과 다름

### P2: 묶어서 수정

- 카드 간격, radius, 색상 tone 불일치
- 설명 문구 밀도 조정
- 문서 보완
- 개발자용 안내 정리

## 다음 실제 수정 입력 형식

가능하면 다음 중 하나 이상을 함께 첨부한다.

```text
build-0.11.53-YYYYMMDD-HHMMSS.txt
runtime-error screenshot
화면 캡처
재현 경로
```

## 확인 대상

- 0.11.45.1에서 보정한 DB driver 로딩 유지
- 작업지시서 목록/상세 DB 데이터 로딩
- 모바일 작업지시서 화면 overflow
- 관리자 저장소 휴지통 모달
- 시스템관리자 storage purge
- i18n 전환 후 주요 문구 표시
