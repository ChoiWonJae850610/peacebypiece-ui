# 0.11.55 QA 수정 — 휴지통 다중 복원/삭제 확인 모달

## 목적

고객관리자 저장소 휴지통에서 여러 항목을 선택한 뒤 복원 또는 삭제를 실행할 때 즉시 처리되지 않도록 확인 모달을 추가한다.

## 반영 내용

- 휴지통 상단 action bar의 `선택 삭제` 버튼명을 `삭제`로 변경했다.
- 선택 복원 실행 전 확인 모달을 추가했다.
- 선택 삭제 실행 전 확인 모달을 추가했다.
- 확인 모달에는 처리 범위를 role 기반으로 표시한다.
  - 작업지시서 n건
  - 문서 n개
  - 디자인 n개
  - 메모 n개
- 처리할 수 없는 선택 항목이 있으면 제외 개수를 안내한다.
- 삭제는 고객관리자 삭제 요청이며 실제 R2 삭제는 시스템관리자 purge 흐름에서 처리한다는 정책 안내를 유지했다.

## 변경하지 않은 것

- `/admin/files` snapshot API
- 복원 action flow
- 삭제 요청 action flow
- 시스템관리자 R2 purge flow
- DB schema

## 테스트 기준

1. `/admin/files` 진입
2. 휴지통 탭에서 작업지시서 1건 이상 선택
3. `복원` 클릭 시 확인 모달 표시 확인
4. `삭제` 클릭 시 확인 모달 표시 확인
5. 모달 문구가 `작업지시서 n건, 문서 n개, 디자인 n개, 메모 n개` 형태로 표시되는지 확인
6. 모달 취소 시 아무 처리도 되지 않는지 확인
7. 모달 확인 시 기존 복원/삭제 요청이 실행되는지 확인
8. 개별 상세 모달 복원/삭제 동작은 기존과 동일한지 확인
