# UI Responsive Work Order Tabs 0.17.88

## 목적

0.17.87에서 원단·부자재 화면의 태블릿/모바일 전환 버튼을 `AppSegmentedTabs`로 정리했다.  
0.17.88에서는 같은 기준을 작업지시서 상세의 태블릿/모바일 화면에 적용한다.

## 적용 범위

- `components/workorder/detail/views/WorkOrderDetailMobileView.tsx`
- `components/workorder/detail/views/WorkOrderDetailTabletView.tsx`

## 변경 기준

### PC

PC 작업지시서 상세 화면은 기존 3분할/상세 섹션 흐름을 유지한다.

### Tablet

태블릿 작업지시서 상세는 다음 탭으로 분리한다.

- 진행 단계
- 공정 정보
- 제품 구성

비용 요약은 관리자에게만 보이는 정보이므로 `진행 단계` 탭 안에 함께 유지한다.

### Mobile

모바일 작업지시서 상세도 같은 탭 구조를 사용한다.

- 진행 단계
- 공정 정보
- 제품 구성

모바일에서는 한 화면에 모든 섹션을 세로로 길게 나열하지 않고, 사용자가 필요한 영역만 전환해서 보도록 한다.

## 영향 범위

- 저장 로직 변경 없음
- 상태 전환 로직 변경 없음
- 첨부/메모/R2 흐름 변경 없음
- 작업지시서 viewModel/editor 구조 변경 없음
- 공통 모달 유지

## 다음 확인 항목

1. 모바일에서 작업지시서 상세 진입 시 `진행 단계` 탭이 먼저 보이는지 확인
2. `공정 정보` 탭에서 기존 편집/저장 흐름이 유지되는지 확인
3. `제품 구성` 탭에서 원단/부자재/외주 편집 흐름이 유지되는지 확인
4. 태블릿에서 탭 전환 시 내용이 잘리지 않는지 확인
5. PC 화면이 변경되지 않았는지 확인
